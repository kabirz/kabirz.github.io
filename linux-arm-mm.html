01 概述

ARM/ARM64 架构的内存管理子系统在通用 Linux MM 框架之上，实现了架构特定的页表操作、TLB 管理、缓存一致性、异常处理等。ARM64（AArch64）是当前主流的 64 位 ARM 架构，广泛用于手机、服务器、嵌入式系统。

ARM64 内存管理特性
  • 48/52 位 VA：支持 LA57 五级页表
  • 4KB/16KB/64KB 页面：可配置页面大小
  • MTE：内存标签扩展（Memory Tagging）
  • FPHP：大页支持（64KB/512MB）
  • PAN：特权访问非特权（Privileged Access Never）
  • UAO：非特权访问覆盖（User Access Override）
  • Contiguous PTE：连续 PTE 优化
  • GCS：Guarded Control Stack

地址转换宏

宏说明
__pa(x)虚拟地址 → 物理地址
__va(x)物理地址 → 虚拟地址
phys_to_virt(x)物理地址 → 虚拟地址（可睡眠）
virt_to_phys(x)虚拟地址 → 物理地址（可睡眠）
page_to_phys(x)页面结构 → 物理地址
phys_to_page(x)物理地址 → 页面结构
线性映射：ARM64 使用线性映射（direct mapping）将物理内存直接映射到虚拟地址空间。映射起始地址为 PAGE_OFFSET，转换公式：__va(pa) = pa + PAGE_OFFSET。这使得内核可以高效访问物理内存。

ARM64 页表格式

// [63:59] PBHA (Page-Based Hardware Attributes)
// [58]    XN (Execute Never)
// [54]    Contiguous hint
// [53]    DBM (Dirty Bit Management)
// [51:12] 物理地址

// 0b11 = 表描述符（Table, PGD/P4D/PUD）
// 0b11 = 页描述符（Page, PTE）

Contiguous PTE

Contiguous PTE：ARM64 硬件支持将 16 个连续的 PTE（4KB 页面）或 16 个连续的 PMD（64KB 大页）标记为一个整体。硬件可以一次性翻译这组条目，减少 TLB miss。Linux 通过 contpte.c 实现。

TLB 刷新流程

// ARM64 TLB 刷新宏（arch/arm64/include/asm/tlbflush.h）

// 刷新当前 CPU
#define __flush_tlb_local() \
    do { \
        dsb(nsh); \
        tlbi(vmalle1); \
        isb(); \
    } while (0);

// 刷新所有 CPU（IS 变体 = Inner Shareable）
#define __flush_tlb_all() \
    do { \
        dsb(ishst); \
        tlbi(vmalle1is); \
        dsb(ish); \
        isb(); \
    } while (0);

// 刷新指定虚拟地址（所有 CPU）
#define __flush_tlb_range(vstart, vend) \
    do { \
        dsb(ishst); \
        tlbi(vae1is, addr_to_vae(vstart)); \
        dsb(ish); \
    } while (0);
ASID：ARM64 支持 ASID（Address Space Identifier），允许不同进程的 TLB 条目共存。Linux 为每个 mm 分配全局 ASID，避免进程切换时全量 TLB flush。

DMA 缓存一致性

DMA 一致性：ARM64 是非一致 DMA架构。设备通过 DMA 访问内存时，CPU 缓存可能包含脏数据。Linux 通过 dma_map_sg() 等 API 自动处理 cache clean/flush。使用 DMA coherent 映射可以避免手动管理（但性能较低）。

07 内存保护

ARM64 安全特性

PAN（Privileged Access Never）

防止内核代码意外访问用户空间内存。设置 PAN 位后，内核使用 uaccess 指令（如 ldtr/sttr）才能访问用户空间。

UAO（User Access Override）

允许内核临时禁用 PAN 以高效访问用户空间内存。通过 access_ok() 和显式 UAO 指令控制。

MTE（Memory Tagging Extension）

MTE 是 ARMv8.5 引入的内存标记机制，为每个 16 字节内存粒度分配 4 位标签，用于检测缓冲区溢出和 use-after-free。

// MTE 工作原理：
// 1. 分配内存时生成随机标签（存储在页表扩展位）
// 2. 分配时将标签存入指针的高位
// 3. 访问时硬件检查指针标签 vs 内存标签
// 4. 不匹配时触发 Tag Check Fault

// Linux MTE 支持：
// CONFIG_ARM64_MTE            — 启用 MTE
// CONFIG_ARM64_MTE_MEMORY_TAGGING — 内存标签扩展
// CONFIG_MTE_ON — 默认启用

// 配置：
// echo 1 > /proc/sys/vm/unprivileged_userfaultfd_disabled
// echo 1 > /proc/sys/kernel/unprivileged_userfaultfd_disabled

FEAT_S1PIE（页表完整性）

页表完整性：ARMv8.9/ARMv9.4 引入 FEAT_S1PIE，为页表条目添加完整性校验位，防止恶意篡改页表。Linux 通过 CONFIG_ARM64_PAN 和相关 Kconfig 选项支持。
