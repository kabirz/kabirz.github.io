01 概述

seccomp（Secure Computing Mode）是 Linux 内核提供的系统调用过滤器，允许进程限制自己可以调用的系统调用。它是 Docker、Chrome、Firefox 等应用的核心安全机制——容器只能使用 ~50 个安全的系统调用（总共 ~450 个）。

02 三种模式

模式行为引入版本
SECCOMP_MODE_STRICT仅允许 read/write/exit/sigreturn2.6.12
SECCOMP_MODE_FILTERBPF 过滤器自定义允许的系统调用3.5
SECCOMP_MODE_USER_NOTIFICATION通知用户态程序做决策5.0
SECCOMP_MODE_FILTER（最常用）：

  进程调用系统调用（如 open、read、write）
        │
        ↓
  内核检查 seccomp 是否启用
        │
        ├── 未启用 → 正常执行
        │
        └── 已启用 → 运行 BPF 过滤器
                │
                ├── BPF 程序返回 ALLOW → 执行系统调用
                ├── BPF 程序返回 KILL → 杀死进程
                ├── BPF 程序返回 ERRNO → 返回错误码
                ├── BPF 程序返回 TRAP → 发送 SIGSYS 信号
                └── BPF 程序返回 TRACE → ptrace 跟踪

Docker 默认 seccomp 配置：

  允许：read, write, open, close, stat, fstat, mmap, ...
  禁止：reboot, mount, umount2, swapon, swapoff, ...
  约 50/450 个系统调用被允许
  覆盖 99% 的正常应用需求

03 过滤流程

seccomp 过滤路径（每次系统调用都会检查）：

  用户态 → syscall(X) → 内核入口
        │
        ↓
  __seccomp() — kernel/seccomp.c
        │
        ├── 1. 检查 seccomp.mode
        │       STRICT → 仅允许 4 个系统调用
        │       FILTER → 运行 BPF
        │       NONE → 正常执行
        │
        ├── 2. 运行 BPF 过滤器链
        │       遍历所有已安装的 BPF 过滤器
        │       每个过滤器返回：ALLOW / KILL / ERRNO / ...
        │
        ├── 3. 如果返回 ALLOW → 执行系统调用
        │
        └── 4. 如果返回 KILL/ERRNO → 拒绝
                KILL → 发送 SIGKILL
                ERRNO → 返回 -1, errno = 指定错误码

04 应用场景

// 用户态代码示例：安装 seccomp 过滤器
#include <linux/seccomp.h>
#include <linux/filter.h>
#include <linux/audit.h>

// BPF 过滤器：只允许 read/write/open/close
struct sock_filter filter[] = {
    BPF_STMT(BPF_LD+BPF_W+BPF_ABS,
             offsetof(struct seccomp_data, nr)),
    BPF_JUMP(BPF_JMP+BPF_JEQ+BPF_K, __NR_read, 0, 1),
    BPF_STMT(BPF_RET+BPF_K, SECCOMP_RET_ALLOW),
    BPF_JUMP(BPF_JMP+BPF_JEQ+BPF_K, __NR_write, 0, 1),
    BPF_STMT(BPF_RET+BPF_K, SECCOMP_RET_ALLOW),
    // ... 更多规则 ...
    BPF_STMT(BPF_RET+BPF_K, SECCOMP_RET_KILL),
};

struct sock_fprog prog = {
    .len = sizeof(filter) / sizeof(filter[0]),
    .filter = filter,
};

prctl(PR_SET_NO_NEW_PRIVS, 1, 0, 0, 0);
prctl(PR_SET_SECCOMP, SECCOMP_MODE_FILTER, &prog);

使用场景

场景说明
Docker 容器默认 seccomp profile 限制 ~50 个系统调用
Chrome 沙箱渲染进程只能使用极少数系统调用
Firefox 沙箱内容进程 seccomp 过滤
systemd 服务SystemCallFilter= 限制服务系统调用
Android 应用应用沙箱 seccomp 限制
