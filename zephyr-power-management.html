02 PM 框架

// PM 框架核心（subsys/pm/pm.c）

// 系统休眠状态
// PM_STATE_STANDBY    - 待机（保留 RAM）
// PM_STATE_SUSPEND    - 挂起（部分关闭）
// PM_STATE_SOFT_OFF   - 软关闭（需要唤醒源）

// PM 上下文
struct pm_state_info {
    enum pm_state state;         // 休眠状态
    uint8_t min_residency;       // 最小停留时间（ms）
    uint8_t exit_latency;        // 退出延迟（ms）
};

// 手动控制 PM
// CONFIG_PM=y
// CONFIG_PM_DEVICE=y
// CONFIG_PM_DEVICE_RUNTIME=y   ← 设备运行时 PM

03 休眠状态

状态功耗唤醒时间说明
Active最高0正常运行
Idle低极短CPU 空闲（WFI）
Standby更低短保留 RAM，关闭外设
Suspend很低中等大部分关闭
Soft Off最低长需要硬件唤醒源

ARM 休眠模式

// ARM Cortex-M 休眠指令
// WFI (Wait For Interrupt) - 等待中断唤醒
// WFE (Wait For Event) - 等待事件唤醒
// WFI 比 WFE 更省电

// 休眠流程
// 1. 关闭设备时钟
// 2. 降低 CPU 频率
// 3. 执行 WFI 指令
// 4. 中断唤醒
// 5. 恢复设备时钟
// 6. 恢复 CPU 频率

// 锁定状态（防止进入特定休眠状态）

设备 PM 实现

// 驱动实现 PM 回调
static int my_driver_pm_action(const struct device *dev,
                               enum pm_device_action action)
{
    switch (action) {
    case PM_DEVICE_ACTION_RESUME:
        // 恢复设备
        enable_clocks(dev);
        break;
    case PM_DEVICE_ACTION_SUSPEND:
        // 挂起设备
        disable_clocks(dev);
        break;
    case PM_DEVICE_ACTION_TURN_OFF:
        // 关闭设备
        power_off(dev);
        break;
    default:
        return -ENOTSUP;
    }
    return 0;
}

06 Tickless 省电

Tickless 模式允许系统在空闲时停止周期性 tick 中断，显著降低功耗。

// Tickless 配置
// CONFIG_TICKLESS_KERNEL=y
// CONFIG_TICKLESS_LOW_POWER=y

// 工作原理：
// 1. 系统空闲时，查找下一个超时事件
// 2. 设置硬件定时器在该时间点唤醒
// 3. 进入深度休眠（WFI/WFE）
// 4. 中断唤醒，继续执行

// 节省的功耗：
// 周期 Tick (1kHz)：CPU 每秒唤醒 1000 次
// Tickless：CPU 只在有事件时唤醒
// 典型节省：50-90% 功耗
