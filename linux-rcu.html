01 概述

RCU（Read-Copy-Update）是 Linux 内核中最重要的并发原语之一，用于实现无锁读取。读取者在 RCU 读临界区内不获取任何锁，写入者在修改数据后等待宽限期（grace period）再释放旧数据。RCU 是网络协议栈、路由表、VFS dcache 等核心子系统的基石。

RCU 核心原理
  • 读者零开销：rcu_read_lock() 仅禁止抢占（非 PREEMPT_RCU）
  • 宽限期：写入者等待所有读者退出临界区
  • 无锁：读者不需要获取任何锁
  • 写时复制：修改数据前先复制一份
  • 延迟释放：旧数据在宽限期后释放
适用场景
场景原因
路由表查找读多写极少
DCache 查找路径查找是热路径
模块列表模块加载/卸载很少
通知链读取者众多
网络包分类高性能需求

02 RCU 类型

类型抢占场景
RCU（Tree）CONFIG_PREEMPT_RCU默认，通用场景
RCU（Tiny）CONFIG_TINY_RCU单处理器
SRCU可选可睡眠的读临界区
Tasks RCU-非抢占上下文切换

PREEMPT_RCU vs 非抢占 RCU

非抢占 RCU（默认）
  • rcu_read_lock() = preempt_disable()
  • 读临界区不可被抢占
  • 宽限期 = 所有 CPU 经过一次上下文切换
  • 开销极低
PREEMPT_RCU
  • rcu_read_lock() 仅递增计数器
  • 读临界区可被抢占
  • 宽限期需要跟踪抢占的读者
  • 开销略高，但允许更低延迟

03 核心 API

读取侧 API

// 进入 RCU 读临界区
rcu_read_lock();
// ... 访问 RCU 保护的数据 ...
p = rcu_dereference(global_ptr);  // 安全读取指针
// ... 使用 p 访问数据 ...
rcu_read_unlock();

更新侧 API

// 1. 分配新版本数据
new = kmalloc(sizeof(*new), GFP_KERNEL);
*new = *old;  // 复制旧数据
new->field = new_value;  // 修改

// 2. 替换指针（发布新版本）
rcu_assign_pointer(global_ptr, new);

// 3. 同步方式释放旧数据
synchronize_rcu();     // 等待宽限期（可睡眠）
kfree(old);

// 或 异步方式释放
call_rcu(&old->rcu_head, my_free_callback); // 回调释放

API 总结

函数类型说明
rcu_read_lock()读进入读临界区
rcu_read_unlock()读退出读临界区
rcu_dereference(p)读安全读取 RCU 保护的指针
rcu_dereference_protected(p, c)读持锁时读取
rcu_assign_pointer(p, v)写发布新指针
synchronize_rcu()写等待宽限期（可睡眠）
call_rcu(head, func)写宽限期后异步回调
rcu_barrier()写等待所有 call_rcu 回调完成
rcu_read_lock_sched()读调度器 RCU 读临界区
rcu_read_lock_bh()读软中断 RCU 读临界区

04 使用模式

模式一：RCU 指针替换

struct my_config __rcu *global_config;

// 更新（写者）
struct my_config *new = kmalloc(sizeof(*new), GFP_KERNEL);
*new = *global_config;
new->timeout = 300;
rcu_assign_pointer(global_config, new);
synchronize_rcu();
kfree(old);

// 读取（读者）
rcu_read_lock();
struct my_config *cfg = rcu_dereference(global_config);
timeout = cfg->timeout;  // 安全访问
rcu_read_unlock();

模式二：RCU 遍历链表

// 遍历 RCU 保护的链表
rcu_read_lock();
list_for_each_entry_rcu(item, &my_list, list) {
    // item 在 RCU 保护下安全访问
    process(item);
}
rcu_read_unlock();

// 安全删除（需要持有写锁）
spin_lock(&my_lock);
list_del_rcu(&item->list);  // 从链表移除
spin_unlock(&my_lock);
call_rcu(&item->rcu_head, my_free);  // 宽限期后释放

模式三：SLAB_TYPESAFE_BY_RCU

// 允许 slab 对象在 RCU 读者访问期间被释放并重用
// 读者必须在访问后检查对象是否仍然有效
rcu_read_lock();
struct my_obj *obj = rcu_dereference(ptr);
// 二次验证：确认 obj 仍属于预期的 slub
if (unlikely(obj->gen != expected_gen)) {
    rcu_read_unlock();
    return -1;
}
process(obj);
rcu_read_unlock();
RCU 使用规则：
• 读临界区内不可睡眠（除非使用 SRCU）
• 读临界区内不可抢占（非 PREEMPT_RCU）
• rcu_dereference() 必须在 rcu_read_lock() 内使用
• rcu_assign_pointer() 必须在修改完新数据后调用
• 释放旧数据必须等待宽限期

05 Tree RCU

Tree RCU 是多处理器系统的默认 RCU 实现，使用层级结构跟踪 CPU 状态，实现高效的宽限期检测。

工作原理

rcu_read_lock()
↓ 仅递增 current->rcu_read_lock_nesting
... 临界区 ...
↓
rcu_read_unlock()
↓
检查是否需要通知 RCU 核心

宽限期（Grace Period）

// 宽限期 = 所有 CPU 至少经历一次上下文切换
// 在非 PREEMPT_RCU 下，这等价于：所有 CPU 都调用了 rcu_read_unlock()

// Tree RCU 的层级结构：
// CPU 0 ─┐
// CPU 1 ─┤── rcu_node (叶节点)
// CPU 2 ─┘         │
// CPU 3 ─┐         │
// CPU 4 ─┤── rcu_node (叶节点)
// CPU 5 ─┘         │
//              rcu_node (内部节点)
//                   │
//              rcu_state (根)

// 宽限期检测：
// 1. 根节点标记新宽限期开始
// 2. 各 CPU 检查自己的 quiescent state
// 3. 叶节点收集所有 CPU 的状态
// 4. 逐级上报到根节点
// 5. 根节点确认宽限期完成

Quiescent State（静默状态）

事件说明
上下文切换任务离开内核态
返回用户态从内核态返回
空闲CPU 进入 idle
离线CPU 热插拔离线
上下文跟踪经过 context_tracking

06 回调机制

call_rcu 回调

struct my_obj {
    struct rcu_head rcu;
    int data;
};

static void my_free_cb(struct rcu_head *head)
{
    struct my_obj *obj = container_of(head, struct my_obj, rcu);
    kfree(obj);
}

// 使用
call_rcu(&old_obj->rcu, my_free_cb);

synchronize_rcu vs call_rcu

方式特点场景
synchronize_rcu()阻塞等待宽限期可睡眠上下文
call_rcu()异步回调不可睡眠上下文
kvfree_rcu()异步 kfree简单释放（无需自定义回调）
性能提示：call_rcu() 将回调放入 per-CPU 链表，批量处理以减少开销。对于大量释放操作，优先使用 call_rcu() 而非 synchronize_rcu() + kfree()。

07 SRCU

SRCU（Sleepable RCU）是 RCU 的扩展，允许读临界区内睡眠。适用于需要长时间持有读锁且可能调用可睡眠函数的场景。

SRCU vs RCU

特性RCUSRCU
读临界区可睡眠否是
宽限期检测基于 quiescent state基于 CPU 上下文切换
开销极低中等
适用场景通用无锁读可睡眠读临界区
嵌套支持需要独立 SRCU 域

SRCU API

// 定义 SRCU 域
DEFINE_STATIC_SRCU(my_srcu);

// 读临界区（可睡眠）
int idx = srcu_read_lock(&my_srcu);
// ... 可以睡眠的操作 ...
p = rcu_dereference(data);
process(p);
srcu_read_unlock(&my_srcu, idx);

// 宽限期等待
synchronize_srcu(&my_srcu);

// 异步释放
call_srcu(&my_srcu, &obj->rcu_head, my_free_cb);
