01 概述

ICMP（Internet Control Message Protocol）是 IP 层的控制和诊断协议，定义在 RFC 792。它不传输用户数据，而是用于网络错误报告（目标不可达、TTL 过期等）和网络诊断（ping、traceroute）。Linux 内核 ICMP 实现在 net/ipv4/icmp.c（1778 行）。

ICMP 核心功能
  • Echo 请求/应答：ping 工具的基础
  • 错误报告：目标不可达、TTL 过期
  • 路由重定向：通知更优的路由
  • 路径 MTU 发现：Frag Needed 消息
  • 时间戳：网络延迟测量（已少用）

Destination Unreachable 子类型

// net/ipv4/icmp.c:124 — ICMP 错误到 errno 映射
const struct icmp_err icmp_err_convert[] = {
    { .errno = ENETUNREACH,   .fatal = 0 },  // 0: Network Unreachable
    { .errno = EHOSTUNREACH,  .fatal = 0 },  // 1: Host Unreachable
    { .errno = ENOPROTOOPT,   .fatal = 1 },  // 2: Protocol Unreachable
    { .errno = ECONNREFUSED,  .fatal = 1 },  // 3: Port Unreachable
    { .errno = EMSGSIZE,      .fatal = 0 },  // 4: Fragmentation Needed
    { .errno = EOPNOTSUPP,    .fatal = 0 },  // 5: Source Route Failed
    { .errno = ENETUNREACH,   .fatal = 1 },  // 6: Network Unknown
    { .errno = EHOSTDOWN,     .fatal = 1 },  // 7: Host Unknown
    { .errno = EHOSTUNREACH,  .fatal = 0 },  // 12: TOS Host Unreachable
};
// fatal=1 → 连接直接终止
// fatal=0 → 可以重试其他路由

04 内核处理

接收 ICMP 消息

  网卡收到 ICMP 数据包
        │
        ↓
  IP 层 → ip_rcv → NF_INET_LOCAL_IN → ip_local_deliver
        │
        ↓
  IP 协议分发：protocol = IPPROTO_ICMP
        │
        ↓
  icmp_rcv() — net/ipv4/icmp.c
        │
        ├── 校验和验证
        │
        ├── 根据 Type 查找 icmp_pointers[] 处理函数
        │
        ├── Type 0 (Echo Reply)  → ping_rcv()
        │       传递给 ping socket
        │       唤醒 ping 进程
        │
        ├── Type 8 (Echo Request) → icmp_echo()
        │       构造 Echo Reply
        │       调用 icmp_reply() 发送
        │
        ├── Type 3 (Dest Unreach) → icmp_unreach()
        │       提取原始数据包头
        │       传递给上层协议（TCP/UDP）
        │       → tcp_v4_err() / udp_err()
        │
        ├── Type 5 (Redirect) → icmp_redirect()
        │       更新路由缓存
        │
        └── Type 11 (Time Exceeded) → icmp_unreach()
                traceroute 使用此消息

发送 ICMP 消息

  icmp_send() — 内核协议栈调用（生成错误消息）
        │
        ├── 被谁调用？
        │   ip_rcv()     → 目标不可达
        │   ip_forward() → 转发失败
        │   ip_frag()    → 需要分片
        │   udp_rcv()    → 端口不可达
        │
        ├── rate limit 检查（防止 ICMP 风暴）
        │       icmpv4_xrlim_allow()
        │       最多每秒 1000 个 ICMP 错误消息
        │
        └── 构造 ICMP 头 + 原始包头
                icmp_reply() → ip_append_data() → ip_push_pending_frames()

  icmp_echo() → icmp_reply() — 处理 ping 请求

05 错误消息与 socket 映射

ICMP 错误消息如何传递到应用层？

  目标主机收到 ICMP Dest Unreach (Port Unreachable)
        │
        ↓
  icmp_unreach() → 提取原始包的 (protocol, sport, dport)
        │
        ├── protocol = TCP → tcp_v4_err()
        │       → 通知 TCP socket "连接被拒绝"
        │       → 应用 recv() 返回 -1, errno = ECONNREFUSED
        │
        ├── protocol = UDP → udp_err()
        │       → 通知 UDP socket "端口不可达"
        │       → 应用 recvfrom() 返回 -1, errno = ECONNREFUSED
        │
        └── protocol = 未知 → 丢弃

典型场景：

  客户端 TCP connect() → SYN → 服务端无监听
  ← RST (TCP) 或 ← ICMP Port Unreachable
  → connect() 返回 ECONNREFUSED

06 ping 与 ping6

  ping 进程
    │
    ├── socket(AF_INET, SOCK_DGRAM, IPPROTO_ICMP) （ping）
    │   或 socket(AF_INET6, SOCK_DGRAM, IPPROTO_ICMPV6) （ping6）
    │
    ├── sendto(fd, echo_request, ...) → 内核 ICMP 层
    │       → icmp_echo() 构造 Echo Request
    │       → ip_output() → 网卡发送
    │
    ├── 等待 Echo Reply
    │       → icmp_rcv() → Type 0 (Echo Reply)
    │       → ping_rcv() → 唤醒 socket
    │
    └── recvfrom(fd, echo_reply, ...) → 计算 RTT

// net/ipv4/icmp.c:1677 — Echo Request 处理
[ICMP_ECHO] = {
    .handler = icmp_echo,  // 收到 Echo Request 后回调
},

// icmp_echo() 核心逻辑：
static void icmp_echo(struct sk_buff *skb)
{
    // 检查 sysctl_icmp_echo_ignore_all
    if (net->ipv4.sysctl_icmp_echo_ignore_all)
        return;  // 丢弃所有 ping

    // 检查广播 ping 是否忽略
    if (skb->pkt_type == PACKET_BROADCAST &&
        net->ipv4.sysctl_icmp_echo_ignore_broadcasts)
        return;

    // 构造并发送 Echo Reply
    icmp_echo(skb);
}

07 安全与调优

# 禁止所有 ping（ICMP Echo Reply）
$ sysctl -w net.ipv4.icmp_echo_ignore_all=1

# 忽略广播 ping
$ sysctl -w net.ipv4.icmp_echo_ignore_broadcasts=1

# 限制 ICMP 错误消息速率（防止 ICMP 风暴）
$ sysctl -w net.ipv4.icmp_ratelimit=1000     # 每秒上限
$ sysctl -w net.ipv4.icmp_ratemask=6168       # 受限的消息类型

# ICMP 重定向控制
$ sysctl -w net.ipv4.conf.all.accept_redirects=0
$ sysctl -w net.ipv4.conf.all.send_redirects=0

# 忽略伪造的 ICMP 错误响应
$ sysctl -w net.ipv4.icmp_ignore_bogus_error_responses=1

关键 sysctl 参数

参数默认值说明
icmp_echo_ignore_all01=禁止所有 ping
icmp_echo_ignore_broadcasts1忽略广播 ping（推荐开启）
icmp_ratelimit1000ICMP 错误消息速率限制（ms）
icmp_ratemask6168哪些 ICMP 类型受速率限制
accept_redirects1是否接受 ICMP 重定向
send_redirects1是否发送 ICMP 重定向
ignore_bogus_error_responses1忽略伪造错误响应
