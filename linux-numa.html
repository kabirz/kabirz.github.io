01 概述

NUMA（Non-Uniform Memory Access）是多路服务器的内存架构。每个 CPU 直连自己的本地内存，访问其他 CPU 的本地内存需要通过互联总线，延迟更高。Linux 内核自动管理 NUMA 拓扑，优化内存分配位置。

02 NUMA 架构

UMA（统一内存访问）：

  CPU 0 ←──→ CPU 1 ←──→ CPU 2 ←──→ CPU 3
    │            │          │          │
    └────────────┴──────────┴──────────┘
                    │
              ┌─────┴─────┐
              │  共享内存   │  所有 CPU 访问延迟相同
              └───────────┘

NUMA（非统一内存访问）：

  ┌─────────────┐  ┌─────────────┐
  │   Node 0    │  │   Node 1    │
  │ ┌───┐ ┌───┐│  │ ┌───┐ ┌───┐│
  │ │C0 │ │C1 ││  │ │C2 │ │C3 ││
  │ └───┘ └───┘│  │ └───┘ └───┘│
  │     │       │  │      │     │
  │ ┌───┴───┐   │  │  ┌───┴───┐ │
  │ │内存 64G│   │  │  │内存 64G│ │
  │ └───────┘   │  │  └───────┘ │
  └──────┬──────┘  └──────┬─────┘
         │                │
         └──── 互联总线 ───┘
               │
         本地访问 ~100ns
         远程访问 ~300ns（3 倍延迟）

04 NUMA 策略

策略说明适用场景
default从当前 CPU 的本地节点分配通用
bind绑定到指定节点性能关键应用
interleave在指定节点间轮询分配大内存应用
firsttouch首次访问时分配到本地节点通用
weight按权重在节点间分配精细控制
# 设置进程的 NUMA 策略
$ numactl --membind=0 ./myapp         # 绑定到 Node 0
$ numactl --interleave=all ./myapp    # 交错分配
$ numactl --cpunodebind=0 ./myapp     # 绑定 CPU 到 Node 0

# 查看 NUMA 拓扑
$ numactl --hardware
available: 2 nodes (0-1)
node 0 cpus: 0 1 2 3
node 0 size: 65536 MB
node 1 cpus: 4 5 6 7
node 1 size: 65536 MB
node 0 dist: 10 21
node 1 dist: 21 10

# 查看进程的 NUMA 内存分布
$ numastat -p <pid>

05 NUMA 平衡

NUMA 自动平衡（kernel numad）：

  进程在 Node 0 运行，频繁访问 Node 1 的内存
        │
        ↓
  NUMA balancing 采样
        │
        ├── 检测到远程访问
        │
        ├── 迁移页面到本地节点
        │   migrate_pages() → 从 Node 1 复制到 Node 0
        │
        └── 更新页表 → 新分配在本地节点

  开启 NUMA balancing：
  $ sysctl -w kernel.numa_balancing=1

  禁用 NUMA balancing：
  $ sysctl -w kernel.numa_balancing=0
  // 对已知 NUMA 优化的应用（如数据库）可能更好
