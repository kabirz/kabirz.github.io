01 PyTorch 简介与安装

PyTorch 是 Meta（原 Facebook）开发的深度学习框架，以动态计算图（Define-by-Run）为核心设计理念。它提供了灵活的张量计算、自动求导机制以及丰富的神经网络 API，在学术研究和工业部署中都得到广泛应用。

PyTorch 2.x 引入了 torch.compile 即时编译技术，通过图捕获和底层算子融合，在保持动态图灵活性的同时大幅提升训练和推理性能。

核心特性:
动态计算图 — 运行时构建，调试友好。
Eager 模式 — 每行代码即时执行，符合 Python 直觉。
TorchScript / torch.compile — 从动态图到静态图的平滑过渡。
Distributed 生态 — DDP / FSDP / DeepSpeed 无缝集成。

安装 PyTorch 推荐使用 pip 或 conda，官方会根据 CUDA 版本生成对应的安装命令：

# CPU 版本（适用于无 GPU 环境）
pip install torch torchvision torchaudio

# CUDA 12.x 版本
pip install torch torchvision torchaudio --index-url \
  https://download.pytorch.org/whl/cu121

# conda 安装（包含 CUDA 依赖）
conda install pytorch torchvision torchaudio pytorch-cuda=12.1 \
  -c pytorch -c nvidia

02 Tensor 基础

Tensor（张量）是 PyTorch 的核心数据结构，可以理解为支持 GPU 加速的多维数组。它在概念上类似 NumPy 的 ndarray，但增加了 requires_grad 等自动求导属性。

创建 Tensor

import torch

# 从 Python 列表创建
a = torch.tensor([[1, 2], [3, 4]], dtype=torch.float32)

# 特殊初始化
z = torch.zeros(3, 4)           # 全零
o = torch.ones(2, 3)            # 全一
e = torch.eye(5)                # 单位矩阵
r = torch.randn(3, 3)           # 标准正态分布
ri = torch.randint(0, 10, (3,))  # 随机整数

# 形状操作
x = torch.empty(2, 3)           # 未初始化（垃圾值）
x = torch.full((2, 3), 7)       # 填充常量

# 从 NumPy 互转
import numpy as np
arr = np.array([1, 2, 3])
t = torch.from_numpy(arr)       # ndarray -> Tensor（共享内存）
arr2 = t.numpy()                # Tensor -> ndarray（共享内存）

Tensor 属性

x = torch.randn(3, 4, 5)
print(x.shape)      # torch.Size([3, 4, 5])
print(x.dtype)      # torch.float32
print(x.device)     # cpu / cuda:0

print(x.stride())   # (20, 5, 1)

03 Tensor 运算与索引

PyTorch 提供了丰富的张量运算，涵盖算术运算、线性代数、归约操作等。多数运算都支持运算符重载和函数式两种调用方式。

算术运算

a = torch.tensor([1, 2, 3])
b = torch.tensor([4, 5, 6])

# 运算符重载
c = a + b    # tensor([5, 7, 9])
c = a - b    # tensor([-3, -3, -3])
c = a * b    # tensor([4, 10, 18])  逐元素乘
c = a / b    # tensor([0.25, 0.40, 0.50])
c = a ** 2   # tensor([1, 4, 9])

# 函数式
c = torch.add(a, b)
c = torch.mul(a, b)
c = torch.matmul(a, b)  # 点积: tensor(32)

归约与统计

x = torch.randn(3, 4)

x.sum()               # 全部元素求和
x.mean(dim=0)         # 沿着 dim=0 求均值
x.std()               # 标准差
x.max()               # 最大值
x.argmax(dim=1)       # 每行最大值索引
x.min(dim=0).values   # 每列最小值
x.clamp(-1, 1)        # 裁剪到 [-1, 1]

索引与切片

x = torch.randn(4, 3, 5)

x[0]                  # 第一张三维切片 (3, 5)
x[:, 0]               # 所有第一行 (4, 5)
x[1:3, :, :2]         # 第二到第三样本，前两列
x[..., -1]            # 最后一个通道（省略号）

# 高级索引
indices = torch.tensor([0, 2, 2])
x[indices]            # 按行索引选取
mask = x > 0
x[mask]               # 布尔掩码索引（返回一维）

04 Tensor View 与 Broadcasting

View / Reshape

view() 和 reshape() 用于改变张量形状而不改变数据。view 要求张量在内存中是连续的，reshape 则自动处理非连续情况。

x = torch.randn(2, 3, 4)

# view（需要连续内存）
y = x.view(2, 12)           # 展平最后两维
y = x.view(-1, 4)           # 自动推导演算维度: (6, 4)

# reshape（自动处理非连续）
y = x.permute(2, 0, 1)      # 转置 -> 非连续
y = y.reshape(-1)           # reshape 不会报错

# 其他形状变换
z = x.transpose(0, 1)       # 交换维度
z = x.permute(2, 0, 1)      # 任意维度重排 (C, N, H)
z = x.squeeze(1)            # 移除大小为 1 的维度
z = x.unsqueeze(0)          # 增加维度 (1, 2, 3, 4)
z = x.flatten(start_dim=1)  # 从 dim=1 开始展平

Broadcasting

Broadcasting 是 PyTorch 自动扩展不同形状张量进行逐元素运算的机制。规则与 NumPy 一致：从尾部维度开始对齐，每个维度必须为 1 或相等。

# 标量广播
a = torch.ones(3, 4)
b = 5.0
c = a + b  # 5 广播到每个元素: tensor([[6, ...]])

# 向量广播
a = torch.ones(3, 4)          # (3, 4)
b = torch.tensor([1, 2, 3, 4])  # (4,) -> 广播为 (3, 4)
c = a + b

# 列向量广播
a = torch.ones(3, 4)
b = torch.tensor([[1], [2], [3]])  # (3, 1) -> 广播为 (3, 4)
c = a + b

# 多维度广播
a = torch.ones(2, 1, 5)       # (2, 1, 5)
b = torch.ones(3, 1)          #      (3, 1) -> (1, 3, 1) -> (2, 3, 5)
c = a + b                     # 结果形状 (2, 3, 5)
注意:
view 和 reshape 返回的张量与原张量共享数据内存，修改会影响原始数据。使用 .clone() 创建独立副本。

05 Autograd 自动求导

PyTorch 的 autograd 引擎通过运行时构建计算图（DAG）并自动执行反向传播。每个 Tensor 有 .grad_fn 属性记录生成该张量的操作，构成计算图的节点。

基本用法

x = torch.randn(3, requires_grad=True)
w = torch.randn(3, requires_grad=True)
b = torch.randn(1, requires_grad=True)

y = (x * w).sum() + b     # 前向：构建计算图
y.backward()              # 反向：计算梯度

print(x.grad)             # dy/dx
print(w.grad)             # dy/dw
print(b.grad)             # dy/db

计算图机制

x = torch.tensor(2.0, requires_grad=True)
y = x ** 2                # grad_fn: PowBackward0
z = y + 3                 # grad_fn: AddBackward0

print(z.grad_fn)          # <AddBackward0 object>
print(z.grad_fn.next_functions)  # 前驱节点

z.backward()
print(x.grad)             # 2 * 2 = 4.0

梯度清零与累积

# .backward() 默认累积梯度，每次需手动清零
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

for data, target in dataloader:
    optimizer.zero_grad()       # 梯度清零
    output = model(data)
    loss = loss_fn(output, target)
    loss.backward()
    optimizer.step()

# 也可以手动清零
model.zero_grad()

梯度控制

# 关闭梯度跟踪（推理时）
with torch.no_grad():
    y = model(x)                # 不构建计算图，节省内存

# 冻结参数
for param in model.parameters():
    param.requires_grad = False

# 查看梯度
for name, param in model.named_parameters():
    if param.grad is not None:
        print(name, param.grad.norm())
性能提示:
训练时使用 loss.backward() 前确保 optimizer.zero_grad() 已调用。推理时始终使用 torch.no_grad() 上下文以避免不必要的计算图构建。

06 nn.Module 与模型搭建

nn.Module 是所有神经网络模块的基类。它提供了参数管理、子模块注册、训练/评估模式切换等基础设施。

自定义模块

import torch.nn as nn
import torch.nn.functional as F

class MLP(nn.Module):
    def __init__(self, in_dim, hidden_dim, out_dim):
        super().__init__()
        self.fc1 = nn.Linear(in_dim, hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, hidden_dim)
        self.fc3 = nn.Linear(hidden_dim, out_dim)
        self.dropout = nn.Dropout(0.3)

    def forward(self, x):
        x = F.relu(self.fc1(x))
        x = self.dropout(x)
        x = F.relu(self.fc2(x))
        x = self.fc3(x)
        return x

model = MLP(784, 256, 10)
print(model)

nn.Sequential

nn.ModuleList / ModuleDict

# 动态层列表
class MultiHeadNet(nn.Module):
    def __init__(self, num_heads=3):
        super().__init__()
        self.heads = nn.ModuleList([
            nn.Linear(128, 10) for _ in range(num_heads)
        ])

    def forward(self, x):
        return [head(x) for head in self.heads]

# ModuleDict（按名称选择）
class Selector(nn.Module):
    def __init__(self):
        super().__init__()
        self.activations = nn.ModuleDict({
            "relu": nn.ReLU(),
            "tanh": nn.Tanh(),
            "sigmoid": nn.Sigmoid(),
        })

    def forward(self, x, act="relu"):
        return self.activations[act](x)

07 DataLoader 与 Dataset

torch.utils.data.Dataset 和 DataLoader 是 PyTorch 数据加载的核心抽象。Dataset 定义数据索引方式，DataLoader 负责批量加载、打乱和多进程并行。

自定义 Dataset

from torch.utils.data import Dataset, DataLoader

class ImageDataset(Dataset):
    def __init__(self, image_paths, labels, transform=None):
        self.image_paths = image_paths
        self.labels = labels
        self.transform = transform

    def __len__(self):
        return len(self.image_paths)

    def __getitem__(self, idx):
        img = Image.open(self.image_paths[idx]).convert("RGB")
        label = self.labels[idx]

        if self.transform:
            img = self.transform(img)

        return img, label

# 使用

    transform=transforms.ToTensor()

08 训练循环模板

PyTorch 的训练循环需要手动组织，这提供了完全的控制权。以下是适用于分类任务的通用模板。

import torch
import torch.nn as nn
from torch.utils.data import DataLoader

def train_one_epoch(model, dataloader, optimizer, criterion,
                    device, epoch, log_interval=100):
    model.train()
    total_loss = 0
    correct = 0
    total = 0

    for batch_idx, (data, target) in enumerate(dataloader):
        data, target = data.to(device), target.to(device)

        optimizer.zero_grad()
        output = model(data)
        loss = criterion(output, target)
        loss.backward()
        optimizer.step()

        total_loss += loss.item()
        pred = output.argmax(dim=1)
        correct += pred.eq(target).sum().item()
        total += target.size(0)

        if batch_idx % log_interval == 0:
            print(f"Train Epoch: {epoch} "
                  f"[{batch_idx * len(data)}/{len(dataloader.dataset)}] "
                  f"Loss: {loss.item():.4f} "
                  f"Acc: {100. * correct / total:.2f}%")

    return total_loss / len(dataloader), correct / total


@torch.no_grad()
def evaluate(model, dataloader, criterion, device):
    model.eval()
    total_loss = 0
    correct = 0

    for data, target in dataloader:
        data, target = data.to(device), target.to(device)
        output = model(data)
        loss = criterion(output, target)

        total_loss += loss.item()
        pred = output.argmax(dim=1)
        correct += pred.eq(target).sum().item()

    avg_loss = total_loss / len(dataloader)
    accuracy = correct / len(dataloader.dataset)
    return avg_loss, accuracy


# 主循环
def main():
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    model = MLP(784, 256, 10).to(device)
    optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
    criterion = nn.CrossEntropyLoss()

    train_loader = DataLoader(train_dataset, batch_size=64, shuffle=True)
    test_loader = DataLoader(test_dataset, batch_size=256)

    for epoch in range(1, 11):
        train_loss, train_acc = train_one_epoch(
            model, train_loader, optimizer, criterion, device, epoch
        )
        test_loss, test_acc = evaluate(
            model, test_loader, criterion, device
        )
        print(f"Epoch {epoch}: "
              f"Train Loss={train_loss:.4f} Acc={train_acc:.4f} | "
              f"Test Loss={test_loss:.4f} Acc={test_acc:.4f}")
最佳实践:
使用 @torch.no_grad() 装饰评估函数，避免构建推理时的计算图。训练/评估模式切换通过 model.train() / model.eval() 控制 BatchNorm 和 Dropout 行为。

torch.cuda.amp 混合精度训练

混合精度训练使用 float16 进行前向和反向传播，同时在关键归约操作中保留 float32 精度。AMP 可带来约 1.5-3 倍训练加速，且对模型精度影响极小。

from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for data, target in dataloader:
    data, target = data.to(device), target.to(device)

    optimizer.zero_grad()

    # 前向在 amp 上下文内
    with autocast(device_type="cuda"):
        output = model(data)
        loss = criterion(output, target)

    # 梯度缩放防止 underflow
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()

# 推理时也可使用
model.eval()
with torch.no_grad(), autocast():
    output = model(data)

torch.compile

# PyTorch 2.x 即时编译
model = MLP(784, 256, 10).to(device)
model = torch.compile(model, mode="default")
# mode: "default" / "reduce-overhead" / "max-autotune"

# 只编译推理
model = torch.compile(model, mode="reduce-overhead")
model.eval()
with torch.no_grad():
    output = model(data)
注意:
torch.compile 目前对动态形状和某些控制流支持有限。在第一次调用时会触发编译（预热），后续调用速度显著提升。CUDA >= 11.6 且新 GPU 架构（Ampere 及以上）效果最佳。

10 保存与加载

PyTorch 支持多种模型保存策略，从简单的参数字典到完整的检查点，满足不同场景需求。

保存 state_dict（推荐）

# 保存模型参数
torch.save(model.state_dict(), "model_weights.pth")

# 加载模型参数
model = MLP(784, 256, 10)   # 必须先创建模型实例
model.load_state_dict(torch.load("model_weights.pth"))
model.eval()

保存完整检查点

checkpoint = {
    "epoch": epoch,
    "model_state_dict": model.state_dict(),

    "loss": loss,
    "lr_scheduler": scheduler.state_dict(),
    "args": args,
}

torch.save(checkpoint, "checkpoint_epoch10.pt")

# 加载检查点

TorchScript 导出

# Tracing（推荐用于简单模型）

最佳实践:
训练过程中定期保存检查点（每 N 个 epoch）。保存时建议保留最佳验证精度对应的权重。使用 torch.save(model.module.state_dict(), ...) 处理 DataParallel 封装的模型。
