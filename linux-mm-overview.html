01 概述

Linux 内存管理是内核中最复杂的子系统之一（超过 20 万行代码）。本文全景梳理所有内存管理组件的关系，从虚拟地址空间到物理页面分配，从页面回收到 Swap。

03 内存分配流程

内核内存分配路径：

  kmalloc(size) — 小对象
        │
        ├── size < 8KB → SLUB 分配器
        │       kmem_cache_alloc()
        │       → 从 slab 缓存取对象
        │
        └── size ≥ 8KB → Buddy System
                __alloc_pages()
                → 从 free_area 取页面

  vmalloc(size) — 大块虚拟连续内存
        │
        └── 分散的物理页面 + 虚拟地址映射

  alloc_pages() — 直接页面分配
        │
        └── Buddy System → __alloc_pages()
                            → 从 free_area 取连续页面

用户态内存分配路径：

  malloc(size)
        │
        ├── size < 128KB → brk() 扩展堆
        │       mm/mmap.c → do_brk_flags()
        │
        └── size ≥ 128KB → mmap()
                mm/mmap.c → do_mmap()
                → 创建 VMA，延迟分配（缺页时分配物理页面）

04 性能特征

分配器对象大小延迟适用场景
kmalloc< 8KB~100ns内核小对象
vmalloc任意~1μs大块虚拟连续内存
alloc_pages≥ 4KB~200ns页面级分配
malloc任意~1μs用户态分配
mmap任意~100ns大文件/共享内存
