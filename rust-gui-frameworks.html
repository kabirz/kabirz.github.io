01 概述

Rust GUI 生态比成熟语言（C++ Qt / Swift UIKit / JavaFX）年轻，但迭代极快。五个主流框架各有特色：

框架范式渲染最佳场景
TauriWeb + NativeWebView混合应用
IcedElm (MVU)wgpu跨平台桌面
eguiImmediate Modewgpu / glow工具、调试
Slint声明式 DSLOpenGL/SK/Wayland嵌入式
DioxusReact (RSX)WebView / Tauri / wgpu跨端（web/desktop/mobile）

范式对比：

  • Retained Mode：构建 UI 树，框架维护状态（Qt、GTK、Iced、Slint）
  • Immediate Mode：每帧重画，无状态（egui）
  • Reactive：声明式 + 状态变化触发重渲染（React、Dioxus）
  • Hybrid：Rust 后端 + Web 前端（Tauri）

02 Tauri（混合架构）

Tauri 用系统 WebView 渲染 HTML/CSS/JS，Rust 作为后端。体积小、内存省（相比 Electron）。

// Cargo.toml + tauri.conf.json
// [dependencies]
// tauri = { version = "2", features = [] }

// src-tauri/src/main.rs
#[tauri::command]
fn greet(name: &str) -> String {
    format!("Hello, {}!", name)
}

fn main() {
    tauri::Builder::default()
        .invoke_handler(tauri::generate_handler![greet])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}

// 前端（React/Vue/Svelte/Vanilla）
// app/index.html:
// <button onclick="greet()">Greet</button>
// <script>
//   import { invoke } from '@tauri-apps/api';
//   function greet() {
//     invoke('greet', { name: 'World' }).then(console.log);
//   }
// </script>

核心特性：

  • 体积小：~5MB（Electron 100MB+）
  • 内存省：~30MB（Electron 200MB+）
  • 跨平台：Windows / macOS / Linux / Mobile（2.0+）
  • 前端自由：React、Vue、Svelte、Vanilla
  • 安全：白名单 IPC，文件系统/网络可控
  • Plugin 丰富：通知、对话框、剪贴板、文件系统、shell

03 Iced（Elm 风格）

Iced 是The Elm Architecture (MVU) 的 Rust 实现。纯 Rust，无 JavaScript。

// [dependencies]
// iced = "0.12"

use iced::widget::{button, column, text};
use iced::{Center, Element};

fn main() -> iced::Result {
    iced::run("Counter", App::update, App::view)
}

#[derive(Default)]
struct App {
    counter: i32,
}

#[derive(Debug, Clone, Copy)]
enum Message {
    IncrementPressed,
    DecrementPressed,
}

impl App {
    fn update(&mut self, message: Message) {
        match message {
            Message::IncrementPressed => self.counter += 1,
            Message::DecrementPressed => self.counter -= 1,
        }
    }
    
    fn view(&self) -> Element<Message> {
        column![
            button("Increment").on_press(Message::IncrementPressed),
            text(self.counter).size(50),
            button("Decrement").on_press(Message::DecrementPressed),
        ]
        .padding(20)
        .align_x(Center)
        .into()
    }
}

核心特性：

  • MVU 架构：Model + View + Update
  • 响应式：状态变化自动重渲染
  • wgpu 渲染：跨平台、GPU 加速
  • 类型安全：Message 是 enum，编译期检查
  • 无运行时：完全 Rust
  • 主题支持：内置 + 自定义
  • 跨平台：桌面 + Web（experimental）

典型应用：

// 复杂状态：
struct AppState {
    todos: Vec<Todo>,
    filter: Filter,
    input: String,
}

enum Message {
    AddTodo,
    ToggleTodo(usize),
    DeleteTodo(usize),
    FilterChanged(Filter),
    InputChanged(String),
}

fn update(state: &mut AppState, msg: Message) -> Task<Message> {
    match msg {
        Message::AddTodo => {
            if !state.input.is_empty() {
                state.todos.push(Todo::new(state.input.clone()));
                state.input.clear();
            }
        }
        // ...
    }
    Task::none()
}

// 异步（Task）：
enum Message {
    DataLoaded(Result<Data, Error>),
}

fn load_data() -> Task<Message> {
    Task::perform(async {
        fetch_data().await
    }, Message::DataLoaded)
}

04 egui（Immediate Mode）

egui 是立即模式 GUI，每帧重新绘制所有 UI。极简、极快、极适合工具应用。

// [dependencies]
// eframe = "0.27"

use eframe::egui;

fn main() -> eframe::Result {
    eframe::run_simple_native("My App", Default::default(), |ctx, _| {
        egui::CentralPanel::default().show(ctx, |ui| {
            ui.heading("Hello");
            if ui.button("Click").clicked() {
                println!("clicked");
            }
            ui.label("Status: ok");
        });
    })
}

// 带状态：
struct App { count: i32 }

impl eframe::App for App {
    fn update(&mut self, ctx: &egui::Context, _: &mut eframe::Frame) {
        egui::CentralPanel::default().show(ctx, |ui| {
            ui.label(format!("Count: {}", self.count));
            if ui.button("+1").clicked() {
                self.count += 1;
            }
        });
    }
}

fn main() {
    eframe::run_native("app", Default::default(), Box::new(|_| Box::new(App { count: 0 }))).unwrap();
}

核心特性：

  • 无状态：每帧重新绘制
  • 极简 API：函数式调用
  • 超快：60FPS + 极低 CPU
  • 调试友好：每帧重新计算
  • 跨平台：Web、桌面、VR
  • 跨框架集成：bevy、wgpu、glium、SDL

典型用例：

  • 调试工具
  • 数据可视化
  • 游戏内 UI
  • 配置面板
  • 开发工具（如 Rust Explorer）

高级控件：

ui.text_edit_singleline(&mut self.name);
ui.checkbox(&mut self.option, "Enable");
ui.slider(&mut self.value, 0..=100);
ui.color_picker_button_rgb(&mut self.color);

// 绘图：
egui::Plot::new("plot")
    .show(ui, |plot_ui| {
        plot_ui.line(Line::new(PlotPoints::from_explicit_function(
            |x| x.sin(),
            0.0..std::f32::consts::TAU,
            100,
        )));
    });

// 自定义绘制：
let (rect, _) = ui.allocate_exact_size(
    Vec2::new(100.0, 100.0),
    egui::Sense::click(),
);
ui.painter().rect_filled(rect, 5.0, egui::Color32::RED);

05 Slint（声明式 DSL）

Slint 用声明式 DSL（类似 QML）描述 UI，Rust 作为后端逻辑。适合嵌入式。

// main.rs
slint::slint! {
    export component App inherits Window {
        in property <int> counter: 0;
        callback increment;
        
        VerticalBox {
            Text { text: "Counter: \{root.counter}"; }
            Button {
                text: "Add";
                clicked => { root.counter += 1; }
            }
        }
    }
}

fn main() {
    let app = App::new().unwrap();
    app.run().unwrap();
}

外部 .slint 文件：

// main.rs
slint::include_modules!();

fn main() {
    let app = App::new().unwrap();
    app.on_increment({
        let app_handle = app.as_weak();
        move || {
            let app = app_handle.unwrap();
            app.set_counter(app.get_counter() + 1);
        }
    });
    app.run().unwrap();
}

// app.slint (separate file):
// export component App inherits Window { ... }

核心特性：

  • DSL 语言：QML 风格，编译器校验
  • 多渲染后端：OpenGL、Skia、Software、Qt
  • 嵌入式友好：极小二进制、低内存
  • 语言绑定：Rust、C++、JavaScript、Python
  • 可视化编辑器：Slint Studio
  • 商业许可：GPLv3 + 商业双授权

06 Dioxus（React 风格）

Dioxus 是React for Rust。RSX 宏、Virtual DOM、Hooks。已有专门文档（rust-dioxus.html）。

// [dependencies]
// dioxus = { version = "0.5", features = ["desktop"] }

use dioxus::prelude::*;

fn main() {
    dioxus::launch(app);
}

fn app() -> Element {
    let mut count = use_signal(|| 0);
    
    rsx! {
        div {
            h1 { "Counter: {count}" }
            button { onclick: move |_| count += 1, "Increment" }
            button { onclick: move |_| count -= 1, "Decrement" }
        }
    }
}

核心特性：

  • React 习惯：use_signal、use_resource、use_effect
  • RSX 宏：类似 JSX，但 Rust 原生
  • 多 renderer：Desktop（WebView/wgpu）、Mobile、Web、SSR、TUI、LiveView
  • 跨端：一套代码多端运行
  • 真 Virtual DOM：diff + patch

更多内容参考 Dioxus 深度文档。

10 选型指南

决策树：

需要现代 UI？
├─ Web 技术栈 → Tauri
│       - 前端团队熟悉
│       - 跨端一致
│       - 复杂富文本（编辑器、富 HTML）
│
├─ React 习惯 → Dioxus
│       - 跨端一套码
│       - 喜欢声明式 + Hooks
│
├─ 纯 Rust + 复杂应用 → Iced
│       - MVU 干净
│       - 无 JS 依赖
│       - 桌面应用
│
├─ 工具 / 调试 → egui
│       - 立即模式简单
│       - 性能极高
│       - 不追求精致 UI
│
├─ 嵌入式 / 资源受限 → Slint
│       - 极小体积
│       - 多后端
│       - 注意商业授权
│
└─ 游戏 UI → egui（调试）+ 自定义（实际）

GUI 生态总结:
1. Tauri：生产成熟，前端友好，最大生态
2. Iced：纯 Rust，Elm 风格，准生产
3. egui：极简极快，工具应用首选
4. Slint：嵌入式优化，DSL 学习
5. Dioxus：React 习惯，跨端一套码
Rust GUI 仍在快速发展，没有"Qt"那样的统一王者，但每个框架都很优秀。
