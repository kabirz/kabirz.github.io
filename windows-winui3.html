01 概述

WinUI 3（Windows UI Library 3）是微软新一代 Windows 原生 UI 框架，它把原本绑定在 Windows 系统中的 UWP XAML 框架剥离出来，作为 Windows App SDK（前身 Project Reunion）的一部分随应用一起分发。它统一了 Win32 桌面应用与 UWP 的开发体验，是微软当前主推的 Windows 客户端 UI 技术栈。

Windows App SDK 的核心使命是解耦：让开发者不再受限于目标 Windows 版本——只要应用携带了 Windows App SDK 运行时，就能在 Windows 10 1809 及以上版本获得一致的现代 API 与控件，而不必等待系统更新。WinUI 3 则是这个 SDK 中负责 XAML UI 层的那一块。

核心特性
  • 系统解耦：XAML 不再随 OS 更新，由 NuGet 包分发
  • 完整 Win32 能力：桌面应用无沙箱，可调用任意 API
  • 现代控件：Mica/Acrylic、InfoBar、NavigationView 等
  • 高性能绑定：{x:Bind} 编译时绑定
  • 双语言一等公民：C# 与 C++/WinRT 均原生支持
  • 多部署模式：MSIX 打包 / 未打包 / 自包含
适用场景
场景适用度
新建 Windows 原生桌面应用强烈推荐
UWP 应用现代化首选迁移目标
需要 Win32 + 现代 UI 混合极佳
WPF 应用升级界面可考虑（XAML 差异较大）
跨平台需求不适用（仅 Windows）
一句话定位：WinUI 3 是「能调用一切 Win32 API 的 UWP XAML」——它既保留了 UWP 现代控件与设计语言，又解除了 UWP 沙箱的一切限制。

02 架构

理解 WinUI 3 必须先理解它「从哪里来」。Windows 10 时代的 UWP 应用使用 Windows.UI.Xaml 命名空间——这套 XAML 引擎编译进 Windows 系统本身（Windows.UI.Xaml.dll），只能通过系统更新演进。WinUI 3 把整套 XAML 引擎、控件、合成（Composition）层从系统中抽取出来，重命名为 Microsoft.UI.Xaml，作为 NuGet 包随应用分发。

新旧命名空间对比
UWP（旧）WinUI 3（新）
Windows.UI.XamlMicrosoft.UI.Xaml
Windows.UI.Xaml.ControlsMicrosoft.UI.Xaml.Controls
Windows.UI.Xaml.DataMicrosoft.UI.Xaml.Data
Windows.UI.CompositionMicrosoft.UI.Composition
Windows.UI.Xaml.MediaMicrosoft.UI.Xaml.Media
随 OS 更新随 App 的 NuGet 包
核心组件
  • XAML 引擎：解析、布局、渲染（DirectX/Direct2D）
  • 控件库：Button/ListView/NavigationView 等
  • Composition：可视化层、动画、效果
  • 语言投影：C# 投影 / C++/WinRT
  • WinAppSDK 运行时：引导、应用生命周期、窗口
  • WinRT 类型系统：底层 COM 互操作基础

应用模型对比

// ── UWP 应用（沙箱，受限） ──
// 在 AppContainer 中运行，无法直接调用大量 Win32 API
// 只能通过 UWP API 子集 + 受限能力声明
sealed partial class App : Windows.UI.Xaml.Application { }

// ── WinUI 3 桌面应用（无沙箱） ──
// 普通 Win32 进程，拥有全部 Win32/COM/.NET 能力
// 同时享受完整的现代 XAML UI
public partial class App : Microsoft.UI.Xaml.Application { }

C++/WinRT 与 C# 投影

WinUI 3 的底层是WinRT 类型系统（同 UWP）。微软提供两种「语言投影」（不是绑定，是代码生成的一等公民接口）：

语言投影特点
C# / .NETC# WinRT 投影（CsWinRT）开发效率高，自然异步语法，主流选择
C++C++/WinRT（头文件投影）零运行时开销，原生性能，可深入系统
Rustwindows-rs（社区）实验性，非官方一等支持
关键事实：WinUI 3 的「桌面应用」（Desktop app）模型就是普通 Win32 EXE——它没有 AppContainer 沙箱，可以调用 CreateFile、RegCreateKey、原生 COM、P/Invoke 任意 Win32 API，同时也保留了 UWP 风格的 XAML UI。这是它相对于 UWP 的决定性自由。

03 应用模型

WinUI 3 应用的入口是 App 类，它继承自 Microsoft.UI.Xaml.Application。Application 派生类负责应用级资源、启动行为、未处理异常。顶级窗口由 Microsoft.UI.Xaml.Window 表示——它比 UWP 的 Window 更强，原生支持多窗口、自定义标题栏、系统菜单等。

// App.xaml.cs —— 应用入口
public partial class App : Application
{
    private MainWindow? m_window;

    protected override void OnLaunched(LaunchActivatedEventArgs args)
    {
        // 创建主窗口并显示。Window 是顶级容器
        m_window = new MainWindow();
        m_window.Activate();          // 激活（不是 Show）
    }
}

// App.xaml —— 应用级资源与 XAML 入口
<!-- <Application root.> -->
<Application
    x:Class="MyApp.App"
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Application.Resources>
        <ResourceDictionary>
            <x:String x:Key="AppName">My App</x:String>
        </ResourceDictionary>
    </Application.Resources>
</Application>

Window 与 AppWindow

WinUI 3 提供两个协同的窗口抽象：Microsoft.UI.Xaml.Window（XAML 树宿主）与 Microsoft.UI.Windowing.AppWindow（WinAppSDK 提供的原生窗口控制器）。二者通过 WindowNative 桥接，覆盖从 UI 内容到窗口外壳的所有需求。

抽象来源职责
Microsoft.UI.Xaml.WindowWinUI 3XAML 内容树宿主，Content 属性
Microsoft.UI.Windowing.AppWindowWinAppSDK标题栏、尺寸、系统菜单、呈现器
WinRT.Interop.WindowNative互操作取底层 HWND，调用 Win32 API
// MainWindow.xaml.cs —— 桌面窗口 + Mica 背景
public sealed partial class MainWindow : Window
{
    public MainWindow()
    {
        InitializeComponent();

        // 1. 拿到 AppWindow（WinAppSDK 的窗口控制器）
        var hwnd = WinRT.Interop.WindowNative.GetWindowHandle(this);
        var windowId = Win32Interop.GetWindowIdFromWindow(hwnd);
        AppWindow appWindow = AppWindow.GetFromWindowId(windowId);

        // 2. 自定义标题栏（延伸内容区到标题栏）
        var titleBar = appWindow.TitleBar;
        titleBar.ExtendsContentIntoTitleBar = true;

        // 3. 启用 Mica 云母背景材质
        SystemBackdrop = new MicaBackdrop();
    }
}
持有窗口引用：OnLaunched 中必须把 Window 存到字段（如 m_window），否则局部变量离开作用域后窗口对象可能被 GC 回收——表现是窗口「凭空消失」。这是 WinUI 3 初学者最常见的坑之一。

04 XAML 控件与布局

WinUI 3 的控件库与 UWP 同源（毕竟从 UWP XAML 剥离而来），命名空间从 Windows.UI.Xaml.Controls 改为 Microsoft.UI.Xaml.Controls。控件语义、属性、事件几乎一一对应，但 WinUI 3 还独占了一批更新的控件与 API（如 InfoBar、PipsPager、ScrollView 新版）。

布局容器

核心布局面板
  • Grid：行列网格（最常用）
  • StackPanel：横向/纵向堆叠
  • RelativePanel：相对定位
  • Canvas：绝对坐标
  • WrapGrid/ItemsWrapGrid：列表虚拟化
  • ScrollViewer：滚动容器
与 WPF/UWP 的差异

// NavigationView —— 现代导航
<NavigationView SelectionChanged="Nav_OnSelectionChanged">
    <NavigationView.MenuItems>
        <NavigationViewItem Icon="Home" Tag="home" Content="首页" />
        <NavigationViewItem Icon="Setting" Tag="settings" Content="设置" />
    </NavigationView.MenuItems>

    <Frame x:Name="ContentFrame" />
</NavigationView>

// 后台代码：根据 Tag 导航
private void Nav_OnSelectionChanged(object sender, NavigationViewSelectionChangedEventArgs e)
{
    if (e.SelectedItemContainer is NavigationViewItem item)
    {
        Type? page = item.Tag.ToString() switch {
            "home"     => typeof(HomePage),
            "settings" => typeof(SettingsPage),
            _ => null
        };
        if (page != null) ContentFrame.Navigate(page);
    }
}

05 数据绑定

WinUI 3 的数据绑定是它的杀手锏之一。除了经典的 {Binding}（运行时反射，与 WPF/UWP 一致），它独有 {x:Bind}——编译时绑定，由 XAML 编译器生成强类型的绑定代码，性能比反射绑定高一个数量级，且能在编译期检测类型错误。

{x:Bind} 编译绑定
  • 编译时生成代码，无运行时反射
  • 强类型，编译期类型检查
  • 支持函数/方法绑定（直接绑到方法）
  • 支持事件绑定（事件 → 方法）
  • 默认 Mode=OneTime（性能优先）
  • 大数据列表性能最佳
{Binding} 经典绑定
  • 运行时反射，灵活但慢
  • 依赖 DataContext（隐式数据上下文）
  • 支持 RelativeSource（找祖先/模板父）
  • 跨视图模型、动态数据场景
  • 与 WPF/UWP 习惯一致
// 数据源：实现 INotifyPropertyChanged
public class UserViewModel : INotifyPropertyChanged
{
    private string _name = "";
    public string Name
    {
        get => _name;
        set { if (_name != value) { _name = value; OnPropertyChanged(); } }
    }

    public event PropertyChangedEventHandler? PropertyChanged;
    protected void OnPropertyChanged([CallerMemberName] string? p = null)
        => PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(p));
}

// —— XAML：{x:Bind} 与 {Binding} 对比 ——
<Page x:Class="MyApp.UserPage" />
    <Grid>
        <<!-- x:Bind：编译期强类型，绑定到页面的属性 -->
        <TextBlock Text="{x:Bind CurrentUser.Name, Mode=OneWay}" />

        <<!-- x:Bind 函数绑定：直接调用方法 -->
        <TextBlock Text="{x:Bind FormatName(CurrentUser.Name), Mode=OneWay}" />

        <<!-- x:Bind 事件绑定：把按钮点击直接绑到方法 -->
        <Button Content="保存" Click="{x:Bind SaveUser}" />

        <<!-- Binding：运行时反射，依赖 DataContext -->
        <TextBox Text="{Binding Name, Mode=TwoWay, UpdateSourceTrigger=PropertyChanged}" />
    </Grid>
</Page>

// 页面代码隐藏：x:Bind 暴露的属性必须是 public 字段或属性
public sealed partial class UserPage : Page
{
    public UserViewModel CurrentUser { get; } = new();

    public UserPage() { InitializeComponent(); }

    public string FormatName(string n) => $"用户：{n}";

    public void SaveUser() { /* 保存逻辑 */ }
}

大数据列表：x:Phase 分阶段渲染

对于超大列表（成千上万项），WinUI 3 提供 x:Phase——XAML 编译器分阶段渲染每个数据模板，先渲染 Phase 0（最关键字段），滚动空闲时再补渲染后续 Phase，保证滚动帧率。

<ListView ItemsSource="{x:Bind LargeItems}">
    <ListView.ItemTemplate>
        <DataTemplate x:DataType="local:Item">
            <StackPanel>
                <!-- Phase 0：最先渲染，保证滚动可见 -->
                <TextBlock Text="{x:Bind Title}" />
                <!-- Phase 1：滚动空闲时补 -->
                <TextBlock Text="{x:Bind Subtitle}" x:Phase="1" />
                <!-- Phase 2：再延迟一档 -->
                <Image Source="{x:Bind Thumbnail}" x:Phase="2" />
            </StackPanel>
        </DataTemplate>
    </ListView.ItemTemplate>
</ListView>

ObservableCollection 与 IValueConverter

// ObservableCollection：列表变化自动通知 UI
public ObservableCollection<Item> Items { get; } = new();
Items.Add(new Item());      // UI 自动刷新

// IValueConverter：值转换器（绑定时转换类型）
public class BoolToVisibilityConverter : IValueConverter
{
    public object Convert(object v, Type t, object p, string l)
        => (bool)v ? Visibility.Visible : Visibility.Collapsed;

    public object ConvertBack(object v, Type t, object p, string l)
        => throw new NotImplementedException();
}

<Page.Resources>
    <local:BoolToVisibilityConverter x:Key="BoolToVis" />
</Page.Resources>
<Button Visibility="{x:Bind IsAdmin, Converter={StaticResource BoolToVis}}" />
x:Bind 默认 OneTime：与 {Binding} 默认 OneWay 不同，{x:Bind} 默认是 OneTime（一次性绑定，不监听变化）。需要双向或单向更新必须显式写 Mode=OneWay 或 Mode=TwoWay，否则数据变了 UI 不更新。

06 现代外观与材质

WinUI 3 把 Windows 11 的 Fluent Design 设计语言作为默认外观。两大招牌材质是 Mica（云母）与 Acrylic（亚克力），它们通过 Composition API 与系统桌面背景实时混合，呈现半透明的现代质感。

Mica 云母背景
  • 主窗口背景，与桌面壁纸混合
  • 不透明但有色调，节省 GPU
  • 窗口失去焦点时变暗
  • 仅窗口本身可用，非浮层
  • SystemBackdrop = new MicaBackdrop()
Acrylic 亚克力材质
  • 半透明模糊，背后内容虚化
  • 用于浮层（Flyout、菜单、命令栏）
  • 性能开销大于 Mica
  • 支持自定义着色与不透明度
  • AcrylicBrush 资源
// Mica 窗口背景（C# 代码，最简方式）
public MainWindow()
{
    InitializeComponent();
    SystemBackdrop = new MicaBackdrop {
        Kind = MicaKind.BaseAlt     // 三档：Base / BaseAlt / Dim
    };
}

// Acrylic 背景（XAML 资源 + 引用）
<Window.Resources>
    <AcrylicBrush x:Key="CustomAcrylic"
                  TintColor="#1F1F1F"
                  TintOpacity=0.7"
                  BackgroundSource="HostBackdrop" />
</Window.Resources>

<Border Background="{StaticResource CustomAcrylic}"
        CornerRadius="8" />

主题跟随系统

WinUI 3 内置亮/暗主题切换，通过 RequestedTheme 控制。配合 ThemeResource，所有控件自动响应主题变化。建议让应用跟随系统主题，使用 UISettings 监听系统颜色变化。

// 根元素指定主题（Application.Resources 或 root）
<Application.Resources>
    <ResourceDictionary>
        <ResourceDictionary.ThemeDictionaries>
            <ResourceDictionary x:Key="Light">
                <Color x:Key="MyBg">#FFFFFFFF</Color>
            </ResourceDictionary>
            <ResourceDictionary x:Key="Dark">
                <Color x:Key="MyBg">#FF1F1F1F</Color>
            </ResourceDictionary>
        </ResourceDictionary.ThemeDictionaries>
    </ResourceDictionary>
</Application.Resources>

// 引用时用 ThemeResource（主题切换自动更新）
<Grid Background="{ThemeResource MyBg}" />

自定义标题栏

WinUI 3 允许把内容区延伸到标题栏区域，画上自己的标题栏控件。配合 SetTitleBar 注册某个元素为「可拖拽区域」，系统按钮（最小化/关闭）依然由系统绘制保证行为正确。

<Grid>
    <Grid.RowDefinitions>
        <RowDefinition Height="32" />   <!-- 标题栏区域 -->
        <RowDefinition Height="*" />     <!-- 主内容 -->
    </Grid.RowDefinitions>

    <Grid x:Name="AppTitleBar" Height="32">
        <TextBlock Text="My App" VerticalAlignment="Center"
                   Margin="16,0,0,0" />
    </Grid>
</Grid>

// 代码：注册自定义标题栏 + 启用延伸
AppWindow.TitleBar.ExtendsContentIntoTitleBar = true;
SetTitleBar(AppTitleBar);          // 注册为可拖拽区域

ConnectedAnimation 连接动画

页面跳转时把元素从源位置「连贯地」动画到目标位置，是 Fluent 设计的招牌效果。WinUI 3 通过 ConnectedAnimationService 实现。

// 源页面：准备动画
ConnectedAnimationService.GetForCurrentView()
    .PrepareToAnimate("itemImage", sourceImage);

// 目标页面：播放动画（连贯过渡）
var anim = ConnectedAnimationService.GetForCurrentView()
              .GetAnimation("itemImage");
await targetImage.TryStartConnectedAnimationAsync(anim);
Reveal 高亮：鼠标悬停时 ListView 项边缘出现的高亮光效，是 Fluent 设计的另一招牌，WinUI 3 内置支持，无需额外代码。

07 互操作

互操作是 WinUI 3 桌面应用的灵魂。由于运行在普通 Win32 进程中（非 AppContainer），它能调用一切 Win32/COM/.NET API。同时微软提供了完整的桥接层，让 WinUI 3 XAML 能与 WPF/WinForms/MFC 等老技术混合使用。

场景API说明
取底层 HWNDWindowNative.GetWindowHandleWinUI 窗口转 HWND
调用 Win32 APIP/Invoke (DllImport)无限制，原生能力
调用 COM 组件[ComImport] / CsWinRTOffice、Shell、DXGI 等
Win32 → XAMLXAML Islands在 MFC/WinForms/Win32 中嵌入 XAML
WPF/WinForms 混用ElementCompositionPreviewComposition 互操作
跨进程通信WinRT + COMWinRT Broker、AppService
// 取 WinUI 3 窗口的 HWND，调用任意 Win32 API
using System.Runtime.InteropServices;

public sealed partial class MainWindow : Window
{
    [DllImport("user32.dll")]
    private static extern bool SetForegroundWindow(IntPtr hWnd);

    public MainWindow()
    {
        InitializeComponent();

        IntPtr hwnd = WinRT.Interop.WindowNative.GetWindowHandle(this);
        SetForegroundWindow(hwnd);    // 调用经典 Win32 API
    }
}
这是关键自由：UWP 沙箱应用调用 SetForegroundWindow、CreateFile 写任意路径、读注册表 HKLM、加载任意 DLL——这些要么被禁要么受限。WinUI 3 桌面应用全部畅通，这就是它「桌面」身份的全部意义。

XAML Islands（反向场景）

除了「WinUI 3 应用调用 Win32」，还有一种反向场景：已有 Win32/MFC/WinForms/WPF 应用想在自己的窗口里嵌入一块 WinUI XAML 控件——这就是 XAML Islands。它用 WindowsXamlManager + DesktopWindowXamlSource 把 XAML 内容树托管到任意 HWND。

// 在 WPF/WinForms/MFC 中承载 WinUI XAML（伪代码，C#）
var xamlSource = new DesktopWindowXamlSource();
xamlSource.Initialize(hostHwnd);           // 宿主窗口 HWND

// 把 XAML 控件填进去
xamlSource.Content = new MyWinUIUserControl();
选型建议：新项目优先「WinUI 3 桌面应用」；只有当巨型遗留 WPF/MFC 项目只想局部现代化、不想重写外壳时，才用 XAML Islands 嵌入。

08 打包与部署

WinUI 3 桌面应用支持两种部署模式：MSIX 打包（推荐，现代部署）与未打包（Unpackaged，传统 EXE）。两种模式都能用全部 Win32 能力，区别在于安装方式、生命周期管理、自动更新支持。

MSIX 打包（Packaged）
  • 现代 Windows 安装格式
  • 干净安装/卸载（系统跟踪文件）
  • 支持 App Installer 自动更新
  • 可发布到 Microsoft Store
  • 声明式能力（Package.appxmanifest）
  • 支持 MSIX 单包或架构包
  • 需 Windows App SDK 框架包
未打包（Unpackaged）
  • 普通 EXE + DLL，绿色部署
  • 无需 MSIX，老系统友好
  • 必须用 bootstrap 初始化 WinAppSDK
  • 无自动更新，需自行实现
  • 不能上 Store
  • 调试简单，单文件发布容易
// 未打包应用：必须调用 bootstrap 初始化 Windows App SDK 运行时
using Microsoft.Windows.ApplicationModel.WindowsAppRuntime;

public static class Program
{
    [STAThread]
    public static void Main(string[] args)
    {
        // 1. 引导：定位并加载 Windows App SDK 运行时
        Bootstrap.Initialize(0x00010006);   // 版本号 1.6

        // 2. 启动 WinUI 应用
        WinRT.ComWrappersSupport.InitializeComWrappers();
        Application.Start(p => {
            var context = new DispatchContext();
            context.Run(() => new App());
        });
    }
}

09 迁移指南

从已有项目迁到 WinUI 3 的难度因来源而异：UWP 迁移相对容易（概念一致、API 改名），WPF 迁移较难（XAML 差异大、需重写）。

从 UWP 迁移
  • 命名空间 Windows.UI.Xaml → Microsoft.UI.Xaml
  • 移除 AppContainer 沙箱（变普通 Win32）
  • 清单改 Package.appxmanifest
  • 部分 UWP 专属 API 需适配（如 FileOpenPicker）
  • 用 .NET Upgrade Assistant 工具半自动迁移
  • 多数 XAML/控件代码可机械替换
从 WPF 迁移
  • 概念相近（XAML + 数据绑定）
  • 但 API 实质不同，需重写 XAML
  • Window/Button 等同名但命名空间不同
  • 绑定默认行为差异（OneTime vs OneWay）
  • 无 Margin 三元简写
  • 主题资源用 ThemeResource
来源迁移成本建议时机
UWP低（机械替换为主）立即迁，微软已停止 UWP 演进
WPF高（XAML 重写）仅在新项目或大版本重构时
WinForms极高（范式不同）不推荐，可考虑 XAML Islands 渐进
MFC/Win32极高新模块用 XAML Islands 嵌入
// UWP 文件选择器（沙箱专用，需适配）
var picker = new FileOpenPicker();
// UWP：picker.PickSingleFileAsync(); 直接调用

// WinUI 3 桌面：必须指定关联的 HWND（无沙箱的副作用）
IntPtr hwnd = WinRT.Interop.WindowNative.GetWindowHandle(this);
WinRT.Interop.InitializeWithWindow.Initialize(picker, hwnd);
var file = await picker.PickSingleFileAsync();
UWP 迁移注意：UWP 的部分 API（如 FileOpenPicker、DataPackage）在 WinUI 3 桌面模型下需要显式传入 HWND——因为桌面应用没有 UWP 的「应用视图」概念。这类 API 不适配会抛 InvalidCastException 或 COMException。
UWP 的未来：微软已明确不再为 UWP 添加新特性，推荐所有 UWP 项目迁移到 WinUI 3 / Windows App SDK。新建项目应直接选 WinUI 3，存量 UWP 应尽早规划迁移。

10 完整示例

下面是一个最小可运行的 WinUI 3 桌面应用骨架：Mica 背景、{x:Bind} 数据绑定、ContentDialog 确认对话框。展示各组件如何协作。

// ── App.xaml.cs ── 应用入口
public partial class App : Application
{
    private MainWindow? m_window;

    protected override void OnLaunched(LaunchActivatedEventArgs args)
    {
        m_window = new MainWindow();   // 必须持有引用，防 GC
        m_window.Activate();
    }
}

// ── ViewModels/MainViewModel.cs ── 数据 + 命令
public class MainViewModel : INotifyPropertyChanged
{
    private string _name = "World";
    public string Name
    {
        get => _name;
        set { if (_name != value) { _name = value; OnPropertyChanged(); } }
    }

    public IAsyncCommand GreetCommand { get; }

    public MainViewModel()
    {
        GreetCommand = new RelayCommand(async () => {
            var dlg = new ContentDialog {
                XamlRoot = App.MainWindow!.Content.XamlRoot,
                Title = "问候",
                Content = $"你好，{Name}！",
                CloseButtonText = "确定"
            };
            await dlg.ShowAsync();
        });
    }

    public event PropertyChangedEventHandler? PropertyChanged;
    private void OnPropertyChanged([CallerMemberName] string? p = null)
        => PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(p));
}
<!-- ── MainWindow.xaml ── Mica 背景 + x:Bind ──
<Window
    x:Class="MyApp.MainWindow"
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
    xmlns:vm="using:MyApp.ViewModels">

    <Grid>
        <StackPanel VerticalAlignment="Center"
                    HorizontalAlignment="Center"
                    Spacing="16"
                    Padding="48">

            <TextBlock Text="WinUI 3 示例"
                       FontSize="32" FontWeight="Bold" />

            <!-- x:Bind 双向绑定到 ViewModel.Name -->
            <TextBox Header="你的名字"
                     Text="{x:Bind ViewModel.Name, Mode=TwoWay, UpdateSourceTrigger=PropertyChanged}"
                     MinWidth="240" />

            <!-- x:Bind 事件绑定到命令（通过方法包装） -->
            <Button Content="打招呼"
                    Click="{x:Bind GreetClick}"
                    HorizontalAlignment="Center"
                    Style="{StaticResource AccentButtonStyle}" />
        </StackPanel>
    </Grid>
</Window>
// ── MainWindow.xaml.cs ── 代码隐藏
public sealed partial class MainWindow : Window
{
    public MainViewModel ViewModel { get; } = new();

    public MainWindow()
    {
        InitializeComponent();
        SystemBackdrop = new MicaBackdrop();   // Mica 云母背景
        Title = "My WinUI App";
    }

    public async void GreetClick()
    {
        await ViewModel.GreetCommand.ExecuteAsync();
    }
}
运行要点：新建项目时选「Packaged (WinUI 3 in Desktop)」模板会自动配置好 Package.appxmanifest、Windows App SDK 引用、App.xaml。直接按 F5 即可启动调试。生产部署再决定 MSIX 还是未打包。
