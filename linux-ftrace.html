01 概述

ftrace（Function Tracer）是 Linux 内核的动态函数跟踪框架。它可以在运行时跟踪内核函数调用、中断处理、软中断、调度器事件等。ftrace 是 trace-cmd、perf ftrace、函数图跟踪等工具的底层基础。

02 核心功能

功能跟踪器说明
函数跟踪function记录所有函数调用
函数图跟踪function_graph记录函数调用和返回时间
中断跟踪irqsoff跟踪中断禁用时间
调度延迟wakeup跟踪进程唤醒延迟
栈跟踪stacktrace记录内核栈
事件跟踪tracepoint静态跟踪点
ftrace 工作原理：

  启用函数跟踪时：
  内核在每个被跟踪函数入口插入 mcount/fentry 调用
  → mcount 调用 ftrace 的回调函数
  → 回调函数记录函数名、时间、参数

  关闭时：
  恢复为 NOP 指令 → 零开销

03 使用方式

# 函数跟踪
$ echo function > /sys/kernel/debug/tracing/current_tracer
$ echo 1 > /sys/kernel/debug/tracing/tracing_on
$ cat /sys/kernel/debug/tracing/trace_pipe
  # 实时输出函数调用

# 只跟踪特定函数
$ echo tcp_sendmsg > /sys/kernel/debug/tracing/set_ftrace_filter
$ echo 1 > /sys/kernel/debug/tracing/tracing_on

# 函数图跟踪（带返回值和时间）
$ echo function_graph > /sys/kernel/debug/tracing/current_tracer
$ echo tcp_sendmsg > /sys/kernel/debug/tracing/set_graph_function
$ echo 1 > /sys/kernel/debug/tracing/tracing_on

# trace-cmd — 更友好的 ftrace 前端
$ trace-cmd record -p function_graph -g tcp_sendmsg sleep 5
$ trace-cmd report

# kprobe — 动态跟踪点
$ echo 'p:myprobe tcp_sendmsg len=%x2' > /sys/kernel/debug/tracing/kprobe_events
$ echo 1 > /sys/kernel/debug/tracing/events/kprobes/myprobe/enable
