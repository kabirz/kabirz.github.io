01 概述

mmap（memory map）是 Linux 中将文件或设备映射到进程地址空间的系统调用。它使得进程可以像访问内存一样访问文件内容，实现零拷贝 I/O。mmap 是众多高性能技术的基石：共享内存（IPC）、TCP zero-copy send、pipe splice、DPDK、XDP 等都依赖它。

mmap 核心能力
  • 文件映射：直接读写文件内容（页缓存共享）
  • 匿名映射：分配匿名内存（类似 malloc）
  • 共享映射：多进程共享同一内存区域
  • 私有映射：写时复制（COW）
  • 设备映射：映射设备 I/O（如显卡显存）
  • MAP_HUGETLB：大页映射

02 映射类型

MAP_SHARED（共享映射）：

  进程 A 和进程 B 映射同一文件
  进程 A 写入 → 页面修改 → 进程 B 看到修改
  修改最终写回文件

  ┌──────────┐          ┌──────────┐
  │ 进程 A    │          │ 进程 B    │
  │ mmap     │          │ mmap     │
  └────┬─────┘          └────┬─────┘
       │                     │
       ↓                     ↓
  ┌─────────────────────────────────┐
  │    共享页（同一物理页面）        │
  │    物理页面 refcount = 2        │
  └─────────────────────────────────┘
       │
       ↓
  ┌──────────┐
  │ 文件     │  （最终写回）
  └──────────┘

MAP_PRIVATE（私有映射 / 写时复制）：

  进程 A 和进程 B 映射同一文件
  初始时共享只读页面
  进程 A 写入 → 触发 COW → 进程 A 获得私有副本
  进程 B 看不到 A 的修改

  ┌──────────┐          ┌──────────┐
  │ 进程 A    │          │ 进程 B    │
  │ mmap     │          │ mmap     │
  └────┬─────┘          └────┬─────┘
       │                     │
       ↓ COW                 ↓
  ┌──────────┐          ┌──────────┐
  │ 私有页面  │          │ 共享页面  │
  │ (A 的副本)│          │ (原文件)  │
  └──────────┘          └──────────┘

常用映射标志

标志说明
MAP_SHARED共享映射，修改对其他进程可见
MAP_PRIVATE私有映射，写时复制
MAP_FIXED映射到指定地址
MAP_ANONYMOUS匿名映射（不关联文件）
MAP_POPULATE预先分配页面
MAP_LOCKED锁定页面（不允许换出）
MAP_HUGETLB使用大页（2MB/1GB）
MAP_UNINITIALIZED不初始化（性能优化）

03 核心数据结构

// include/linux/mm_types.h:913 — 虚拟内存区域
struct vm_area_struct {
    unsigned long  vm_start;       // 映射起始地址
    unsigned long  vm_end;         // 映射结束地址
    struct mm_struct  *vm_mm;       // 所属进程地址空间
    pgprot_t       vm_page_prot;  // 页面访问权限
    vm_flags_t     vm_flags;      // VM_READ/VM_WRITE/VM_EXEC/VM_SHARED
    struct list_head anon_vma_chain;
    struct anon_vma  *anon_vma;    // 匿名页面反向映射
    const struct vm_operations_struct *vm_ops; // 操作函数表
    struct file     *vm_file;      // ★ 关联的文件（匿名映射为 NULL）
    unsigned long  vm_pgoff;      // 文件偏移（以页面为单位）
};

数据结构关系

  进程地址空间 mm_struct
    │
    ├── VMA 链表（按地址排序）
    │     ┌──────────┐    ┌──────────┐    ┌──────────┐
    │     │ VMA 1    │───→│ VMA 2    │───→│ VMA 3    │
    │     │ code     │    │ data     │    │ heap     │
    │     │ RO+Exec  │    │ RW+Shared│    │ RW+Anon  │
    │     └──────────┘    └──────────┘    └──────────┘
    │
    ├── mm_pgtable (页表)
    │     PGD → P4D → PUD → PMD → PTE
    │
    └── mmap_lock（保护 VMA 的读写锁）

  mmap() 调用后：
  1. 创建新的 VMA
  2. 插入 VMA 链表和红黑树
  3. 不立即分配物理页面！
  4. 访问时触发缺页异常 → 按需分配

04 mmap 流程

mmap() 系统调用完整流程：

  用户态：ptr = mmap(NULL, 4096, PROT_READ|PROT_WRITE,
                     MAP_SHARED, fd, 0);
        │
        ↓
  do_mmap() — mm/mmap.c:335
        │
        ├── 1. 参数检查
        │       len > 0, addr 合法, fd 有效
        │
        ├── 2. 计算映射参数
        │       offset 页对齐
        │       addr = get_unmapped_area()  ← 找到空闲虚拟地址空间
        │
        ├── 3. 创建 VMA
        │       kmem_cache_alloc(vm_area_cachep)
        │       设置 vm_start, vm_end, vm_flags, vm_file
        │
        ├── 4. 调用文件的 mmap 方法
        │       file->f_op->mmap(file, vma)
        │       │
        │       ├── 普通文件 → generic_file_mmap()
        │       │       设置 vm_ops = &generic_file_vm_ops
        │       │
        │       ├── 匿名映射 → anon_vma_prepare()
        │       │
        │       └── 设备文件 → 设备自己的 mmap 实现
        │           （如 /dev/dri/card0 → GPU 显存映射）
        │
        ├── 5. 插入 VMA 到进程地址空间
        │       vma_link() → 红黑树 + 链表
        │
        └── 6. 返回用户态虚拟地址
                注意：此时没有分配物理页面！

首次访问映射区域 → 缺页异常：

  用户态访问 ptr[0]
        │
        ↓
  CPU → 缺页异常（页表项不存在）
        │
        ↓
  do_page_fault() → handle_mm_fault()
        │
        ├── 文件映射：从磁盘读取页面（page cache 命中则直接使用）
        ├── 匿名映射：分配零页面（COW 时分配新页面）
        └── 建立页表映射
        │
        ↓
  返回用户态，ptr[0] 正常读取
  后续访问不再缺页（页面已映射）

05 缺页异常

缺页异常处理流程（handle_mm_fault）：

  CPU 访问虚拟地址 → 页表项无效 → 缺页异常
        │
        ↓
  handle_pte_fault()
        │
        ├── PTE 不存在（首次访问）
        │     │
        │     ├── 匿名页面 → do_anonymous_page()
        │     │       分配零页面，建立映射
        │     │       写时触发 COW → do_wp_page()
        │     │
        │     ├── 文件映射页面 → do_fault()
        │     │       ├── 读取页面（read fault）
        │     │       │   从 page cache 或磁盘读取
        │     │       ├── 写共享页面 → COW
        │     │       └── 写私有页面 → 分配新页面 + 复制
        │     │
        │     └── 设备映射 → vm_ops->fault()
        │             设备驱动处理（如 GPU 页面）
        │
        ├── PTE 存在但权限不足
        │     │
        │     └── 写只读页面 → COW
        │           do_wp_page() → 分配新页面 + 复制内容
        │
        └── 已在交换区（swap）
              do_swap_page() → 从 swap 恢复
mmap + 缺页异常的性能优势
mmap 延迟分配物理页面（直到首次访问才分配），这带来两个好处：
① 更快的进程创建：fork() 时 COW 共享父进程页面，无需立即复制
② 按需分配：映射了 1GB 但只用了 1MB，只分配实际使用的页面

06 应用场景

共享内存（IPC）
// POSIX 共享内存
int fd = shm_open("/myshm", O_RDWR|O_CREAT, 0666);
ftruncate(fd, 4096);
void *ptr = mmap(NULL, 4096,
    PROT_READ|PROT_WRITE,
    MAP_SHARED, fd, 0);
// 多个进程映射同一文件 → 共享内存
TCP Zero-Copy Send
// sendfile / MSG_ZEROCOPY
// 文件 → page cache → mmap
// → NIC DMA 读取页面
// 无 memcpy！
// 网卡直接 DMA 读文件页
pipe splice
// splice(fd_in, ..., pipe, ..., SPLICE_F_MOVE)
// 仅修改页面引用计数
// 不复制实际数据
// 依赖 mmap 的页面管理机制
动态链接库
// libc.so 被 mmap 到进程地址空间
// .text 段共享（多个进程共享同一份）
// .data 段私有（COW 各自独立）
// 节省内存 + 加快加载
