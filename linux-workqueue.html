01 工作队列概览

Linux 内核提供了 9 个预定义的系统工作队列，用于不同的场景。它们在 kernel/workqueue.c 中定义和初始化。

系统工作队列一览

工作队列内核线程名关键标志用途
system_wqeventsWQ_PERCPU默认通用工作队列
system_highpri_wqevents_highpriWQ_HIGHPRI | WQ_PERCPU高优先级工作
system_long_wqevents_longWQ_PERCPU长时间运行的工作
system_unbound_wqevents_unboundWQ_UNBOUND不绑定 CPU 的工作
system_freezable_wqevents_freezableWQ_FREEZABLE | WQ_UNBOUND可冻结的工作
system_power_efficient_wqevents_power_efficientWQ_POWER_EFFICIENT | WQ_UNBOUND省电工作
system_freezable_power_efficient_wqevents_freezable_pwr_efficientWQ_FREEZABLE | WQ_POWER_EFFICIENT | WQ_UNBOUND可冻结 + 省电
system_bh_wqevents_bhWQ_BH | WQ_PERCPU软中断上下文工作
system_bh_highpri_wqevents_bh_highpriWQ_BH | WQ_HIGHPRI | WQ_PERCPU高优先级软中断工作

02 WQ 标志位

每个系统工作队列由一组 WQ 标志位定义其行为。

标志位定义

// include/linux/workqueue.h
// WQ_ 标志位定义

// 执行上下文
WQ_BH               = 1 << 0,   // 在软中断(bottom half)上下文执行
WQ_UNBOUND           = 1 << 1,   // 不绑定到任何 CPU（可迁移）
WQ_PERCPU            = 1 << 8,   // 绑定到特定 CPU（每 CPU 一个 worker）

// 电源管理
WQ_FREEZABLE         = 1 << 2,   // 系统挂起时冻结
WQ_POWER_EFFICIENT   = 1 << 7,   // 省电模式（可转为 unbound）

// 内存管理
WQ_MEM_RECLAIM        = 1 << 3,   // 可用于内存回收（保证有 worker）

// 优先级
WQ_HIGHPRI            = 1 << 4,   // 高优先级（nice -20）

// CPU 密集
WQ_CPU_INTENSIVE      = 1 << 5,   // CPU 密集型（影响并发 worker 数量）

标志位关系图

WQ_PERCPU (绑定CPU)          WQ_UNBOUND (不绑定CPU)
    │                              │
    ├── system_wq                  ├── system_unbound_wq
    ├── system_highpri_wq          ├── system_freezable_wq
    ├── system_long_wq             ├── system_power_efficient_wq
    ├── system_bh_wq               └── system_freezable_power_efficient_wq
    └── system_bh_highpri_wq

03 system_wq

默认的通用工作队列，适用于大多数场景。

// kernel/workqueue.c 第 8002 行
system_wq = alloc_workqueue("events", WQ_PERCPU, 0);

特性

属性值说明
标志WQ_PERCPU每 CPU 一个 worker 池
优先级默认 (nice 0)普通优先级
冻结否系统挂起时不冻结
省电否不进入省电模式
BH否在线程上下文执行
CPU 绑定绑定work 在提交 CPU 上执行

使用场景

// 最常用的场景
queue_work(system_wq, &my_work);

// 适用于：
// - 大多数驱动程序的延迟处理
// - 不需要特殊优先级或电源管理的工作
// - 需要访问 per-CPU 数据的工作

04 system_highpri_wq

高优先级工作队列，worker 线程以 nice -20 运行。

// kernel/workqueue.c 第 8004 行
system_highpri_wq = alloc_workqueue("events_highpri",
                                     WQ_HIGHPRI | WQ_PERCPU, 0);

特性

属性值说明
标志WQ_HIGHPRI | WQ_PERCPU高优先级 + 绑定 CPU
优先级nice -20最高普通优先级
worker 线程高优先级会抢占普通优先级线程

使用场景

// 需要快速响应的场景
queue_work(system_highpri_wq, &my_work);

// 适用于：
// - 网络驱动的 NAPI 处理
// - 需要低延迟的中断下半部
// - 实时性要求高的工作

// 注意：
// - 不要用于长时间运行的工作
// - 会抢占其他线程，影响系统响应性

05 system_long_wq

专门用于长时间运行工作的队列，防止阻塞其他 workqueue worker。

// kernel/workqueue.c 第 8006 行
system_long_wq = alloc_workqueue("events_long", WQ_PERCPU, 0);

特性

属性值说明
标志WQ_PERCPU绑定 CPU
优先级默认普通优先级
worker 数量受限防止长时间工作占用所有 worker

使用场景

// 长时间运行的工作
queue_work(system_long_wq, &my_work);

// 适用于：
// - 大文件 I/O
// - 批量数据处理
// - 可能耗时数百毫秒的操作

// 为什么不用 system_wq？
// system_wq 的 worker 数量有限（默认 256）
// 长时间工作会占用 worker，导致其他 work 无法执行
// system_long_wq 有独立的 worker 池，不会影响其他工作

06 system_unbound_wq

不绑定到任何 CPU 的工作队列，work 可以在任意 CPU 上执行。

// kernel/workqueue.c 第 8007 行
system_unbound_wq = alloc_workqueue("events_unbound", WQ_UNBOUND, WQ_MAX_ACTIVE);

特性

属性值说明
标志WQ_UNBOUND不绑定 CPU
优先级默认普通优先级
worker 数量WQ_MAX_ACTIVE最大并发 worker 数
CPU 亲和性无可在任意 CPU 上执行

使用场景

// 不需要绑定 CPU 的工作
queue_work(system_unbound_wq, &my_work);

// 适用于：
// - 跨 CPU 的工作
// - 不访问 per-CPU 数据的工作
// - 需要负载均衡的工作

// vs system_wq：
// system_wq：work 绑定到提交它的 CPU
// system_unbound_wq：work 可以在任意 CPU 上执行

07 system_freezable_wq

可冻结的工作队列，在系统挂起（suspend/hibernate）时自动冻结。

// kernel/workqueue.c 第 8009 行
system_freezable_wq = alloc_workqueue("events_freezable",
                                       WQ_FREEZABLE | WQ_UNBOUND, 1);

特性

属性值说明
标志WQ_FREEZABLE | WQ_UNBOUND可冻结 + 不绑定 CPU
冻结行为系统挂起时冻结阻止新 work 提交
worker 状态冻结worker 线程被冻结

使用场景

// 系统挂起时需要停止的工作
queue_work(system_freezable_wq, &my_work);

// 适用于：
// - 驱动程序的后台工作
// - 不需要在系统挂起时运行的工作
// - 可以安全延迟的工作

// 冻结流程：
// 1. 系统进入 freeze 阶段
// 2. 所有 pending work 被标记
// 3. worker 线程被冻结（不能执行新 work）
// 4. 系统挂起
// 5. 唤醒后 worker 解冻，继续执行

08 system_power_efficient_wq

省电工作队列，可以进入深度空闲状态以节省功耗。

// kernel/workqueue.c 第 8011 行
system_power_efficient_wq = alloc_workqueue("events_power_efficient",
                                             WQ_POWER_EFFICIENT | WQ_UNBOUND, 1);

特性

属性值说明
标志WQ_POWER_EFFICIENT | WQ_UNBOUND省电 + 不绑定 CPU
电源管理可进入深度空闲CPU 可以进入更深的 C-state
worker 数量1减少 worker 数量以省电

使用场景

// 不需要频繁唤醒 CPU 的工作
queue_work(system_power_efficient_wq, &my_work);

// 适用于：
// - 定时器轮询（不需要精确时间）
//- 后台数据同步
// - 低频传感器读取

// vs system_wq：
// system_wq：保持 CPU 活跃，快速响应
// system_power_efficient_wq：允许 CPU 进入深度空闲，节省功耗

09 system_bh_wq / system_bh_highpri_wq

在软中断（bottom half）上下文执行的工作队列，适用于需要从中断上下文延迟处理的场景。

// kernel/workqueue.c 第 8015-8016 行
system_bh_wq = alloc_workqueue("events_bh", WQ_BH | WQ_PERCPU, 0);
system_bh_highpri_wq = alloc_workqueue("events_bh_highpri",
                                        WQ_BH | WQ_HIGHPRI | WQ_PERCPU, 0);

特性

属性system_bh_wqsystem_bh_highpri_wq
标志WQ_BH | WQ_PERCPUWQ_BH | WQ_HIGHPRI | WQ_PERCPU
执行上下文软中断软中断
优先级默认高优先级
worker 类型BH workerBH worker（高优先级）

BH Worker 机制

// BH worker 在软中断上下文执行
// 不需要创建内核线程
// 直接在 softirq 上下文中执行 work

// 工作流程：
// 1. queue_work(system_bh_wq, &work)
// 2. work 被添加到 per-CPU 的 BH worker 队列
// 3. 在软中断上下文中执行 work
// 4. 执行完成后标记 BH worker 空闲

// 优势：
// - 不需要创建内核线程（节省内存）
// - 执行延迟更低（不需要线程调度）
// - 可以在中断上下文中使用（但 work 本身在 BH 上下文）

使用场景

// 需要快速延迟处理的场景
queue_work(system_bh_wq, &my_work);

// 适用于：
// - 网络驱动的 NAPI poll
// - 需要从中断上下文延迟到 BH 上下文的工作
// - 不需要创建额外线程的场景

// 注意：
// - BH worker 在软中断上下文，不能睡眠
// - 不能调用可能睡眠的函数（kmalloc(GFP_KERNEL)、mutex_lock 等）
// - 适合短时间执行的工作

10 对比总结

执行上下文对比

工作队列执行上下文是否可睡眠典型延迟
system_wq内核线程✅ 可以微秒级
system_highpri_wq高优先级线程✅ 可以更低
system_long_wq内核线程✅ 可以较高（worker 有限）
system_unbound_wq内核线程✅ 可以微秒级
system_freezable_wq内核线程（可冻结）✅ 可以微秒级
system_power_efficient_wq内核线程（省电）✅ 可以较高
system_bh_wq软中断❌ 不行极低
system_bh_highpri_wq软中断（高优先级）❌ 不行极低

选择决策树

需要在软中断上下文执行？
├── 是 → system_bh_wq 或 system_bh_highpri_wq
└── 否 → 继续判断
    │
    需要高优先级？
    ├── 是 → system_highpri_wq
    └── 否 → 继续判断
        │
        长时间运行？
        ├── 是 → system_long_wq
        └── 否 → 继续判断
            │
            需要冻结/省电？
            ├── 可冻结 → system_freezable_wq
            ├── 省电 → system_power_efficient_wq
            ├── 都要 → system_freezable_power_efficient_wq
            └── 不需要 → 继续判断
                │
                需要跨 CPU 执行？
                ├── 是 → system_unbound_wq
                └── 否 → system_wq（默认）

常见错误

// ❌ 错误：在 system_bh_wq 的 work 中睡眠
void my_bh_work(struct work_struct *work)
{
    kmalloc(1024, GFP_KERNEL);  // 可能睡眠！BH 上下文不允许
    mutex_lock(&my_mutex);     // 可能睡眠！
}

// ✅ 正确：使用 GFP_ATOMIC 或不睡眠
void my_bh_work(struct work_struct *work)
{
    kmalloc(1024, GFP_ATOMIC);  // 不睡眠
    // 或者使用 system_wq 代替 system_bh_wq
}

// ❌ 错误：在 system_power_efficient_wq 中放 CPU 密集工作
void my_power_work(struct work_struct *work)
{
    heavy_computation();  // 会长时间占用 CPU，阻止省电
}

// ✅ 正确：CPU 密集工作使用 system_wq
void my_cpu_work(struct work_struct *work)
{
    heavy_computation();  // 使用 system_wq
}
