01 概述

工作队列（Workqueue）是 Linux 内核的延迟执行框架，允许在进程上下文中异步执行工作函数。它将工作推迟到专门的 worker 线程中执行，避免在中断上下文中执行可能睡眠的操作。

核心特性
  • 进程上下文：worker 线程中执行，可睡眠
  • 延迟执行：中断 → 调度 → worker 执行
  • CPU 绑定：可绑定到特定 CPU
  • 并发控制：可配置最大并发 worker
  • 系统化：cmwq 框架自动管理 worker
  • flush 支持：等待所有工作完成
适用场景
场景说明
中断下半部替代 softirq/tasklet
设备事件热插拔、配置变更
延迟初始化模块加载后执行
定时任务周期性检查/清理
USB 异步操作USB 核心大量使用

05 worker 线程

cmwq（Concurrency Managed Workqueue）框架自动管理 worker 线程，根据负载动态创建/销毁。

cmwq 机制

schedule_work(my_work)
↓
加入 pool_workqueue
关联到 worker_pool
↓
worker_thread1
执行 work_func()
worker_thread2
空闲或执行其他
↓
work_func() 执行完毕
work 标记为完成

Worker 类型

类型说明
普通 worker按需创建，空闲时销毁
bound worker绑定到特定 CPU
unbound worker不绑定 CPU，可调度到任意 CPU

06 使用模式

// 模式一：一次性工作
static void my_work_func(struct work_struct *work)
{
    // 在进程上下文中执行，可睡眠
    msleep(100);
    pr_info("work completed\n");
}

struct work_struct my_work;
INIT_WORK(&my_work, my_work_func);
schedule_work(&my_work);
// 模式二：延迟工作（定时器）
static void my_delayed_func(struct delayed_work *dwork)
{
    pr_info("delayed work done\n");
}

struct delayed_work my_dwork;
INIT_DELAYED_WORK(&my_dwork, my_delayed_func);

// 5 秒后执行
schedule_delayed_work(&my_dwork, msecs_to_jiffies(5000));

// 修改延迟
mod_delayed_work(system_wq, &my_dwork, msecs_to_jiffies(10000));

// 取消
cancel_delayed_work_sync(&my_dwork);
// 模式三：中断下半部
static irqreturn_t my_irq_handler(int irq, void *dev_id)
{
    struct my_device *dev = dev_id;
    disable_irq_nosync(irq);
    schedule_work(&dev->bottom_half);  // 推迟到 worker 执行
    return IRQ_HANDLED;
}

// worker 中处理（可睡眠）
static void my_bottom_half(struct work_struct *work)
{
    struct my_device *dev = container_of(work, ...);
    // 处理数据（可睡眠）
    enable_irq(dev->irq);
}
