01 概述

Softirq（软中断）是 Linux 中断处理的延迟执行机制。硬件中断处理程序（上半部）执行时间短，将耗时操作推迟到软中断（下半部）中处理。Softirq 在中断上下文之外执行，可以被硬件中断打断，但不能睡眠。Tasklet 是基于 softirq 的更高层抽象，保证同一 tasklet 不会在多 CPU 上并发执行。

02 软中断类型

软中断编号用途
HI_SOFTIRQ0高优先级 tasklet
TIMER_SOFTIRQ1内核定时器
NET_TX_SOFTIRQ2网络发送
NET_RX_SOFTIRQ3网络接收（NAPI）
BLOCK_SOFTIRQ4块设备 I/O
IRQ_POLL_SOFTIRQ5中断轮询
TASKLET_SOFTIRQ6普通 tasklet
SCHED_SOFTIRQ7调度器负载均衡
HRTIMER_SOFTIRQ8高精度定时器
RCU_SOFTIRQ9RCU 回调

03 执行流程

软中断完整执行流程：

  触发阶段：
  硬件中断处理程序 / 内核代码
        │
        ├── raise_softirq(NET_RX_SOFTIRQ)
        │   设置 softirq_pending 位图
        │
        ├── 或 __raise_softirq_irqoff(NET_RX_SOFTIRQ)
        │   在禁用中断时使用
        │
        └── 返回（不立即执行软中断）

  执行阶段：
  硬件中断返回时检查 softirq_pending
        │
        ↓
  __do_softirq() — kernel/softirq.c
        │
        ├── 1. 记录最大迭代次数（MAX_SOFTIRQ_RESTART=10）
        │
        ├── 2. while (softirq_pending) {
        │       pending = local_softirq_pending();
        │       local_irq_enable();  // 开中断！允许被硬中断打断
        │
        │       遍历 pending 位图：
        │       for_each_set_bit(nr, &pending, NR_SOFTIRQS) {
        │           h->action();  // 调用对应的 softirq 处理函数
        │       }
        │
        │       local_irq_disable();
        │       restart--;
        │   }
        │
        ├── 3. 如果重启次数用完
        │       → raise_softirq(TASKLET_SOFTIRQ)
        │       → 让 ksoftirqd 后续处理
        │
        └── 4. 返回

04 Tasklet

Tasklet vs Softirq：

  Softirq:
  - 同一 softirq 可以在多个 CPU 并发执行
  - 不能睡眠
  - 适合高频、性能关键的场景（如网络收包）

  Tasklet:
  - 同一 tasklet 在任意时刻只在一个 CPU 执行
  - 基于 softirq（TASKLET_SOFTIRQ）实现
  - 不能睡眠
  - 适合不需要并发的驱动延迟处理

  Tasklet 内部使用 per-CPU 的链表：
  CPU 0: tasklet_A → tasklet_B → tasklet_C
  CPU 1: tasklet_D → tasklet_E
  每个 tasklet 同时只能在一个 CPU 上执行

// Tasklet 使用示例
struct tasklet_struct my_tasklet;

static void my_tasklet_func(unsigned long data)
{
    printk(KERN_INFO "tasklet running on CPU %d\n",
           smp_processor_id());
}

// 初始化
tasklet_init(&my_tasklet, my_tasklet_func, 0);

// 调度（中断上下文中调用）
tasklet_schedule(&my_tasklet);

// 禁用/启用
tasklet_disable(&my_tasklet);
tasklet_enable(&my_tasklet);

// 销毁
tasklet_kill(&my_tasklet);

05 Bottom Half 机制对比

机制并发睡眠适用场景
Softirq多 CPU 并发否网络收发、定时器
Tasklet单 CPU 串行否驱动延迟处理
Workqueue可配置是可睡眠的延迟工作
Threaded IRQ专用线程是驱动中断处理
选择建议
① 需要高性能、无睡眠、可并发 → Softirq（网络子系统）
② 需要无睡眠、但不需要并发 → Tasklet（简单驱动）
③ 需要睡眠或长时间运行 → Workqueue（推荐）
④ Tasklet 已不推荐新代码使用，优先使用 Workqueue
