01 概述

原子操作是内核中无锁并发的基础，用于在多处理器环境下安全地操作共享数据。Linux 提供了从底层硬件原子指令到上层引用计数的完整层次化并发原语。

核心并发原语
  • atomic_t / atomic64_t：原子计数器
  • refcount_t：引用计数（饱和语义）
  • spinlock_t：自旋锁
  • rwlock_t：读写锁
  • mutex：互斥锁（可睡眠）
  • completion：完成量
  • RCU：读-拷贝-更新
内存序（Memory Ordering）
类型说明
Relaxed无序（最快）
Acquire读屏障（阻止后续操作重排到此之前）
Release写屏障（阻止之前操作重排到此之后）
Fully ordered完全有序（最慢，最安全）

03 原子操作

atomic_t API

// 初始化
atomic_t counter = ATOMIC_INIT(0);

// 基本操作
atomic_set(&counter, 100);        // 设置值
atomic_read(&counter);            // 读取值
atomic_add(1, &counter);          // 加（无返回值）
atomic_sub(1, &counter);          // 减
atomic_inc(&counter);            // 加 1
atomic_dec(&counter);            // 减 1

// 返回值版本（完全有序）
atomic_add_return(1, &counter);  // 返回新值
atomic_sub_return(1, &counter);
atomic_inc_return(&counter);
atomic_dec_return(&counter);

// fetch 版本（返回旧值）
atomic_fetch_add(1, &counter);  // 返回旧值
atomic_fetch_sub(1, &counter);

// 比较并交换
atomic_cmpxchg(&counter, old, new);  // CAS 操作
atomic_try_cmpxchg(&counter, &old, new); // 安全 CAS

内存序变体

include/linux/atomic.h:10-78

// 四种变体：

// 1. Relaxed（无序）— 最快，无排序保证
atomic_add_unless_relaxed(v, 1, 0);
atomic_fetch_add_relaxed(1, v);

// 2. Acquire（读屏障）
atomic_add_unless_acquire(v, 1, 0);
atomic_fetch_add_acquire(1, v);
// 等价于：__atomic_acquire_fence(); fetch_relaxed();

// 3. Release（写屏障）
atomic_add_unless_release(v, 1, 0);
// 等价于：__atomic_release_fence(); add_relaxed();

// 4. Fully ordered（默认，完全有序）
atomic_add(1, v);
atomic_fetch_add(1, v);
// 等价于：pre_fence(); add_relaxed(); post_fence();
选择原则：默认使用完全有序版本。只有在性能关键路径且经过仔细分析后才使用 relaxed/acquire/release 变体。错误的内存序是极难调试的并发 bug 来源。

04 引用计数

refcount_t 是专为引用计数设计的类型，相比 atomic_t 具有饱和语义（溢出时不回绕）。

与 atomic_t 的区别

特性atomic_trefcount_t
溢出行为回绕到负数饱和到 REFCOUNT_SATURATED
下溢行为回绕到大正数饱和 + WARN
检查函数无refcount_inc_not_zero()
用途通用计数器专用引用计数
安全需手动保护内置防溢出保护

refcount_t API

// 初始化
refcount_set(&ref, 1);           // 设置初始引用

// 增加引用（无检查）
refcount_inc(&ref);             // 增加（必须 > 0）

// 增加引用（检查是否为零）
bool ok = refcount_inc_not_zero(&ref); // 如果为 0 则失败

// 减少引用
refcount_dec(&ref);             // 减少（必须 > 0）

// 减少并检查是否归零
bool last = refcount_dec_and_test(&ref); // 最后一个引用

// 查询
bool alive = refcount_read(&ref) != 0;
bool saturated = refcount_is_saturated(&ref);
饱和语义：当引用计数溢出或下溢时，refcount_t 不会回绕，而是饱和到 REFCOUNT_SATURATED (0xC0000000)。这避免了经典的 use-after-free 和 double-free 安全漏洞。检测到异常时会打印 WARN 并进入内核 oops。

自旋锁类型

类型说明使用场景
spinlock_t普通自旋锁短临界区
rwlock_t读写锁读多写少

使用示例

// spinlock_t — 保护短临界区
static DEFINE_SPINLOCK(my_lock);

spin_lock(&my_lock);
// ... 临界区（不可睡眠） ...
spin_unlock(&my_lock);

// rwlock_t — 读多写少
static DEFINE_RWLOCK(my_rwlock);

read_lock(&my_rwlock);     // 多个读者可以同时持有
// ... 只读操作 ...
read_unlock(&my_rwlock);

write_lock(&my_rwlock);    // 写者独占
// ... 写操作 ...
write_unlock(&my_rwlock);

06 内存屏障

内存屏障（Memory Barrier）用于控制指令和内存操作的重排序，确保多处理器系统中的内存一致性。

屏障类型

屏障指令说明
smp_mb()全屏障阻止所有重排序
smp_wmb()写屏障阻止写-写重排序
smp_rmb()读屏障阻止读-读重排序
smp_store_release()释放屏障之前的读写不会重排到此之后
smp_load_acquire()获取屏障此后的读写不会重排到此之前
mb()完全屏障包含编译器屏障 + 硬件屏障
rmb()读屏障编译器 + 硬件读屏障
wmb()写屏障编译器 + 硬件写屏障

使用示例

// 生产者-消费者模式
// 生产者：
data = 42;
smp_wmb();             // 确保 data 在 ready 之前写入
ready = 1;

// 消费者：
if (ready) {
    smp_rmb();         // 确保读 data 在检查 ready 之后
    x = data;
}

// 更简洁的版本：
smp_store_release(&ready, 1);  // 替代 smp_wmb + store
if (smp_load_acquire(&ready)) {  // 替代 load + smp_rmb
    x = data;
}
Acquire/Release 模型：Linux 推荐使用 acquire/release 语义而非显式屏障。Acquire 保证此后的操作不会重排到此之前，Release 保证此之前的操作不会重排到此之后。这比全屏障更高效。

08 使用模式

引用计数模式

// 典型的引用计数生命周期
struct my_object {
    struct refcount_t refcount;
    // ... 其他字段 ...
};

// 分配
struct my_object *obj = kzalloc(sizeof(*obj), GFP_KERNEL);
refcount_set(&obj->refcount, 1);  // 初始引用 = 1

// 获取引用
bool ok = refcount_inc_not_zero(&obj->refcount);
if (!ok) return;  // 对象可能已释放

// 释放引用
if (refcount_dec_and_test(&obj->refcount)) {
    // 最后一个引用，释放对象
    kfree(obj);
}

per-CPU 计数器模式

// 高性能 per-CPU 计数器
DEFINE_PER_CPU(unsigned long, my_counter);

// 仅访问当前 CPU（无锁）
this_cpu_inc(my_counter);
this_cpu_dec(my_counter);
unsigned long val = this_cpu_read(my_counter);

// 需要汇总时遍历所有 CPU
unsigned long total = 0;
for_each_possible_cpu(cpu)
    total += per_cpu(my_counter, cpu);

常见错误

不要混用 atomic_t 和 refcount_t：refcount_t 具有饱和语义，不能替代 atomic_t 的任意计数。反过来，atomic_t 没有溢出保护，不应用于引用计数。

不要在持有锁时调用可能睡眠的函数：spinlock 临界区内不能调用 kmalloc(GFP_KERNEL)、mutex_lock()、msleep() 等。

不要忘记 release 语义：多处理器系统中，分配对象后的初始化必须在引用计数传递给其他 CPU 之前完成。
