01 概述

ext4（Fourth Extended Filesystem）是 Linux 最广泛使用的文件系统，也是大多数发行版的默认选择。它在 ext3 基础上增加了 extent、延迟分配、在线碎片整理、纳秒时间戳等特性。ext4 代码分布在 fs/ext4/ 目录下，仅 super.c 就有 7598 行。

02 磁盘布局

ext4 磁盘布局：

┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ Boot     │ Block    │ Block    │ Block    │  Data    │
│ Block    │ Group 0  │ Group 1  │ Group 2  │  Blocks  │
│ (1KB)    │          │          │          │  ...     │
└──────────┴──────────┴──────────┴──────────┴──────────┘

每个 Block Group 包含：
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ Super    │ Group    │ Block    │ Inode    │  Data    │
│ Block    │ Desc     │ Bitmap   │ Bitmap   │  Blocks  │
│ (副本)   │ Table    │          │          │          │
└──────────┴──────────┴──────────┴──────────┴──────────┘

  Super Block：文件系统元数据（块大小、inode 数量、状态）
  Group Descriptor：块组描述符
  Block Bitmap：哪些块已使用
  Inode Bitmap：哪些 inode 已使用
  Inode Table：inode 结构数组
  Data Blocks：实际文件数据

03 核心数据结构

// 磁盘上的 inode 结构（128 字节）
struct ext4_inode {
    __le16   i_mode;       // 文件类型 + 权限
    __le16   i_uid;        // 所有者 UID
    __le32   i_size_lo;    // 文件大小（低 32 位）
    __le32   i_atime;      // 访问时间
    __le32   i_ctime;      // 变更时间
    __le32   i_mtime;      // 修改时间
    __le32   i_dtime;      // 删除时间
    __le16   i_gid;        // 组 ID
    __le16   i_links_count; // 硬链接计数
    __le32   i_blocks_lo;  // 占用块数
    __le32   i_flags;      // 文件标志
    __le32   i_block[60/4]; // ★ 块指针（15 个）或 Extent 树
    __le32   i_generation; // 文件版本（NFS 用）
    __le32   i_size_high;  // 文件大小高 32 位
};

// 内核中的 ext4 inode
struct ext4_inode_info {
    struct inode      vfs_inode;    // 继承自 VFS inode
    __le32            i_data[15];  // 原始块指针
    struct ext4_extent *i_block;   // Extent 树（大文件）
    unsigned int      i_state;     // 状态标志
    struct ext4_journal_inode *jinode; // 日志 inode
};

04 日志（Journal）

ext4 日志机制（保证崩溃一致性）：

  传统写入（无日志）：
  修改 inode → 修改 bitmap → 修改数据 → 写超级块
  崩溃时任何步骤中断 → 文件系统不一致 → 需要 fsck

  日志模式（Journaling）：
  1. 将所有修改写入日志区（顺序写入，高性能）
  2. fsync() / 定期提交 → 日志标记为 committed
  3. 将日志中的修改应用到实际位置
  4. 崩溃恢复：重放已 committed 的日志 → 一致性保证

  ┌──────────────────────────────────────────┐
  │            ext4 日志模式                  │
  ├──────────────────────────────────────────┤
  │  data=writeback  │ 只记录元数据           │
  │  data=ordered    │ 先写数据，再写元数据（默认）│
  │  data=journal    │ 元数据和数据都记录（最安全）│
  └──────────────────────────────────────────┘

05 Extent 树

ext4 用 Extent 替代了传统的间接块指针：

传统（ext2/ext3）：
  inode.i_block[0..11]  → 直接块指针（12 个）
  inode.i_block[12]     → 一级间接
  inode.i_block[13]     → 二级间接
  inode.i_block[14]     → 三级间接
  大文件需要多级索引，效率低

ext4 Extent 树：
  inode.i_block[0] 存储根 Extent 树
  每个 Extent 记录一段连续块的范围：
  ┌─────────┬─────────┬──────────┐
  │ ee_block│ ee_len  │ ee_start │
  │ 逻辑起始│ 块数量  │ 物理起始  │
  └─────────┴─────────┴──────────┘

  小文件：一个 Extent 就够（1 个 extent 描述 128MB）
  大文件：树形 Extent（类似 B-tree）
  碎片减少：连续分配效率高

06 关键特性

特性说明
Extent连续块范围，减少间接块开销
延迟分配写入时暂不分配块，flush 时才分配，减少碎片
多块分配一次分配多个连续块
日志崩溃后快速恢复（无需全盘 fsck）
在线碎片整理e4defrag 在线整理
大文件支持最大 16TB 文件
纳秒时间戳比 ext3 的秒级精度高
barrier确保日志写入磁盘（数据安全）
