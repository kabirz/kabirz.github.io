01 SMP 概览

Zephyr SMP（Symmetric Multi-Processing）支持在多核处理器上运行，每个 CPU 维护独立的就绪队列。

SMP 支持架构

架构SoC核心数
ARM Cortex-MSTM32H7双核（M7+M4）
ARM Cortex-MnRF5340双核（M33+M33）
ARM Cortex-Ai.MX 8多核
x86多核 x86多核
ARCARC HS多核

03 SMP 调度

// SMP 调度核心逻辑（kernel/smp/）

// 1. 每 CPU 独立就绪队列
_kernel.cpus[cpu_id].ready_q.runq;

// 2. 调度决策（kernel/sched.c next_up）
static ALWAYS_INLINE struct k_thread *next_up(void)
{
#ifdef CONFIG_SMP
    // 处理当前线程挂起/中止
    if (z_is_thread_halting(_current)) {
        halt_thread(_current, ...);
    }
#endif

    // 从本 CPU 就绪队列选择最高优先级线程
    struct k_thread *thread = runq_best();

    // 元 IRQ 抢占恢复
    thread = metairq_preempt_recover(thread);

#ifndef CONFIG_SMP
    // 单核：如果当前线程仍在队列中，不切换
    if (thread == _current) {
        return thread;
    }
#endif

    return thread;
}

04 CPU 亲和性

// 设置线程 CPU 亲和性
k_thread_cpu_mask_clear(thread);
k_thread_cpu_mask_set(thread, 0);  // 绑定到 CPU 0

// 允许在 CPU 0 和 CPU 1 上运行
k_thread_cpu_mask_clear(thread);
k_thread_cpu_mask_set(thread, 0);
k_thread_cpu_mask_set(thread, 1);

// 查询线程 CPU 亲和性
uint32_t mask;
k_thread_cpu_mask_get(thread, &mask);

// 注意：
// CONFIG_SCHED_CPU_MASK_PIN_ONLY=y
// 仅允许绑定到单个 CPU（简化调度）

05 处理器间中断 (IPI)

IPI 用于 CPU 间通信，通知其他 CPU 重新调度。

07 SMP 配置

Kconfig 选项

Kconfig说明
CONFIG_SMP=y启用 SMP
CONFIG_MP_MAX_NUM_CPUS=2最大 CPU 数
CONFIG_SCHED_CPU_MASK_PIN_ONLY=y仅允许绑定单 CPU
CONFIG_IPI_HAS_UNIQUE_IRQ=yIPI 使用独立中断

每 CPU 数据

// 每 CPU 数据（kernel/include/ksched.h）
struct z_kernel {
    struct {
        struct _priq_run runq;  // 本 CPU 就绪队列
    } cpus[CONFIG_MP_MAX_NUM_CPUS];

    struct k_thread *current;  // 本 CPU 当前线程
    struct k_thread *idle;     // 本 CPU 空闲线程
