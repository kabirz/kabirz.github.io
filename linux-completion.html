01 概述

Completion（完成量）是 Linux 内核中的一对一等待/通知原语。一个线程等待某个事件完成，另一个线程完成事件后通知等待者。与 semaphore 不同，completion 默认阻塞等待（而非非阻塞），并且设计用于同步点而非排他点。

核心特性
  • FIFO 顺序：按等待顺序唤醒
  • 可重复使用：complete_all 后需 reinit
  • 多种等待：可中断、可杀死、超时
  • 完整内存屏障：complete() 包含 full barrier
  • 非排他性：用于同步点，不用于互斥
适用场景
场景说明
模块加载等待初始化完成
异步 I/O等待 I/O 完成
设备初始化等待固件加载
线程同步等待子线程完成
RCU 同步等待 grace period

> 0已完成，等待者立即返回
UINT_MAXcomplete_all() 设置，永久完成

初始化方式

// 静态初始化
DECLARE_COMPLETION(my_comp);         // done=0

// 栈上初始化
DECLARE_COMPLETION_ONSTACK(my_comp); // 用于局部变量

// 动态初始化
struct completion comp;
init_completion(&comp);               // done=0

// 重置（complete_all 后必须调用）
reinit_completion(&comp);              // done=0

03 核心 API

等待 API

函数阻塞超时可中断
wait_for_completion()是否否
wait_for_completion_timeout()是是否
wait_for_completion_interruptible()是否是
wait_for_completion_interruptible_timeout()是是是
wait_for_completion_killable()是否致命信号
wait_for_completion_killable_timeout()是是致命信号
try_wait_for_completion()否--
completion_done()否--

通知 API

函数说明
complete(x)唤醒一个等待者
complete_all(x)唤醒所有等待者（设 done=UINT_MAX）
complete_on_current_cpu(x)在当前 CPU 上唤醒（WF_CURRENT_CPU）

完整 API 示例

// 完整的等待 API
// 返回值：
//   0 = 完成
//   -ERESTARTSYS = 被信号中断（interruptible）
//   -ERESTARTSYS = 被致命信号杀死（killable）
//   超时值 = 剩余时间（timeout 版本）
//   false = 未完成（try_wait）

// 同步等待
wait_for_completion(&comp);

// 带超时等待
unsigned long ret = wait_for_completion_timeout(&comp, 100);
if (ret == 0) { /* 超时 */ }

// 可中断等待
int ret = wait_for_completion_interruptible(&comp);
if (ret == -ERESTARTSYS) { /* 被信号中断 */ }

// 非阻塞检查
if (try_wait_for_completion(&comp)) {
    /* 已完成 */
}

04 内部实现

// kernel/sched/completion.c
// complete() 的核心实现

static void complete_with_flags(struct completion *x, int wake_flags)
{
    unsigned long flags;
    raw_spin_lock_irqsave(&x->wait.lock, flags);

    if (x->done != UINT_MAX)
        x->done++;           // done 计数 +1
    swake_up_locked(&x->wait, wake_flags); // 唤醒一个等待者

    raw_spin_unlock_irqrestore(&x->wait.lock, flags);
}

void complete(struct completion *x)
{
    complete_with_flags(x, 0);  // 无特殊唤醒标志
}

void complete_all(struct completion *x)
{
    raw_spin_lock_irqsave(&x->wait.lock, flags);
    x->done = UINT_MAX;           // 设为永久完成
    swake_up_all_locked(&x->wait); // 唤醒所有等待者
    raw_spin_unlock_irqrestore(&x->wait.lock, flags);
}
FIFO 语义：completion 使用 FIFO 队列管理等待者。complete() 唤醒最早等待的线程。complete_all() 唤醒所有等待者。这确保了公平性——先等待的线程先被唤醒。

05 对比其他原语

特性completionsemaphoremutexwait_event
默认行为阻塞非阻塞（down_trylock）阻塞阻塞
通知方式complete/complete_allupunlockwake_up
计数单次/全部可计数仅排他条件
语义同步点资源访问互斥条件等待
典型场景等待异步操作完成资源池临界区保护事件等待
completion vs wait_event：completion 是一次性通知（done++），wait_event 是条件循环检查（while (!condition)）。completion 适合"等待某个操作完成"，wait_event 适合"等待某个条件变为真"。如果操作可以重复触发，使用 completion + reinit_completion()。

06 使用模式

// 模式一：单次同步（最常见）
struct completion comp;
init_completion(&comp);

// 线程 A：发起异步操作
do_async_work(&comp);

// 线程 B：等待完成
wait_for_completion(&comp);
// 模式二：可中断等待
int ret = wait_for_completion_interruptible(&comp);
if (ret == -ERESTARTSYS) {
    // 用户进程收到信号，优雅退出
    return -EINTR;
}
// 模式三：带超时
unsigned long ret;
ret = wait_for_completion_timeout(&comp, msecs_to_jiffies(1000));
if (ret == 0) {
    dev_err(&dev->dev, "timeout\n");
    return -ETIMEDOUT;
}
// 模式四：重复使用（complete_all + reinit）
struct completion comp;
init_completion(&comp);

// 第一轮
do_work1(&comp);
wait_for_completion(&comp);
reinit_completion(&comp);  // 必须重置！

// 第二轮
do_work2(&comp);
wait_for_completion(&comp);
