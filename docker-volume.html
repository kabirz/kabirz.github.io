01 概述

容器默认无状态：容器删除后，其文件系统（容器层）一起删除。要持久化数据，需要挂载外部存储。

Docker 提供三种挂载方式：

类型存储位置管理方典型场景
Volume/var/lib/docker/volumes/Docker数据库、应用数据
Bind Mount任意宿主路径用户开发、配置注入
tmpfs宿主内存内核敏感数据、临时

命令对比：

# 1. Volume（命名卷）
docker run -v myvol:/data nginx
docker run --mount type=volume,src=myvol,dst=/data nginx

# 2. Bind Mount
docker run -v /home/me/config:/etc/nginx nginx
docker run --mount type=bind,src=/home/me/config,dst=/etc/nginx nginx

# 3. tmpfs
docker run --tmpfs /cache nginx
docker run --mount type=tmpfs,dst=/cache nginx

# 推荐使用 --mount（更清晰、可读性更好）
# -v 是老语法，简写但有歧义

03 Volume（命名卷）

# 创建
docker volume create myvol

# 列表
docker volume ls
DRIVER    VOLUME NAME
local     myvol

# 详情
docker volume inspect myvol
[
  {
    "CreatedAt": "2024-...",
    "Driver": "local",
    "Labels": {},
    "Mountpoint": "/var/lib/docker/volumes/myvol/_data",
    "Name": "myvol",
    "Options": {},
    "Scope": "local"
  }
]

# 删除
docker volume rm myvol

# 清理无用卷
docker volume prune

使用：

# 1. 显式使用
docker run -d \
  --name db \
  -v myvol:/var/lib/mysql \
  mysql:8

# 2. 自动创建匿名卷
# Dockerfile: VOLUME /var/lib/mysql
# 启动时自动创建随机名卷
docker run mysql:8
# 创建类似 6c4f0a8b... 的匿名卷

# 3. 只读挂载
docker run -v myvol:/data:ro nginx
docker run --mount type=volume,src=myvol,dst=/data,ro nginx

# 4. 卷共享（容器间）
docker run -d -v sharedvol:/data --name app1 nginx
docker run -d --volumes-from app1 --name app2 nginx
# app2 共享 app1 的所有卷

Volume Driver：

// 默认 driver: local
// 卷存在 /var/lib/docker/volumes/<name>/_data

// 其他 driver（通过 plugin）：
//   local + nfs      - 通过 NFS 挂载
//   local + tmpfs    - 内存卷
//   local + btrfs    - btrfs 子卷
//   local + zfs      - zfs 数据集

// 使用 NFS：
docker volume create \
  --driver local \
  --opt type=nfs \
  --opt o=addr=192.168.1.10,rw \
  --opt device=:/path/to/share \
  nfsvol

// CSI 驱动（Cloud Volume）：
//   - AWS EBS
//   - GCE Persistent Disk
//   - Azure Disk
//   - Ceph RBD
//   - Longhorn

04 Bind Mount

# 1. 基本用法
docker run -v /home/me/app:/app nginx
docker run --mount type=bind,src=/home/me/app,dst=/app nginx

# 2. 只读
docker run -v /home/me/config:/etc/nginx:ro nginx
docker run --mount type=bind,src=/home/me/config,dst=/etc/nginx,ro nginx

# 3. SELinux 标签（:z / :Z）
docker run -v /home/me/config:/etc/nginx:z nginx
# :z  - 共享标签（多个容器共享）
# :Z  - 私有标签（仅此容器）

# 4. 单文件 bind
docker run -v /home/me/nginx.conf:/etc/nginx/nginx.conf:ro nginx
Bind Mount 的坑:
1. 路径必须存在（否则 Docker 创建空目录，导致容器看到空文件）
2. 权限：宿主文件 owner 与容器内不一致时，可能无法读写
3. 覆盖容器内容：bind mount 覆盖容器内目录（如果目标有镜像文件）
4. 跨主机不可移植：路径在每台机器上不同
5. 安全：bind mount 可让容器写宿主敏感路径
推荐：开发环境用 bind mount（hot reload），生产用 volume。

典型开发场景：

# Web 开发：源代码挂载到容器
docker run -d \
  -v $(pwd)/src:/app/src \
  -v $(pwd)/config:/app/config \
  -p 3000:3000 \
  node-app

// 容器内修改文件 → 宿主同步
// 宿主修改文件 → 容器看到（hot reload）
// 无需重建镜像

05 tmpfs Mount

# 基本用法
docker run -d --tmpfs /cache nginx
docker run -d --mount type=tmpfs,dst=/cache nginx

# 选项
docker run -d \
  --mount type=tmpfs,dst=/cache,tmpfs-size=1g,tmpfs-mode=1777 \
  nginx

# 特点：
# - 数据在内存（不写磁盘）
# - 容器停止后数据消失
# - 性能极高
# - 不计入镜像

# 用途：
# - 敏感数据（密码、密钥）
# - 临时缓存
# - 防止磁盘写入（安全）

# 与容器内 /tmp 区别：
# - /tmp 默认在 overlayfs（写时复制）
# - tmpfs 是独立的内存文件系统
# - 容器删除后，/tmp 数据消失
# - tmpfs 数据也消失，但更快

06 image-packed 卷

Docker 26+ / BuildKit 引入的新特性：从镜像中打包卷：

# 卷创建时从镜像复制内容
docker volume create myvol --opt type=image --opt o=subpath=/data --opt device=myimage:latest

# 用途：
# - 预填充配置数据
# - 预填充基础数据（如 GeoIP 数据库）
# - 跨容器共享只读内容

# 老办法（仍在用）：
# 1. 镜像构建时 COPY 数据到容器
# 2. 启动容器时 cp 数据到卷
# 3. 删除原文件

# 新办法：
# 直接从镜像创建卷，跳过手动复制

07 集群存储

跨主机的卷（Swarm 或 K8s）：

// Docker Swarm：
//   集群卷通过 driver 实现
//   例：使用 NFS / GlusterFS / Ceph

docker volume create \
  --driver nfs \
  --opt type=nfs \
  --opt o=addr=nfs.local,rw \
  --opt device=:/share \
  shared

// 在任何节点都能挂载
docker service create --mount type=volume,src=shared,dst=/data web

// Kubernetes：
//   通过 CSI（Container Storage Interface）插件
//   - PVC (Persistent Volume Claim)
//   - PV  (Persistent Volume)
//   - StorageClass

// yaml 示例：
// apiVersion: v1
// kind: PersistentVolumeClaim
// metadata:
//   name: my-pvc
// spec:
//   accessModes: [ReadWriteOnce]
//   resources:
//     requests:
//       storage: 10Gi
//   storageClassName: standard

08 Volume Plugins

// Volume plugin 接口（HTTP API）：
// 路径：/VolumeDriver.{Action}

// 主要 action：
POST /VolumeDriver.Create
{ "Name": "myvol", "Opts": {} }

POST /VolumeDriver.Remove
{ "Name": "myvol" }

POST /VolumeDriver.Mount
{ "Name": "myvol" }
// 返回: { "Mountpoint": "/path/to/mount" }

POST /VolumeDriver.Unmount
{ "Name": "myvol" }

POST /VolumeDriver.Path
{ "Name": "myvol" }
// 返回: { "Mountpoint": "/path/to/mount" }

POST /VolumeDriver.Get
{ "Name": "myvol" }
// 返回卷详细信息

POST /VolumeDriver.List
// 返回所有卷

POST /VolumeDriver.Capabilities
// 返回 driver 能力

// Plugin 注册：
// /etc/docker/plugins/mydriver.spec
// 或 docker plugin install

// 知名 Volume Plugins：
//   - blockbridge
//   - convoy
//   - rexray (多种 backend)
//   - purestorage
//   - portworx
//   - storageos

CSI（Container Storage Interface）：

// CSI 是 K8s/CRI-O/Containerd 共同的标准
// 三种 gRPC 服务：
//   IdentityServer  - 插件信息
//   ControllerServer - 卷管理（create/delete/attach）
//   NodeServer       - 节点端（mount/unmount）

// 知名 CSI：
//   - AWS EBS CSI
//   - GCE PD CSI
//   - Azure Disk CSI
//   - Ceph CSI
//   - Longhorn (Rancher)
//   - OpenEBS
//   - Portworx
//   - Rook

// CSI 与 Docker Volume Plugin 区别：
//   Docker Volume Plugin - HTTP API，仅 Docker
//   CSI                  - gRPC，跨平台（K8s/Mesos/Containerd）
