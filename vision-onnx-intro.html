01 ONNX 简介

ONNX（Open Neural Network Exchange）是一种开放的神经网络模型交换格式，由 Microsoft 和 Facebook 于 2017 年联合推出。它的核心目标是让深度学习模型可以在不同框架（PyTorch、TensorFlow、MXNet 等）之间自由迁移。

ONNX 不仅是一个文件格式，更是一个完整的生态系统，包括 ONNX Runtime（高性能推理引擎）、ONNX opset（算子规范集）以及丰富的模型优化工具链。

ONNX 的核心优势

  • 框架互操作性 — 在 PyTorch 中训练，导出为 ONNX，在 TensorRT 或 ONNX Runtime 中推理
  • 硬件加速 — ONNX Runtime 支持 CPU、GPU、NPU 等多种硬件后端，并提供算子融合等优化
  • 模型优化 — 通过 ONNX simplifier、onnxoptimizer 等工具进行图优化、常量折叠和量化
  • 部署友好 — 统一的推理接口，适合生产环境和边缘设备部署

典型转换链

# 典型工作流：PyTorch 训练 ONNX 导出 TensorRT 部署

# PyTorch --torch.onnx.export--> ONNX
# ONNX --onnx2trt / trtexec--> TensorRT engine
# ONNX --ONNX Runtime--> 跨平台推理

# 或：TensorFlow Keras --tf2onnx--> ONNX --onnxruntime--> 推理
ONNX Runtime:
ONNX Runtime 是官方跨平台推理引擎，支持 C++/Python/C#/Java API，适用于生产环境。它提供图优化、量化、内核调优等功能，在 CPU 和 GPU 上均有良好性能。

02 ONNX 规范

ONNX 基于 Protocol Buffers（protobuf）序列化，使用 .onnx 作为文件扩展名。一个 ONNX 模型本质上是一个 protobuf 二进制文件，顶层结构为 ModelProto。

核心数据结构

// ModelProto -- 模型根结构
message ModelProto {
  int64 ir_version = 1;           // ONNX IR 版本
  repeated OperatorSetIdProto opset_import = 2;
  string producer_name = 3;       // 导出框架名称（"pytorch"）
  string producer_version = 4;    // 导出框架版本
  string domain = 5;              // 模型域
  int64 model_version = 6;
  string doc_string = 7;
  GraphProto graph = 8;           // 计算图（核心）
  repeated StringStringEntryProto metadata_props = 14;
}

// GraphProto -- 计算图
message GraphProto {
  repeated NodeProto node = 1;    // 算子节点列表
  string name = 2;               // 图名称
  repeated TensorProto initializer = 5;  // 常量/权重
  repeated ValueInfoProto input = 11;    // 输入信息
  repeated ValueInfoProto output = 12;   // 输出信息
  repeated ValueInfoProto value_info = 13; // 中间张量信息
}

// NodeProto -- 算子节点
message NodeProto {
  repeated string input = 1;     // 输入张量名称
  repeated string output = 2;    // 输出张量名称
  string op_type = 4;            // 算子类型（"Conv", "Relu"）
  string domain = 7;             // 算子域（"" 为标准 opset）
  string name = 3;
  repeated AttributeProto attribute = 5;  // 属性
}

// TensorProto -- 张量数据
message TensorProto {
  enum DataType {
    FLOAT = 1;    // float32
    UINT8 = 2;    // uint8
    INT8 = 3;     // int8
    UINT16 = 4;
    INT16 = 5;
    INT32 = 6;
    INT64 = 7;
    STRING = 8;
    BOOL = 9;
    FLOAT16 = 10; // half precision
    DOUBLE = 11;  // float64
    BFLOAT16 = 16; // bfloat16
  }
  DataType data_type = 4;
  repeated int64 dims = 1;        // 形状
  // raw_data 或 xxx_data 字段存储值
  bytes raw_data = 9;
}

protobuf 大小限制:
大型 ONNX 文件（>2GB）需要启用 protobuf 的 C++ 解析器。Python 默认 protobuf 限制为 2GB，可通过 onnx.load(..., load_external_data=False) 处理外部数据的大型模型。

03 常用算子

ONNX 定义了标准的算子集合（opset），涵盖神经网络推理所需的常见操作。以下是视觉模型中常用的关键算子。

核心视觉算子

# Conv — 卷积
# 输入: (N, C, H, W)  权重: (M, C, kh, kw)  偏置: (M,)
# 属性: dilations, group, kernel_shape, pads, strides
node = onnx.helper.make_node(
    "Conv",
    inputs=["input", "weight", "bias"],
    outputs=["output"],
    kernel_shape=[3, 3],
    strides=[1, 1],
    pads=[1, 1, 1, 1],
    dilations=[1, 1],
    group=1,
)

# Relu — ReLU 激活
# 输入: X  输出: Y
node = onnx.helper.make_node(
    "Relu",
    inputs=["X"],
    outputs=["Y"],
)

# BatchNormalization — 批归一化
# 输入: X, scale, bias, running_mean, running_var
# 输出: Y, running_mean, running_var, saved_mean, saved_var
node = onnx.helper.make_node(
    "BatchNormalization",
    inputs=["X", "scale", "bias", "mean", "var"],
    outputs=["Y"],
    epsilon=1e-5,
    momentum=0.9,
)

# MaxPool / AveragePool — 池化
node = onnx.helper.make_node(
    "MaxPool",
    inputs=["X"],
    outputs=["Y"],
    kernel_shape=[2, 2],
    strides=[2, 2],
    pads=[0, 0, 0, 0],
)

形状操作算子

# Reshape — 重塑形状
# 输入: data, shape  输出: reshaped
node = onnx.helper.make_node(
    "Reshape",
    inputs=["data", "shape"],
    outputs=["reshaped"],
)

# Transpose — 转置/维度重排
# perm 指定输出各维度对应输入维度的索引
node = onnx.helper.make_node(
    "Transpose",
    inputs=["X"],
    outputs=["Y"],
    perm=[0, 2, 3, 1],  # NCHW -> NHWC
)

# Concat — 沿指定轴拼接
node = onnx.helper.make_node(
    "Concat",
    inputs=["A", "B", "C"],
    outputs=["concat"],
    axis=1,
)

# Flatten — 展平（从指定轴开始）
node = onnx.helper.make_node(
    "Flatten",
    inputs=["X"],
    outputs=["Y"],
    axis=1,  # (N, C, H, W) -> (N, C*H*W)
)

# Slice — 切片
# 输入: data, starts, ends, axes, steps
node = onnx.helper.make_node(
    "Slice",
    inputs=["X", "starts", "ends", "axes", "steps"],
    outputs=["Y"],
)

# Gather / Unsqueeze — 索引/扩展维度
node = onnx.helper.make_node("Unsqueeze",
    inputs=["X", "axes"], outputs=["Y"])

归约与归一化算子

# Softmax — 归一化指数
node = onnx.helper.make_node(
    "Softmax",
    inputs=["X"],
    outputs=["Y"],
    axis=1,
)

# GlobalAveragePool — 全局平均池化
node = onnx.helper.make_node(
    "GlobalAveragePool",
    inputs=["X"],
    outputs=["Y"],
)

# ReduceMean — 归约求均值
node = onnx.helper.make_node(
    "ReduceMean",
    inputs=["X"],
    outputs=["Y"],
    axes=[2, 3],
    keepdims=1,
)

# MatMul — 矩阵乘（常用于全连接层）
node = onnx.helper.make_node(
    "MatMul",
    inputs=["A", "B"],
    outputs=["Y"],
)

# Add / Sub / Mul — 逐元素算术运算
node = onnx.helper.make_node("Add",
    inputs=["A", "B"], outputs=["C"])
算子版本:
每个算子在不同 opset 中可能有不同的输入/输出签名或新增属性。opset 19+ 的迭代重点是支持更动态的形状、更丰富的数据类型和更灵活的注意力机制。建议在导出时使用目标部署环境支持的最高 opset 版本。

04 PyTorch 导出 ONNX

torch.onnx.export 是 PyTorch 导出 ONNX 模型的核心 API。它通过 tracing（追踪）或 scripting（脚本化）将 PyTorch 模型转换为 ONNX 图。

基本导出

import torch
import torch.onnx

class SimpleModel(torch.nn.Module):
    def __init__(self):
        super().__init__()
        self.conv = torch.nn.Conv2d(3, 16, 3, padding=1)
        self.relu = torch.nn.ReLU()
        self.pool = torch.nn.AdaptiveAvgPool2d(1)
        self.fc = torch.nn.Linear(16, 10)

    def forward(self, x):
        x = self.conv(x)
        x = self.relu(x)
        x = self.pool(x)
        x = x.view(x.size(0), -1)
        x = self.fc(x)
        return x

model = SimpleModel()
model.eval()

# 创建示例输入
dummy_input = torch.randn(1, 3, 32, 32)

# 基本导出
torch.onnx.export(
    model,                      # PyTorch 模型
    dummy_input,                # 示例输入（用于 tracing）
    "simple_model.onnx",        # 输出文件路径
    export_params=True,         # 导出训练好的权重
    opset_version=19,           # ONNX opset 版本
    do_constant_folding=True,   # 常量折叠优化
    input_names=["input"],      # 输入名称
    output_names=["output"],    # 输出名称
    verbose=False,
)

动态轴（Dynamic Axes）

# 生产环境中输入 batch size 和分辨率通常是动态的
# 通过 dynamic_axes 参数指定哪些维度是动态的

torch.onnx.export(
    model,
    dummy_input,
    "dynamic_model.onnx",
    opset_version=19,
    input_names=["input"],
    output_names=["output"],
    dynamic_axes={
        "input": {
            0: "batch_size",    # 动态 batch
            2: "height",        # 动态高度
            3: "width",         # 动态宽度
        },
        "output": {
            0: "batch_size",    # 输出 batch 与输入一致
        },
    },
)

自定义算子与符号函数

# 对于 PyTorch 未注册 ONNX 转换的算子
# 需要手动注册符号函数

from torch.onnx import register_custom_op_symbolic

def my_op_symbolic(g, input, alpha):
    """自定义算子映射到 ONNX LeakyRelu"""
    return g.op("LeakyRelu", input, alpha_f=alpha)

register_custom_op_symbolic(
    "aten::leaky_relu", my_op_symbolic, opset=19
)

# 更复杂的自定义映射
from torch.onnx.symbolic_helper import _parse_args

@_parse_args("v", "f", "i")
def my_custom_op(g, input, scale_factor, mode):
    """将自定义 resize 映射到 ONNX Resize 算子"""
    roi = g.op("Constant", value_t=torch.tensor([]))
    scales = g.op("Constant",
                  value_t=torch.tensor([1, 1, scale_factor,
                                        scale_factor]))
    return g.op("Resize", input, roi, scales,
                coordinate_transformation_mode="asymmetric")

导出完整检测模型

# 导出 torchvision 检测模型
import torchvision.models.detection as det

model = det.fasterrcnn_resnet50_fpn(
    weights=det.FasterRCNN_ResNet50_FPN_Weights.COCO_V1
)
model.eval()

dummy_input = torch.randn(1, 3, 800, 800)

# 检测模型输出是字典，需要适配
class DetectionWrapper(torch.nn.Module):
    def __init__(self, model):
        super().__init__()
        self.model = model

    def forward(self, x):
        out = self.model(x)
        # 将字典输出转换为元组
        return out["boxes"], out["scores"], out["labels"]

wrapped = DetectionWrapper(model)

torch.onnx.export(
    wrapped,
    dummy_input,
    "faster_rcnn.onnx",
    opset_version=17,          # 检测模型建议 opset 16/17
    input_names=["input"],
    output_names=["boxes", "scores", "labels"],
    dynamic_axes={
        "input": {0: "batch_size"},
        "boxes": {0: "num_detections"},
        "scores": {0: "num_detections"},
        "labels": {0: "num_detections"},
    },
)
导出注意事项:
导出前务必调用 model.eval() 禁用 dropout 和 batch norm 的训练行为。
确保 opset 版本与目标推理后端兼容。
控制流（if/for）不会在 tracing 中被完整导出，需使用 torch.jit.script 或 torch.jit.trace 的 scripting 模式。
动态形状需要额外测试以确保正确性。

05 TensorFlow 导出 ONNX

TensorFlow 模型可以通过 tf2onnx 工具导出为 ONNX 格式，支持 Keras SavedModel 和 concrete function 等多种输入格式。

安装 tf2onnx

# 安装 tf2onnx
pip install tf2onnx onnx onnxruntime

# 验证安装
python -c "import tf2onnx; print(tf2onnx.__version__)"

命令行转换

# 从 SavedModel 转换
python -m tf2onnx.convert \
  --saved-model saved_model/my_model \
  --output model.onnx \
  --opset 19 \
  --inputs "serving_default_input:0[1,224,224,3]" \
  --outputs "StatefulPartitionedCall:0" \
  --verbose

# 从 Keras H5 文件转换
python -m tf2onnx.convert \
  --keras model.h5 \
  --output model.onnx \
  --opset 18

# 从 checkpoint 转换
python -m tf2onnx.convert \
  --checkpoint training/ckpt-1000 \
  --output model.onnx \
  --inputs "input:0" \
  --outputs "output:0" \
  --inputs-as-nchw "input:0"  # NHWC -> NCHW 转换

Python API 转换

import tf2onnx
import tensorflow as tf

# 加载 Keras 模型
model = tf.keras.models.load_model("model.h5")

# 指定输入签名
spec = (tf.TensorSpec(
    (None, 224, 224, 3), tf.float32, name="input"
),)

# 转换
output_path = "model.onnx"
model_proto, _ = tf2onnx.convert.from_keras(
    model,
    input_signature=spec,
    opset=19,
    output_path=output_path,
)

print(f"Model converted: {output_path}")
print(f"Graph inputs: {[i.name for i in model_proto.graph.input]}")
print(f"Graph outputs: {[o.name for o in model_proto.graph.output]}")

# 从 concrete function 转换
@tf.function
def predict(x):
    return model(x, training=False)

concrete_fn = predict.get_concrete_function(
    tf.TensorSpec((None, 224, 224, 3), tf.float32)
)

model_proto, _ = tf2onnx.convert.from_function(
    concrete_fn,
    opset=19,
    output_path="model_fn.onnx",
)

NCHW 与 NHWC 布局

# TensorFlow 使用 NHWC 格式，ONNX 标准使用 NCHW
# tf2onnx 默认会处理布局转换，但某些情况需要手动干预

# 方法 1：让 tf2onnx 自动处理 NCHW 转换
# 使用 inputs-as-nchw 标记输入
python -m tf2onnx.convert \
  --saved-model model \
  --output model.onnx \
  --inputs-as-nchw "input:0"

# 方法 2：在模型层面转换
model = tf.keras.Sequential([
    tf.keras.layers.InputLayer(input_shape=(224, 224, 3)),
    tf.keras.layers.Lambda(
        lambda x: tf.transpose(x, [0, 3, 1, 2]),
        name="to_nchw"
    ),
    # ... 后续层按 NCHW 处理
    tf.keras.layers.Lambda(
        lambda x: tf.transpose(x, [0, 2, 3, 1]),
        name="to_nhwc"
    ),
])

模型验证

import onnx
import onnxruntime as ort
import numpy as np

# 加载 ONNX 模型
onnx_model = onnx.load("model.onnx")
onnx.checker.check_model(onnx_model)

# 使用 ONNX Runtime 验证
sess = ort.InferenceSession("model.onnx")
input_name = sess.get_inputs()[0].name
output_name = sess.get_outputs()[0].name

# 准备测试数据
test_input = np.random.randn(1, 3, 224, 224).astype(np.float32)

# ONNX 推理
onnx_output = sess.run([output_name],
                       {input_name: test_input})[0]

# 与原始 TensorFlow 输出对比
tf_output = model.predict(
    np.transpose(test_input, [0, 2, 3, 1])
)

# 计算误差
diff = np.abs(onnx_output - tf_output).max()
print(f"Max difference: {diff:.6f}")
验证建议:
转换后务必运行 onnx.checker.check_model() 验证模型结构正确性。
使用 ONNX Runtime 进行推理与原始框架输出对比，差异应小于 1e-5。
若输出差异较大，检查 layout 转换（NHWC vs NCHW）和算子兼容性。

07 opset 版本兼容性

opset（算子集版本）是 ONNX 规范演进的核心标志。每个新版本可能引入新算子、修改现有算子签名或废弃旧算子。版本选择直接决定了 ONNX 模型的可移植性和导出成功概率。

opset 版本演进

# 关键版本里程碑
# opset 7   (2018)  -- LSTM/GRU 支持，BatchNorm 升级
# opset 9   (2018)  -- 新增空间注意力、Pad 升级
# opset 10  (2019)  -- 新增 Resize、ReverseSequence
# opset 11  (2020)  -- Resize 重大升级（4个输入版本）
# opset 12  (2020)  -- 新增 Round、QLinearMatMul

opset 选择建议

# 部署环境 vs opset 版本

# TensorRT 8.x  -- opset 16+（推荐 opset 16）
# TensorRT 9.x  -- opset 19+（推荐 opset 18）
# ONNX Runtime 1.15+ -- opset 19+
# ONNX Runtime Mobile -- opset 16（算子有限集）
# PyTorch 2.0 -- opset 17 默认
# PyTorch 2.1+ -- opset 19 默认

def choose_opset(target_backend):
    """根据目标推理后端选择 opset 版本"""
    if target_backend == "tensorrt":
        return 16  # TensorRT 8.x 支持最稳定
    elif target_backend == "onnxruntime_gpu":
        return 19  # 最新 opset 有更好优化
    elif target_backend == "onnxruntime_mobile":
        return 16  # 移动端算子集有限
    elif target_backend == "openvino":
        return 17  # OpenVINO 支持
    elif target_backend == "coreml":
        return 13  # CoreML 兼容性
    else:
        return 18  # 通用推荐

    except Exception as e:
        print(f"opset {version}: FAIL - {e}")

Resize 算子在 opset 10 中只有 2 个输入，opset 11+ 要求 4 个输入（X, roi, scales, sizes）。

from torch.onnx import register_custom_op_symbolic

def resize_symbolic(g, input, size=None, scale_factor=None,
                    mode="nearest", align_corners=None):
    """自定义 interpolate 的 ONNX 映射"""
    if scale_factor is not None:
        scales = g.op("Constant",
                      value_t=torch.tensor(
                          [1.0, 1.0, scale_factor, scale_factor]
                      ))
    else:
        # 根据 target size 计算 scale
        h, w = size[0], size[1]
        scales = g.op("Div",
                      g.op("Cast", g.op("Shape", input),
                           to_i=onnx.TensorProto.FLOAT),
                      g.op("Shape", input))

    roi = g.op("Constant", value_t=torch.tensor([]))
    return g.op("Resize", input, roi, scales,
                coordinate_transformation_mode="asymmetric",
                cubic_coeff_a=-0.75,
                mode=mode,
                nearest_mode="floor")

register_custom_op_symbolic(
    "aten::upsample_nearest2d",
    resize_symbolic,
    opset=16
)

class BadModel(torch.nn.Module):
    def forward(self, x):
        if x.shape[2] > 100:  # 这个分支在 tracing 时只走一条
            x = self.branch_a(x)
        else:
            x = self.branch_b(x)
        return x

# 解决方案 1：使用 torch.jit.script

torch.onnx.export(
    model, dummy_input, "model.onnx",
    opset_version=17,
)

# 解决方案 2：使用 torch.where 替代 if
class GoodModel(torch.nn.Module):
    def forward(self, x):
        # 使用条件计算替代分支
        cond = (x.shape[2] > 100).float()
        a_out = self.branch_a(x)
        b_out = self.branch_b(x)
        return cond * a_out + (1 - cond) * b_out

def validate_onnx_compatibility(model_path, backend="onnxruntime"):
    """完整的 ONNX 兼容性检查"""
    import onnx
    import onnxruntime as ort
    import numpy as np

    # 1. 结构检查
    model = onnx.load(model_path)
    onnx.checker.check_model(model)

    # 2. opset 检查
    opset = model.opset_import[0].version
    print(f"opset: {opset}")

    # 3. 算子检查
    unsupported = check_operator_support(model, backend)
    if unsupported:
        print(f"WARNING: Unsupported ops: {unsupported}")

    # 4. ONNX Runtime 推理验证
    try:
        sess = ort.InferenceSession(model_path)
        for inp in sess.get_inputs():
            test = np.random.randn(*inp.shape).astype(
                np.float32
            )
            output = sess.run(None, {inp.name: test})
        print("ONNX Runtime inference: OK")
    except Exception as e:
        print(f"ONNX Runtime inference: FAIL - {e}")

    # 5. TensorRT 验证（可选）
    try:
        import tensorrt as trt
        logger = trt.Logger(trt.Logger.WARNING)
        builder = trt.Builder(logger)
        network = builder.create_network(
            1 << int(trt.NetworkDefinitionCreationFlag.EXPLICIT_BATCH)
        )
        parser = trt.OnnxParser(network, logger)
        with open(model_path, "rb") as f:
            parser.parse(f.read())
        print("TensorRT parser: OK")
    except Exception as e:
        print(f"TensorRT parser: FAIL - {e}")

    print("Validation complete")

常见问题速查表

问题速查:
Constant folding 失败 降低 opset 版本或关闭 do_constant_folding。
Resize coordinate_transformation_mode 设置 half_pixel 或 asymmetric。
TensorRT 不支持动态形状 使用固定输入形状或设置 optimization_profile。
onnxruntime 找不到输入 检查 input_names 是否匹配。
onnxsim 后精度变化 使用 --check-n 参数验证。
NHWC vs NCHW 维度混乱 确认 TensorFlow 的布局并启用 inputs-as-nchw。
超大模型加载慢 使用 onnx.load_external_data 分割外部数据。
