01 概述

Kernel Livepatch 允许在不重启系统的情况下修复内核漏洞。它通过运行时替换内核函数实现热补丁，避免了重启带来的服务中断。Kpatch（Red Hat）、kGraft（SUSE）和内核内置的 livepatch 框架都基于此机制。

02 架构

Livepatch 工作原理：

  补丁前：
  函数 vulnerable_func() 在内核中运行

  应用 Livepatch：

  1. 加载补丁模块
  │   包含修复后的函数 patched_func()
  │
  2. 注册补丁
  │   klp_enable_patch()
  │   记录：vulnerable_func → patched_func 的映射
  │
  3. 等待安全切换点
  │   所有执行 vulnerable_func 的任务完成
  │
  4. 修改函数入口
  │   vulnerable_func 的入口 NOP → JMP patched_func
  │
  5. 后续调用直接跳转到 patched_func

  安全切换机制（KLP）：
  - 使用栈检查确保没有任务在旧函数中
  - 设置切换点（stack check）
  - 所有任务通过切换点后才激活补丁

03 补丁流程

Livepatch 加载流程：

  管理员：katp install livepatch-5.10.0-15-foo.ko
        │
        ↓
  klp_enable_patch() — kernel/livepatch/core.c
        │
        ├── 1. 验证补丁
        │       检查符号版本匹配
        │       检查补丁函数签名
        │
        ├── 2. 注册补丁
        │       将补丁添加到全局补丁列表
        │       设置函数替换映射
        │
        ├── 3. 设置切换点
        │       klp_schedule_transition()
        │       等待所有任务到达安全点
        │
        ├── 4. 激活补丁
        │       修改函数入口 NOP → JMP
        │       所有新调用跳转到修复函数
        │
        └── 5. 完成
            补丁生效，无需重启

  管理员：katp remove livepatch-5.10.0-15-foo.ko
        │
        └── 回滚：修改函数入口 JMP → NOP
