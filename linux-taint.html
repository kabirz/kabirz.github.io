01 概述

当内核检测到可能影响稳定性的事件时，会将自己标记为"污染"（tainted）。污染是一个诊断标记，告诉技术支持人员内核可能不可靠。一个被污染的内核在提交 bug 报告时需要先清除污染状态。

02 污染标志

位标志说明
0TAINT_PROPRIETARY_MODULE加载了非 GPL 模块
1TAINT_FORCED_MODULE强制加载了不兼容模块
2TAINT_CPU_OUT_OF_SPECCPU 运行在规格外
3TAINT_MODULE模块被强制加载
4TAINT_UNSIGNED_MODULE加载了未签名模块
5TAINT_SOFTLOCKUP检测到软锁定
6TAINT_DIE内核 oops/panic
7TAINT_OVERRIDDEN_ACPI_TABLEACPI 表被覆盖
8TAINT_WARNWARN_ON 触发
9TAINT_CUSTOM_*厂商自定义标记

03 污染触发流程

污染触发示例：

  加载非 GPL 模块
  $ insmod proprietary_driver.ko
        │
        ↓
  mark_module_tainted(module)
        │
        ├── 检查 MODULE_LICENSE
        │   不是 "GPL" 或 "GPL v2" → 污染
        │
        ├── add_taint(TAINT_PROPRIETARY_MODULE, LOCKDEP_STILL_OK)
        │   设置 taint_flags 对应位
        │
        └── printk(KERN_WARNING "Loading module taints kernel")

  查看污染状态
  $ cat /proc/sys/kernel/tainted
  12288  # 二进制：11000000000000 = 位 12 + 位 13

  $ cat /proc/sys/kernel/tainted
  0  # 未污染

04 诊断与排查

# 查看内核污染状态
$ cat /proc/sys/kernel/tainted
0  # 0 = 干净，非 0 = 被污染

# 解释污染标志
$ echo 12288 | python3 -c "
import sys
taint = int(sys.stdin.read())
flags = {
    0: 'proprietary_module', 1: 'forced_module',
    2: 'cpu_out_of_spec', 4: 'unsigned_module',
    5: 'softlockup', 6: 'die', 8: 'warn',
}
for bit, name in sorted(flags.items()):
    if taint & (1 << bit):
        print(f'  Bit {bit}: {name}')
"

# 清除污染（仅调试用）
$ echo 0 > /proc/sys/kernel/tainted  # 不推荐在生产环境

# 内核配置
CONFIG_MODULE_SIG=n    # 禁用模块签名（会污染）
CONFIG_MODULE_SIG_FORCE=y  # 强制模块签名（防止未签名污染）
