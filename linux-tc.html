01 概述

Traffic Control（tc）是 Linux 内核中的流量控制框架，位于网络设备层和协议栈之间。它控制数据包的发送顺序和速率，实现 QoS（服务质量）、带宽限制、流量整形、优先级调度等功能。tc 框架总计 63124 行代码，是内核网络栈中最庞大的子系统之一。

tc 核心能力
  • QoS：保证关键流量的带宽
  • 限速：限制接口发送速率
  • 整形：平滑突发流量
  • 调度：多队列优先级调度
  • 过滤：按流分类处理
  • 丢包策略：智能丢包（如 RED、CoDel）
核心概念
组件说明
Qdisc队列规则（调度器）
Class分类（逻辑子队列）
Filter过滤器（按条件分类）
Action动作（对匹配包的处理）

struct Qdisc {
    int             (*enqueue)(struct sk_buff *skb,
                               struct Qdisc *sch,

    struct sk_buff *(*dequeue)(struct Qdisc *sch); // ★ 出队
    u32             limit;       // 队列长度上限

};

    const struct Qdisc_class_ops *cl_ops;

04 Qdisc 调度器

Qdisc 是 tc 的核心组件，决定数据包的发送顺序和速率。

常见 Qdisc 类型

Qdisc类型功能适用场景
pfifo_fast无类3 队列优先级 FIFO默认调度器
fq无类Fair Queuing（公平队列）Google BBR 推荐
fq_codel无类FQ + CoDel 主动队列管理抗 bufferbloat
tbf无类Token Bucket Filter（令牌桶限速）带宽限制
htb有类Hierarchy Token Bucket（层级令牌桶）精细 QoS
prio有类多优先级队列简单优先级
sfq无类Stochastic Fair Queuing公平带宽
red无类Random Early Detection（随机早期丢弃）抗拥塞

有类 vs 无类 Qdisc

无类 Qdisc（简单）：

  所有数据包进入同一个队列
  按统一策略处理（FIFO/限速/公平）

  tbf (Token Bucket Filter)：
  ┌──────────────────────────────────────┐
  │  令牌桶：令牌以固定速率生成            │
  │  包需要获取令牌才能发送                │
  │  桶满 → 令牌被丢弃（限速）            │
  │  桶空 → 包等待或丢弃                  │
  └──────────────────────────────────────┘

有类 Qdisc（精细）：

  根 Qdisc 可以有多个子类（Class）
  每个 Class 可以有独立的 Qdisc
  过滤器决定包进入哪个 Class

  htb (Hierarchy Token Bucket)：
  ┌──────────────────────────────────────────┐
  │           Root Qdisc (htb)               │
  │     ┌──────────┬──────────┬──────────┐   │
  │     │ Class 1  │ Class 2  │ Class 3  │   │
  │     │ 10Mbps   │ 80Mbps   │ 10Mbps   │   │
  │     │ (保证)   │ (借用)   │ (低优先) │   │
  │     └────┬─────┘──────────┘──────────┘   │
  │          │                                │
  │     ┌────┴────┐                          │
  │     │ qdisc   │                          │
  │     │ (sfq)   │                          │
  │     └─────────┘                          │
  └──────────────────────────────────────────┘

05 分类器与动作

数据包分类流程：

  数据包进入 Qdisc
        │
        ↓
  Filter 分类（匹配条件）
        │
        ├── u32 匹配：IP 头/端口/协议
        │       tc filter add dev eth0 parent 1:0 protocol ip u32
        │       match ip dport 80 0xffff flowid 1:10
        │
        ├── flower 匹配：5 元组 + MAC + VLAN
        │       tc filter add dev eth0 parent 1:0 protocol ip flower
        │       dst_port 80 ip_proto tcp flowid 1:10
        │
        └── bpf 匹配：自定义 BPF 程序
                tc filter add dev eth0 parent 1:0 bpf da
                obj my_filter.o sec classifier

Action 处理（对匹配包的操作）：

  → mirred：重定向到其他设备（镜像）
  → vlan：修改 VLAN 标签
  → pedit：修改包头字段
  → gact：丢弃/通过/随机
  → police：限速
  → bpf：执行 BPF 程序

06 应用场景

带宽限制
# 限制 eth0 下行 100Mbps
tc qdisc add dev eth0 root tbf \
  rate 100mbit burst 64kb latency 50ms
优先级队列
# SSH 优先，HTTP 次之，其他最低
tc qdisc add dev eth0 root handle 1: prio
tc filter add dev eth0 parent 1:0 protocol ip u32 \
  match ip dport 22 0xffff flowid 1:1
tc filter add dev eth0 parent 1:0 protocol ip u32 \
  match ip dport 80 0xffff flowid 1:2
抗 Bufferbloat
# 使用 fq_codel 替代默认 qdisc
tc qdisc replace dev eth0 root fq_codel
# fq_codel 主动管理队列深度
# 防止缓冲区膨胀导致的高延迟
流量镜像
# 镜像所有流量到 tap0（抓包）
tc qdisc add dev eth0 ingress
tc filter add dev eth0 parent ffff: \
  protocol all u32 match u32 0 0 \
  action mirred egress mirror dev tap0

07 配置命令

# 查看 qdisc
$ tc qdisc show dev eth0

# 查看 filter
$ tc filter show dev eth0

# 查看 class
$ tc class show dev eth0

# 添加 qdisc
$ tc qdisc add dev eth0 root handle 1: prio

# 删除 qdisc
$ tc qdisc del dev eth0 root

# 替换 qdisc
$ tc qdisc replace dev eth0 root fq_codel

# 添加 filter
$ tc filter add dev eth0 parent 1:0 protocol ip \
  u32 match ip dport 22 0xffff flowid 1:1

# 查看统计
$ tc -s qdisc show dev eth0
