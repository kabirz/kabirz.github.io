01 概述

Linux 内核提供了丰富的调试工具，从最简单的 printk 到复杂的 KASAN/KCSAN。选择正确的调试工具是高效排查内核问题的关键。

02 调试工具

工具用途开销
printk内核日志输出低
pr_debug条件编译调试零（编译时关闭）
dynamic_debug运行时动态调试低
ftrace函数跟踪中
tracepoints静态跟踪点低（未启用时零开销）
perf性能分析低
KASAN内存错误检测高
KCSAN数据竞争检测高
lockdep死锁检测中
KGDB内核调试器高
crash转储分析离线

03 调试流程

内核调试决策树：

  问题类型？
  │
  ├── 内存错误（use-after-free/overflow）→ KASAN
  ├── 数据竞争 → KCSAN
  ├── 死锁 → lockdep
  ├── 性能瓶颈 → perf + ftrace
  ├── 函数调用追踪 → ftrace
  ├── 事件追踪 → tracepoints + perf
  ├── 内核崩溃 → Kdump + crash
  ├── 驱动调试 → debugfs + printk
  └── 安全审计 → audit + auditd

04 场景选择

# printk — 最基本的调试
$ dmesg -T              # 查看内核日志（带时间戳）
$ dmesg -w              # 实时监控
$ journalctl -k         # systemd 查看内核日志

# dynamic_debug — 运行时调试
$ echo 'file drivers/net/veth.c +p' > /sys/kernel/debug/dynamic_debug/control
$ dmesg | tail -20      # 查看调试输出

# KASAN — 内存错误
$ echo 1 > /sys/kernel/debug/kasan/enable

# lockdep — 死锁检测
$ echo 1 > /sys/kernel/debug/lockdep/lock_stat

# ftrace — 函数跟踪
$ echo function > /sys/kernel/debug/tracing/current_tracer
$ echo 1 > /sys/kernel/debug/tracing/tracing_on
