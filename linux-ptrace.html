01 概述

ptrace（process trace）是 Linux 提供的进程跟踪系统调用。它允许一个进程（tracer）观察和控制另一个进程（tracee）的执行，包括读写内存、寄存器、拦截系统调用。strace、ltrace、gdb 的内核基础都是 ptrace。

02 ptrace API

#include <sys/ptrace.h>

// 请求类型
PTRACE_TRACEME      // 子进程请求被跟踪（通常在 exec 前调用）
PTRACE_ATTACH       // 附加到已运行的进程
PTRACE_DETACH       // 分离，恢复被跟踪进程
PTRACE_PEEKTEXT     // 读取被跟踪进程的内存
PTRACE_POKETEXT     // 写入被跟踪进程的内存
PTRACE_GETREGS      // 获取寄存器
PTRACE_SETREGS      // 设置寄存器
PTRACE_SINGLESTEP   // 单步执行（执行一条指令）
PTRACE_SYSCALL      // 在系统调用入口/出口停止
PTRACE_CONT         // 继续执行
PTRACE_KILL         // 终止被跟踪进程

// 示例：strace 简化版
if (fork() == 0) {
    ptrace(PTRACE_TRACEME, 0, NULL, NULL);
    execlp("ls", "ls", NULL);
} else {
    int status;
    wait(&status);
    while (!WIFEXITED(status)) {
        ptrace(PTRACE_SYSCALL, child, NULL, NULL);
        wait(&status);
        // 在系统调用入口/出口获得控制
    }
}

03 跟踪流程

ptrace 跟踪流程：

  tracer 进程（如 strace）        tracee 进程（如 ls）
  │                                │
  ├── ptrace(PTRACE_TRACEME)       │
  │   标记为被跟踪                  │
  │                                ├── execve("ls")
  │                                │   触发 SIGTRAP
  │   wait() ← 等待               │
  │                                │
  ├── ptrace(PTRACE_SYSCALL)       │
  │   让 tracee 继续执行            │
  │                                ├── 执行系统调用
  │                                │   触发 SIGTRAP（入口）
  │   wait() ← 获得控制            │
  ├── 获取系统调用号               │
  ├── ptrace(PTRACE_SYSCALL)       │
  │                                ├── 执行系统调用
  │                                │   触发 SIGTRAP（出口）
  │   wait() ← 获得控制            │
  ├── 获取返回值                   │
  └── 继续循环...

04 应用场景

工具用途ptrace 用法
