01 概述

FIFO（命名管道）是 Pipe 的文件系统版本。它在文件系统中创建一个特殊文件（类型为 p），任意进程（无需亲缘关系）通过 open 该文件即可建立通信。FIFO 的内核实现复用了 Pipe 的 pipe_inode_info，只是多了一层文件系统接口。

02 创建与通信流程

FIFO 通信流程：

  进程 A（写端）                         进程 B（读端）
  │                                      │
  ├── mkfifo("/tmp/myfifo", 0644)        │  // 创建 FIFO 文件
  │   → 在文件系统中创建特殊节点          │
  │   → inode 类型 = S_IFIFO             │
  │                                      │
  ├── fd = open("/tmp/myfifo", O_WRONLY) │
  │   → 阻塞直到有读端打开               │
  │                                      │ fd = open("/tmp/myfifo", O_RDONLY)
  │   → 连接建立！                        │
  │                                      │
  ├── write(fd, data, len)               │  // 写入数据
  │   → 数据进入内核 pipe buffer          │
  │                                      │ read(fd, buf, len)  // 读取数据
  │                                      │
  ├── close(fd)                          │ close(fd)
  │                                      │
  └── unlink("/tmp/myfifo")              │  // 删除 FIFO 文件

03 FIFO vs Pipe

特性Pipe (pipe())FIFO (mkfifo)
文件系统不可见可见（ls -l 显示 p 类型）
进程关系必须 fork 继承 fd任意进程（通过路径 open）
持久性进程退出即销毁需手动 unlink
阻塞默认阻塞默认阻塞（O_NONBLOCK 可改变）
内核实现pipe_inode_infopipe_inode_info（复用）
关键点：FIFO 底层完全复用了 Pipe 的实现（pipe_inode_info），只是在文件系统层添加了 open/close 路径。打开 FIFO 时，内核创建一个 pipe 实例并关联到 inode。
