01 概述

Kernel Lockdown 是 Linux 内核的安全功能，限制内核自我修改的能力。即使以 root 权限运行，也无法加载未签名模块、修改内核映射、访问特定硬件寄存器。它是 UEFI 安全启动和 Secure Boot 的内核侧实现。

02 锁定模式

模式sysctl 值限制
0 (无锁定)kernel.lockdown=0不限制（默认）
1 (完整性)kernel.lockdown=1阻止修改内核映射和模块签名验证
2 (机密性)kernel.lockdown=2阻止所有内核自我修改
完整性模式（Integrity）限制：

  ✗ 禁止：加载未签名的内核模块
  ✗ 禁止：直接修改内核内存
  ✓ 允许：读取 /dev/mem（有限制）
  ✓ 允许：kexec 加载已签名内核
  ✓ 允许：休眠/挂起

机密性模式（Confidentiality）限制：

  ✗ 禁止：所有完整性模式的限制
  ✗ 禁止：访问 /dev/mem
  ✗ 禁止：访问 /dev/kmem
  ✗ 禁止：debugfs 中的敏感信息
  ✗ 禁止：性能计数器（perf_event）
  ✗ 禁止：kexec

03 检查流程

Lockdown 检查流程：

  内核代码尝试执行受限操作
        │
        ├── 尝试加载模块
        │   → security_locked_down(LOCKDOWN_MODULE_LOAD)
        │   → 检查模块签名
        │   → 未签名 → 返回 -EPERM
        │
        ├── 尝试访问 /dev/mem
        │   → security_locked_down(LOCKDOWN_DEV_MEM)
        │   → 机密性模式 → 返回 -EPERM
        │
        ├── 尝试修改内核映射
        │   → security_locked_down(LOCKDOWN_KCORE)
        │   → 返回 -EPERM
        │
        └── 尝试 perf_event
            → security_locked_down(LOCKDOWN_PERF)
            → 机密性模式 → 返回 -EPERM

04 应用场景

场景锁定模式说明
UEFI Secure Boot完整性确保只运行签名的内核
容器安全机密性防止容器逃逸攻击
DRM/KMS完整性防止 GPU 固件被篡改
合规环境机密性PCI-DSS、HIPAA 等要求
