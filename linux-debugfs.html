01 概述

debugfs 是 Linux 内核提供的调试专用虚拟文件系统，挂载在 /sys/kernel/debug。它允许驱动开发者暴露任意调试信息和控制接口，不受 sysfs 的结构化约束。生产环境通常关闭 debugfs 以减少攻击面。

03 API 与使用

// debugfs API
#include <linux/debugfs.h>

// 创建目录
struct dentry *dir = debugfs_create_dir("my_driver", NULL);

// 创建文件（可读写）
u32 counter = 0;
debugfs_create_u32("counter", 0644, dir, &counter);

// 创建文件（只读）
debugfs_create_u32("status", 0444, dir, &status);

// 创建 blob 文件（大块数据）
struct debugfs_blob_wrapper blob = {
    .data = my_buffer,
    .size = my_buffer_size,
};
debugfs_create_blob("dump", 0444, dir, &blob);

// 创建符号链接
debugfs_create_symlink("link", dir, "/sys/class/net/eth0");

// 清理
debugfs_remove_recursive(dir);

04 应用场景

# 查看 debugfs 内容
$ sudo mount -t debugfs none /sys/kernel/debug
$ ls /sys/kernel/debug/
$ ls /sys/kernel/debug/dri/      # GPU 驱动调试
$ ls /sys/kernel/debug/btrfs/    # Btrfs 调试
$ ls /sys/kernel/debug/tracing/  # ftrace（已在 ftrace 文章介绍）

典型用途
