01 概述

信号量（Semaphore）是一种可计数的睡眠锁，允许多个持有者（计数信号量）或作为排他锁使用。在 Linux 内核中，基本信号量已较少使用，但其思想体现在 rwsem（读写信号量）和 percpu-rwsem 中。

核心特性
  • 可计数：计数器 > 1 允许多个持有者
  • 可睡眠：等待时不消耗 CPU
  • 无 owner：不记录持有者（与 mutex 不同）
  • 可从中断调用：down_trylock()

03 核心 API

// 初始化
sema_init(&sem, 1);     // 计数=1（排他）
sema_init(&sem, 3);     // 计数=3（3个持有者）

// 获取/释放
down(&sem);              // 获取（计数-1）
up(&sem);                // 释放（计数+1）

// 可中断版本
int ret = down_interruptible(&sem);

// 非阻塞版本
int ret = down_trylock(&sem);  // 可在中断中使用

// 超时版本
int ret = down_timeout(&sem, 100);  // 100 jiffies
注意：新代码应优先使用 mutex（排他）或 rw_semaphore（读写）。基本 semaphore 已较少直接使用，主要通过 rwsem 间接使用。

04 读写信号量

rw_semaphore 是最重要的信号量变体，允许多个读者同时持有，写者独占。

// 初始化
static DEFINE_RWSEM(my_rwsem);

// 读者（可睡眠）
down_read(&my_rwsem);
// ... 只读操作 ...
up_read(&my_rwsem);

// 写者（可睡眠）
down_write(&my_rwsem);
// ... 写操作 ...
up_write(&my_rwsem);

// 尝试获取
int ret = down_read_trylock(&my_rwsem);
int ret = down_write_trylock(&my_rwsem);

05 使用模式

// rw_semaphore：mmap_lock 典型用法
down_read(&mm->mmap_lock);     // 遍历 VMA
for_each_vma(vma) {
    // 只读遍历 VMA 链表
}
up_read(&mm->mmap_lock);

down_write(&mm->mmap_lock);    // 修改 VMA
