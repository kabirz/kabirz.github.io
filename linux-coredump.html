01 概述

Coredump 是进程崩溃时内核将进程内存映像保存到文件的机制。当进程收到 SIGSEGV（段错误）等致命信号时，内核可以生成 core 文件，包含进程的内存内容、寄存器状态和堆栈信息。GDB 可以加载 core 文件进行事后分析（post-mortem debugging）。

02 Coredump 流程

Coredump 触发流程：

  进程收到致命信号（如 SIGSEGV）
        │
        ├── 检查是否生成 core dump
        │   ├── core_pattern 是否配置？
        │   ├── 进程的 RLIMIT_CORE 是否足够？
        │   ├── 进程是否有 CAP_SYS_RESOURCE？
        │   └── 是否被 seccomp 阻止？
        │
        ├── do_coredump() — fs/coredump.c
        │   │
        │   ├── 1. 打开 core 文件
        │   │       根据 core_pattern 决定路径
        │   │       /proc/sys/kernel/core_pattern
        │   │
        │   ├── 2. 写入 ELF 头
        │   │       ET_CORE 类型
        │   │
        │   ├── 3. 遍历所有 VMA（虚拟内存区域）
        │   │       对每个 VMA 生成 PT_LOAD 段
        │   │       包含内存内容
        │   │
        │   ├── 4. 写入寄存器状态
        │   │       NT_PRSTATUS（通用寄存器）
        │   │       NT_FPREGSET（浮点寄存器）
        │   │
        │   ├── 5. 写入辅助信息
        │   │       /proc/pid/maps（内存映射）
        │   │       /proc/pid/auxv（环境变量）
        │   │
        │   └── 6. 关闭文件
        │
        └── 进程被杀死

03 ELF Core 格式

ELF Core 文件结构：

  ┌──────────────────────────────┐
  │ ELF Header (ET_CORE)        │  文件类型标识
  ├──────────────────────────────┤
  │ Program Header 1            │  PT_NOTE（寄存器等）
  │ - NT_PRSTATUS                │  进程状态
  │ - NT_PRPSINFO               │  进程信息
  ├──────────────────────────────┤
  │ Program Header 2            │  PT_LOAD（.text 段）
  │ - addr: 0x400000            │  代码段内存内容
  ├──────────────────────────────┤
  │ Program Header 3            │  PT_LOAD（.data 段）
  │ - addr: 0x600000            │  数据段内存内容
  ├──────────────────────────────┤
  │ Program Header 4            │  PT_LOAD（.bss 段）
  │ - addr: 0x800000            │  BSS 段内存内容
  ├──────────────────────────────┤
  │ Program Header 5            │  PT_LOAD（栈）
  │ - addr: 0x7fff...           │  进程栈内存内容
  ├──────────────────────────────┤
  │ Program Header 6            │  PT_LOAD（堆）
  │ - addr: 0x5555...           │  进程堆内存内容
  ├──────────────────────────────┤
  │ VMA 数据区域                 │  实际内存内容
  └──────────────────────────────┘

04 配置与使用

# 启用 core dump
$ ulimit -c unlimited  # 设置 core 文件大小无限制

# 配置 core 文件名
$ cat /proc/sys/kernel/core_pattern
core

# 使用 systemd-coredump（推荐）
$ echo '|/usr/lib/systemd/systemd-coredump %P %g %u %t %e %h %s' \
  > /proc/sys/kernel/core_pattern

# 查看 core 文件列表
$ coredumpctl list

# 用 GDB 分析 core 文件
$ gdb /path/to/program /path/to/core
(gdb) bt              # 查看崩溃调用栈
(gdb) info registers  # 查看寄存器
(gdb) info locals     # 查看局部变量
(gdb) thread apply all bt  # 所有线程的调用栈

# 禁用 core dump（安全）
$ ulimit -c 0
$ echo '/proc/sys/kernel/core_pattern = /dev/null' > /etc/sysctl.d/core.conf

常见 core 文件位置

配置文件名
默认./core（当前目录）
systemd/var/lib/systemd/coredump/
自定义core.%e.%p（进程名.PID）
