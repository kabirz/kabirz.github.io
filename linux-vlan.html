01 概述

VLAN（Virtual Local Area Network）是 IEEE 802.1Q 标准定义的虚拟局域网技术。通过在以太网帧中插入 4 字节的 VLAN 标签，将一个物理网络划分为多个逻辑隔离的广播域。Linux 内核原生支持 VLAN，在数据中心和云环境中广泛使用。

VLAN 核心价值
  • 广播域隔离：不同 VLAN 间不互通
  • 安全性：流量隔离减少攻击面
  • 灵活性：逻辑划分不受物理位置限制
  • QoS：通过 VLAN 优先级字段实现流量分级
Linux VLAN 实现
特性说明
VLAN 过滤bridge 内置 VLAN filtering
802.1Q 嵌套支持 QinQ（双层标签）
硬件卸载支持网卡 VLAN offload
QoS 映射内核优先级 → VLAN PCP

02 VLAN 帧格式

802.1Q VLAN 标签

标准以太网帧：

+----------+----------+---------+-----+
| DST MAC  | SRC MAC  | Type    | Data|  FCS
| (6 字节) | (6 字节) | (2 字节)|     |
+----------+----------+---------+-----+

802.1Q VLAN 帧（插入 4 字节标签）：

+----------+----------+---------+----------+----------+---------+-----+
| DST MAC  | SRC MAC  | TPID    | TCI      | Type     | Data    | FCS
| (6 字节) | (6 字节) | 0x8100  | PCP|DEI|VID | (2 字节)|         |
+----------+----------+---------+----+---+----+----------+---------+
                           │     │   │    │
                           │     │   │    └── VLAN ID (12 bit, 0-4095)
                           │     │    └── Drop Eligible Indicator (1 bit)
                           │     └── Priority Code Point (3 bit, QoS)
                           └── Tag Protocol ID (0x8100)

内核结构 vlan_ethhdr：

+----------+----------+---------+----------+----------+
| dst      | h_dest   | h_source| h_vlan_proto | h_vlan_TCI | h_vlan_encapsulated_proto |
| (6)      | (6)      | (2)     | (2)          | (2)        |
+----------+----------+---------+--------------+------------+
  ether_header (14 bytes)     +  VLAN tag (4 bytes)   + payload

VLAN ID 范围

VID用途说明
0Priority Tag仅优先级，无 VLAN 隔离
1-4094用户 VLAN可用的 VLAN ID
4095保留不可使用

vlan_dev_priv.real_dev = eth0  ← 每个 VLAN 设备指向底层物理网卡

VLAN 过滤模式（bridge VLAN filtering）

传统 VLAN（端口-based）：

  VLAN 10 的包只能从 eth0.10 进出
  VLAN 20 的包只能从 eth0.20 进出

Bridge VLAN Filtering（802.1Q aware bridge）：

  一个 br0 接口上可以同时有多个 VLAN
  每个端口配置允许哪些 VLAN 通过
  不需要创建多个 VLAN 子接口

  $ ip link add br0 type bridge vlan_filtering 1
  $ bridge vlan add dev eth0 vid 10
  $ bridge vlan add dev eth0 vid 20
  $ bridge vlan add dev eth0 pvid untagged vid 10

04 核心数据结构

// include/linux/if_vlan.h:178 — VLAN 设备私有数据
struct vlan_dev_priv {
    unsigned int    nr_ingress_mappings;    // 入方向优先级映射数
    u32             ingress_priority_map[8]; // PCP → 内核优先级映射
    unsigned int    nr_egress_mappings;     // 出方向优先级映射数
    struct vlan_priority_tci_mapping __rcu
                    *egress_priority_map[16]; // 内核优先级 → PCP 映射

    __be16          vlan_proto;     // VLAN 协议（0x8100 / 0x88A8）
    u16             vlan_id;        // VLAN ID (0-4095)
    u16             flags;          // VLAN_REORDER_HDR / VLAN_GVRP 等

    struct net_device *real_dev;     // ★ 底层物理网卡
    unsigned char   real_dev_addr[ETH_ALEN]; // 物理网卡 MAC 地址

    struct vlan_pcpu_stats __percpu *vlan_pcpu_stats; // 统计
};

05 收发流程

发送流程

  应用发送数据到 eth0.10
        │
        ↓
  vlan_dev_hard_start_xmit() — net/8021q/vlan_dev.c
        │
        ├── 1. 检查 VLAN 标签
        │       如果 skb 已有 VLAN tag → 重新打标签
        │       否则 → 构造 4 字节 VLAN 头
        │
        ├── 2. 插入 VLAN 头
        │       __vlan_insert_tag() → 在以太网头中插入 TCI
        │
        ├── 3. 交给底层网卡
        │       dev_queue_xmit(skb, vlan_dev_priv->real_dev)
        │       → eth0 的 ndo_start_xmit
        │
        ↓
  物理网卡发送带 VLAN tag 的帧

接收流程

  物理网卡 eth0 收到带 VLAN tag 的帧
        │
        ↓
  网卡驱动检测到 VLAN tag
        │
        ├── 硬件卸载（VLAN RX offload）
        │   驱动剥离 VLAN tag，设置 skb->vlan_tci
        │
        └── 软件处理
            __vlan_hwaccel_rx() 或 __vlan_vlan_receive()
        │
        ↓
  __netif_receive_skb_core()
        │
        ├── 检查 skb->vlan_tci
        │
        ├── vlan_tci != 0 → 查找对应的 VLAN 设备
        │       vlan_find_dev()
        │       找到 eth0.10（vid=10）
        │       重新设置 skb->dev = eth0.10
        │
        ├── 再次进入 __netif_receive_skb_core()
        │       此时 skb->dev = eth0.10
        │       数据包交给 eth0.10 的协议栈
        │
        ↓
  应用从 eth0.10 读取数据（无 VLAN tag）
VLAN 标签处理的两种方式
① 硬件卸载：网卡自动剥离/添加 VLAN tag（高效，推荐）
② 软件处理：内核在 vlan_dev 中插入/剥离（通用，兼容所有网卡）

06 VLAN + Bridge

两种 VLAN + Bridge 模式

模式 1：VLAN 子接口 + Bridge（传统）

  需要为每个 VLAN 创建子接口
  每个 VLAN 独立的 bridge

  $ ip link add eth0.10 type vlan id 10
  $ ip link add br10 type bridge
  $ ip link set eth0.10 master br10

  缺点：N 个 VLAN → N 个 bridge，管理复杂

模式 2：Bridge VLAN Filtering（推荐）

  一个 bridge 管理所有 VLAN
  每个端口配置允许哪些 VLAN

  $ ip link add br0 type bridge vlan_filtering 1
  $ bridge vlan add dev eth0 vid 10 pvid untagged
  $ bridge vlan add dev eth0 vid 20
  $ bridge vlan add dev veth0 vid 10 pvid untagged
  $ bridge vlan add dev veth1 vid 20 pvid untagged

  优势：一个 bridge 管理所有 VLAN，简洁高效

Bridge VLAN Filtering 架构

┌────────────────────────────────────────────────────┐
│               br0 (vlan_filtering=1)                │
│                                                     │
│  Port: eth0          VLANs: 10, 20                  │
│  Port: veth0         VLANs: 10 (pvid, untagged)    │
│  Port: veth1         VLANs: 20 (pvid, untagged)    │
│                                                     │
│  eth0 收到 VLAN 10 的帧 → 转发到 veth0（strip tag） │
│  eth0 收到 VLAN 20 的帧 → 转发到 veth1（strip tag） │
│  veth0 发送的帧 → 打 VLAN 10 tag → 从 eth0 发出    │
│  veth1 发送的帧 → 打 VLAN 20 tag → 从 eth0 发出    │
└────────────────────────────────────────────────────┘

07 配置命令

# 创建 VLAN 子接口
$ ip link add link eth0 name eth0.10 type vlan id 10
$ ip addr add 10.0.0.1/24 dev eth0.10
$ ip link set eth0.10 up

# 查看 VLAN 信息
$ ip -d link show eth0.10
    vlan protocol 802.1Q id 10 

# Bridge VLAN Filtering
$ ip link add br0 type bridge vlan_filtering 1
$ ip link set eth0 master br0
$ ip link set veth0 master br0

# 添加 VLAN 到端口
$ bridge vlan add dev eth0 vid 10
$ bridge vlan add dev eth0 vid 20 pvid untagged
$ bridge vlan add dev veth0 vid 10 pvid untagged

# 查看 VLAN 表
$ bridge vlan show
port    vlan ids
eth0    10 PVID Egress Untagged
        20
veth0   10 PVID Egress Untagged
veth1   20 PVID Egress Untagged

# 删除 VLAN
$ bridge vlan del dev eth0 vid 10
$ ip link del eth0.10

# pvid：收到未标记帧时，打上此 VLAN ID
# untagged：发包时剥离 VLAN tag
