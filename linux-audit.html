01 概述

Linux Audit 是内核提供的安全审计框架，记录系统调用、文件访问、用户认证等安全相关事件。它是 Linux 安全合规（CIS、PCI-DSS、HIPAA）的基础工具，也是 SELinux、AppArmor 的审计后端。

02 审计事件类型

类型系统调用类别示例
SYSCALL系统调用open, execve, connect, kill
PATH文件路径文件访问的完整路径
EXECVE程序执行execve 的参数和环境变量
CWD当前工作目录进程的 cwd
SOCKADDR网络地址connect/sendto 的目标地址
CRED凭证UID/GID/SEUID/SEGID
USER_LOGIN用户登录登录/登出事件
CONFIG_CHANGE配置变更规则添加/删除

03 审计流程

审计事件产生流程：

  进程执行系统调用（如 open）
        │
        ↓
  内核检查是否有匹配的审计规则
        │
        ├── 无规则 → 正常执行，不记录
        │
        └── 有规则 → audit_syscall_entry()
                │
                ├── 1. 构造审计上下文
                │       audit_context：syscall号、参数、返回值
                │
                ├── 2. 记录相关信息
                │       PATH（文件路径）
                │       CWD（当前目录）
                │       SOCKADDR（网络地址）
                │
                ├── 3. 系统调用执行完毕
                │
                ├── 4. audit_syscall_exit()
                │       记录返回值和错误码
                │
                └── 5. 发送审计记录
                        → auditd 守护进程
                        → 写入 /var/log/audit/audit.log

04 使用方式

# 启动审计守护进程
$ sudo systemctl start auditd

# 添加审计规则
# 监控 /etc/passwd 的所有访问
$ sudo auditctl -w /etc/passwd -p rwa -k passwd_access

# 监控所有 execve 系统调用
$ sudo auditctl -a always,exit -F arch=b64 -S execve -k exec_log

# 监控特定用户的所有系统调用
$ sudo auditctl -a always,exit -F uid=1000 -F arch=b64 -S all -k user_action

# 查询审计日志
$ ausearch -k passwd_access -ts recent
$ ausearch -ua 1000 -ts today

# 生成审计报告
$ aureport --auth          # 认证报告
$ aureport --login         # 登录报告
$ aureport --failed        # 失败操作报告

# 删除审计规则
$ sudo auditctl -d -w /etc/passwd -p rwa -k passwd_access

# 查看当前规则
$ sudo auditctl -l
