01 概述

进程组和会话是 Linux 进程层级管理的两大概念。进程组是一组共享同一 PGID 的进程，用于信号广播（如 Ctrl+C 终止整个管道链）。会话是一组进程组的集合，关联到一个控制终端。它们是 shell 作业控制（job control）的基础。

02 层级结构

进程层级：

  会话（Session）
  ├── 会话 ID（SID）
  ├── 控制终端（tty/pts）
  └── 进程组列表
        ├── 前台进程组（终端前台）
        │     ├── 进程 A（PGID=100）
        │     └── 进程 B（PGID=100）
        │
        └── 后台进程组
              ├── 进程 C（PGID=200）
              └── 进程 D（PGID=200）

典型场景 — shell 管道：

  $ cat file | grep pattern | wc -l

  创建三个进程，共享同一 PGID
  ├── cat   (PID=100, PGID=100, SID=1)
  ├── grep  (PID=101, PGID=100, SID=1)
  └── wc    (PID=102, PGID=100, SID=1)

  Ctrl+C 发送 SIGINT 到 PGID=100 → 三个进程全部终止

03 信号投递与进程组

信号投递方式：

  kill(pid, SIGTERM)  → 发送给指定 PID
  kill(-pgid, SIGTERM) → 发送给整个进程组
  kill(0, SIGTERM)    → 发送给当前进程组所有进程

  Ctrl+C → 内核发送 SIGINT 到前台进程组
  Ctrl+Z → 内核发送 SIGTSTP 到前台进程组
  Ctrl+\ → 内核发送 SIGQUIT 到前台进程组

  tcsetpgrp() → 设置前台进程组
  → shell 执行前台命令时调用
  → 终端信号转发到该进程组

04 作业控制

# Shell 作业控制命令
$ sleep 100 &             # 后台运行（新进程组）
[1] 12345

$ jobs                      # 查看作业列表
[1]+ Running                 sleep 100 &

$ fg %1                     # 调到前台
$ Ctrl+Z                    # 挂起（SIGTSTP）
[1]+ Stopped                 sleep 100

$ bg %1                     # 放到后台继续运行
[1]+ Running                 sleep 100 &

$ kill %1                   # 终止作业

关键系统调用

系统调用功能
setpgid(pid, pgid)设置进程的进程组
getpgrp()获取当前进程组 ID
setsid()创建新会话（进程成为会话领导）
getsid(pid)获取进程的会话 ID
tcsetpgrp(fd, pgrp)设置前台进程组
