01 概述

Writeback（回写）是将脏页（dirty pages）从内存写回磁盘的机制。Linux 采用延迟写入策略：write() 仅将数据写入 Page Cache（标记为脏页），后台回写线程定期将脏页刷入磁盘。这是 Page Cache 性能优化的关键——写操作立即返回，磁盘 I/O 异步执行。

02 回写流程

脏页回写触发条件：

  ① 脏页比例超过 dirty_background_ratio（默认 10%）
     → 触发后台回写（background writeback）

  ② 脏页比例超过 dirty_ratio（默认 20%）
     → 触发前台回写（foreground writeback）
     → 写入进程被阻塞直到脏页减少

  ③ 脏页存活时间超过 dirty_expire_centisecs（默认 30 秒）
     → 定时回写

  ④ 用户调用 fsync() / fdatasync()
     → 立即回写指定文件的所有脏页

  ⑤ 用户调用 sync()
     → 立即回写所有脏页

回写执行流程：

  回写线程被唤醒
        │
        ├── 遍历所有脏 inode
        │   writeback_sb_inodes()
        │
        ├── 对每个脏 inode
        │   │
        │   ├── 调用 a_ops->writepages()
        │   │   文件系统特定的回写实现
        │   │
        │   ├── 提交 I/O 请求
        │   │   submit_bio() → Block Layer → 网卡驱动
        │   │
        │   └── 标记页面为非脏

        │
        └── 更新回写统计

03 pdflush 与回写线程

回写机制线程说明
flushflush-N:0每个 inode 一个回写线程
writebackwriteback后台回写线程
dirty_writebackdirty_writeback_centisecs定期唤醒回写
fsync进程上下文同步回写
pdflush 已废弃
Linux 2.6.32+ 取消了 pdflush，改为 per-inode 的 flush 线程。每个脏 inode 有独立的回写线程，避免了 pdflush 的全局锁竞争问题。

04 配置与调优

# 查看脏页统计
$ cat /proc/vmstat | grep -E "nr_dirty|nr_writeback"
nr_dirty     12345    # 当前脏页数
nr_writeback 678      # 正在回写的页数

# 调整回写参数
$ sysctl -w vm.dirty_ratio=40                # 脏页比例上限
$ sysctl -w vm.dirty_background_ratio=5     # 后台回写触发阈值
$ sysctl -w vm.dirty_expire_centisecs=3000  # 脏页最大存活时间
$ sysctl -w vm.dirty_writeback_centisecs=500 # 回写线程唤醒间隔

# 强制回写
$ sync                      # 回写所有脏页
$ echo 3 > /proc/sys/vm/drop_caches  # 释放 Page Cache

# 数据库服务器推荐配置
$ sysctl -w vm.dirty_ratio=10
$ sysctl -w vm.dirty_background_ratio=3
$ sysctl -w vm.dirty_expire_centisecs=500
