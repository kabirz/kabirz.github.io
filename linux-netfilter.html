01 概述

Netfilter 是 Linux 内核中的数据包处理框架，iptables 是其用户空间的配置工具。Netfilter 在协议栈的 IP 层设置了 5 个钩子点，允许内核模块在这些点注册回调函数，对经过的数据包进行修改、过滤或丢弃。

Netfilter 项目
  • 内核框架：net/netfilter/ (99623 行代码)
  • 5 个钩子：PRE_ROUTING / LOCAL_IN / FORWARD / LOCAL_OUT / POST_ROUTING
  • 表驱动：filter / nat / mangle / raw / security
  • 连接追踪：nf_conntrack 模块
  • 动态模块：自动加载 NAT、conntrack 等内核模块
iptables 工具
  • 用户空间程序：通过 Netlink 与内核通信
  • 规则管理：配置表、链、规则
  • 命令：iptables / ip6tables（IPv6）
  • nftables：下一代替代品（Linux 3.13+）

                                    +---------------+          +-----------------+

                                    +---------------+          +-----------------+

                                             ↓                          |
                                    +------------------+                 |
                                    |                  |                 |

                                    +--------------------+
                                             |
                                             ↓
                                           Outgoing
钩子位置触发条件
NF_INET_PRE_ROUTING数据包刚进入 IP 层所有进入的数据包（路由选择之前）
NF_INET_LOCAL_IN路由选择后，目的 IP 是本机发给本机的数据包
NF_INET_FORWARD路由选择后，目的 IP 不是本机转发的数据包
NF_INET_LOCAL_OUT本地程序发出的数据包刚到 IP 层本机发出（路由选择之前）
NF_INET_POST_ROUTING即将交给下层发送所有出去的数据包（路由选择之后）

数据包的三条路径

// 路径 1：本机收到的数据包（目的 IP 是本机）
PRE_ROUTING → LOCAL_IN → local processes

// 路径 2：转发的数据包（目的 IP 不是本机）
PRE_ROUTING → FORWARD → POST_ROUTING

// 路径 3：本机发出的数据包
LOCAL_OUT → POST_ROUTING

03 表（Tables）

iptables 用表来分类管理规则。每个表包含一组规则，根据功能划分：

表功能典型用途
filter数据包过滤防火墙（允许/拒绝访问）
nat网络地址转换SNAT / DNAT / MASQUERADE
mangle修改数据包头修改 TTL、TOS、MARK
raw跳过连接追踪高性能场景（跳过 conntrack）
securitySELinux 标记与 SELinux 模块配合

各表可注册的链

                    PRE_ROUTING    LOCAL_IN    FORWARD    LOCAL_OUT    POST_ROUTING
                    ──────────    ────────    ───────    ─────────    ────────────
raw                    ✓                                      ✓
conntrack              ✓             ✓           ✓          ✓              ✓
mangle                 ✓             ✓           ✓          ✓              ✓
nat (DNAT)             ✓                                      ✓
nat (SNAT)                          ✓                        ✓              ✓
filter                            ✓           ✓          ✓
security                          ✓           ✓          ✓

04 链（Chains）

链（Chain）决定了什么时候触发规则。每个链对应一个钩子点。

链对应钩子说明
PREROUTINGNF_IP_PRE_ROUTING所有进入的数据包
INPUTNF_IP_LOCAL_IN发给本机的数据包
FORWARDNF_IP_FORWARD转发的数据包
OUTPUTNF_IP_LOCAL_OUT本机发出的数据包
POSTROUTINGNF_IP_POST_ROUTING即将发送出去的数据包
用户自定义链
除了 5 个内置链，用户可以在表中定义自定义链。自定义链没有和钩子绑定，不会自动触发，只能从其他链的规则中跳转过来。这类似于函数调用——自定义链是函数，内置链是入口。

05 规则（Rules）

规则（Rule）存放在特定表的特定链上，由两部分组成：

匹配条件（Matching）
  • 协议类型（TCP/UDP/ICMP）
  • 源/目的 IP 地址
  • 源/目的端口
  • 输入/输出接口（eth0、br0）
  • 包头数据
  • 连接状态（ESTABLISHED / NEW）
目标动作（Target）
Target说明
ACCEPT放行数据包
DROP丢弃数据包
REJECT丢弃并返回 ICMP 错误
RETURN跳出当前链
QUEUE送往用户空间处理
SNAT源地址转换
DNAT目的地址转换
MASQUERADE动态 SNAT

06 表优先级与执行顺序

当多个表在同一个钩子上都有规则时，按优先级从高到低执行：

NF_IP_PRE_ROUTING 钩子上的执行顺序：

    ┌──────────────────────────────────────────┐
    │  1. raw.PREROUTING                       │ ← 最高优先级
    │  2. conntrack（自动，不可配置）             │
    │  3. mangle.PREROUTING                    │
    │  4. nat.DNAT                             │ ← 最低优先级
    └──────────────────────────────────────────┘

NF_IP_LOCAL_IN 钩子上的执行顺序：

    ┌──────────────────────────────────────────┐
    │  1. mangle.INPUT                         │
    │  2. filter.INPUT                         │
    │  3. security.INPUT                       │
    │  4. nat.SNAT                             │
    └──────────────────────────────────────────┘

NF_IP_LOCAL_OUT 钩子上的执行顺序：

    ┌──────────────────────────────────────────┐
    │  1. raw.OUTPUT                           │
    │  2. conntrack（自动）                     │
    │  3. mangle.OUTPUT                        │
    │  4. nat.DNAT                             │
    │  5. filter.OUTPUT                        │
    │  6. security.OUTPUT                      │
    └──────────────────────────────────────────┘

完整执行路径（本机收到数据包）

数据包到达 PRE_ROUTING：
    → raw.PREROUTING      (1) 检查是否跳过 conntrack
    → conntrack            (2) 记录连接状态
    → mangle.PREROUTING   (3) 修改包头（如 MARK）
    → nat.DNAT            (4) 目的地址转换

路由选择 → 目的 IP 是本机 → 进入 LOCAL_IN：

    → mangle.INPUT        (5) 修改包头
    → filter.INPUT        (6) 防火墙过滤（ACCEPT/DROP）
    → security.INPUT      (7) SELinux 检查
    → nat.SNAT            (8) 源地址转换

→ 交给本地进程

07 连接追踪（Connection Tracking）

Conntrack 是 Netfilter 最强大也最消耗资源的功能之一。它追踪所有经过的数据包的连接状态（不仅是 TCP，UDP 和 ICMP 也被视为"连接"）。

连接状态

状态说明示例
NEW新检测到的连接的第一个包TCP SYN 包、任意 UDP 包
ESTABLISHED连接已建立（收到反方向的包）TCP 收到 SYN/ACK、UDP 收到回复
RELATED与已有 ESTABLISHED 连接相关的新连接FTP 数据连接、ICMP 错误
INVALID不属于任何连接，且不是合法的新连接无关联的 TCP RST
UNTRACKED被 raw 表标记为不追踪raw 表中 NOTRACK 规则
Conntrack 在协议栈中的位置：

PRE_ROUTING  ──→  conntrack  ──→  mangle  ──→  nat.DNAT
                      │
                      │ 检查数据包是否匹配已有连接
                      │
                      ├── 匹配 → 更新连接状态
                      ├── NEW   → 创建新连接记录
                      └── INVALID → 标记为无效

LOCAL_OUT  ──→  conntrack  ──→  mangle  ──→  nat.DNAT  ──→  filter
查看连接追踪
cat /proc/net/nf_conntrack — 查看所有追踪的连接
conntrack -L — 用户空间查看工具
iptables -t raw -A PREROUTING -p tcp --dport 80 -j NOTRACK — 跳过追踪

08 内核实现

nf_hook_ops — 钩子操作结构

// include/linux/netfilter.h:98
struct nf_hook_ops {
    struct list_head    list;       // 链表节点
    struct rcu_head      rcu;        // RCU 释放
    nf_hookfn           *hook;       // 回调函数指针
    struct net_device   *dev;        // 绑定的网络设备
    void                *priv;       // 私有数据
    u8                  pf;          // 协议族（AF_INET / AF_INET6）
    unsigned int        hooknum;     // 钩子编号（NF_INET_PRE_ROUTING 等）
    int                 priority;    // 优先级（越小越先执行）
};

// 钩子回调函数签名
typedef unsigned int (*nf_hookfn)(
    unsigned int hooknum,           // 钩子编号
    struct sk_buff *skb,            // 数据包
    const struct nf_hook_state *state  // 钩子状态
);

// 返回值：
// NF_ACCEPT  — 继续处理
// NF_DROP    — 丢弃数据包
// NF_STOLEN  — 接管数据包（不再由协议栈处理）
// NF_QUEUE   — 送往用户空间
// NF_REPEAT  — 重新调用此钩子

nf_inet_hooks 枚举

// include/uapi/linux/netfilter.h:42
enum nf_inet_hooks {
    NF_INET_PRE_ROUTING,   // = 0
    NF_INET_LOCAL_IN,      // = 1
    NF_INET_FORWARD,       // = 2
    NF_INET_LOCAL_OUT,     // = 3
    NF_INET_POST_ROUTING,  // = 4
    NF_INET_NUMHOOKS,      // = 5
    NF_INET_INGRESS,       // = 5 ( ingress 特殊钩子 )
};

iptables 规则注册流程

$ iptables -A INPUT -s 10.0.0.0/8 -j DROP
        │
        ↓
iptables 用户空间程序
        │
        │  Netlink (NETFILTER_SUBSYS_*)
        ↓
内核 nfnetlink 模块 — net/netfilter/nfnetlink.c
        │
        ↓
xt_table.c / nf_tables — 解析规则并插入链
        │
        ↓
数据包到达 NF_INET_LOCAL_IN 钩子
        │
        ↓
遍历 filter.INPUT 链上的所有规则
        │
        ├── 规则 1: -s 10.0.0.0/8 -j DROP → 匹配！→ NF_DROP（丢弃）
        │
        └── 规则 2: ... → 不到达
