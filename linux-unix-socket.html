01 概述

UNIX Domain Socket（AF_UNIX）是 Linux 中同一主机进程间通信的高性能机制。它使用 Socket API，但不经过网络协议栈——数据直接在内核中从一个 socket 的发送队列复制到另一个 socket 的接收队列。Docker、Systemd、X11、D-Bus 等大量系统服务使用它。

UNIX Domain Socket 特性
  • 不经过协议栈：数据不走 IP/TCP/UDP
  • 支持 SOCK_STREAM：可靠有序（类似 TCP）
  • 支持 SOCK_DGRAM：无连接（类似 UDP）
  • 文件描述符传递：SCM_RIGHTS（独特能力）
  • 路径寻址：/var/run/docker.sock
  • 凭证传递：SCM_CREDENTIALS（UID/GID）

02 类型与寻址

SOCK_STREAM（流式，可靠有序）：

  服务端                          客户端
  socket(AF_UNIX, SOCK_STREAM)    socket(AF_UNIX, SOCK_STREAM)
  bind("/var/run/app.sock")       connect("/var/run/app.sock")
  listen()                        ↓
  accept()  ← 连接建立 →         send()/recv()
                                   ↓
                                 数据双向流动（内核复制 sk_buff）

SOCK_DGRAM（数据报，无连接）：

  服务端                          客户端
  socket(AF_UNIX, SOCK_DGRAM)    socket(AF_UNIX, SOCK_DGRAM)
  bind("/var/run/app.sock")       sendto("/var/run/app.sock", data)
  recvfrom() ← 数据报 ←         （无需 listen/accept）
  注意：无连接但仍然有序（同一对 socket 之间）

抽象命名空间（无文件系统路径）：

  bind() 时 addr->sun_path[0] = '\0'
  → 不在文件系统中创建文件
  → 仅通过进程传名可见（/proc/pid/fd/）
  → 自动清理，无残留 socket 文件

地址结构

// struct sockaddr_un — UNIX Domain Socket 地址
struct sockaddr_un {
    sa_family_t sun_family;  // AF_UNIX
    char        sun_path[108]; // 文件系统路径（如 /var/run/docker.sock）
};

// 绑定示例
struct sockaddr_un addr;
addr.sun_family = AF_UNIX;
strcpy(addr.sun_path, "/var/run/myapp.sock");
bind(fd, (struct sockaddr *)&addr, sizeof(addr));

03 核心数据结构

// include/net/af_unix.h:36 — UNIX socket
struct unix_sock {
    struct sock         sk;          // 继承自 sock
    struct unix_address *addr;       // 绑定的地址（路径）
    struct path         path;        // 文件系统路径
    struct sock         *peer;       // ★ 对端 socket（连接后指向对方）
    struct sock         *listener;   // 监听 socket（accept 前的 socket）
    struct socket_wq    peer_wq;     // 对端等待队列
    struct wait_queue_entry_t peer_wake; // 对端唤醒
    struct scm_stat     scm_stat;    // 凭证统计
};

数据结构关系

  服务端 unix_sock                    客户端 unix_sock
  ┌──────────────┐                   ┌──────────────┐
  │ sk           │                   │ sk           │
  │ addr → /var/...│                 │ addr → /var/...│
  │ peer ─────────┼──────────────────┼── peer       │
  │ listener → sk │                   │              │
  └──────────────┘                   └──────────────┘

  连接建立后：
  服务端 sk->peer = 客户端 sk
  客户端 sk->peer = 服务端 sk
  双向指针，直接通信

  数据发送路径：
  服务端 send() → sk->sk_write_queue → sk->peer->sk_receive_queue
  客户端 recv() ← sk->sk_receive_queue（从对端的写队列取出）
  无协议栈参与，纯内核内存复制！

  │                                         │
  │ socket(AF_UNIX, SOCK_STREAM, 0)         │ socket(AF_UNIX, SOCK_STREAM, 0)
  │ bind(path)                              │
  │ listen(backlog)                         │
  │              ┌──────────────┐           │
  │ accept() ←── │ 三次握手模拟  │ ←── connect(path)
  │              └──────────────┘           │
  │                                         │
  │ ←──────── 双向数据流 ─────────→         │
  │ send() → sk_write_queue → peer recv()  │

  │                                         │
  │ close()                                 │ close()
  │              ┌──────────────┐           │
  │              │ 半关闭通知    │           │

05 SCM_RIGHTS 文件描述符传递

UNIX Domain Socket 最独特的功能是跨进程传递文件描述符——发送方可以将一个 fd 通过 socket 发送给接收方，接收方获得一个指向同一个内核文件对象的 fd。这是其他 IPC 机制无法做到的。

SCM_RIGHTS 文件描述符传递：

  进程 A（发送方）                        进程 B（接收方）
  ┌─────────────────┐                    ┌─────────────────┐
  │  fd = open(...)  │                    │                 │
  │  ↓              │                    │                 │
  │  sendmsg(fd)    │ ─── SCM_RIGHTS ──→ │ recvmsg()      │
  │  + 辅助数据      │     内核传递       │ → new_fd        │
  │  (fd=3)         │                    │ (fd=7)         │
  └─────────────────┘                    └─────────────────┘

  关键：
  - 进程 A 的 fd=3 和进程 B 的 fd=7 指向同一个内核 file
  - 不是复制文件描述符值，而是共享内核对象
  - 接收方的 fd 号可以不同

  使用场景：
  - systemd 传递 listening socket 给子进程（socket activation）
  - Docker 传递 pipe fd 给容器
  - X11 传递 X socket 给应用
  - privilege separation（sshd 传递 socket 给特权进程）

// 发送方：通过 SCM_RIGHTS 发送 fd
struct msghdr msg = { 0 };
struct cmsghdr *cmsg;
char buf[CMSG_SPACE(sizeof(int))];
int fd = open("/tmp/file", O_RDONLY);

msg.msg_control = buf;
msg.msg_controllen = sizeof(buf);
cmsg = CMSG_FIRSTHDR(&msg);
cmsg->cmsg_level = SOL_SOCKET;
cmsg->cmsg_type = SCM_RIGHTS;
cmsg->cmsg_len = CMSG_LEN(sizeof(int));
*CMSG_DATA(cmsg) = fd;

sendmsg(sock_fd, &msg, 0);

// 接收方：接收 fd
recvmsg(sock_fd, &msg, 0);
cmsg = CMSG_FIRSTHDR(&msg);
int new_fd = *CMSG_DATA(cmsg);  // 新的 fd 编号

什么时候用 UNIX Socket 而不是 TCP？
当通信双方在同一主机上时，UNIX Domain Socket 性能更好（无协议栈开销），并且支持 fd 传递和凭证传递。典型场景：数据库本地连接、Docker API、systemd 通信。
