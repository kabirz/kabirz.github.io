01 概述

内存热插拔（Memory Hotplug）允许在系统运行时动态添加或移除内存，无需重启。它用于云服务器按需分配内存（如 KVM 热迁移）、虚拟机内存热添加、以及内存故障隔离。

02 热添加流程

内存热添加流程：

  物理内存插入（或虚拟机增加内存）
        │
        ↓
  固件/驱动通知内核新内存可用
        │
        ↓
  memory_block_probe() — mm/memory_hotplug.c
        │
        ├── 1. 创建 memory_block 结构
        │       每个 memory_block = 128MB（默认）
        │
        ├── 2. 初始化 Zone
        │       将新内存添加到对应 NUMA 节点的 Zone
        │
        ├── 3. 注册页面到伙伴系统
        │       __add_pages() → 添加页面到 free_area
        │
        ├── 4. 通知子系统
        │       通知 PCI、ACPI 等子系统新内存可用
        │
        └── 5. 完成
            新内存可用于分配

内存热移除流程：

  $ echo offline > /sys/devices/system/memory/memory90/state
        │
        ├── 1. 将 memory_block 标记为 offline
        │       offline_pages()
        │
        ├── 2. 迁移页面到其他内存
        │       migrate_pages() → 移动所有页面
        │
        ├── 3. 从伙伴系统移除页面
        │       __remove_pages()
        │
        └── 4. 通知子系统
            新内存不再可用

03 应用场景

场景说明
云服务器按需增加 VM 内存（KVM 热迁移）
内存故障检测到坏内存 → 迁移页面 → 热移除
弹性伸缩负载增加时热添加内存
内存压缩热移除前先压缩页面

04 配置与使用

# 查看内存块状态
$ ls /sys/devices/system/memory/
memory0  memory1  memory2  ...  memory127

$ cat /sys/devices/system/memory/memory0/state
online

# 热移除内存块
$ echo offline > /sys/devices/system/memory/memory90/state
$ cat /sys/devices/system/memory/memory90/state
offline

# 热添加内存（需要硬件/固件支持）
$ echo online > /sys/devices/system/memory/memory128/state

# 查看内存热插拔统计
$ cat /proc/vmstat | grep hotplug
$ dmesg | grep -i "memory\|hotplug"

# 内核配置
CONFIG_MEMORY_HOTPLUG=y        # 启用内存热插拔
CONFIG_MEMORY_HOTREMOVE=y      # 启用内存热移除
CONFIG_ACPI_HOTPLUG_MEMORY=y   # ACPI 内存热插拔
