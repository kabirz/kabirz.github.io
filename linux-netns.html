01 概述

网络命名空间（Network Namespace）是 Linux 内核提供的一种网络栈隔离机制。每个命名空间拥有独立的网络设备、IP 地址、路由表、iptables 规则、/proc/net 目录等。它是 Docker、Kubernetes 等容器技术实现网络隔离的基石。

每个命名空间独立拥有
  • 网络设备（eth0、lo、br0...）
  • IPv4/IPv6 协议栈
  • 路由表
  • iptables / netfilter 规则
  • /proc/net 目录
  • socket 端口空间
  • ARP 表
  • Netfilter 连接追踪
典型场景
场景说明
Docker 容器每个容器一个独立网络命名空间
Kubernetes PodPod 内容器共享网络命名空间
网络测试模拟复杂网络拓扑
VPN隔离不同 VPN 隧道的网络
防火墙隔离不同安全域的网络

03 核心数据结构

// include/net/net_namespace.h:62 — 网络命名空间核心结构
struct net {
    refcount_t     passive;          // 引用计数（决定何时释放）
    struct ns_common  ns;            // 命名空间公共结构（含 ns_ops）

    /* ──── 网络设备管理 ──── */
    struct list_head  dev_base_head; // 所有网络设备链表
    struct hlist_head *dev_name_head; // 按名字索引的设备哈希表
    struct hlist_head *dev_index_head;// 按 ifindex 索引的设备哈希表
    struct xarray     dev_by_index;  // 按索引快速查找
    struct net_device *loopback_dev; // 该命名空间的 loopback 设备

    /* ──── 协议栈子系统 ──── */
    struct netns_ipv4  ipv4;         // IPv4 协议栈状态
    struct netns_ipv6  ipv6;         // IPv6 协议栈状态

    /* ──── Netfilter ──── */
#ifdef CONFIG_NETFILTER
    struct netns_nf    nf;           // iptables 框架
    struct netns_ct    ct;           // 连接追踪
    struct netns_nftables nft;      // nftables
#endif

    /* ──── 其他 ──── */
    struct proc_dir_entry *proc_net; // /proc/net/
    struct sock *rtnl;              // rtnetlink socket
    struct user_namespace *user_ns; // 所属用户命名空间

    // ... 还有 netns_mib, netns_packet, netns_xfrm 等
};

04 生命周期

创建 → 使用 → 销毁

1. 创建网络命名空间

  $ ip netns add myns
        │
        ↓
  clone(CLONE_NEWNET) 或 unshare(CLONE_NEWNET)
        │
        ↓
  内核创建新的 struct net
  ├── 分配独立的 loopback 设备
  ├── 初始化 IPv4/IPv6 协议栈
  ├── 初始化 iptables 框架
  ├── 创建 /proc/net/ 目录
  └── 加入命名空间链表

2. 在命名空间中操作

  $ ip netns exec myns bash
        │
        ↓
  当前进程切换到 myns 命名空间
  ├── 所有网络操作使用 myns 的网络栈
  ├── ip addr / ip route 只影响 myns
  └── iptables 只影响 myns

3. 销毁网络命名空间

  $ ip netns delete myns
        │
        ↓
  从链表中移除
  ├── 等待所有引用计数归零
  ├── 释放所有网络设备
  ├── 释放协议栈资源
  └── 释放 struct net 内存

05 默认命名空间

init_net — 全局默认网络命名空间

  所有网络命名空间形成一棵树：
  init_net 是根节点
  容器的网络命名空间是子节点

  ┌─────────────────────┐
  │     init_net        │  ← 宿主机的网络命名空间
  │   (默认命名空间)      │
  │                     │
  │   eth0 (物理网卡)    │
  │   docker0 (bridge)  │
  │   路由表、iptables   │
  └─────┬───────┬───────┘
        │       │
        ↓       ↓
  ┌──────────┐ ┌──────────┐
  │ netns_1  │ │ netns_2  │  ← 容器网络命名空间
  │ (容器1)   │ │ (容器2)   │
  │          │ │          │
  │ eth0     │ │ eth0     │
  │ (veth)   │ │ (veth)   │
  │ lo       │ │ lo       │
  └──────────┘ └──────────┘

  init_net 在内核启动时创建
  所有未进入其他命名空间的进程都在 init_net 中

06 Docker 中的应用

Docker 容器网络创建流程

$ docker run -d --name web -p 8080:80 nginx
        │
        ↓
1. 创建新的网络命名空间
        │
        ├── clone(CLONE_NEWNET)
        └── 创建 struct net
        │
        ↓
2. 创建 veth pair
        │
        ├── ip link add veth0 type veth peer name veth1
        └── veth0 在宿主机，veth1 在容器
        │
        ↓
3. 将 veth1 移入容器命名空间
        │
        ├── ip link set veth1 netns <container-pid>
        └── veth1 出现在容器的 eth0
        │
        ↓
4. 在宿主机命名空间
        │
        ├── ip link set veth0 master docker0
        ├── ip addr add 172.17.0.2/16 dev veth0
        └── ip link set veth0 up
        │
        ↓
5. 在容器命名空间
        │
        ├── ip addr add 172.17.0.2/16 dev eth0
        ├── ip link set eth0 up
        ├── ip route add default via 172.17.0.1
        └── 容器网络就绪！

容器网络流量路径

容器内应用 → eth0 (172.17.0.2) → 容器网络命名空间
    │
    ↓  veth pair
    │
宿主机 veth0 → docker0 bridge → 宿主机网络栈
    │
    ↓  NAT (iptables POSTROUTING)
    │
物理网卡 eth0 → 外部网络
Pod 网络
Kubernetes Pod 中的所有容器共享同一个网络命名空间（同一个 IP 地址）。这通过 pause 容器实现——pause 容器创建网络命名空间，其他容器通过 --net=container:pause 加入。

07 配置命令

# 创建命名空间
$ ip netns add myns

# 列出所有命名空间
$ ip netns list
myns

# 在命名空间中执行命令
$ ip netns exec myns ip addr
1: lo: <LOOPBACK> mtu 65536 qdisc noop state DOWN
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00

# 创建 veth pair 并连接两个命名空间
$ ip link add veth0 type veth peer name veth1
$ ip link set veth1 netns myns
$ ip addr add 10.0.0.1/24 dev veth0
$ ip link set veth0 up
$ ip netns exec myns ip addr add 10.0.0.2/24 dev veth1
$ ip netns exec myns ip link set veth1 up

# 在命名空间中添加路由
$ ip netns exec myns ip route add default via 10.0.0.1

# 在命名空间中配置 iptables
$ ip netns exec myns iptables -L

# 删除命名空间
$ ip netns delete myns

# 查看进程所属的网络命名空间
$ ls -la /proc/<pid>/ns/net
lrwxrwxrwx 1 root root 0 ... /proc/<pid>/ns/net -> 'net:[4026531992]'

# 在不同命名空间间建立 veth 连接
$ ip link add ns0-ns1 type veth peer name ns1-ns0
$ ip link set ns0-ns1 netns ns0
$ ip link set ns1-ns0 netns ns1
$ ip netns exec ns0 ip addr add 10.0.0.1/24 dev ns0-ns1
$ ip netns exec ns0 ip link set ns0-ns1 up
$ ip netns exec ns1 ip addr add 10.0.0.2/24 dev ns1-ns0
$ ip netns exec ns1 ip link set ns1-ns0 up

完整网络拓扑示例

$ ip netns add ns1
$ ip netns add ns2
$ ip link add veth1 type veth peer name veth2
$ ip link set veth1 netns ns1
$ ip link set veth2 netns ns2
$ ip netns exec ns1 ip link set lo up
$ ip netns exec ns1 ip addr add 10.0.0.1/24 dev veth1
$ ip netns exec ns1 ip link set veth1 up
$ ip netns exec ns2 ip link set lo up
$ ip netns exec ns2 ip addr add 10.0.0.2/24 dev veth2
$ ip netns exec ns2 ip link set veth2 up

$ ip netns exec ns1 ping 10.0.0.2
PING 10.0.0.2: 64 bytes from 10.0.0.2: icmp_seq=1 ttl=64 time=0.041 ms  ✓
