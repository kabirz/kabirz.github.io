01 概述

tun/tap 是 Linux 内核中最经典的虚拟网络设备。与 veth 不同，tun/tap 的另一端是用户空间程序，允许用户态应用直接读写网络数据包。它是 VPN（如 OpenVPN、WireGuard）和虚拟化（如 KVM、QEMU）的核心基础。

核心特性
  • 用户态接口：通过 fd 读写数据包
  • tun 模式：IP 层数据包
  • tap 模式：以太网帧数据包
  • 可配置路由：控制哪些流量走 tun/tap
  • VPN 基础：隧道、IPSec 等应用
  • 虚拟化基础：KVM/QEMU 网络后端

KVM/QEMU虚拟机网络后端

03 工作原理

数据包流程（VPN 场景）

1. 应用程序 A 通过 socket A 发送数据包（目的 IP: 192.168.3.1）
        │
        ▼
2. 协议栈匹配路由表 → 走 tun0
        │
        ▼
3. tun0 将数据包传递给应用程序 B
        │
        ▼
4. 应用程序 B 封装原始数据包（隧道）
        │
        ▼
5. 通过 socket B 发送给协议栈
        │
        ▼
6. 协议栈路由 → 走 eth0 → 物理网络
        │
        ▼
7. 远端 10.33.0.1 解封装 → 转发给 192.168.3.1

关键点：

  • tun0 的一端是协议栈，另一端是用户态程序 B
  • 协议栈发给 tun0 的数据包，应用程序 B 可以通过 read() 读取
  • 应用程序 B 写入 tun0 的数据包，协议栈会通过 write() 接收
  • 数据包走哪个网络设备由路由表控制

tun 设备打开流程

// 用户态程序打开 /dev/net/tun
int fd = open("/dev/net/tun", O_RDWR);

// 通过 ioctl 配置 tun 设备
struct ifreq ifr;
ifr.ifr_flags = IFF_TUN | IFF_NO_PI;  // TUN 模式，无包信息
ioctl(fd, TUNSETIFF, &ifr);

// 内核创建 tun 设备，绑定到 fd
// 用户态通过 read/write 与协议栈交换数据包
nread = read(fd, buffer, sizeof(buffer)); // 读取数据包
write(fd, buffer, nread);  // 发送数据包

05 实验演示

示例程序

// tun_test.c — 简单的 tun 设备测试程序
#include <net/if.h>
#include <sys/ioctl.h>
#include <fcntl.h>
#include <linux/if_tun.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int tun_alloc(int flags)
{
    struct ifreq ifr;
    int fd;

    fd = open("/dev/net/tun", O_RDWR);
    if (fd < 0) return fd;

    memset(&ifr, 0, sizeof(ifr));
    ifr.ifr_flags = flags;
    ioctl(fd, TUNSETIFF, &ifr);

    printf("Opened tun/tap device: %s\n", ifr.ifr_name);
    return fd;
}

int main()
{
    int tun_fd, nread;
    char buffer[1500];

    tun_fd = tun_alloc(IFF_TUN | IFF_NO_PI);
    if (tun_fd < 0) { perror("tun"); return 1; }

    while (1) {
        nread = read(tun_fd, buffer, sizeof(buffer));
        printf("Read %d bytes from tun/tap device\n", nread);
    }
}

运行演示

# 窗口 1：启动 tun 程序
$ gcc tun.c -o tun && sudo ./tun
Open tun/tap device tun1 for reading...
Read 84 bytes from tun/tap device    # 收到 ICMP echo request

# 窗口 2：配置 tun 设备
$ sudo ip addr add 192.168.3.11/24 dev tun1
$ sudo ip link set tun1 up

# 窗口 3：ping 测试
$ ping -c 4 192.168.3.12
PING 192.168.3.12 ... 4 packets transmitted, 0 received  # 无应答（程序没处理）
观察：ping 的数据包确实到达了应用程序（显示 84 字节），但因为程序只是打印数据包长度不做处理，所以没有返回 ICMP echo reply，ping 失败。

06 VPN 应用

VPN 工作原理

本地 VPN 客户端
┌────────────────────────────────────────────┐
│                                            │
│  ┌──────────┐    ┌──────────┐              │
│  │ 应用程序 │───→│ tun0     │              │
│  └──────────┘    └────┬─────┘              │
│                       │ IP 包              │
│                  ┌────┴────┐               │
│                  │  VPN    │  ← 加密/封装  │
│                  │  程序   │               │
│                  └────┬────┘               │
│                       │ 加密后的数据包     │
│                  ┌────┴────┐               │
│                  │  eth0   │               │
│                  └────┬────┘               │
└───────────────────────│────────────────────┘
                        ↓
              ┌─────────────────┐
              │  Internet       │
              └────────┬────────┘
                       ↓
              ┌─────────────────┐
              │ 远端 VPN 服务器 │
              └────────┬────────┘
                       ↓ 解密
              ┌─────────────────┐
              │     目标网络    │
              └─────────────────┘
OpenVPN 原理：OpenVPN 使用 tun 设备创建虚拟网卡，所有发往 tun0 的数据包被 OpenVPN 进程加密后通过 eth0 发送到远端 VPN 服务器，远端解密后转发给目标。返回路径反之。
