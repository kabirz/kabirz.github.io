01 概述

Kprobe 和 Uprobe 是 Linux 的动态探针机制。Kprobe 在内核函数入口/返回处插入断点，Uprobe 在用户态函数入口/返回处插入断点。它们允许在不修改代码的情况下，运行时监控任意函数的调用。

02 Kprobe

Kprobe 工作原理：

  在目标函数入口插入断点指令（INT3/x86）
        │
        ├── pre_handler() — 函数执行前调用
        │   可以查看/修改寄存器和参数
        │
        ├── 原始指令执行
        │
        ├── post_handler() — 函数执行后调用
        │   可以查看返回值
        │
        └── 恢复执行

Uprobe 工作原理：

  在用户态二进制文件的函数入口插入断点
  类似 GDB 的断点机制
  用于监控用户态程序的函数调用

// 内核模块中使用 kprobe
#include <linux/kprobes.h>

static kprobe_opcode_t kprobe_addr;

static int handler_pre(struct kprobe *p, struct pt_regs *regs)
{
    printk(KERN_INFO "kprobe: function called, ax=%lx\n", regs->ax);
    return 0;
}

static struct kprobe kp = {
    .symbol_name = "tcp_sendmsg",
    .pre_handler = handler_pre,
};

int init_module(void) {
    register_kprobe(&kp);
    return 0;
}

03 Uprobe

# Uprobe — 用户态函数探针

# 在 /usr/bin/myapp 的 main 函数处插入探针
$ echo 'p:myprobe /usr/bin/myapp:main' > /sys/kernel/debug/tracing/kprobe_events
$ echo 1 > /sys/kernel/debug/tracing/events/kprobes/myprobe/enable

# 或使用 perf
$ perf probe -x /usr/bin/myapp -a main
$ perf record -e probe_myapp -aR sleep 10

# Uprobe 用于监控
# - 库函数调用
# - 系统调用入口
# - 应用特定函数
