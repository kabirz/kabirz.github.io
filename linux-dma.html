01 概述

DMA（Direct Memory Access）是设备不经过 CPU 直接访问系统内存的技术。Linux DMA 子系统提供了统一的地址映射框架，处理 CPU 物理地址与设备总线地址之间的转换，以及缓存一致性管理。

DMA 的必要性
  • 性能：设备直接读写内存，释放 CPU
  • 批量传输：网卡收包、磁盘读写等高吞吐场景
  • 零拷贝：减少 CPU 参与的数据复制
  • 一致性：确保 CPU 和设备看到一致的内存数据
DMA 映射类型
类型说明
一致性 DMA分配即映射，始终一致
流式 DMA映射已有内存，需要手动同步
IOMMU 映射通过 IOMMU 进行地址翻译
swiotlb软件 IO TLB（bounce buffer）

DMA 地址空间

// 三种地址概念
// 1. CPU 物理地址（phys_addr_t）— CPU 看到的物理地址
// 2. DMA 总线地址（dma_addr_t）— 设备看到的地址
// 3. 虚拟地址（void *）— 内核映射的虚拟地址

// 在简单系统中，三者可能相等
// 在有 IOMMU 或 PCI 桥的系统中，需要翻译

// bus_dma_region 描述 CPU 物理地址到 DMA 地址的映射
struct bus_dma_region {
    phys_addr_t cpu_start;    // CPU 物理地址起始
    dma_addr_t dma_start;    // DMA 总线地址起始
    u64 size;                // 区域大小
};

典型使用模式

// 一致性 DMA（DMA 读写频繁，如 descriptor ring）
dma_addr_t dma_handle;
void *vaddr;
vaddr = dma_alloc_coherent(&pdev->dev, 4096, &dma_handle, GFP_KERNEL);
// dma_handle → 给设备使用
// vaddr → CPU 使用（自动保持一致）
dma_free_coherent(&pdev->dev, 4096, vaddr, dma_handle);

// 流式 DMA（批量传输，如网络包）
dma_addr_t dma_addr;
dma_addr = dma_map_single(&pdev->dev, skb->data, skb->len, DMA_TO_DEVICE);
// dma_addr → 给设备使用
// ... 设备传输 ...
dma_unmap_single(&pdev->dev, dma_addr, skb->len, DMA_TO_DEVICE);

04 一致性 DMA

一致性 DMA 缓冲区（coherent buffer）在分配时即建立映射，CPU 和设备始终看到一致的数据。适用于描述符环、中断状态等需要频繁同步的场景。

实现方式

CMA 分配
连续物理内存
↓
DMA 域映射
设备可见地址
IOVA 分配
IOMMU 管理
↓
IOMMU 映射
设备地址翻译
swiotlb
bounce buffer
↓
bounce buffer
非一致性 DMA

特点与限制

优点
  • 分配即映射，无需手动同步
  • CPU 和设备始终一致
  • 适用于高频同步场景
  • 可以从中断上下文访问
限制
  • 分配可能失败（内存不足）
  • 物理内存可能不连续
  • 无法用于大块数据传输
  • 性能低于流式 DMA

05 流式 DMA

流式 DMA（streaming DMA）用于映射已有的内核缓冲区给设备使用。需要手动进行缓存同步。

操作流程

准备数据
填充缓冲区
↓
dma_map_single(dev, buf, len, DMA_TO_DEVICE)
Clean cache → 设备可读
↓
设备 DMA 传输
直接访问内存
↓
dma_unmap_single(dev, addr, len, DMA_TO_DEVICE)
完成后可选同步

scatterlist DMA

// 散列表 DMA：映射多个不连续的页面
struct scatterlist sg[NR_PAGES];

// 设置 scatterlist
sg_init_table(sg, nr_pages);
for (i = 0; i < nr_pages; i++)
    sg_set_page(&sg[i], pages[i], PAGE_SIZE, 0);

// 映射
int nents = dma_map_sg(dev, sg, nr_pages, DMA_BIDIRECTIONAL);

// nents 可能 <= nr_pages（硬件合并了相邻段）
// 设备使用 sg_dma_address(sg[i]) 和 sg_dma_len(sg[i])

// 取消映射
dma_unmap_sg(dev, sg, nents, DMA_BIDIRECTIONAL);
缓存同步：流式 DMA 需要在合适的时机执行缓存同步。使用 DMA_TO_DEVICE 时在 map 前 clean cache；使用 DMA_FROM_DEVICE 时在 unmap 后 invalidate cache。使用 DMA_BIDIRECTIONAL 时两者都需要。

06 SWIOTLB

SWIOTLB（Software IO TLB）是 Linux 的软件 bounce buffer 机制，用于设备无法直接访问所有物理内存时的数据中转。

工作原理

设备 DMA 地址空间有限
无法访问高位物理地址
↓
swiotlb bounce buffer
低位物理内存缓冲区
↓ 数据复制
CPU 物理内存
高位地址
↔
swiotlb buffer
低位地址
↔
设备 DMA
总线地址

配置

参数说明
swiotlb=nobounce禁用 swiotlb
swiotlb=force强制使用 swiotlb
swiotlb=nnounceded不播报缓冲区大小
CONFIG_SWIOTLB编译时启用
CONFIG_SWIOTLB_DYNAMIC动态分配（节省内存）
何时需要 swiotlb：当设备的 DMA 地址位宽小于 CPU 物理地址位宽时（如 32 位设备在 64 位系统上），或者设备无法访问所有物理内存时。ARM64 上的 DMA32 设备通常需要 swiotlb。

07 IOMMU

IOMMU（I/O Memory Management Unit）为设备提供独立的地址翻译，类似于 CPU 的 MMU。支持 DMA 重映射、设备直通（passthrough）等。

IOMMU 功能

功能说明
DMA 重映射设备看到的地址可以不同于物理地址
设备直通虚拟机直接使用物理设备
中断重映射防止恶意中断
SVA共享虚拟地址（设备直接使用进程地址空间）
S1PTW设备页表行走（Page Table Walk）

ARM SMMU（System MMU）

drivers/iommu/arm/

// ARM SMMU 是 ARM 平台的标准 IOMMU
// 支持：DMA 重映射、SVA、IOVA 分配

// SMMU 工作流程：
// 1. 设备发出 DMA 请求（设备地址）
// 2. SMMU 查找 IOVA → 物理地址映射
// 3. 通过页表翻译为物理地址
// 4. 访问物理内存

// Linux IOMMU API：
iommu_map(domain, iova, paddr, size, prot);  // 建立映射
iommu_unmap(domain, iova, size);             // 移除映射
iommu_domain_alloc(dev);                     // 分配 IOMMU 域
VT-d / SMMU：x86 使用 Intel VT-d，ARM 使用 SMMU。两者都提供 DMA 重映射，但实现细节不同。Linux 通过 IOMMU 框架统一抽象。
