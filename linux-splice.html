01 概述

splice 是 Linux 提供的零拷贝数据传输系统调用，可以在文件描述符和管道之间移动数据，无需在用户态和内核态之间复制数据。它是 nginx、redis、kafka 等高性能应用的核心优化手段。

02 零拷贝原理

传统 read/write（两次拷贝）：

  磁盘 → 内核 Page Cache → copy_to_user() → 用户态 buf
  写：用户态 buf → copy_from_user() → 内核 socket 缓冲区 → 网卡
  总共 2-4 次数据拷贝

splice（零拷贝）：

  文件 → Page Cache → 页面引用移动 → pipe buffer → 网卡
  总共 0 次数据拷贝！
  只修改 page 引用计数和指针

sendfile（零拷贝发送）：

  文件 fd → Page Cache → 零拷贝 → socket
  nginx sendfile on; 配置启用

性能对比

方式拷贝次数上下文切换适用场景
read + write4 次4 次通用
mmap + write3 次4 次大文件
sendfile2 次2 次文件→socket
splice0 次2 次任意 fd 对

03 API 与使用

// splice — 在 fd_in 和 pipe 之间移动数据
ssize_t splice(int fd_in, loff_t *off_in,
               int pipe_out, loff_t *off_out,
               size_t len, unsigned int flags);

// 典型用法：文件 → pipe → socket
int pipefd[2];
pipe(pipefd);

// 文件 → pipe（零拷贝）
splice(filefd, NULL, pipefd[1], NULL, len, SPLICE_F_MOVE);

// pipe → socket（零拷贝）
splice(pipefd[0], NULL, sockfd, NULL, len, SPLICE_F_MOVE);

// 总共：文件 → socket 零拷贝传输！
splice 内部机制（只移动页面引用）：

  splice(filefd → pipe)
        │
        ↓
  从文件 Page Cache 获取页面
        │
        ↓
  pipe_buffer.page = file_page
  get_page(file_page)  ← 仅增加引用计数
        │
        ↓
  splice(pipe → socket)
        │
        ↓
  从 pipe_buffer 取出页面
        │
        ↓
  传递给 socket 发送
  put_page(file_page) ← 仅减少引用计数

  整个过程不复制任何数据！
  只是页面指针在不同缓冲区之间传递

04 sendfile 与 vmsplice

系统调用功能限制
splice任意 fd ↔ pipe需要 pipe 作为中间人
sendfile文件 → socket（专用）只能发到 socket
vmsplice用户态内存 → pipe只能到 pipe
nginx 如何使用 sendfile
nginx 配置 sendfile on; 后，发送静态文件时：
① 读取文件 → Page Cache
② sendfile(fd_file, fd_socket, ...) → 零拷贝发送
③ 不经过 nginx 用户态 → 减少内存拷贝和 CPU 开销
