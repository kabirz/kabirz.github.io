01 概述

Linux 内核提供了丰富的调试技术来定位问题。本文全景梳理所有调试技术的关系和使用场景，帮助开发者快速定位和修复内核问题。

03 调试流程

调试决策树：

  问题类型？
  │
  ├── 内存错误 → KASAN / KCSAN / UBSAN
  ├── 死锁 → lockdep
  ├── 性能问题 → perf / ftrace / BPF
  ├── 函数追踪 → ftrace / kprobe
  ├── 内核崩溃 → Kdump / crash
  ├── 驱动问题 → debugfs / printk / dynamic_debug
  └── 安全问题 → audit / seccomp / BPF

04 场景选择

问题类型推荐技术开销
use-after-freeKASAN~3x 减速
数据竞争KCSAN~2x 减速
死锁lockdep~5% 减速
性能瓶颈perf + ftrace~1-5% 减速
内核崩溃Kdump + crash离线分析
驱动调试debugfs + printk低
安全审计audit + seccomp低
