01 概述

CPUFreq 是 Linux 的CPU 频率动态调节框架。它根据系统负载自动调整 CPU 频率，在性能和功耗之间取得平衡。笔记本电脑使用它延长续航，服务器使用它降低散热成本。

02 频率调节器

调节器策略适用场景
performance始终最高频率基准测试、低延迟
powersave始终最低频率省电、低负载
ondemand按需快速升频通用（传统）
conservative渐进式升频/降频保守策略
schedutil基于调度器负载推荐（现代内核）
schedutil（推荐的现代调节器）：

  调度器检测到 CPU 负载变化
        │
        ├── 负载高 → 请求升频
        │       → cpufreq 驱动设置新频率
        │       → CPU 运行更快
        │
        └── 负载低 → 请求降频
                → cpufreq 驱动设置新频率
                → CPU 功耗降低

  优势：直接使用调度器的利用率信息
  比 ondemand/conservative 更准确、响应更快

03 调节流程

  调节器检测到负载变化
        │
        ↓
  计算目标频率
  target_freq = f(load, policy)
        │
        ↓
  调用 cpufreq_driver->target_index()
        │
        ├── 软件调节（如 ACPI P-state）
        │       写入 ACPI 寄存器 → CPU 切换频率
        │
        └── 硬件调节（如 Intel HWP）
            硬件自动根据负载调节
        │
        ↓
  CPU 在新频率下运行
        │
        ↓
  监控功耗/温度
        │
        ├── 过热 → 降低频率（thermal throttling）
        └── 正常 → 继续调节

04 配置与调优

# 查看当前频率
$ cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_cur_freq
2400000  # 2.4 GHz

# 查看可用频率
$ cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_available_frequencies
800000 1200000 1600000 2000000 2400000

# 查看当前调节器
$ cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor
schedutil

# 切换调节器
$ sudo cpupower frequency-set -g performance
$ sudo cpupower frequency-set -g powersave

# 设置最大/最小频率
$ sudo cpupower -c 0 frequency-set -f 2400000  # 最大
$ sudo cpupower -c 0 frequency-set -f 800000   # 最小

# 查看 CPU 信息
$ cpupower frequency-info
