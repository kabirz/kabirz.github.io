01 概述

VXLAN（Virtual eXtensible LAN）是一种 overlay 网络技术，通过在 UDP 数据包中封装二层以太网帧，实现跨三层网络的二层互通。它是 Docker Swarm、Kubernetes（Flannel/Calico VXLAN 模式）等容器网络方案的底层技术。

VXLAN 核心特性
  • 24 位 VNI：支持 1600 万个虚拟网络（VLAN 仅 4096）
  • UDP 封装：默认端口 4789
  • 跨三层：可以穿越路由器和防火墙
  • MAC 学习：FDB 表管理远端 VTEP
  • 多播支持：BUM 流量可走多播

02 封包与解包

封装格式

原始以太网帧（容器 A 发出）：

+----------+----------+---------+-----------+
| DST MAC  | SRC MAC  | Type    | IP Header |  Data
| (6 字节) | (6 字节) | (2)     |           |
+----------+----------+---------+-----------+

VXLAN 封装后（通过物理网络传输）：

+------+----------+----------+--------+----------+---------+----------+---------+-----------+
| Outer| Outer    | Outer    | Outer  | VXLAN    | Inner   | Inner    | Inner   | Inner     |
| DST  | SRC      | UDP      | UDP    | Header   | DST MAC | SRC MAC  | Type    | IP+Data   |
| MAC  | MAC      | Port     | Len    | (8 字节) | MAC     | MAC      |         |           |
+------+----------+----------+--------+----------+---------+----------+---------+-----------+
| ←───── 外层以太网头 ──────→| ←UDP→ | ← VXLAN →| ←────────── 内层以太网帧 ──────────────→ |
                                 4789    VNI=100

VXLAN Header (8 字节):
+--------+--------+--------+--------+
| Flags  | Reserved|    VNI (24 bit) |
| (8 bit)|         |    = 0x000064   |
+--------+--------+--------+--------+

VTEP（VXLAN Tunnel Endpoint）

  VTEP 是 VXLAN 隧道的端点，负责封装和解封数据包

04 核心数据结构

// include/net/vxlan.h:284 — VXLAN 设备
struct vxlan_dev {
    struct vxlan_dev_node  hlist4;     // VNI 哈希表节点（IPv4）
    struct list_head       next;       // 命名空间链表
    struct vxlan_sock __rcu *vn4_sock;  // IPv4 UDP 监听 socket
    struct net_device      *dev;       // VXLAN 网络设备
    struct net             *net;       // 网络命名空间
    struct vxlan_rdst      default_dst;// 默认转发目的地

    struct rhashtable       fdb_hash_tbl; // ★ FDB 转发表
    struct rhashtable       mdb_tbl;       // 多播组表
    struct vxlan_config     cfg;          // 配置参数
};
// VXLAN 配置结构
struct vxlan_config {
    __be32              saddr;     // 源 IP（外层）
    __be16              dst_port;  // 目的端口（默认 4789）
    __be16              port_min;  // 端口范围最小值
    __be16              port_max;  // 端口范围最大值
    u32                vni;       // ★ VXLAN Network Identifier
    u32                flags;     // VXLAN_F_LEARN 等
    u8                 tos;       // DSCP/TOS
    u8                 ttl;       // 外层 TTL
};

05 发送流程

  容器 A 发送数据到 10.0.0.3
        │
        ↓
  协议栈路由 → 10.0.0.3 走 vxlan0
        │
        ↓
  vxlan_xmit() — vxlan_core.c
        │
        ├── 1. 查 FDB
        │       根据目的 MAC 查找转发表
        │       找到 → 获取远端 VTEP IP (172.17.0.2)
        │       找不到 → 多播泛洪 / 丢弃
        │
        ├── 2. 构造 VXLAN 头
        │       设置 VNI（24 位虚拟网络 ID）
        │       设置 UDP 端口 4789
        │
        ├── 3. 封装外层 IP 头
        │       src=172.17.0.1, dst=172.17.0.2
        │       protocol=UDP
        │
        ├── 4. 发送
        │       udp_send_skb() → IP 层 → 物理网卡
        │
        ↓
  外层 UDP 包通过物理网络到达 Host B

06 接收流程

  Host B 物理网卡收到 UDP:4789 数据包
        │
        ↓
  协议栈 → UDP 层 → 匹配 VXLAN socket
        │
        ↓
  vxlan_udp_encap() — vxlan_core.c
        │
        ├── 1. 验证 VXLAN 头
        │       检查 VNI 是否匹配本机 VXLAN 设备
        │       不匹配 → 丢弃
        │
        ├── 2. 解封
        │       剥离外层 UDP/IP 头
        │       提取内层以太网帧
        │
        ├── 3. MAC 学习
        │       根据内层源 MAC + 外层源 IP 更新 FDB
        │       记录：MAC → VTEP IP 映射
        │
        ├── 4. 转发
        │       查找内层目的 MAC 对应的本地端口
        │       交给 bridge / 本地网卡处理
        │
        ↓
  容器 B 从 VXLAN 设备收到原始以太网帧

07 FDB 转发表

VXLAN 的 FDB（Forwarding Database）记录 远端 MAC → 远端 VTEP IP 的映射关系。

FDB 表项：

  容器 MAC 地址          远端 VTEP IP         VNI
  ─────────────          ─────────────        ───
  aa:bb:cc:dd:ee:01  →   172.17.0.2          100
  aa:bb:cc:dd:ee:02  →   172.17.0.3          100
  aa:bb:cc:dd:ee:03  →   172.17.0.4          200

学习过程：

  1. 收到远端 VXLAN 包 → 学习 (src MAC → outer src IP)
  2. 静态添加：bridge fdb append aa:bb:cc:dd:ee:01 dst 172.17.0.2 dev vxlan0 vni 100
  3. 多播/泛洪学习：收到 BUM 包时自动学习

老化机制：

  FDB 条目有老化时间（默认 300 秒）
  超时未使用 → 自动删除
  避免过期条目占用内存
FDB 查询优先级
① 先查本机 FDB（静态 + 动态学习的条目）
② 未找到 → 检查是否有多播组（BUM 流量）
③ 未找到 → 丢弃（单播未知目的 MAC）

Pod A → Pod B 数据路径：
10.244.1.2 → cni0 → flannel.1（封装 VXLAN）
→ eth0（192.168.1.100 → 192.168.1.200）
→ Node B eth0 → flannel.1（解封 VXLAN）
→ cni0 → 10.244.2.3

其他场景

数据中心 L2 Overlay

跨物理交换机实现二层互通，VM 迁移时 IP 地址不变。VNI 的 1600 万容量远超 VLAN 的 4096。

多租户隔离

不同租户使用不同 VNI，在同一物理网络上完全隔离。云平台（OpenStack/Neutron）使用 VXLAN 实现租户网络。
