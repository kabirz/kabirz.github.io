01 概述

Static Keys（静态键）是 Linux 内核的零开销条件分支优化机制。传统 if (static_var) 每次执行都要读取全局变量（缓存行未命中），而 static keys 通过运行时修改代码路径（NOP ↔ JMP），使条件分支在不执行时完全无开销。它是 tracepoint、安全模块检查等高频路径的核心优化。

02 实现原理

传统条件分支（每次执行）：

  if (static_branch_unlikely(&my_key)) {
      do_something();  // 每次都执行检查
  }

  编译为：
  CMP [static_key_value], 0    // 读取全局变量（缓存行未命中 ~100ns）
  JZ  skip                     // 条件跳转
  CALL do_something
  skip:

Static Key（零开销）：

  静态键关闭时（默认状态）：
  do_something 的入口是 NOP 指令
  → 执行时直接跳过，无任何检查开销

  静态键开启时：
  static_branch_enable(&my_key)
  → 修改代码：NOP → JMP do_something
  → 执行时直接跳转，无条件检查开销

  关闭时：
  static_branch_disable(&my_key)
  → 修改代码：JMP → NOP

03 API 与使用

#include <linux/jump_label.h>

// 定义静态键（默认关闭）
DEFINE_STATIC_KEY_FALSE(my_feature_key);

// 使用静态键
if (static_branch_unlikely(&my_feature_key)) {
    // 这段代码在静态键关闭时完全不会执行
    // 编译为 NOP → 零开销
    do_expensive_check();
}

// 运行时启用/禁用
static_branch_enable(&my_feature_key);   // 启用
static_branch_disable(&my_feature_key);  // 禁用

// 用法示例：性能计数器开关
DEFINE_STATIC_KEY_FALSE(perf_counters_enabled);

void on_packet(struct sk_buff *skb) {
    if (static_branch_unlikely(&perf_counters_enabled)) {
        update_perf_counters(skb);  // 仅在启用时执行
    }
}

04 应用场景

场景Static Key说明
tracepointtracepoint static keys动态启用/禁用 tracepoint
安全模块LSM static keysSELinux/AppArmor 检查开关
网络过滤netfilter static keys动态启用 Netfilter 规则
调试开关debug static keys运行时启用调试代码
性能计数perf static keys运行时启用性能监控
vs. 全局变量
全局变量：if (global_flag) → 每次读取内存（~100ns cache miss）
Static Key：static_branch_unlikely() → NOP 指令（0ns）
在高频路径（如每个数据包、每个系统调用）中，static key 的性能提升显著。
