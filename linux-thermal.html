01 概述

Linux Thermal 框架管理 CPU 和设备的温度控制。当温度超过阈值时，触发节流（throttling）降低性能以保护硬件。它与 cpufreq 紧密协作——温度过高时降低 CPU 频率，是笔记本、服务器、嵌入式系统的关键保护机制。

02 架构

Thermal 框架架构：

  ┌─────────────────────────────────────────────┐
  │            Thermal Zone（温度传感器）         │
  │  /sys/class/thermal/thermal_zone0/          │
  │  ├── temp: 当前温度（毫摄氏度）              │
  │  ├── type: 传感器类型（x86_pkg_temp 等）     │
  │  ├── trip_point_0_temp: 第一阈值            │
  │  └── trip_point_0_type: critical/hot/passive│
  └───────────────────┬─────────────────────────┘
                      │ 温度超过阈值
                      ↓
  ┌─────────────────────────────────────────────┐
  │            Thermal Governor（调节器）        │
  │  step_wise: 步进式节流                      │
  │  power_allocator: 功率分配（IPA）           │
  │  bang_bang: 开关式控制                      │
  └───────────────────┬─────────────────────────┘
                      │ 请求降温
                      ↓
  ┌─────────────────────────────────────────────┐
  │            Cooling Device（冷却设备）        │
  │  cpufreq cooling: 降低 CPU 频率            │
  │  fan: 控制风扇转速                          │
  │  devfreq cooling: 降低 GPU 频率            │
  │  power_allocator: 限制 CPU 功率            │
  └─────────────────────────────────────────────┘

03 Thermal Governor

Governor策略适用场景
step_wise温度每升一级，降低一个冷却状态通用
power_allocatorPID 控制器分配功耗预算多核 CPU
bang_bang阈值判断开关风扇控制
user_space通知用户态程序自定义策略

04 应用场景

# 查看热区信息
$ cat /sys/class/thermal/thermal_zone0/temp
52000  # 52°C

$ cat /sys/class/thermal/thermal_zone0/type
x86_pkg_temp

# 查看 trip 点
$ cat /sys/class/thermal/thermal_zone0/trip_point_0_temp
100000  # 100°C（临界温度）

$ cat /sys/class/thermal/thermal_zone0/trip_point_0_type
critical

# 查看冷却设备
$ ls /sys/class/thermal/cooling_device0/
$ cat /sys/class/thermal/cooling_device0/cur_state
0  # 当前冷却等级（0=无节流）

# 使用 thermald 守护进程（Intel 推荐）
$ sudo systemctl start thermald
