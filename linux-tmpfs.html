01 概述

tmpfs 是 Linux 的基于内存的文件系统。文件数据存储在 RAM 中（或 swap），不写磁盘。它是 POSIX 共享内存（/dev/shm）、/tmp、容器 overlayfs 下层等多个机制的基础。核心实现在 mm/shmem.c（6022 行），复用了 Page Cache 的 address_space 机制。

02 架构

tmpfs 架构（基于 Page Cache + shmem）：

  用户态：write(fd, data, len)
        │
        ↓
  VFS → tmpfs 文件操作
        │
        ↓
  shmem_write_begin() → 分配页面
        │
        ├── 从 page cache 查找/分配页面
        │   get_user_pages_fast() 或 shmem_get_folio()
        │
        ├── 将用户数据复制到 Page Cache 页面
        │
        └── shmem_write_end() → 标记为脏页

  用户态：read(fd, buf, len)
        │
        ↓
  VFS → tmpfs 文件操作
        │
        ↓
  shmem_read_iter() → 从 Page Cache 读取
        │
        └── 数据已经在内存中，直接复制到用户态

  关键：tmpfs 的所有数据都在 Page Cache 中
  不写磁盘（除非显式 swap out）
  进程退出后释放（除非持久挂载）

03 应用场景

/dev/shm — POSIX 共享内存

  $ ls -la /dev/shm
  -rw------- 1 user user 4096 Jun 12 10:00 myshm

  shm_open("/myshm") → 在 /dev/shm 创建文件
  mmap(MAP_SHARED) → 映射到进程地址空间
  多进程映射同一文件 → 共享内存

/tmp — 临时文件

  $ mount | grep tmpfs
  tmpfs on /tmp type tmpfs (rw,nosuid,nodev)

  程序写入 /tmp → 数据在 RAM 中
  重启 → 数据消失（安全，无需清理）
  systemd-tmpfiles 定期清理过期文件

/run — 运行时数据

  $ mount | grep /run
  tmpfs on /run type tmpfs (rw,nosuid,nodev)

  PID 文件、socket 文件、systemd 状态
  重启自动清除

容器 — OverlayFS 下层

  Docker overlay2 存储驱动：
  upperdir（容器层，ext4）
  lowerdir（镜像层，只读）
  workdir → tmpfs（工作目录）

  容器删除后 tmpfs 自动释放

tmpfs 挂载选项

# 创建 tmpfs 挂载点
$ mount -t tmpfs -o size=256M tmpfs /mnt/ramdisk

# 常用选项
size=256M       # 最大容量限制
nr_inodes=4096  # 最大 inode 数量
mode=1777       # 权限（/tmp 通常 1777）
uid=0           # 所有者
gid=0           # 所属组

# 查看 tmpfs 使用情况
$ df -h /dev/shm /tmp /run
Filesystem      Size  Used Avail Use%
tmpfs           2.0G  128M  1.9G   6% /dev/shm
tmpfs           2.0G  4.2M  2.0G   1% /tmp
tmpfs           2.0G  3.5M  2.0G   1% /run
