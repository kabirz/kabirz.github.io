01 概述

KASAN（Kernel Address Sanitizer）是 Linux 内核的内存错误检测工具，能够检测 use-after-free、buffer-overflow、out-of-bounds 等内存问题。它是内核开发和驱动调试的重要工具，类似于用户态的 AddressSanitizer（ASan）。

03 影子内存

KASAN 影子内存（Generic 模式）：

  每 8 字节的内核内存对应 1 字节的影子内存

  内存地址:  0x0000  0x0008  0x0010  0x0018  ...
  影子内存:    [0]     [0]     [0]     [0]     ...
               全0 = 全部可访问

  分配 16 字节（0x0000-0x000F）后：
  影子内存:    [0]     [0]     [1]     [1]     ...
               ↑ 前 8 字节      ↑ 后 8 字节
               可访问            可访问

  释放后标记为 freed：
  影子内存:    [0]     [0]     [F]     [F]     ...
                            [F] = freed（use-after-free 检测）

  越界访问时：
  访问 0x0020 → 影子内存值 != 0 → 检测到越界访问！

04 检测流程

KASAN 检测流程：

  内核代码执行内存访问
        │
        ├── 1. KASAN 检查影子内存
        │       __asan_loadN() / __asan_storeN()
        │
        ├── 2. 影子内存值 == 0 → 正常访问，继续
        │
        └── 3. 影子内存值 != 0 → 检测到错误
                │
                ├── 4. 判断错误类型
                │   [F] = use-after-free
                │   [1..7] = buffer-overflow
                │   [K] = kernel-address-use-after-scope
                │
                └── 5. 打印报告
                    KASAN: use-after-free
                    ...
                    堆栈跟踪
                    分配/释放位置

05 配置与使用

# 内核配置
CONFIG_KASAN=y
CONFIG_KASAN_GENERIC=y  # 或 CONFIG_KASAN_SW_TAGS
CONFIG_KASAN_STACK=y    # 栈缓冲区检测
CONFIG_KASAN_VMALLOC=y  # vmalloc 检测

# 运行时控制
$ cat /sys/kernel/debug/kasan/report
# 查看 KASAN 报告
