01 概述

VDSO（Virtual Dynamic Shared Object）是内核映射到每个进程地址空间的一小段共享库代码。它允许用户态程序直接读取内核维护的时间和时钟信息，无需系统调用。gettimeofday()、clock_gettime()、time() 等高频调用都通过 VDSO 实现，性能提升 10-100 倍。

02 架构

传统 gettimeofday（系统调用）：

  应用 → gettimeofday()
         → int 0x80 / syscall 指令
         → 进入内核态
         → 读取时间
         → 返回用户态
  开销：~200ns（上下文切换 + 内核处理）

VDSO gettimeofday（零系统调用）：

  应用 → gettimeofday()
         → 直接读取 VDSO 共享内存中的时间值
         → 无需进入内核态
  开销：~20ns（仅内存读取）

VDSO 实现原理：

  内核启动时：
  1. 编译一段共享库代码（vdso.so）
  2. 映射到每个进程的地址空间（固定地址）
  3. 共享库直接读取内核维护的时钟数据

  时钟数据更新：
  - 定时器中断更新 VDSO 映射区域中的时间值
  - 应用读取时直接获取最新值
  - 无需系统调用

03 应用场景

系统调用VDSO 替代性能提升
gettimeofday()直接读 VDSO~10x
clock_gettime()直接读 VDSO~10x
time()直接读 VDSO~10x
clock_getres()直接读 VDSO~10x
getcpu()直接读 VDSO~5x
# 查看进程的 VDSO 映射
$ cat /proc/self/maps | grep vdso
7f8b8c1fe000-7f8b8c200000 r-xp 00000000 00:00 0 [vdso]

# 查看 VDSO 内容（x86_64）
$ objdump -d /proc/self/exe 2>/dev/null | grep gettimeofday
