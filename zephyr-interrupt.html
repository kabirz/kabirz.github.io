02 中断控制器

Zephyr 支持多种中断控制器架构，通过统一的 IRQ 子系统抽象。

架构控制器特点
ARM Cortex-MNVIC最高 240 个中断，可编程优先级
ARM Cortex-A/RGIC最多 1020 个 SPI，支持 SGI
RISC-VPLIC / CLINT平台级中断控制器
x86APIC / IOAPIC高级可编程中断控制器
ARCICU中断控制单元

NVIC 配置（ARM Cortex-M）

// NVIC 中断优先级配置
// CONFIG_NUM_IRQS = 系统支持的中断数
// CONFIG_NUM_PREEMPT_PRIORITIES = 抢占优先级数
// 中断优先级：0（最高）~ N-1（最低）

// 中断分组（ARM 特有）
// 分组决定抢占优先级和子优先级的位分配
// 通常配置为 GROUP4（4位抢占，0位子优先级）

03 ISR 注册与调用

ISR 通过 IRQ_CONNECT 宏注册，在编译时链接到中断向量表。

ISR 注册

// 编译时注册 ISR
// IRQ_CONNECT(IRQn, priority, isr_handler, parameter, flags)

// 示例：注册 UART 中断
#define UART_IRQ_PRIO  1
#define UART_IRQ_FLAGS 0

// ISR 处理函数
void uart_isr(const struct device *dev, void *user_data)
{
    // 处理 UART 事件
}

// 注册 ISR
IRQ_CONNECT(DT_INST_IRQN(0), UART_IRQ_PRIO, uart_isr, NULL, UART_IRQ_FLAGS);
irq_enable(DT_INST_IRQN(0));

// 运行时动态注册（不推荐）
// irq_connect_dynamic(IRQn, priority, isr, arg, flags);

IRQ_CONNECT 宏展开

// IRQ_CONNECT 在编译时展开为：
// 1. 定义 ISR 包装函数（保存/恢复上下文）
// 2. 将 ISR 函数指针写入中断向量表
// 3. 设置中断优先级

// 对于 ARM Cortex-M：
// 1. 填充 NVIC->ICER[IRQn] 使能位
// 2. 填充 NVIC->IPR[IRQn] 优先级
// 3. 在向量表中设置 ISR 地址

04 中断上下文

ISR 在中断栈上执行，有特殊的限制。

中断栈配置

// 中断栈大小配置
// CONFIG_ISR_STACK_SIZE = 中断栈大小（字节）
// 默认值：2048 字节

// 每 CPU 独立中断栈（SMP）
// CONFIG_ARCH_IRQ_STACK_SIZE

// 中断栈内存分配
// 静态分配在 .bss 段
K_THREAD_DEFINE(isr_stack, CONFIG_ISR_STACK_SIZE, ...);

ISR 中的限制

操作ISR 中可用说明
k_sem_give()是可唤醒线程
k_fifo_put()是可向 FIFO 放入数据
k_event_post()是可发布事件
k_mutex_lock()否不能阻塞
k_sem_take()否不能等待
k_sleep()否不能休眠
k_malloc()否不能动态分配
printk()否不能输出
irq_lock()是可屏蔽中断
k_spin_lock()是可获取自旋锁

ISR 中检测

// 检查是否在 ISR 中执行
if (k_is_in_isr()) {
    // 在 ISR 中，使用 ISR 安全的 API
    k_sem_give(&my_sem);
} else {
    // 在线程中，可以使用阻塞 API
    k_sem_take(&my_sem, K_FOREVER);
}

05 嵌套中断

Zephyr 支持中断嵌套，高优先级中断可以打断低优先级中断。

// ARM Cortex-M 嵌套中断流程：
// 1. 低优先级中断触发
// 2. 保存上下文到中断栈
// 3. 执行 ISR（低优先级）
// 4. 高优先级中断触发
// 5. 保存低优先级 ISR 上下文
// 6. 执行高优先级 ISR
// 7. 恢复低优先级 ISR 上下文
// 8. 低优先级 ISR 完成
// 9. 恢复线程上下文

// 注意：嵌套中断需要足够的中断栈空间
// CONFIG_ISR_STACK_SIZE 应考虑最坏情况

06 致命异常处理

Zephyr 定义了多种致命异常类型，用于处理不可恢复的错误。

异常类型

类型值说明
K_ERR_CPU_EXCEPTION0CPU 异常（如除零、非法指令）
K_ERR_SPURIOUS_IRQ1伪中断
K_ERR_KERNEL_OOPS2内核 Oops
K_ERR_KERNEL_PANIC3内核 Panic
K_ERR_STACK_CHK_FAIL4栈溢出检测

异常处理

// kernel/fatal.c

// 致命错误处理函数
void z_fatal_error(unsigned int reason, const struct arch_esf *esf)
{
    // 1. 打印异常信息
    z_print_fatal_reason(reason, esf);

    // 2. 调用用户注册的回调
    // k_fatal_halt 或自定义处理

    // 3. 系统复位或停机
    z_system_reset();
}

// CONFIG_FATAL_ERRORS_HANDLER=y

07 ISR 延迟处理

ISR 中不能执行耗时操作，Zephyr 提供多种机制将工作延迟到线程上下文。

延迟处理机制

机制说明适用场景
k_work工作队列通用延迟处理
k_sem_give()信号量通知简单事件通知
k_fifo_put()FIFO 队列传递数据到线程
k_event_post()事件发布位掩码事件通知
k_work_schedule()延迟工作延迟执行任务
// 工作队列延迟处理
K_WORK_DEFINE(my_work, my_work_handler);

void my_isr(void *arg, void *arg2)
{
    // ISR 中只提交工作
    k_work_submit(&my_work);
}

void my_work_handler(struct k_work *work)
{
    // 在线程上下文中执行耗时操作
    process_data();
    update_display();
}
