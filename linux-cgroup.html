01 概述

cgroup（Control Group）是 Linux 内核提供的进程分组与资源限制机制。它允许将进程组织成层级结构，并对每个组的 CPU、内存、IO、网络等资源进行精细控制。Docker、Kubernetes、systemd 等容器和系统管理工具都以 cgroup 为基础。

cgroup 的核心能力
  • 资源限制：限制进程组的 CPU、内存、IO 使用量
  • 优先级分配：不同进程组获得不同的资源份额
  • 资源统计：监控进程组的资源使用情况
  • 进程控制：冻结/恢复/杀死进程组
  • 资源隔离：容器的核心隔离机制之一
设计哲学
  • 层级结构：进程组形成树状层级
  • 控制器可插拔：每个子系统独立注册
  • 文件系统接口：通过 VFS 暴露控制接口
  • kernfs 驱动：底层使用 kernfs 实现
  • 统一层级：v2 支持所有控制器挂载在同一层级
典型用途：容器运行时通过 cgroup 限制容器的 CPU 时间、内存上限、磁盘 IO 带宽；systemd 通过 cgroup 管理系统服务的资源分配；桌面环境通过 cgroup 实现 OOM 保护。

02 v1 vs v2

v1 cgroup v1
  • 每个控制器可挂载在不同层级
  • 同一进程可在不同层级有不同 cgroup
  • 控制器间互不感知
  • 接口：每个控制器有自己的文件
  • 挂载：mount -t cgroup -o cpu,name=cpu none /sys/fs/cgroup/cpu

fs/proc/cgroup.c · kernel/cgroup/cgroup-v1.c

v2 cgroup v2
  • 所有控制器挂载在同一层级
  • 统一的 cgroup.procs 和 cgroup.controllers
  • 更好的委托（delegation）支持
  • 接口：cgroup.procs、cgroup.subtree_control
  • 挂载：mount -t cgroup2 none /sys/fs/cgroup

kernel/cgroup/cgroup.c · cgroup2_fs_type

线程模式

v2 线程模式：通过 cgroup.type 文件设置为 "threaded"，允许在同一 cgroup 内按线程粒度分配资源（而不仅是进程）。这对于需要线程级 CPU 控制的应用（如 Java 服务器）非常有用。

css_set 的作用：当多个任务属于同一组 cgroup 时，它们共享同一个 css_set。这避免了在 fork/exit 时为每个任务单独更新所有子系统的引用计数。一个 css_set 代表一种"任务-控制器"绑定组合。

cgroup BPF 数据结构

// include/linux/bpf-cgroup-defs.h
struct cgroup_bpf {
    struct bpf_prog_array __rcu *effective[MAX_CGROUP_ATTACH_TYPE];
    struct list_head progs[MAX_CGROUP_ATTACH_TYPE];
    struct bpf_prog *connect_prog4;     // 连接程序
    struct bpf_prog *connect_prog6;
    // ...
};
有效程序数组：cgroup 的 effective 数组包含从根到当前 cgroup 路径上所有 BPF 程序的组合。当任务执行网络操作时，内核遍历此数组执行所有挂载的 BPF 程序。这实现了类似"策略叠加"的效果。

08 命名空间

cgroup 命名空间

cgroup 命名空间为容器提供了 /proc/self/cgroup 的虚拟化视图，使容器内的进程只能看到自己的 cgroup 子树。

// 容器内视图（cgroup 命名空间隔离后）
$ cat /proc/self/cgroup

委托（Delegation）

v2 委托模型：cgroup v2 简化了委托机制，允许非特权用户管理自己的 cgroup 子树。通过 CGRP_ROOT_NS_DELEGATE 标志，命名空间边界内的控制器接口文件可从内部写入。这使得容器运行时可以安全地管理容器内的资源限制。

CLONE_INTO_CGROUP

// 新进程直接进入指定 cgroup（无需迁移）
int fd = open("/sys/fs/cgroup/mygroup", O_RDONLY);
clone3(&(struct kernel_clone_args) {
    .flags = CLONE_INTO_CGROUP,
    .exit_signal = SIGCHLD,
    .cgroup = fd,
}, sizeof(args));
