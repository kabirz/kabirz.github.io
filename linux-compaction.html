01 概述

内存压缩（Compaction）是 Linux 的内存碎片整理机制。当系统运行一段时间后，空闲页面变得碎片化，无法找到连续的高阶页面（如 2MB 大页）。Compaction 通过移动已分配的页面，将碎片化的空闲页面合并为连续区域。

02 压缩流程

内存压缩流程：

  碎片化前：
  页面: [已用][空闲][已用][空闲][已用][空闲][已用]
  无法分配连续 4 个空闲页面

  Compaction 执行：

  1. 扫描页面，找到可迁移页面（movable）
  │   匿名页面、Page Cache 页面
  │
  2. 找到空闲页面的目标位置
  │   在可迁移页面之后寻找空闲位置
  │
  3. 迁移页面
  │   旧位置：标记为空闲
  │   新位置：复制数据，更新页表
  │
  4. 重复直到满足需求

  碎片化后：
  页面: [空闲][空闲][空闲][空闲][已用][已用][已用]
  可以分配连续 4 个空闲页面

两种模式：

  同步压缩（Synchronous）：
  - 在内存分配失败时触发
  - 当前进程阻塞等待压缩完成
  - 及时但影响延迟

  异步压缩（Asynchronous）：
  - kcompactd 守护进程后台运行
  - 主动整理内存碎片
  - 不影响进程延迟

03 触发时机

触发条件类型说明
高阶分配失败同步__alloc_pages 失败时触发
kcompactd 定时器异步定期检查并压缩
内存压力异步空闲内存低于水位线
透明大页合并异步khugepaged 需要连续内存
内存热插拔同步新内存加入时整理

04 配置与调优

# 查看碎片状态
$ cat /proc/buddyinfo
Node 0, zone   Normal  3762  2049  1042   512   256   128    64     32     16      8      4      2      1

# 查看压缩统计
$ cat /proc/vmstat | grep compact
compact_stall       12       # 同步压缩触发次数
compact_fail        3        # 压缩失败次数
compact_success     9        # 压缩成功次数

# 启动 kcompactd
$ echo 1 > /proc/sys/vm/compact_memory  # 手动触发压缩

# 调整压缩行为
$ sysctl -w vm.compact_unevictable_allowed=1  # 允许压缩不可驱逐页面
