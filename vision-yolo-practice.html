01 环境安装

1.1 安装 Ultralytics

YOLOv8 由 Ultralytics 团队维护，通过 pip 即可安装。推荐使用 Python 3.8-3.11 版本。

# 基础安装（推荐）
pip install ultralytics

# 指定版本安装
pip install ultralytics==8.3.0

# 从源码安装（最新开发版）
pip install git+https://github.com/ultralytics/ultralytics.git

# 验证安装
python -c "import ultralytics; print(ultralytics.__version__)"

# 输出示例: 8.3.0

1.2 依赖与环境

Ultralytics 自动安装 PyTorch 和相关依赖。如果需要 CUDA 加速，建议提前安装 GPU 版本的 PyTorch：

# 查看 PyTorch 版本和 CUDA 可用性
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"

# 如果需要手动安装 GPU 版 PyTorch（根据 CUDA 版本选择）
# CUDA 12.1
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

# CUDA 11.8
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118
硬件建议:
训练 YOLOv8 至少需要 4GB 显存的 GPU。YOLOv8n/yolov8s 可以在 4-6GB 显存上训练（batch=16）；YOLOv8m 需要 8GB 以上；YOLOv8l/x 建议 16GB 以上。如果没有 GPU，可以使用 Google Colab 的免费 T4 GPU（约 15GB 显存），或使用 CPU 训练小模型（会慢 10-50 倍）。

02 COCO 数据集

2.1 数据集概述

COCO（Common Objects in Context）是目标检测领域最权威的数据集之一，由微软在 2014 年发布。YOLOv8 的预训练权重即基于 COCO 数据集训练。

版本训练集图像验证集图像类别数标注类型
COCO 201482,78340,50480检测 + 分割 + 关键点
COCO 2017118,2875,00080检测 + 分割 + 关键点
COCO 2017（测试集）—40,67080无公开标注

COCO 的 80 个类别分为 12 个大类，包括：person、vehicle（bicycle、car、motorcycle 等）、animal（bird、cat、dog 等）、food（apple、orange、banana 等）等日常物体。

3.3 LabelMe 转换为 YOLO 格式

import json
import os

def labelme_to_yolo(labelme_json, img_w, img_h, class_map):
    """LabelMe JSON 转换为 YOLO 格式标注"""
    with open(labelme_json) as f:
        data = json.load(f)

    yolo_lines = []
    for shape in data["shapes"]:
        label = shape["label"]
        if label not in class_map:
            continue

        # Polygon → bbox
        points = shape["points"]
        xs = [p[0] for p in points]
        ys = [p[1] for p in points]
        x1, y1 = min(xs), min(ys)
        x2, y2 = max(xs), max(ys)

        # 归一化
        x_center = (x1 + x2) / 2.0 / img_w
        y_center = (y1 + y2) / 2.0 / img_h
        width    = (x2 - x1) / img_w
        height   = (y2 - y1) / img_h

        class_id = class_map[label]
        yolo_lines.append(
            f"{class_id} {x_center:.6f} {y_center:.6f} "
            f"{width:.6f} {height:.6f}"
        )

    return yolo_lines

数据集制作建议:
每个类别至少 300 张标注图像。数据应覆盖不同光照、角度、尺度和背景。训练集:验证集:测试集比例建议 7:2:1 或 8:1:1。标注质量比数量更重要，边界框应紧密贴合物体。使用 Roboflow 可以自动完成数据分割、格式转换和增强。

4.2 核心参数详解

参数默认值说明建议
model—模型权重文件（.pt）或配置文件（.yaml）从预训练开始，yolov8n/s/m/l/x
data—数据集配置文件路径见上节 dataset.yaml
epochs100训练轮次小数据集 100-300, 大数据集 300-600
imgsz640输入图像尺寸（像素）320/416/640/1280, 大尺寸提高小物体检测
batch16批次大小根据显存调整，越大越好（梯度更稳定）
lr00.01初始学习率小模型 0.01, 大模型 0.001
lrf0.01最终学习率（lr0 * lrf）cos_lr=True 时有效
patience50早停耐心值（轮次）验证集 mAP 持续不提升时停止
cos_lrFalse余弦退火学习率调度推荐开启，收敛更平稳
ampTrue自动混合精度训练显存减半，速度提升 1.5-2x
device—训练设备（0/1/cpu）多卡: "0,1,2,3"
workers8数据加载线程数根据 CPU 核数调整
warmup_epochs3线性预热轮次前 3 轮从低 lr 升高到 lr0
patience20早停耐心值验证 mAP 连续 20 轮不提升则停止
optimizerauto优化器（auto/SGD/Adam/AdamW）auto 自动选择 AdamW

4.4 迁移学习策略

YOLOv8 的迁移学习（Transfer Learning）策略：

  • 冻结骨干训练：在前 10-20 轮冻结 backbone 权重（设置 freeze=10），仅训练检测头，加速收敛
  • 分阶段学习率：head 层使用 lr0，backbone 层使用 lr0 × 0.1
  • 自动锚点计算：YOLOv8 会根据新数据集自动聚类锚点框尺寸
  • 标签平滑：设置 label_smoothing=0.1 减少过拟合
# 冻结训练示例
from ultralytics import YOLO

model = YOLO("yolov8m.pt")

# 1. 第一阶段：只训练检测头（冻结 backbone）
model.train(
    data="dataset.yaml",
    epochs=30,
    freeze=10,          // 冻结前 10 层（backbone）
    lr0=0.005,          // 稍高的学习率
    imgsz=640,
    batch=16,
)

# 2. 第二阶段：解冻全体微调
model.train(
    data="dataset.yaml",
    epochs=70,
    freeze=0,           // 不冻结
    lr0=0.001,          // 较小的学习率
    imgsz=640,
    batch=16,
)
训练实践建议:
从 COCO 预训练权重开始（迁移学习），在 100 个类别上，YOLOv8n 在单卡上约 8 小时收敛。小数据集（500 张以下）需加强数据增强，建议开启 Mosaic（0.75-1.0）和 MixUp（0.1-0.3）。注意 Mosaic 增强会在最后 10 个 epoch 自动关闭。

5.4 训练曲线解读

YOLOv8 自动生成的 results.png 包含以下曲线：

曲线说明理想趋势
train/box_loss边界框回归损失不断下降并收敛
train/cls_loss分类损失不断下降并收敛
train/dfl_loss分布聚焦损失不断下降并收敛
metrics/precision精确率（验证集）趋向稳定值
metrics/recall召回率（验证集）趋向稳定值
metrics/mAP50IoU=0.5 时的 mAP趋向稳定值
metrics/mAP50-95IoU 0.5-0.95 的 mAP趋向稳定值
val/box_loss验证集边界框损失下降后不回升（回升=过拟合）
训练诊断:
如果训练损失下降但验证损失上升 → 过拟合，减少 epochs 或增加数据增强。如果训练和验证损失都不下降 → 学习率太低或模型容量太小。如果 mAP 波动剧烈 → 批次大小太小或学习率太高。建议用 patience=20 早停防止过拟合。

6.4 模型调优建议

问题指标表现解决方法
精确率低（误检多）P 低, R 高提高置信度阈值 conf, 增加负样本, 提高 IoU 阈值
召回率低（漏检多）P 高, R 低降低置信度阈值, 增加数据增强, 提高输入分辨率
小物体检测差AP_s 远低于 AP_l增大 imgsz, 使用更深的模型, 增加小物体数据
类别混淆非对角线值高检查标签是否正确, 增加类别间区分性数据
过拟合训练 mAP 远高于验证增加数据增强, 减少 epochs, 增大 dropout
欠拟合训练和验证 mAP 都低增加 epoch, 增加模型容量, 检查学习率

07 预测与导出

7.1 图像预测

from ultralytics import YOLO

# 加载训练好的模型
model = YOLO("runs/detect/train/weights/best.pt")

# 单张图像预测
results = model("test.jpg")

# 图像列表

# 文件夹中所有图像
results = model("path/to/images/")

# URL 图像
results = model("https://example.com/image.jpg")

# 解析结果
for i, result in enumerate(results):
    boxes = result.boxes                   // 边界框
    print(f"图像 {i}: {len(boxes)} 个目标")

    for box in boxes:
        cls_id = int(box.cls[0])           // 类别 ID
        conf = float(box.conf[0])          // 置信度
        xyxy = box.xyxy[0].tolist()        // [x1, y1, x2, y2]
        xywh = box.xywh[0].tolist()        // [x, y, w, h]

        print(f"  类别={cls_id}, 置信度={conf:.2f}, "
              f"框={xyxy}")

    // 可视化结果
    annotated = result.plot()              // BGR numpy 数组
    result.save(f"result_{i}.jpg")      // 保存标注后的图像

7.3 模型导出

YOLOv8 支持导出为多种部署格式，适应不同的硬件平台：

from ultralytics import YOLO

model = YOLO("best.pt")

# 导出为 ONNX (最通用)
model.export(format="onnx", dynamic=True, simplify=True)
# 输出: best.onnx

# 导出为 TensorRT (NVIDIA GPU 最优)
model.export(format="engine", half=True, imgsz=640)
# 输出: best.engine (FP16 精度)

# 导出为 OpenVINO (Intel CPU/VPU)
model.export(format="openvino", half=False)
# 输出: best.xml + best.bin

# 导出为 CoreML (Apple 设备)
model.export(format="coreml")
# 输出: best.mlpackage

# 导出为 TFLite (移动端/嵌入式)
model.export(format="tflite", int8=True, data="dataset.yaml")
# 输出: best_float32.tflite / best_int8.tflite

# 导出为 PaddlePaddle
model.export(format="paddle")
# 输出: model.pdmodel + model.pdiparams

7.4 导出格式对比

格式设备速度提升精度损失使用场景
PyTorch (.pt)任何1x无训练、验证、开发
ONNX任何1-2x极小跨平台部署、框架无关
TensorRT (FP16)NVIDIA GPU3-5x极小生产级 GPU 推理
TensorRT (INT8)NVIDIA GPU5-8x1-2%推理延迟要求极高
OpenVINOIntel CPU/VPU2-4x极小Intel 平台部署
TFLite (INT8)ARM/Edge TPU3-5x2-5%移动端、嵌入式
CoreMLApple2-4x极小iOS/macOS
导出注意事项:
ONNX 导出后可以用 ONNXRuntime 推理，获得稳定的生产级性能。TensorRT 导出需要 NVIDIA GPU 且耗时较长（INT8 量化需要校准数据集）。TFLite INT8 量化需要代表性数据集用于校准。对于边缘设备（Jetson），推荐 ONNX → TensorRT 路线；对于 Intel 服务器，推荐 OpenVINO 路线。

08 完整端到端示例

以下是一个完整的 YOLOv8 工作流程脚本，从数据准备到模型部署全覆盖：

#!/usr/bin/env python3
"""YOLOv8 端到端训练与部署流水线"""

import argparse
import sys
from pathlib import Path
from ultralytics import YOLO


class YOLOPipeline:
    """YOLOv8 训练与部署流水线"""

    def __init__(self, model_name="yolov8m.pt"):
        self.model_name = model_name
        self.model = None

    def train(self, data_yaml, epochs=100, imgsz=640,
                batch=16, device="0"):
        """训练模型"""
        print(f"[INFO] 加载模型: {self.model_name}")
        self.model = YOLO(self.model_name)

        print("[INFO] 开始训练...")
        self.model.train(
            data=data_yaml,
            epochs=epochs,
            imgsz=imgsz,
            batch=batch,
            device=device,
            workers=8,
            pretrained=True,
            optimizer="auto",
            cos_lr=True,
            amp=True,
            patience=20,
            save=True,
            save_period=10,
            val=True,
            plots=True,
        )
        print("[INFO] 训练完成")

    def evaluate(self, weight_path, data_yaml, imgsz=640):
        """评估模型"""
        self.model = YOLO(weight_path)
        metrics = self.model.val(
            data=data_yaml,
            imgsz=imgsz,
            batch=16,
            plots=True,
        )
        print(f"[RESULT] mAP@0.5: {metrics.box.map50:.4f}")
        print(f"[RESULT] mAP@0.5:0.95: {metrics.box.map:.4f}")
        print(f"[RESULT] Precision: {metrics.box.p:.4f}")
        print(f"[RESULT] Recall: {metrics.box.r:.4f}")
        return metrics

    def export(self, weight_path, format="onnx",
                half=True, imgsz=640):
        """导出模型到指定格式"""
        self.model = YOLO(weight_path)
        export_path = self.model.export(
            format=format,
            half=half,
            imgsz=imgsz,
            simplify=True,
            dynamic=True,
        )
        print(f"[INFO] 导出成功: {export_path}")
        return export_path

    def predict(self, weight_path, source, conf=0.25, iou=0.45):
        """运行推理"""
        self.model = YOLO(weight_path)
        results = self.model(
            source=source,
            conf=conf,
            iou=iou,
            save=True,
            save_txt=True,
            save_conf=True,
        )
        print(f"[INFO] 处理了 {len(results)} 个结果")
        return results


def main():
    parser = argparse.ArgumentParser(
        description="YOLOv8 Pipeline"
    )
    parser.add_argument("--mode", required=True,
                        choices=["train", "eval",
                                 "export", "predict"])
    parser.add_argument("--model", default="yolov8m.pt")
    parser.add_argument("--data", default="dataset.yaml")
    parser.add_argument("--weights", default="best.pt")
    parser.add_argument("--source", default="test.jpg")
    parser.add_argument("--epochs", type=int, default=100)
    parser.add_argument("--imgsz", type=int, default=640)
    parser.add_argument("--batch", type=int, default=16)
    parser.add_argument("--format", default="onnx")
    args = parser.parse_args()

    pipeline = YOLOPipeline(model_name=args.model)

    if args.mode == "train":
        pipeline.train(args.data, args.epochs,
                       args.imgsz, args.batch)
    elif args.mode == "eval":
        pipeline.evaluate(args.weights, args.data, args.imgsz)
    elif args.mode == "export":
        pipeline.export(args.weights, args.format)
    elif args.mode == "predict":
        pipeline.predict(args.weights, args.source)


if __name__ == "__main__":
    main()
