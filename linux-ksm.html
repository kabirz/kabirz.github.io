01 概述

KSM（Kernel Same-page Merging）是 Linux 的内存去重机制。它定期扫描进程的匿名页面，找到内容相同的页面并将它们合并为一个共享页面（写时复制）。在虚拟化环境中，多个 VM 运行相同操作系统时，KSM 可以节省 30-70% 的内存。

02 合并流程

KSM 合并流程：

  ksmd 守护进程定期运行
        │
        ├── 1. 收集候选页面
        │       扫描进程地址空间中被标记为可合并的页面
        │       madvise(MADV_MERGEABLE) 标记
        │
        ├── 2. 计算页面校验和
        │       对每个页面计算 fasthash
        │
        ├── 3. 查找匹配
        │       校验和相同 → 逐字节比较
        │       确认内容完全相同
        │
        ├── 4. 合并页面
        │       多个相同页面 → 合并为一个
        │       所有映射指向同一物理页面
        │       页面标记为 COW（写时复制）
        │
        └── 5. 释放多余物理页面
            返回给伙伴系统

合并效果示例：

  合并前：
  VM 1: [page A] [page B] [page C]  （3 个不同页面）
  VM 2: [page A] [page B] [page D]  （3 个不同页面）
  共 6 个物理页面

  合并后：
  VM 1: [page A] [page B] [page C]  （page A, B 共享）
  VM 2: [page A] [page B] [page D]  （page A, B 共享）
  共 4 个物理页面（节省 33%）

03 应用场景

场景说明节省
虚拟化（KVM/QEMU）多 VM 运行相同 OS30-70%
容器（Docker）多容器共享基础镜像页面20-50%
桌面系统多进程共享库页面10-30%
科学计算多进程处理相同数据视数据而定
KSM 安全风险
KSM 合并的页面可以通过侧信道攻击（如缓存定时攻击）检测其他进程的内存内容。因此在多租户环境中，需要权衡内存节省和安全风险。

04 配置与调优

# 启用 KSM
$ echo 1 > /sys/kernel/mm/ksm/run

# 查看 KSM 统计
$ cat /sys/kernel/mm/ksm/pages_shared     # 共享页面数
$ cat /sys/kernel/mm/ksm/pages_sharing    # 引用共享页面的页面数
$ cat /sys/kernel/mm/ksm/pages_to_scan    # 每次扫描的页面数
$ cat /sys/kernel/mm/ksm/sleep_millisecs  # 扫描间隔（毫秒）

# 调整扫描参数
$ echo 100 > /sys/kernel/mm/ksm/pages_to_scan     # 每次扫描 100 页
$ echo 20 > /sys/kernel/mm/ksm/sleep_millisecs    # 每 20ms 扫描一次

# 标记进程页面为可合并
$ madvise(addr, len, MADV_MERGEABLE);  // 系统调用

# QEMU/KVM 使用 KSM
$ echo 1 > /sys/kernel/mm/ksm/run
$ virsh qemu-monitor-command vm --hmp --cmd "info ksm"
