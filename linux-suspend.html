01 概述

Linux 支持多种系统休眠状态：挂起到内存（Suspend-to-RAM，即睡眠）、挂起到磁盘（Suspend-to-Disk，即休眠）、混合休眠等。这些机制是笔记本电池续航、服务器节能的关键。

02 休眠状态

状态系统调用恢复速度功耗说明
suspend (S3)echo mem > /sys/power/state快（秒级）极低挂起到内存（笔记本睡眠）
hibernate (S4)echo disk > /sys/power/state慢（分钟级）零挂起到磁盘（完全关机状态）
hybridecho disk > /sys/power/state + 检查快/慢极低同时写入内存和磁盘
freezeecho freeze > /sys/power/state极快中低仅冻结用户进程
idleCPU idle states立即低仅 CPU 空闲（已有 cpuidle 文章）

03 挂起流程

Suspend-to-RAM (S3) 流程：

  用户态：echo mem > /sys/power/state
        │
        ↓
  pm_suspend() — kernel/power/suspend.c
        │
        ├── 1. 冻结用户进程
        │       freeze_processes()
        │       所有用户态进程挂起（TASK_FROZEN）
        │
        ├── 2. 冻结内核线程
        │       freeze_kernel_threads()
        │       仅允许 suspend 专用线程运行
        │
        ├── 3. 设备挂起
        │       dpm_suspend()
        │       遍历所有设备，调用 dev->pm_ops->suspend()
        │       ├── 网卡：停止 DMA，保存寄存器
        │       ├── 磁盘：停止 I/O，刷新缓存
        │       ├── USB：断开连接
        │       └── GPU：保存状态
        │
        ├── 4. 平台挂起
        │       platform_suspend_ops->suspend()
        │       写入 ACPI S3 状态
        │       CPU 停止，内存保持刷新
        │
        └── 5. 系统进入睡眠
            仅内存和唤醒设备保持供电

唤醒流程：

  唤醒事件（按键/网络包/定时器）
        │
        ↓
  1. CPU 恢复执行
  2. 平台恢复：platform_suspend_ops->resume()
  3. 设备恢复：dpm_resume()
  4. 解冻内核线程
  5. 解冻用户进程
  6. 系统恢复正常运行

04 配置与调优

# 查看支持的休眠状态
$ cat /sys/power/state
mem freeze disk

# 查看唤醒设备
$ cat /proc/acpi/wakeup
Device  S-state   Status   Sysfs node
LID     S3    *enabled   LNXPWRGBBTN:00
EHC1    S3    *enabled   EHCI:00
XHC     S3    *enabled   xhci_hcd:00

# 禁用特定设备唤醒
$ echo EHC1 > /proc/acpi/wakeup  # 切换 enable/disable

# 查看挂起统计
$ sudo pm-suspend --verbose   # 详细挂起日志
$ dmesg | grep -i "suspend\|resume\|freeze"

# systemd 挂起
$ systemctl suspend           # S3 挂起
$ systemctl hibernate         # S4 休眠
$ systemctl hybrid-sleep     # 混合休眠
