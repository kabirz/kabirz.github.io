01 概述

sk_buff（socket buffer）是 Linux 内核网络栈中最核心的数据结构。每一个网络数据包——无论是在收包路径、发包路径、协议栈处理、Netfilter 钩子、还是设备驱动——都以 sk_buff 的形式在内核中流转。理解 sk_buff 就是理解 Linux 网络栈的关键。

sk_buff 无处不在
  • 网卡驱动 → sk_buff → 协议栈
  • TCP/UDP → sk_buff → IP 层 → 设备层
  • Netfilter 在 sk_buff 上执行过滤
  • bridge 在 sk_buff 上转发
  • iptables 在 sk_buff 上修改/NAT
  • tc (Traffic Control) 排队管理 sk_buff

02 内存布局

四指针模型

sk_buff 的数据区域使用四个指针管理：

  +------+------+------+------+------+------+------+------+
  | head |      | data |      |      | tail |      | end  |
  +------+------+------+------+------+------+------+------+
  ↑               ↑                       ↑              ↑
  head            data                    tail           end
  (分配起点)      (数据起点)              (数据终点)      (分配终点)

  head → data  : headroom（预留空间，可 prepend 头部）
  data → tail  : 实际数据（以太网帧 / IP 包 / TCP 段）
  tail → end   : tailroom（尾部空间，可 append 数据）

  使用场景：
  ┌──────────────────────────────────────────────────────────┐
  │ headroom (可 prepend)   │  data (实际数据)  │ tailroom   │
  │                         │                   │            │
  │ ETH HDR prepended here  │ ← skb_push() →   │ ← append   │
  │ → skb_put() 尾部扩展    │    skb_pull() 前移│            │
  └──────────────────────────────────────────────────────────┘

  举例：收到一个 1500 字节的以太网帧
  head = 0xffff880012340000
  data = 0xffff88001234002e  (data - head = 46 bytes headroom)
  tail = 0xffff880012340614  (tail - data = 1500 bytes)
  end  = 0xffff880012341000  (end - tail = tailroom)

四指针操作 API

操作函数效果
头部预留skb_reserve(skb, len)data 和 tail 同时前移，预留头部空间
头部扩展skb_push(skb, len)data 前移 len 字节，增长数据头部
头部收缩skb_pull(skb, len)data 后移 len 字节，削减数据头部
尾部扩展skb_put(skb, len)tail 后移 len 字节，增长数据尾部
尾部收缩skb_trim(skb, len)tail 前移到 data+len，截断数据

03 核心字段

// include/linux/skbuff.h:886 — sk_buff 核心字段
struct sk_buff {
    /* ──── 链表 / 树节点 ──── */
    union {
        struct { struct sk_buff *next, *prev; }; // 队列链表
        struct rb_node    rbnode;     // 红黑树节点（TCP 重传队列）
        struct list_head list;       // 通用链表
    };

    /* ──── 关联的 socket 和设备 ──── */
    struct sock        *sk;         // 所属 socket
    struct net_device  *dev;         // 关联的网络设备

    /* ──── 时间戳 ──── */
    ktime_t            tstamp;      // 收发时间戳

    /* ──── 控制缓冲区（每层私有数据） ──── */
    char               cb[48] __aligned(8); // 48 字节，各层可用

    /* ──── 数据长度 ──── */
    unsigned int       len;         // 数据总长度（含 frag）
    unsigned int       data_len;    // 非线性数据长度

    /* ──── 协议头偏移量 ──── */
    __u16              transport_header; // L4（TCP/UDP）头偏移
    __u16              network_header;   // L3（IP）头偏移
    __u16              mac_header;       // L2（以太网）头偏移
    __u16              inner_transport_header; // 内层 L4（隧道）
    __u16              inner_network_header;   // 内层 L3（隧道）
    __u16              inner_mac_header;       // 内层 L2（隧道）

    /* ──── 校验和 ──── */
    __u32              csum;        // 校验和
    __u8               ip_summed;   // CHECKSUM_NONE/UNNECESSARY/PARTIAL

    /* ──── 数据指针（关键！） ──── */
    sk_buff_data_t     tail;        // 数据尾指针
    sk_buff_data_t     end;         // 缓冲区尾指针
    unsigned char      *head;       // 缓冲区头指针
    unsigned char      *data;       // ★ 数据起始指针

    /* ──── 元数据 ──── */
    __u16              protocol;    // 协议类型（ETH_P_IP 等）
    __u16              vlan_tci;    // VLAN 标签
    __u32              mark;        // Netfilter 标记
    __u32              priority;    // 优先级
    int                skb_iif;     // 入接口索引
    u32                hash;        // 流哈希（RPS/ECMP）

    /* ──── 引用计数 ──── */
    refcount_t         users;       // 引用计数
    unsigned int       truesize;    // 实际占用内存（含 skb 结构）
};

04 生命周期

收包路径中 sk_buff 的生命周期：

  1. 分配
  网卡驱动调用 __netdev_alloc_skb()
  → __alloc_skb() → kmalloc 头 + alloc_page 数据
  → 初始化 skb->head, data, tail, end
  → skb->dev = 网卡
  → skb_reserve(skb, NET_SKB_PAD)  // 预留头部空间

  2. DMA 写入
  网卡将数据 DMA 到 skb 的 data 区域
  → skb_put(skb, pkt_len)  // 设置数据长度

  3. 协议栈处理
  每层协议处理时修改 skb：
  → eth_type_trans()       // skb_pull 剥离以太网头
  → skb_pull(skb, ip_hdr_len) // 剥离 IP 头
  → skb_pull(skb, tcp_hdr_len) // 剥离 TCP 头
  → 到达 socket 时 skb->data 指向应用数据

  4. 用户态读取
  应用调用 recv() / read()
  → skb_copy_datagram() 将数据拷贝到用户态

  5. 释放
  kfree_skb() / __kfree_skb()
  → 减少引用计数
  → 引用为 0 时释放 skb 和数据缓冲区
  → 回收 page / slab

发包路径中 sk_buff 的生命周期：

  1. 应用调用 send() → 内核分配 skb
  2. TCP/UDP 层添加协议头（skb_push）
  3. IP 层添加 IP 头（skb_push）
  4. 设备层添加以太网头（skb_push）
  5. 驱动 DMA 发送
  6. 发送完成 → 释放 skb

05 常用操作

// 头部操作 — 逐层剥离协议头

// 收包：从 L2 到 L4
skb->protocol = eth_type_trans(skb, dev); // 剥离以太网头，设置协议
// skb->data 现在指向 IP 头

ip_hdr = ip_hdr(skb);                    // 获取 IP 头指针
skb_pull(skb, ip_hdrlen(skb));            // 剥离 IP 头
// skb->data 现在指向 TCP/UDP 头

tcp_hdr = tcp_hdr(skb);                   // 获取 TCP 头指针

// 发包：逐层添加协议头
skb_reserve(skb, header_len);             // 预留头部空间
skb_put(skb, payload_len);                // 设置负载数据
skb_push(skb, ip_hdr_len);               // 添加 IP 头
skb_push(skb, eth_hdr_len);              // 添加以太网头

// 隧道封装：添加内外层头部
skb->inner_network_header = skb->network_header;
skb_push(skb, outer_hdr_len);             // 添加外层头
skb->encapsulation = 1;                  // 标记有内层封装

06 克隆与复制

skb_clone() — 浅拷贝（共享数据区）

  原始 sk_buff                    克隆的 sk_buff
  ┌──────────┐                   ┌──────────┐
  │ skb 头部  │                   │ skb 头部  │
  │ data ────┼──→ 共享数据区 ←────┼─── data  │
  │ ...      │                   │ ...      │
  └──────────┘                   └──────────┘
       ↓                              ↓
  head ──────────→ [共享数据缓冲区] ←────────── head

  用途：
  - Netfilter 需要同时查看和修改数据包
  - TCP 重传时保留原始数据
  - tc 分类时多路径处理

skb_copy() — 深拷贝（独立数据区）

  原始 sk_buff                    复制的 sk_buff
  ┌──────────┐                   ┌──────────┐
  │ skb 头部  │                   │ skb 头部  │
  │ data ────┼──→ 数据区 A        │ data ────┼──→ 数据区 B（独立副本）
  └──────────┘                   └──────────┘

  用途：
  - 需要修改数据但不影响原始包
  - 向用户态发送数据副本
操作函数数据共享性能
浅拷贝skb_clone()共享数据区快（仅分配 skb 头部）
深拷贝skb_copy()独立副本慢（分配 skb + 复制数据）
头部分离skb_cow_head()按需分离中（仅在需要写头时复制）
skb_cow_head() 的巧妙设计
当协议栈需要修改 skb 头部（如 TCP 重传修改序号）时，调用 skb_cow_head()。它会检查头部空间是否充足且 skb 是否被共享（cloned）。如果不满足，才执行 skb_copy_head() 分离。这避免了不必要的复制，是"写时复制"（COW）思想在内核网络栈中的应用。
