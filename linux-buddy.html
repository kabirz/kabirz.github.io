01 概述

伙伴系统（Buddy System）是 Linux 内核的物理页面分配器，负责管理所有物理内存。它将空闲页面组织成 11 个阶（order 0~10），每个阶管理 2^order 个连续页面。当内核需要分配内存时（如 sk_buff、inode、内核模块），最终都会走到伙伴系统分配物理页面。

02 Zone 与伙伴系统

Linux 物理内存管理层次：

  物理内存
    │
    ├── Zone DMA（0~16MB）— ISA 设备 DMA 区域
    ├── Zone DMA32（0~4GB）— 32 位设备 DMA
    └── Zone Normal（4GB+）— 常规内存

  每个 Zone 包含 11 个 free_list（伙伴系统核心）：

  Zone Normal
  ├── free_area[0] → 2^0 = 1 页面 (4KB)
  ├── free_area[1] → 2^1 = 2 页面 (8KB)
  ├── free_area[2] → 2^2 = 4 页面 (16KB)
  ├── ...
  ├── free_area[9] → 2^9 = 512 页面 (2MB)
  └── free_area[10] → 2^10 = 1024 页面 (4MB)

  每个 free_area 包含一个链表
  链表上的块都是 2^order 个连续页面

03 分配与释放流程

分配页面 __alloc_pages()：

  需要 order=2（4 个连续页面）
        │
        ├── 1. 检查 watermarks（水位线）
        │       high → 低水位 → 唤醒 kswapd
        │       low → 紧急，直接回收
        │       min → 危险，可能失败
        │
        ├── 2. 查找 free_area[2]
        │       有 → 取出一个块，分配成功
        │
        ├── 3. 没有 → 拆分更大的块
        │       找到 free_area[3]（8 页面）
        │       拆分为两个 free_area[2]
        │       一个分配，一个放回 free_area[2]
        │
        ├── 4. 如果都找不到 → 内存回收
        │       kswapd / direct reclaim
        │       回收 Page Cache / slab
        │
        └── 5. 仍然找不到 → OOM Killer

释放页面 __free_pages()：

  释放 order=2 的块（4 页面）
        │
        ├── 1. 检查伙伴（buddy）是否存在
        │       伙伴 = 地址 ^ (4KB << order)
        │
        ├── 2. 伙伴空闲 → 合并
        │       两个 order=2 合并为 order=3
        │       继续检查 order=3 的伙伴
        │       递归合并直到无法合并
        │
        └── 3. 放入对应 free_area[order]

04 SLUB 分配器

SLUB（Slab Unqueued）是 Linux 内核的小对象分配器，管理小于页面大小的内核对象（如 struct inode、struct task_struct）。

伙伴系统 vs SLUB：

  伙伴系统：分配整页（4KB、8KB、...）
  SLUB：分配对象（如 192B 的 inode、6KB 的 task_struct）

  SLUB 工作方式：
  1. 从伙伴系统分配整页
  2. 将页面切分为固定大小的对象槽位
  3. 分配时从 freelist 取一个空闲槽位
  4. 释放时将槽位放回 freelist

  ┌──────────────────────────────────────────┐
  │ SLUB 页（4KB）                            │
  ├──────────┬──────────┬──────────┬──────────┤
  │ object 0 │ object 1 │ object 2 │ object 3 │
  │ (已分配)  │ (空闲)   │ (已分配)  │ (空闲)   │
  └──────────┴───┬──────┴──────────┴───┬──────┘
                  │                      │
              freelist              freelist
              (空闲对象链表)

  /proc/slabinfo 查看所有 slab 缓存
  /sys/kernel/slab/ 查看单个缓存详情

05 OOM Killer

OOM Killer 工作流程：

  内存不足 → __alloc_pages 失败
        │
        ↓
  out_of_memory() — mm/oom_kill.c
        │
        ├── 1. 检查是否 OOM 允许
        │       system 例外、OOM_SCORE_ADJ_MAX
        │
        ├── 2. 选择牺牲进程
        │       oom_badness() 计算 oom_score
        │       分数 = RSS + swap + 页表 + oom_score_adj
        │       分数最高 → 被选中
        │
        ├── 3. 发送 SIGKILL
        │       do_send_sig_info(SIGKILL, ...)
        │
        └── 4. 进程退出 → 释放内存 → 恢复
