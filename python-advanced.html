01 描述器协议

描述器是 Python 中属性访问的核心机制。任何定义了 __get__、__set__ 或 __delete__ 中至少一个方法的对象，都是一个描述器。当类属性被访问时，Python 会按 数据描述器 > 实例属性 > 非数据描述器 的优先级顺序查找。

描述器协议定义

提示:
描述器协议是 property、classmethod、staticmethod 以及 slot 的底层实现基础。理解描述器就等于理解了 Python 的属性机制。
class Descriptor:
    """一个典型的数据描述器"""
    def __get__(self, obj, objtype=None):
        print(f"GET from {objtype.__name__}")
        return getattr(obj, '_value', None)

    def __set__(self, obj, value):
        print(f"SET {value} on {obj}")
        obj._value = value

    def __delete__(self, obj):
        print(f"DELETE from {obj}")
        del obj._value

class MyClass:
    attr = Descriptor()

obj = MyClass()
obj.attr = 42    # SET 42 on ...
print(obj.attr)  # GET from MyClass
del obj.attr     # DELETE from ...

数据描述器 vs 非数据描述器

描述器按是否定义 __set__ 分为两类。数据描述器（定义了 __set__ 或 __delete__）优先级高于实例属性。非数据描述器（只定义 __get__）的优先级低于实例属性，这正是方法的工作方式。

class NonDataDescriptor:
    """非数据描述器——只有 __get__"""
    def __get__(self, obj, objtype=None):
        return "from descriptor"

class Demo:
    attr = NonDataDescriptor()

d = Demo()
print(d.attr)    # "from descriptor"
d.attr = "实例值"
print(d.attr)    # "实例值"——实例属性覆盖了非数据描述器
del d.attr
print(d.attr)    # "from descriptor"——删除实例属性后恢复

property 的实现原理

property 是 Python 内置的数据描述器。它包装了 getter、setter 和 deleter 函数，在 __get__ 中调用 getter，在 __set__ 中调用 setter。

class Property:
    """简化版 property 实现"""
    def __init__(self, fget=None, fset=None, fdel=None):
        self.fget = fget
        self.fset = fset
        self.fdel = fdel

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        if self.fget is None:
            raise AttributeError
        return self.fget(obj)

    def __set__(self, obj, value):
        if self.fset is None:
            raise AttributeError
        self.fset(obj, value)

    def setter(self, func):
        return type(self)(self.fget, func, self.fdel)

classmethod 实现原理

classmethod 是一个非数据描述器。它在 __get__ 中将函数绑定到类，而不是实例。无论通过类还是实例访问，第一个参数始终是类本身。

class ClassMethod:
    """简化版 classmethod"""
    def __init__(self, func):
        self.func = func

    def __get__(self, obj, objtype=None):
        if objtype is None:
            objtype = type(obj)
        return lambda *args, **kw: self.func(objtype, *args, **kw)

staticmethod 实现原理

staticmethod 是最简单的描述器。它直接在 __get__ 中返回原始函数，不做任何绑定。

class StaticMethod:
    """简化版 staticmethod"""
    def __init__(self, func):
        self.func = func

    def __get__(self, obj, objtype=None):
        return self.func

描述器在 Python 中的应用

描述器无处不在。函数本身就是非数据描述器，所以方法调用会隐式传入 self。@property 用于计算属性，__slots__ 背后也使用了描述器来管理实例属性。ORM 框架如 SQLAlchemy 大量使用描述器实现声明式映射。

class ValidatedField:
    """ORM 风格的验证字段描述器"""
    def __init__(self, validator):
        self.validator = validator
        self.data = {}

    def __set_name__(self, owner, name):
        self.name = name

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return self.data.get(id(obj))

    def __set__(self, obj, value):
        self.validator(value)
        self.data[id(obj)] = value

def non_empty(val):
    if not val:
        raise ValueError("值不能为空")

class User:
    name = ValidatedField(non_empty)
    email = ValidatedField(non_empty)

02 元类

元类是创建类的类。在 Python 中，type 是所有元类的基类。当定义一个类时，Python 会调用元类的 __new__ 和 __init__ 来创建类对象。理解元类，就是理解 Python 类创建过程的终极控制权。

type 创建类

type(name, bases, dict) 是动态创建类的标准方式。三个参数分别代表类名、父类元组和命名空间字典。

# 用 type 动态创建类等价于 class 语句
def init(self, name):
    self.name = name

def greet(self):
    return f"Hello, {self.name}"

DynamicClass = type('DynamicClass', (), {
    '__init__': init,
    'greet': greet
})

obj = DynamicClass("World")
print(obj.greet())  # Hello, World

__new__ vs __init__

元类的 __new__ 在类创建之前调用，负责创建并返回类对象。__init__ 在新类创建之后调用，负责初始化。__new__ 是类工厂的核心，__init__ 适合做后处理。

class Meta(type):
    def __new__(mcs, name, bases, namespace):
        print(f"__new__: 创建类 {name}")
        # 可以在 __new__ 中修改类的定义
        namespace['created_by'] = mcs.__name__
        return super().__new__(mcs, name, bases, namespace)

    def __init__(cls, name, bases, namespace):
        print(f"__init__: 初始化类 {name}")
        super().__init__(name, bases, namespace)

class MyClass(metaclass=Meta):
    pass

print(MyClass.created_by)  # Meta

__call__ 和类创建钩子

元类的 __call__ 在 ClassName() 被调用时触发。这意味着元类可以控制实例创建的全过程，常用于实现单例模式或对象池。

class SingletonMeta(type):
    _instances = {}

    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            instance = super().__call__(*args, **kwargs)
            cls._instances[cls] = instance
        return cls._instances[cls]

class Logger(metaclass=SingletonMeta):
    def __init__(self):
        self.level = "INFO"

a = Logger()
b = Logger()
print(a is b)  # True

注册模式

元类可以自动收集子类信息，实现注册表模式。这在插件系统和框架中非常常见。

class RegistryMeta(type):
    registry = {}

    def __new__(mcs, name, bases, namespace):
        cls = super().__new__(mcs, name, bases, namespace)
        if not name.startswith('Base'):
            mcs.registry[name] = cls
        return cls

class BasePlugin(metaclass=RegistryMeta):
    """所有插件的基类，不会被注册"""
    def process(self, data):
        raise NotImplementedError

class JsonPlugin(BasePlugin):
    def process(self, data):
        return {"data": data}

class XmlPlugin(BasePlugin):
    def process(self, data):
        return f"{data}"

print(BasePlugin.registry)
# {'JsonPlugin': , 'XmlPlugin': }

ORM 中的应用

元类在 ORM 框架中扮演核心角色。SQLAlchemy 的 declarative_base 背后就是元类，它负责将声明式类定义转换为数据库表映射。

class ModelMeta(type):
    def __new__(mcs, name, bases, namespace):
        if name == 'Model':
            return super().__new__(mcs, name, bases, namespace)

        fields = {}
        for key, val in namespace.items():
            if isinstance(val, Field):
                fields[key] = val
                namespace[key] = None

        namespace['_fields'] = fields
        namespace['_table'] = name.lower()
        return super().__new__(mcs, name, bases, namespace)

class Field:
    def __init__(self, ftype):
        self.ftype = ftype

class Model(metaclass=ModelMeta):
    def save(self):
        cols = ', '.join(self._fields.keys())
        vals = ', '.join(f"'{getattr(self, k)}'" for k in self._fields)
        return f"INSERT INTO {self._table} ({cols}) VALUES ({vals})"

class User(Model):
    name = Field(str)
    email = Field(str)

user = User()
user.name = "Alice"
user.email = "alice@example.com"
print(user.save())
# INSERT INTO user (name, email) VALUES ('Alice', 'alice@example.com')

元类的限制与最佳实践

元类是非常强大的工具，但同时也增加了代码的复杂度。一般规则是：能用装饰器、类装饰器或 __init_subclass__ 解决的问题，就不要用元类。Python 3.6 引入的 __init_subclass__ 可以在父类中拦截子类创建，是元类的一个轻量替代。

最佳实践:
元类的使用场景包括：ORM 映射、单例模式、注册模式、API 验证框架、DSL 构建。在框架设计之外，大多数情况下不需要自定义元类。

03 抽象基类 ABC

抽象基类（Abstract Base Class）是 Python 中定义接口和规范的标准化方式。Python 的 abc 模块提供了 ABC 类和 @abstractmethod 装饰器，用于声明抽象方法，强制子类实现。

abc.ABC 与 @abstractmethod

从 ABC 继承的类不能直接实例化（如果包含抽象方法）。子类必须实现所有抽象方法才能实例化。这是实现模板方法模式和策略模式的理想方式。

from abc import ABC, abstractmethod

class Stream(ABC):
    @abstractmethod
    def read(self, size: int) -> bytes:
        ...

    @abstractmethod
    def write(self, data: bytes) -> int:
        ...

    def close(self):
        """非抽象方法——子类可以选择性重写"""
        pass

class FileStream(Stream):
    def read(self, size: int) -> bytes:
        return b"data" * size

    def write(self, data: bytes) -> int:
        return len(data)

s = FileStream()
print(s.read(10))  # b'datadatadata...'

# Stream()  # TypeError: Can't instantiate abstract class

register() 方法

register() 可以将一个类注册为 ABC 的虚拟子类，而无需实际继承。虚拟子类通过 isinstance() 和 issubclass() 检查时会返回 True，但不会继承 ABC 的任何方法。

from abc import ABC

class Iterable(ABC):
    @abstractmethod
    def __iter__(self):
        ...

# 将内置 list 注册为 Iterable 的虚拟子类
Iterable.register(list)

print(isinstance([1, 2, 3], Iterable))  # True

# 自定义类注册
@Iterable.register
class MyList:
    def __iter__(self):
        return iter([1, 2, 3])

print(issubclass(MyList, Iterable))  # True

__subclasshook__

__subclasshook__ 是 ABC 上最灵活的机制。它允许基于方法存在性（而非继承关系）来判断子类关系。collections.abc 中的许多抽象基类都利用了这个机制实现了鸭子类型兼容。

from abc import ABC, abstractmethod

class Sized(ABC):
    @abstractmethod
    def __len__(self):
        ...

    @classmethod
    def __subclasshook__(cls, other):
        if cls is Sized:
            if any("__len__" in B.__dict__ for B in other.__mro__):
                return True
        return NotImplemented

class MyContainer:
    def __len__(self):
        return 42

print(issubclass(MyContainer, Sized))  # True
print(isinstance(MyContainer(), Sized))  # True

collections.abc 标准库中的 ABC

标准库的 collections.abc 模块定义了大量有用的 ABC，包括 Container、Hashable、Iterable、Iterator、Reversible、Generator、Callable、Sequence、MutableSequence、Set、Mapping、MutableMapping 等。这些 ABC 不仅规范了接口，还通过 mixin 方法提供了大量免费实现。

from collections.abc import Sequence

class MySequence(Sequence):
    def __getitem__(self, index):
        return [1, 2, 3][index]

    def __len__(self):
        return 3

s = MySequence()
print(s[0])        # 1
print(s.count(1))  # 1 —— mixin 方法
print(s.index(2))  # 1 —— mixin 方法
print(list(reversed(s)))  # [3, 2, 1]

04 协议 Protocol

PEP 544 在 Python 3.8 中引入了 typing.Protocol，实现了结构子类型（structural subtyping）或称为静态鸭子类型。与 ABC 的名义子类型（nominal subtyping）不同，Protocol 只关心对象是否具有所需的属性和方法签名。

结构子类型 vs 名义子类型

名义子类型要求显式声明继承关系（如 class Dog(Animal)）。结构子类型则根据形状（shape）来判断——如果你的类有 quack() 方法，你就被视为鸭子，而不需要继承 Duck。

from typing import Protocol

class Quackable(Protocol):
    def quack(self) -> str:
        ...

class Duck:
    def quack(self) -> str:
        return "嘎嘎嘎"

class Person:
    def quack(self) -> str:
        return "假装是鸭子"

def make_it_quack(d: Quackable) -> None:
    print(d.quack())

# 不需要显式继承 Quackable！
make_it_quack(Duck())    # 嘎嘎嘎
make_it_quack(Person())  # 假装是鸭子

静态鸭子类型

Protocol 主要在类型检查时发挥作用。mypy 和 pyright 会检查传入的对象是否满足 Protocol 定义的接口。在运行时，如果没有 @runtime_checkable，Protocol 不会对 isinstance() 产生影响。

from typing import Protocol, runtime_checkable

@runtime_checkable
class SupportsClose(Protocol):
    def close(self) -> None:
        ...

class FileHandler:
    def close(self) -> None:
        print("关闭文件")

class Connection:
    def close(self) -> None:
        print("关闭连接")

def cleanup(obj: SupportsClose) -> None:
    obj.close()
    if isinstance(obj, SupportsClose):
        print("实现了 SupportsClose 协议")

cleanup(FileHandler())
cleanup(Connection())

协议组合

Protocol 支持继承和组合，可以构建复杂的接口层次。这对于定义上下文管理器、迭代器、异步协议等复杂的接口规范特别有用。

from typing import Protocol, Iterator

class Readable(Protocol):
    def read(self, size: int = -1) -> bytes:
        ...

class Writable(Protocol):
    def write(self, data: bytes) -> int:
        ...

class ReadWritable(Readable, Writable, Protocol):
    """组合协议"""
    def seek(self, offset: int, whence: int = 0) -> int:
        ...

class Buffer:
    def read(self, size: int = -1) -> bytes:
        return b"data"

    def write(self, data: bytes) -> int:
        return len(data)

    def seek(self, offset: int, whence: int = 0) -> int:
        return offset

def process(stream: ReadWritable) -> None:
    data = stream.read()
    stream.write(data)

process(Buffer())

runtime_checkable 与性能

使用 @runtime_checkable 装饰的 Protocol 支持 isinstance() 检查。但需要注意，运行时检查会迭代所有 Protocol 中的方法，对于大型 Protocol 可能会带来性能开销。建议只在需要 isinstance 支持时才使用 runtime_checkable。

提示:
Protocol 是 Python 类型系统向 Go 接口和 Rust trait 风格靠拢的重要一步。在现代 Python 代码库中，优先使用 Protocol 而不是 ABC 来定义接口，可以获得更好的解耦和更灵活的复用。

05 类型注解进化

Python 的类型注解自 PEP 484（Python 3.5）引入以来，经历了快速的进化。每个新版本都带来了更简洁、更强大的类型表达方式。

PEP 484 → PEP 585: 原生泛型

PEP 484 最初使用 typing.List[int] 等泛型容器。PEP 585（Python 3.9）允许直接使用内置类型的泛型版本，大幅简化了注解。

# PEP 484 风格 (Python 3.5-3.8, 3.9+仍兼容)
from typing import List, Dict, Tuple, Set

def process(items: List[int]) -> Dict[str, Tuple[int, ...]]:
    ...

# PEP 585 风格 (Python 3.9+)
def process(items: list[int]) -> dict[str, tuple[int, ...]]:
    ...

PEP 604: 联合类型简化

PEP 604（Python 3.10）引入了 X | Y 语法替代 Union[X, Y]，以及 X | None 替代 Optional[X]。

# 之前
from typing import Union, Optional
def parse(val: Union[str, int]) -> Optional[float]:
    ...

# 之后 (Python 3.10+)
def parse(val: str | int) -> float | None:
    ...

PEP 646: Variadic Generics

PEP 646（Python 3.11）引入了 *args 风格的类型变量，用于表达任意数量的类型参数。这对 NumPy 和 Tensor 库特别重要。

from typing import Generic, TypeVarTuple

Ts = TypeVarTuple('Ts')

class Array(Generic[*Ts]):
    def __init__(self, shape: tuple[*Ts]):
        self.shape = shape

# 使用
arr = Array((3, 4))           # Array[*tuple[int, int]]
arr2 = Array((2, 3, 4))       # Array[*tuple[int, int, int]]

PEP 695: Type Parameter Syntax

PEP 695（Python 3.12）引入了更简洁的类型参数声明语法，不再需要显式定义 TypeVar。

# 之前
from typing import TypeVar, Generic
T = TypeVar('T')
class Stack(Generic[T]):
    def push(self, item: T) -> None: ...
    def pop(self) -> T: ...

# 之后 (Python 3.12+)
class Stack[T]:
    def push(self, item: T) -> None: ...
    def pop(self) -> T: ...

# 函数也支持
def first[T](items: list[T]) -> T:
    return items[0]

PEP 692: TypedDict 与 **kwargs

PEP 692（Python 3.12）允许使用 TypedDict 注解 **kwargs，精确表达关键字参数的类型结构。

from typing import TypedDict, Unpack

class Person(TypedDict):
    name: str
    age: int

def greet(**kwargs: Unpack[Person]) -> str:
    return f"{kwargs['name']} is {kwargs['age']}"

greet(name="Alice", age=30)  # OK
# greet(name="Bob")  # type error: missing 'age'

PEP 698: Override 装饰器

PEP 698（Python 3.12）引入了 @override 装饰器，用于显式标记方法重写。类型检查器可以检测到重写但签名不匹配的情况。

from typing import override

class Base:
    def process(self, data: str) -> int:
        return len(data)

class Derived(Base):
    @override
    def process(self, data: str) -> int:
        return super().process(data) + 1

    # @override
    # def proces(self, data: str) -> int:  # type error: 方法名错误
    #     ...

泛型 Self 与 TypeVar

PEP 673（Python 3.11）引入了 Self 类型，用于注解返回自身类型的工厂方法或链式调用。而 TypeVar 和 ParamSpec 则为高阶函数和装饰器提供了精确的类型约束。

from typing import Self, TypeVar, ParamSpec, Callable
import functools

P = ParamSpec('P')
T = TypeVar('T')

class Builder:
    def set_name(self, name: str) -> Self:
        self.name = name
        return self

    def set_age(self, age: int) -> Self:
        self.age = age
        return self

class SpecialBuilder(Builder):
    def extra(self) -> Self:
        return self

# 链式调用保留子类类型
sb = SpecialBuilder().set_name("X").set_age(10).extra()

# ParamSpec 用于装饰器类型注解
def logged(func: Callable[P, T]) -> Callable[P, T]:
    @functools.wraps(func)
    def wrapper(*args: P.args, **kwargs: P.kwargs) -> T:
        print(f"调用 {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

06 match 语句模式匹配

PEP 634-636 在 Python 3.10 中引入了 match 语句，这是 Python 有史以来最重大的语法增强之一。它借鉴了 Rust、Scala 和 Haskell 等语言中的模式匹配思想，为 Python 带来了声明式的条件分支能力。

基本语法与字面量模式

最简单的模式匹配使用字面量值进行精确匹配。每个 case 可以有一个守卫条件（guard）来添加额外约束。

def http_status(code: int) -> str:
    match code:
        case 200:
            return "OK"
        case 201:
            return "Created"
        case 301 | 302:
            return "Redirect"
        case 400:
            return "Bad Request"
        case 404:
            return "Not Found"
        case 500 | 502 | 503:
            return "Server Error"
        case _:
            return "Unknown"

print(http_status(404))  # Not Found

通配符与绑定

通配符（_）匹配任何值但不绑定。普通的变量名会匹配任何值并绑定到该变量。这可用于解包序列元素。

def process_point(point):
    match point:
        case (0, 0):
            return "原点"
        case (0, y):
            return f"在 Y 轴上: y={y}"
        case (x, 0):
            return f"在 X 轴上: x={x}"
        case (x, y):
            return f"坐标: ({x}, {y})"
        case _:
            return "不是二维坐标"

print(process_point((3, 5)))   # 坐标: (3, 5)
print(process_point((0, 7)))   # 在 Y 轴上: y=7

类模式

类模式允许按对象的属性进行匹配，类似于数据类的解构。这是 match 语句中最强大的特性之一。

from dataclasses import dataclass

@dataclass
class Command:
    name: str
    args: list

@dataclass
class Result:
    code: int
    data: str

def handle(obj) -> str:
    match obj:
        case Command(name="quit"):
            return "退出程序"
        case Command(name="greet", args=[name]):
            return f"你好, {name}!"
        case Command():
            return f"未知命令: {obj.name}"
        case Result(code=0, data=msg):
            return f"成功: {msg}"
        case Result(code=err):
            return f"错误代码: {err}"
        case _:
            return "未知类型"

print(handle(Command("greet", ["World"])))  # 你好, World!
print(handle(Result(0, "操作完成")))          # 成功: 操作完成

守卫条件

守卫（guard）是在模式之后添加的 if 表达式，在模式匹配的基础上增加额外约束。只有模式和守卫同时满足时才进入该分支。

def classify_number(n: int) -> str:
    match n:
        case _ if n < 0:
            return "负数"
        case 0:
            return "零"
        case _ if n % 2 == 0:
            return "正偶数"
        case _:
            return "正奇数"

print(classify_number(-5))  # 负数
print(classify_number(42))  # 正偶数

序列与映射模式

match 语句支持匹配列表（序列模式）和字典（映射模式）。映射模式使用 {} 语法并按键匹配。

def parse_config(config: dict) -> str:
    match config:
        case {"debug": True, "port": port}:
            return f"调试模式，端口 {port}"
        case {"port": port, "host": host}:
            return f"生产模式: {host}:{port}"
        case {}:
            return "配置为空"
        case _:
            return "无效配置"

def parse_ast(nodes):
    match nodes:
        case ["+", left, right]:
            return f"({parse_ast(left)} + {parse_ast(right)})"
        case ["*", left, right]:
            return f"({parse_ast(left)} * {parse_ast(right)})"
        case [("num", val)]:
            return str(val)
        case _:
            return str(nodes)

print(parse_ast(["+", ["*", ("num", 3), ("num", 4)], ("num", 5)]))
# ((3 * 4) + 5)

嵌套模式与 OR 模式

模式可以任意嵌套，OR 模式用 | 连接多个备选。还可以使用 as 关键字绑定子模式的值。

def analyze_shape(shape) -> str:
    match shape:
        case ("circle", float(r) as radius):
            return f"圆形，半径 {radius}"
        case ("rect", w, h) | ("rectangle", w, h):
            return f"矩形 {w}x{h}"
        case ("polygon", *points) if len(points) >= 3:
            return f"多边形，{len(points)} 个顶点"
        case _:
            return "未知形状"

print(analyze_shape(("circle", 5.0)))           # 圆形，半径 5.0
print(analyze_shape(("rect", 100, 200)))        # 矩形 100x200
print(analyze_shape(("polygon", (0,0), (1,0), (1,1))))  # 多边形，3 个顶点

match 在 AST 处理中的应用

模式匹配在编译器、解释器和 AST 处理中最能体现其威力。以下是一个简单的表达式求值器，展示了 match 如何简化树形结构的处理。

def evaluate(expr) -> int:
    match expr:
        case ("num", int(n)):
            return n
        case ("add", left, right):
            return evaluate(left) + evaluate(right)
        case ("sub", left, right):
            return evaluate(left) - evaluate(right)
        case ("mul", left, right):
            return evaluate(left) * evaluate(right)
        case ("div", left, right):
            return evaluate(left) // evaluate(right)
        case _:
            raise ValueError(f"未知表达式: {expr}")

expr = ("add",
         ("mul", ("num", 3), ("num", 4)),
         ("num", 5))
print(evaluate(expr))  # 17
提示:
模式匹配不是 switch/case 的简单替代。它是声明式的结构性解构。在 Python 3.10+ 项目中，所有涉及条件分支 + 数据解构的场景都应优先考虑 match 语句。
