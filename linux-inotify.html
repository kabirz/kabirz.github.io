01 概述

inotify 是 Linux 内核提供的文件系统事件监控机制。它允许进程监控文件/目录的变化（创建、删除、修改、属性变更等），无需轮询。VSCode 文件监视、logrotate、IDE 项目刷新等都依赖 inotify。

02 API 与使用

// inotify API
#include <sys/inotify.h>

// 1. 创建 inotify 实例
int fd = inotify_init();      // 阻塞模式
int fd = inotify_init1(IN_NONBLOCK); // 非阻塞

// 2. 添加监控
int wd = inotify_add_watch(fd, "/var/log",
    IN_CREATE | IN_DELETE | IN_MODIFY | IN_MOVED_TO);

// 3. 等待事件
struct inotify_event event;
read(fd, &event, sizeof(event));
// event.wd = 监控描述符
// event.mask = 事件类型
// event.len = 名称长度
// event.name = 变化的文件名（目录监控时）

// 4. 移除监控
inotify_rm_watch(fd, wd);

// 5. 关闭
close(fd);

03 事件类型

事件说明
IN_CREATE在监控目录中创建文件/目录
IN_DELETE在监控目录中删除文件/目录
IN_MODIFY文件内容被修改
IN_ATTRIB文件属性被修改（权限、时间戳等）
IN_MOVED_FROM文件从监控目录移出
IN_MOVED_TO文件移入监控目录
IN_CLOSE_WRITE可写文件被关闭
IN_OPEN文件被打开
IN_ISDIR变化的对象是目录（标志位）

# inotifywait（from inotify-tools 包）
$ inotifywait -m -r /var/log -e create,delete,modify

04 inotify vs fanotify

特性inotifyfanotify
监控粒度单个文件/目录整个挂载点
权限需要读权限CAP_SYS_ADMIN
事件类型文件操作文件访问 + 权限
响应速度快略慢
典型用途文件监视（IDE）杀毒软件、DLP
容量用户可调（/proc/sys/fs/inotify/）系统级
