01 概述

互斥锁（Mutex）是 Linux 内核中可睡眠的排他锁。与自旋锁不同，mutex 在等待时会让出 CPU，适用于需要长时间持有锁或临界区可能睡眠的场景。

核心特性
  • 可睡眠：等待时不消耗 CPU
  • 排他性：同一时刻只有一个持有者
  • 优先级继承：防止优先级反转
  • 调试支持：Lockdep 检测死锁
  • owner 跟踪：记录持有者信息

03 核心 API

// 静态初始化
static DEFINE_MUTEX(my_mutex);

// 运行时初始化
mutex_init(&my_mutex);

// 获取/释放
mutex_lock(&my_mutex);         // 获取（可睡眠）
mutex_trylock(&my_mutex);       // 尝试获取（非阻塞）
mutex_unlock(&my_mutex);       // 释放

// 可中断版本
int ret = mutex_lock_interruptible(&my_mutex);

// 可杀死版本
int ret = mutex_lock_killable(&my_mutex);

04 内部实现

struct mutex {
    atomic_long_t owner;         // 持有者 task_struct 指针
    raw_spinlock_t wait_lock;     // 保护等待队列
    struct list_head wait_list;  // 等待者链表
    bool optimized_mqs_ok;
#ifdef CONFIG_DEBUG_MUTEXES
    struct mutex_waiter *cur_waiter;
    void *magic;
    owner_cpu;
    void *owner;
#endif
};
乐观自旋：mutex 实现使用了乐观自旋优化 — 当持有者正在 CPU 上运行时，等待者会短暂自旋而非立即睡眠，减少上下文切换开销。

05 使用模式

// 标准模式
mutex_lock(&dev->mutex);
// ... 可睡眠操作 ...
mutex_unlock(&dev->mutex);

// 可中断模式（驱动中常用）
int my_read(struct file *f, char __user *buf, size_t len, loff_t *ppos)
{
    int ret;
    ret = mutex_lock_interruptible(&dev->mutex);
    if (ret) return ret;
    // ... 读操作 ...
    mutex_unlock(&dev->mutex);
    return len;
}
