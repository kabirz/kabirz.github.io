01 概述

Linux 网络子系统是内核中最复杂的子系统之一，实现了完整的 TCP/IP 协议栈、多种网络设备驱动、高性能数据路径（XDP/NAPI/GRO/GSO）以及网络虚拟化（netns、bridge、vlan）。

核心职责
  • 协议栈：TCP、UDP、ICMP、IP、ARP 等
  • 网络设备：统一的 net_device 抽象
  • 数据路径：sk_buff 在协议栈间传递
  • 高性能：NAPI、GRO、GSO、XDP
  • 安全：Netfilter、nftables、eBPF
  • 虚拟化：veth、bridge、vlan、tun/tap

03 sk_buff

struct sk_buff 是网络子系统的核心数据结构，代表一个网络数据包在内核中的传递。

struct sk_buff

include/linux/skbuff.h:886

struct sk_buff {
    // 链表/树节点
    union {
        struct { struct sk_buff *next, *prev;
                 struct net_device *dev; };
        struct rb_node rbnode;    // TCP 重传队列
    };

    struct sock     *sk;           // 所属 socket
    ktime_t          tstamp;        // 时间戳
    char             cb[48];        // 控制缓冲（每层自由使用）

    unsigned int     len;           // 总长度（含线性区+分片）
    unsigned int     data_len;      // 非线性部分长度
    __u16            mac_len;       // MAC 头长度
    __u16            hdr_len;       // 可写头长度（克隆时）

    // 协议头指针
    union {
        struct tcphdr *th;     // TCP 头
        struct iphdr  *iph;    // IP 头
        struct udphdr *uh;    // UDP 头
        unsigned char *raw;
    } h;                          // 传输层头

    union {
        struct iphdr  *iph;    // 网络层头
        struct ipv6hdr *ipv6h;
        struct arphdr *arph;
    } nh;                         // 网络层头

    // 数据指针
    unsigned char *head;      // 分配的缓冲区起始
    unsigned char *data;      // 有效数据起始
    unsigned char *tail;      // 有效数据结束
    unsigned char *end;       // 分配的缓冲区结束
};

05 Socket

Socket 是用户空间与网络协议栈的接口。内核通过 struct socket（BSD 套接字）和 struct sock（传输层套接字）两层抽象。

两层 Socket

    socket_state    state;   // SS_FREE/UNCONNECTED/...
    short           type;    // SOCK_STREAM/DGRAM/RAW

struct sock（传输层）
struct sock {
    struct sock_common __sk_common;

    //   skc_dport (远端端口)

NAPI（New API）

NAPI 是 Linux 的混合中断/轮询收包机制：

// 中断驱动模式：每个包触发中断 → 高负载时中断风暴
// 纯轮询模式：CPU 持续轮询 → 低负载时浪费 CPU
// NAPI：中断后切换到轮询，处理完一批包后恢复中断

static int my_poll(struct napi_struct *napi, int budget)
{
    struct my_ring *ring = container_of(napi, ...);
    int work_done = 0;

    while (work_done < budget && ring_has_data(ring)) {
        struct sk_buff *skb = ring_next_skb(ring);
        napi_gro_receive(napi, skb);
        work_done++;
    }

    if (!ring_has_data(ring)) {
        napi_complete_done(napi, work_done);
        // 重新启用中断
    }
    return work_done;
}

GRO（Generic Receive Offload）

GRO：在收包路径中将多个小包合并为大包，减少协议栈处理开销。例如将多个 TCP 段合并为一个大 skb，再交给协议栈处理。通过 napi_gro_receive() 自动触发。

09 XDP / NAPI

XDP（eXpress Data Path）提供在驱动层绕过协议栈的高性能数据路径。

XDP 执行点

模式执行位置说明
Native驱动 NAPI poll最快，需驱动支持
Offloaded网卡硬件可编程网卡（如 SmartNIC）
Generic协议栈入口无需驱动支持，但慢

XDP 动作

动作说明
XDP_ABORTED丢弃并告警
XDP_DROP直接丢弃
XDP_PASS正常传递到协议栈
XDP_TX从同一网卡发回
XDP_REDIRECT重定向到其他网卡/CPU
XDP_MULTIPASS批量处理（AF_XDP）
XDP vs NAPI：NAPI 在驱动层处理包，仍经过 sk_buff。XDP 更底层，直接操作 xdp_buff（不是 sk_buff），性能更高但功能受限。两者可以共存。
