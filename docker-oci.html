01 概述

OCI (Open Container Initiative) 是 2015 年由 Docker、CoreOS、Google、IBM 等共同成立的开放容器标准组织，Linux 基金会项目。

OCI 定义三大规范：

规范定义主要实现
Runtime Spec容器如何被运行（生命周期、操作、状态）runc, crun, kata, runsc
Image Spec镜像格式（manifest、layer、config）docker, buildah, umoci
Distribution Spec镜像仓库 API（pull/push）docker registry, harbor, quay

核心目标：

  • 可移植：任何 OCI 镜像在任何 OCI runtime 上运行
  • 标准化：避免厂商锁定
  • 稳定性：API 向后兼容
  • 最小化：仅规定必要的接口，不限制实现

演进时间线：

2013  Docker 引入容器概念（私有 API）
2014  CoreOS 推出 rkt（竞争）
2015  OCI 成立，统一标准
2016  Runtime Spec v1.0
2017  Image Spec v1.0
2018  runc 成为标准 runtime
2021  Distribution Spec v1.0
2024  Runtime/Image Spec v1.1

02 Runtime Spec

Runtime Spec 定义了容器运行时的行为：什么是容器、如何创建、操作、终止。

// runtime-spec/schema/config-schema.json
// 一个 bundle（容器包）的目录结构：
./mycontainer/
├── config.json   ← runtime spec 配置
└── rootfs/       ← 容器根文件系统

// 创建容器：
$ runc create mycontainer
// runc 读取 config.json，按定义创建容器

03 容器生命周期

Runtime Spec 定义容器的状态和状态转换：

// 容器状态：
//   creating     - 创建中（spec 已解析，namespace 还没切换）
//   created      - 已创建（用户进程未启动）
//   running      - 运行中
//   stopped      - 已停止

// 状态机：
                     ┌──────────┐
                     │ creating │ ← create
                     └────┬─────┘
                          │
                     ┌────▼─────┐
        ┌─ kill ─────│  created │
        │            └────┬─────┘
        │                 │ start
        │            ┌────▼─────┐
        │            │ running  │
        │            └────┬─────┘
        │                 │ exit
        │            ┌────▼─────┐
        └───────────►│ stopped  │
                     └──────────┘

// state.json 文件（runtime 维护）：
{
  "ociVersion": "1.1.0",
  "id": "my-container-id",
  "pid": 12345,           // 容器进程 PID（在宿主 ns 中）
  "status": "running",
  "bundle": "/path/to/bundle",
  "annotations": {...}
}

// runc 把 state 存在 /run/runc/<id>/state.json

04 Operations 操作

Runtime Spec 规定了 runtime CLI 必须支持的操作：

// CLI 命令格式：runtime <command> <id>

// 1. create — 创建容器（不启动）
runc create my-container

// 2. start — 启动已创建的容器（执行 process）
runc start my-container

// 3. state — 查询容器状态
runc state my-container
// → 输出 state.json 内容

// 4. kill — 发送信号
runc kill my-container KILL

// 5. delete — 删除容器（仅 stopped 状态）
runc delete my-container

// 6. exec — 在运行中的容器执行新进程
runc exec my-container /bin/bash
//   读 process.json，启动新进程
//   process.json 类似 config.json 的 process 部分

// 7. run — 便捷命令（create + start 等价）
runc run my-container

// 8. list — 列出所有容器
runc list
// → ID, PID, STATUS, BUNDLE, CREATED, OWNER

// 9. pause / resume — 暂停/恢复（cgroup freezer）
runc pause my-container
runc resume my-container

// 10. ps — 列出容器内进程
runc ps my-container

// 11. spec — 生成默认 spec
runc spec
// → 输出默认 config.json

Hooks（钩子）：runtime 在特定时刻执行用户脚本

// Runtime Spec 定义三种 hooks：

// 1. Prestart（已废弃，被三个钩子替代）
// 2. CreateRuntime    - create 之后、start 之前（在容器 ns 中）
// 3. CreateContainer  - 容器 ns 已设置、process 未启动
// 4. StartContainer   - 容器启动前最后一刻

// config.json:
{
  "hooks": {
    "createRuntime": [
      {
        "path": "/usr/bin/my-hook",
        "args": ["my-hook", "--arg1=value"],
        "env": ["HOOK_ENV=value"],
        "timeout": 5
      }
    ],
    "startContainer": [...]
  }
}

// 容器 init 进程 stdin 收到 spec JSON

06 Image Spec

OCI Image Spec 定义镜像格式。一个镜像由 manifest + config + layers 组成：

// 镜像文件组成：
// 1. Manifest         — 入口清单
// 2. Config           — 镜像配置（环境、cmd、entrypoint、layer 顺序）
// 3. Layer            — 文件系统层（tar+gzip）
// 4. Index            — 多平台镜像清单

// 各文件用 SHA256 作为内容寻址

// ┌─────────────────┐
// │     Index       │  ←─ docker pull 时先拿这个
// └────────┬────────┘
//          │
//   ┌──────┴──────┐
//   │             │
// ┌─▼────────┐ ┌──▼─────────┐
// │ Manifest │ │ Manifest   │  ←─ 不同平台
// │ (amd64)  │ │ (arm64)    │
// └────┬─────┘ └────────────┘
//      │
// ┌────▼──────────────────┐
// │ Config (JSON)          │  ←─ 配置、env、cmd
// │ SHA:abc123...          │
// └────────────────────────┘
//      │
// ┌────▼──────────────────┐
// │ Layer 1 (tar.gz)       │  ←─ 实际文件
// │ SHA:def456...          │
// ├────────────────────────┤
// │ Layer 2                │
// │ SHA:ghi789...          │
// ├────────────────────────┤
// │ Layer N                │
// └────────────────────────┘

// 文件名规则（registry 存储）：
// /var/lib/registry/docker/registry/v2/blobs/sha256/ab/abc123.../data

Image Manifest 示例：

// manifest.json
{
  "schemaVersion": 2,
  "mediaType": "application/vnd.oci.image.manifest.v1+json",
  "config": {
    "mediaType": "application/vnd.oci.image.config.v1+json",
    "digest": "sha256:b6cef1...,",
    "size": 7023
  },
  "layers": [
    {
      "mediaType": "application/vnd.oci.image.layer.v1.tar+gzip",
      "digest": "sha256:9759...,",
      "size": 31234898
    },
    {
      "mediaType": "application/vnd.oci.image.layer.v1.tar+gzip",
      "digest": "sha256:abc12...,",
      "size": 4567
    }
  ],
  "annotations": {
    "org.opencontainers.image.created": "2024-01-01T00:00:00Z"
  }
}

Image Config 示例：

// config JSON 内容：
{
  "created": "2024-01-01T00:00:00Z",
  "author": "Maintainer",
  "architecture": "amd64",
  "os": "linux",
  "config": {
    "User": "app",
    "ExposedPorts": {"8080/tcp": {}},
    "Env": ["PATH=/usr/bin", "LANG=C.UTF-8"],
    "Entrypoint": ["/docker-entrypoint.sh"],
    "Cmd": ["nginx", "-g", "daemon off;"],
    "Volumes": {"/var/cache/nginx": {}},
    "WorkingDir": "/usr/share/nginx",
    "Labels": {"maintainer": "NGINX"}
  },
  "rootfs": {
    "type": "layers",
    "diff_ids": [
      "sha256:...",       // 解压后的 layer SHA（去 gzip）
      "sha256:..."
    ]
  },
  "history": [
    {"created": "...", "created_by": "ADD file:... "},
    {"created": "...", "created_by": "CMD [\"nginx\"]"},
    {"created": "...", "created_by": "RUN apt-get update"}
  ]
}

07 Image Manifest

多平台镜像通过 Image Index 支持：

// Image Index (类比 Docker Manifest List)
{
  "schemaVersion": 2,
  "mediaType": "application/vnd.oci.image.index.v1+json",
  "manifests": [
    {
      "mediaType": "application/vnd.oci.image.manifest.v1+json",
      "digest": "sha256:amd64-manifest-hash",
      "size": 1234,
      "platform": {
        "architecture": "amd64",
        "os": "linux"
      }
    },
    {
      "mediaType": "application/vnd.oci.image.manifest.v1+json",
      "digest": "sha256:arm64-manifest-hash",
      "size": 1234,
      "platform": {
        "architecture": "arm64",
        "os": "linux",
        "variant": "v8"
      }
    },
    {
      "mediaType": "application/vnd.oci.image.manifest.v1+json",
      "digest": "sha256:windows-manifest-hash",
      "platform": {
        "architecture": "amd64",
        "os": "windows"
      }
    }
  ],
  "annotations": {
    "org.opencontainers.image.created": "2024-..."
  }
}

// docker pull nginx 默认按当前架构选择
// docker pull --platform=linux/arm64 nginx 强制选 arm64

Layer 格式：

// 1. 标准格式：tar.gz
//    mediaType: application/vnd.oci.image.layer.v1.tar+gzip

// 2. 无压缩：tar
//    mediaType: application/vnd.oci.image.layer.v1.tar

// 3. zstd 压缩（v1.1+）：
//    mediaType: application/vnd.oci.image.layer.v1.tar+zstd

// 4. 非分层（替代方案）：
//    mediaType: application/vnd.oci.image.layer.nondistributable.v1.tar+gzip
//    用于 vendor-specific binary（如 Windows sys files）

// 每个 layer 是一个 tarball：
//   - 通常包含文件系统的增量
//   - 删除的文件用 .wh.* 标记（参考 OverlayFS）
//   - 必须保持顺序（按 history 顺序）

08 Distribution Spec

Distribution Spec 定义镜像仓库的HTTP API：

// 基础 URL: https://registry/v2/

// 1. 版本检查
GET /v2/
// 200 OK 表示支持 v2 API

// 2. 拉取 manifest
GET /v2/<name>/manifests/<reference>
// name: 镜像名（如 library/nginx）
// reference: tag 或 digest（sha256:abc...）
// Accept: application/vnd.oci.image.manifest.v1+json
//        application/vnd.oci.image.index.v1+json
//        application/vnd.docker.distribution.manifest.v2+json
//        ...

// 3. 拉取 blob（config / layer）
GET /v2/<name>/blobs/<digest>

// 4. 上传 blob（两步）
// 4.1 启动上传
POST /v2/<name>/blobs/uploads/
// 返回 Location: /v2/.../uploads/uuid
//
// 4.2 PUT 数据
PUT <location>&digest=sha256:...
// Content-Type: application/octet-stream
// body: blob 数据

// 5. 推送 manifest
PUT /v2/<name>/manifests/<reference>

// 6. 列出 tag
GET /v2/<name>/tags/list

// 7. 删除 manifest（如果允许）
DELETE /v2/<name>/manifests/<reference>

// 8. catalog
GET /v2/_catalog

认证流程：

// 1. 客户端请求
GET /v2/library/nginx/manifests/latest

// 2. 服务器返回 401 + WWW-Authenticate header
401 Unauthorized
WWW-Authenticate: Bearer realm="https://auth.docker.io/token",
                  service="registry.docker.io",
                  scope="repository:library/nginx:pull"

// 3. 客户端请求 token
GET https://auth.docker.io/token?service=...&scope=...
// 返回:
{"token": "JWT...", "expires_in": 300}

// 4. 重试带 token
GET /v2/.../manifests/...
Authorization: Bearer <token>

内容寻址：

// 所有数据用 SHA256 摘要标识
// manifest digest = sha256(manifest 内容)
// blob digest = sha256(blob 内容)
// 引用通过 digest 完全可重现

// 验证完整性：
// docker trust sign / verify（使用 cosign 等工具）

// 镜像签名：
// cosign sign <image>
// 生成 sigs/ 目录的特殊 manifest
