01 概述

Pipe（管道）是 UNIX/Linux 中最古老最简单的进程间通信机制。管道提供单向字节流，写入端发送数据，读取端接收数据，内核在中间维护一个环形缓冲区。Pipe 是 cmd1 | cmd2 shell 管道操作符的底层实现。

Pipe 核心特性
  • 单向通信：一端读、一端写
  • 字节流：无消息边界
  • 阻塞 I/O：满时写阻塞，空时读阻塞
  • 自动清理：所有 fd 关闭后数据丢弃
  • 页面缓存：零拷贝传输（pipe→pipe splice）
  • 跨 fork：父进程 pipe，子进程继承 fd

02 管道类型

匿名管道（pipe）：

  int fd[2];
  pipe(fd);
  // fd[0] = 读端
  // fd[1] = 写端

  fork() → 子进程继承 fd
  父进程写 fd[1] → 内核缓冲区 → 子进程读 fd[0]
  ┌──────────────────────────────────────────────┐
  │  进程 A (写端)    内核管道缓冲区    进程 B (读端) │
  │  write(fd[1])  → [ring buffer] → read(fd[0]) │
  └──────────────────────────────────────────────┘

命名管道（FIFO）：

  mkfifo /tmp/myfifo 0644
  // 创建文件系统中的管道节点

  进程 A (写端)              进程 B (读端)
  open("/tmp/myfifo", O_WRONLY) → open("/tmp/myfifo", O_RDONLY)
  write(data) → [内核 ring buffer] → read(data)

  优势：无亲缘关系的进程也能通信

对比：

  匿名管道：只能在父子进程间使用（fork 继承 fd）
  命名管道：任意进程可通过路径访问
  两者底层都使用 pipe_inode_info

管道缓冲区大小

属性默认值说明
PIPE_BUF4096 字节原子写入保证（≤此大小的写入不会与并发写混合）
管道容量65536 字节（64KB）Linux 3.16+ 默认环形缓冲区大小
可配置/proc/sys/fs/pipe-max-size可通过 fcntl(F_SETPIPE_SZ) 调整

03 核心数据结构

// include/linux/pipe_fs_i.h:84 — 管道核心结构
struct pipe_inode_info {
    struct mutex           mutex;       // 保护管道的互斥锁
    struct wait_queue_head_t rd_wait, wr_wait; // 读/写等待队列
    unsigned int          max_usage;   // 当前已使用槽位数
    unsigned int          ring_size;   // 环形缓冲区大小（槽位数）
    unsigned int          readers;     // 读端 fd 数量
    unsigned int          writers;     // 写端 fd 数量
    struct pipe_buffer    *bufs;       // ★ 环形缓冲区数组
    struct user_struct    *user;       // 所属用户（内存计数）
};
// pipe_buffer — 单个缓冲区槽位
struct pipe_buffer {
    struct page                *page;     // 数据页面
    struct pipe_buf_operations *ops;      // 操作函数表
    unsigned int               offset;    // 页内偏移
    unsigned int               len;       // 数据长度
    unsigned int               flags;     // 标志
    struct pipe_buf_operations *private;  // 私有操作
};

// pipe_buf_operations — 操作函数表
struct pipe_buf_operations {
    int   (*confirm)(struct pipe_inode_info *, struct pipe_buffer *);
    void (*release)(struct pipe_inode_info *, struct pipe_buffer *);
    bool (*try_steal)(struct pipe_inode_info *, struct pipe_buffer *);
    bool (*get)(struct pipe_inode_info *, struct pipe_buffer *);
};

环形缓冲区

  管道的内部数据结构：

  pipe_inode_info.bufs → [ slot0 | slot1 | slot2 | ... | slot31 ]
                          ↑                                ↑
                       ring_size = 32 个槽位
                       每个槽位 = pipe_buffer（page + offset + len）

  写入：bufs[head % ring_size] = 新数据，head++
  读取：bufs[tail % ring_size] 取数据，tail++
  环绕：head/tail 递增，通过 % ring_size 自动环绕

  ┌───┬───┬───┬───┬───┬───┬───┬───┐
  │ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │...│31 │ ← 32 个 slot
  └───┴───┴───┴───┴───┴───┴───┴───┘
    ↑           ↑
   tail        head
  (读位置)    (写位置)
  max_usage = head - tail = 已写入但未读取的槽位数

04 读写流程

写入流程 write(fd[1], buf, len)：

  1. pipe_write() — fs/pipe.c
        │
        ├── 检查写端是否有效（writers > 0）
        │
        ├── 计算可用空间（ring_size - max_usage）
        │
        ├── 空间不足 → 等待
        │       wait_event_interruptible(wr_wait, 空间可用)
        │
        ├── 有空间 → 分配页面
        │       alloc_page() 或 复用已有页面
        │
        ├── 将数据复制到 pipe_buffer
        │       memcpy(buf->page + offset, data, len)
        │
        ├── 更新 pipe_buffer（offset += len, len = 写入量）
        │       max_usage++ (head++)
        │
        └── 唤醒读端
                wake_up_interruptible(&pipe->rd_wait)

读取流程 read(fd[0], buf, len)：

  1. pipe_read() — fs/pipe.c
        │
        ├── 检查读端是否有效（readers > 0）
        │
        ├── 检查是否有数据（max_usage > 0）
        │
        ├── 无数据 → 等待
        │       wait_event_interruptible(rd_wait, 数据可用)
        │
        ├── 有数据 → 从 pipe_buffer 复制到用户态
        │       memcpy(data, page + offset, copy_len)
        │
        ├── 处理 partial read（可能一次读不完）
        │
        ├── 消费完毕 → 释放 pipe_buffer
        │       pipe_buf_release() → put_page()
        │       max_usage-- (tail++)
        │
        └── 唤醒写端（如果有等待的写者）
                wake_up_interruptible(&pipe->wr_wait)
pipe 的阻塞行为
写阻塞：管道满（max_usage == ring_size）时 write() 阻塞
读阻塞：管道空（max_usage == 0）时 read() 阻塞
O_NONBLOCK：设置非阻塞模式后，满时 write 返回 -1(EAGAIN)，空时 read 返回 0

05 splice 与零拷贝

传统 write + read（两次拷贝）：

  用户态 buffer → memcpy → pipe → memcpy → 用户态 buffer
                        ↑              ↑
                    内核缓冲区     内核缓冲区
                    拷贝 1         拷贝 2

splice（零拷贝）：

  splice(fd_in, off_in, pipefd[1], NULL, len, SPLICE_F_MOVE)
  splice(pipefd[0], NULL, fd_out, off_out, len, SPLICE_F_MOVE)

  仅修改页面引用计数，不复制数据！
  内核页 → pipe buffer → 内核页（只改指针，不拷贝字节）

  适用场景：
  - cat file | grep pattern（文件→管道→文件/终端）
  - nginx/sendfile（文件→socket 零拷贝发送）
操作拷贝次数性能
write() + read()2 次低
splice()0 次（页面引用）极高
sendfile()0 次极高（仅文件→socket）
