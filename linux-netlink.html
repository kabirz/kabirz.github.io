01 概述

Netlink 是 Linux 内核提供的用户态与内核态之间的异步通信机制。它是 ip、bridge、iptables、nft 等所有现代网络配置工具的底层通信通道。Netlink 使用 socket 接口，通过消息传递（而非 ioctl/sysctl）实现高效的内核-用户态交互。

Netlink 核心优势
  • 消息驱动：TLV 格式，可扩展
  • 异步通信：双向消息流
  • 组播通知：内核主动通知用户态事件
  • 多协议：不同子系统有独立协议族
  • 高性能：零拷贝 + 批量消息
  • 替代 ioctl：现代内核首选方式

02 Netlink vs 传统方式

特性ioctlsysctlNetlink
通信方式同步请求-响应文件读写异步消息
数据格式固定结构体字符串/整数TLV（可扩展）
内核通知不支持不支持组播通知
并发性全局锁简单每消息独立
扩展性差（需新增 ioctl 号）有限优（新属性无需改接口）
使用工具ifconfig, routesysctlip, bridge, nft
为什么 ip 取代了 ifconfig？
ifconfig 使用 ioctl，每个操作需要单独的 ioctl 号，无法扩展。当内核添加新功能（如 VLAN、bridge、多队列）时，ioctl 无法轻松支持。
ip 使用 Netlink，新的属性（attribute）可以追加到消息中，内核和用户态向前兼容。

04 核心数据结构

// net/netlink/af_netlink.h:24 — Netlink socket 内核结构
struct netlink_sock {
    struct sock     sk;            // 继承自 sock
    unsigned long   flags;         // 标志
    u32             portid;        // 本端 port ID（类似端口号）
    u32             dst_portid;    // 目标 port ID
    u32             dst_group;     // 目标组播组
    u32             subscriptions; // 订阅的组播组
    struct netlink_callback cb;  // dump 回调
    void (*netlink_rcv)(struct sk_buff *skb); // 接收回调
    int  (*netlink_bind)(struct net *net, int group); // 绑定回调
};

// 每个 Netlink 协议族对应一个 netlink_table
struct netlink_table {
    struct rhashtable  hash;         // portid → sock 哈希表
    struct listeners __rcu *listeners; // 组播监听者位图
    unsigned int     groups;       // 组播组数量
    int             (*bind)(...);  // 绑定回调
};

Netlink 消息格式

Netlink 消息 = 消息头 + 有效载荷

  +-------------------+
  | nlmsghdr (16 字节) |  ← 消息头
  +-------------------+
  | 有效载荷           |  ← TLV 格式的属性
  | (NLA: Type+Len+Val)|
  +-------------------+

  nlmsghdr 结构：
  +--------+--------+--------+--------+--------+
  | len    | type   | flags  | seq    | pid    |
  | (4B)   | (2B)   | (2B)   | (4B)   | (4B)   |
  +--------+--------+--------+--------+--------+

  len:   消息总长度（包含头）
  type:  消息类型（NLMSG_DONE / NLMSG_ERROR / 业务类型）
  flags: NLM_F_REQUEST / NLM_F_ACK / NLM_F_DUMP / NLM_F_CREATE 等
  seq:   序列号（用于匹配请求-响应）
  pid:   发送者的 portid

TLV 属性格式：

  +--------+--------+--------+
  | len    | type   | data   |
  | (2B)   | (2B)   | (变长)  |
  +--------+--------+--------+

  嵌套属性：
  +--------+--------+------------------+
  | len    | type=NLA_NESTED | 子属性...|
  +--------+--------+------------------+

05 通信流程

用户态 → 内核（ip link show）：

  用户执行 ip link show
        │
        ↓
  libnl / iproute2
        │
        ├── socket(AF_NETLINK, SOCK_RAW, NETLINK_ROUTE)
        │
        ├── 构造 Netlink 消息
        │       nlmsghdr: type=RTM_GETLINK, flags=NLM_F_REQUEST|NLM_F_DUMP
        │       payload: { ifinfomsg }
        │
        ├── sendto(nlsock, msg, ...) → 内核
        │
        ↓
  内核 Netlink 处理
        │
        ├── af_netlink.c: netlink_rcv()
        │       → nlmsg_rcv() 解析消息
        │       → 根据 type 分发到子系统
        │
        ├── NETLINK_ROUTE → rtnetlink_rcv()
        │       → 根据 RTM_GETLINK 分发
        │       → 遍历所有网络设备
        │       → 构造响应消息（RTM_NEWLINK × N）
        │
        └── netlink_unicast() → 回复用户态
                │
                ↓
  用户态 recvmsg() → 解析响应 → 显示设备列表

内核 → 用户态（网络设备变化通知）：

  网络设备状态变化（UP/DOWN/IP 变更）
        │
        ↓
  内核调用 rtmsg_ifinfo() → 构造 RTM_NEWLINK 消息
        │
        ↓
  netlink_broadcast() → 发送给所有订阅了 RTNLGRP_LINK 的 socket
        │
        ↓
  用户态 ip monitor / NetworkManager 接收通知

06 Generic Netlink

Generic Netlink（GENL）是 Netlink 的扩展框架，解决了传统 Netlink 协议号有限的问题。它使用动态分配的协议号，允许任意子系统注册自己的 Generic Netlink 族。

传统 Netlink：
  每个子系统占用一个协议号
  协议号有限（~32 个）
  子系统：NETLINK_ROUTE(0), NETLINK_NETFILTER(12), ...

Generic Netlink：
  所有子系统共享一个协议号 (NETLINK_GENERIC=16)
  通过 "族名" 区分不同子系统
  内核动态分配族 ID

  用户态                              内核态
  ┌──────────────────┐            ┌──────────────────┐
  │ socket(NETLINK_  │            │ Generic Netlink   │
  │        GENERIC)  │            │ 多路分发器        │
  │                  │            │                  │
  │ 族名: nl80211    │─── 消息 →  │ → nl80211 处理器  │
  │ 族名: team       │            │ → team 处理器    │
  │ 族名: nlctrl     │            │ → nlctrl 管理器  │
  └──────────────────┘            └──────────────────┘

常见的 Generic Netlink 族
