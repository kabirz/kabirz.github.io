01 概述

Linux 内核提供密码学安全的伪随机数生成器（CSPRNG），是 SSL/TLS、SSH、密钥生成等安全机制的基础。内核从多种硬件熵源（CPU RDRAND、中断时间、磁盘 I/O 延迟）收集熵，通过 CRNG（ChaCha20）算法生成高质量随机数。

02 随机数接口

接口行为阻塞适用场景
/dev/random仅在熵足够时返回数据是（旧版）密钥生成
/dev/urandom始终返回（CSPRNG）否通用随机数
getrandom()根据 flags 选择阻塞模式可选推荐使用
get_random_bytes()内核态 API否（启动后）内核模块
get_random_u32()快速无锁随机数否快速随机数
/dev/random vs /dev/urandom
Linux 5.6+ 中，/dev/random 和 /dev/urandom 使用相同的 CSPRNG（ChaCha20），区别仅在于初始化阶段（系统启动初期熵不足时 /dev/random 会阻塞）。启动完成后两者完全等价。

03 熵源

内核随机数熵源：

  ┌──────────────────────────────────────────┐
  │              熵源池                       │
  ├──────────────────────────────────────────┤
  │  CPU 熵源                                 │
  │  ├── RDRAND/RDSEED（Intel/AMD）          │
  │  ├── ARM V8.5 RNDR（ARM64）             │
  │  └── 硬件随机数生成器（TRNG）             │
  │                                          │
  │  设备熵源                                 │
  │  ├── 中断时间间隔（IRQ timing）           │
  │  ├── 磁盘 I/O 延时                       │
  │  ├── 网络包到达时间                       │
  │  ├── 键盘/鼠标事件时间                    │
  │  └── 音频输入噪声                        │
  │                                          │
  │  混合器（Fast Key Erasable CSPRNG）       │
  │  ├── 输入：各熵源                         │
  │  ├── 算法：ChaCha20                      │
  │  └── 输出：密码学安全随机数              │
  └──────────────────────────────────────────┘

# 查看系统熵状态
$ cat /proc/sys/kernel/random/entropy_avail
256  # 可用熵位数

# 熵池大小
$ cat /proc/sys/kernel/random/poolsize
256  # 熵池大小（位）

# 推荐：使用 getrandom() 而非 /dev/urandom
# getrandom 在系统启动初期会等待熵池初始化完成

04 生成流程

getrandom() 系统调用流程：

  getrandom(buf, len, flags)
        │
        ├── flags 包含 GRND_RANDOM？
        │   是 → 使用 /dev/random 语义（熵不足时等待）
        │   否 → 使用 /dev/urandom 语义（始终返回）
        │
        ↓
  crng_reseed() — 检查 CSPRNG 状态
        │
        ├── 未初始化 → 等待熵池就绪（启动早期）
        │
        └── 已初始化 → 立即返回
                │
                ↓
        ChaCha20 加密生成随机字节
        → 复制到用户态 buffer
