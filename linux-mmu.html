01 概述

MMU（Memory Management Unit）是 CPU 中负责虚拟地址到物理地址转换的硬件单元。Linux 内核通过软件页表管理这一映射关系，实现了进程地址空间隔离、按需分页（demand paging）、写时复制（COW）、内存映射文件等核心功能。

MMU 的核心职责
  • 地址翻译：虚拟地址 → 物理地址
  • 权限检查：读/写/执行权限控制
  • 缺页触发：无效/写保护时触发异常
  • TLB 管理：加速地址翻译缓存
Linux MMU 子系统架构
  • 通用层：mm/memory.c、include/linux/pgtable.h
  • x86 架构层：arch/x86/mm/
  • 页表操作：arch/x86/include/asm/pgtable.h
  • TLB 管理：arch/x86/mm/tlb.c
设计哲学：Linux 采用按需分页（lazy allocation）— 进程的虚拟页面在首次访问时才分配物理页面。这极大提高了内存利用率，因为许多分配的页面从未被实际使用。

02 硬件基础

x86-64 地址空间

x86-64 架构支持 48 位（经典）或 57 位（LA57）虚拟地址空间，物理地址最大 52 位。

模式虚拟地址位数虚拟地址空间物理地址位数页表级数
4 级页表48 位256 TB52 位4 (PGD→PUD→PMD→PTE)
5 级页表 (LA57)57 位128 PB52 位5 (PGD→P4D→PUD→PMD→PTE)

关键控制寄存器

寄存器用途
CR3页表基址寄存器（PDBR），包含当前进程的顶层页表物理地址 + PCID
CR4.PAE启用物理地址扩展（PAE），支持 64 位物理地址
CR4.PCE允许用户空间访问性能计数器
CR4.PCIDE启用 Process Context ID，避免进程切换时 TLB flush
CR4.LA57启用 5 级页表
EFFLAGS.ACAlignment Check，与 SMAP 配合控制内核访问用户页

PCID（Process Context Identifier）

PCID 是 x86 硬件提供的 ASID（Address Space ID）机制，允许 TLB 同时缓存多个进程的地址映射。Linux 使用 per-CPU 的 ASID 数组来管理：

  • 动态 ASID（[0, TLB_NR_DYN_ASIDS-1]）：per-CPU 小数组，缓存最近使用的 mm
  • 全局 ASID（[TLB_NR_DYN_ASIDS, MAX_ASID_AVAILABLE-1]）：跨 CPU 唯一标识
  • kPCID：写入 CR3 的值 = ASID + 1（PCID 0 保留）
  • uPCID：KPTI 下用户态 PCID = kPCID + 2048

每级 512 个条目（9 位索引） · 4KB 页面 · 48 位虚拟地址 = 4×9 + 12 = 48

页表条目格式（x86-64 PTE，64 位）

Bit 63: NX
Bit 62-59: PKEY[3:0]
Bit 58-52: 软件位
Bit 51-12: 物理地址
Bit 11-0: 标志位

Bit 0: Present Bit 1: R/W Bit 2: U/S Bit 3: PWT Bit 4: PCD Bit 5: A Bit 6: D Bit 7: PAT/PSE Bit 8: G

关键标志位详解

位宏含义说明
0_PAGE_PRESENT页面存在CPU 硬件检查，为 0 触发缺页
1_PAGE_RW可写0=只读，1=可写。写保护时触发 COW
2_PAGE_USER用户可访问0=仅内核，1=用户+内核
5_PAGE_ACCESSED已访问 (A bit)CPU 自动置位，用于页面置换算法
6_PAGE_DIRTY已写入 (D bit)CPU 自动置位，标记页面需要回写
7_PAGE_PSE大页4KB 页面=PAT，2MB/1GB=PSE
8_PAGE_GLOBAL全局TLB flush 时保留此条目
63_PAGE_NX不可执行防止代码注入攻击
软件可用位：PTE 中有 6 个软件位（bit 9-11, 52-58）不被硬件使用，内核利用它们实现：
special uffd-wp soft-dirty saved-dirty kernel-4k

// mm/memory.c:6384-6485
static vm_fault_t __handle_mm_fault(struct vm_area_struct *vma,
        unsigned long address, unsigned int flags)
{
    struct vm_fault vmf = { .vma = vma, .address = address & PAGE_MASK, ... };

    pgd = pgd_offset(mm, address);      // 1. 获取 PGD
    p4d = p4d_alloc(mm, pgd, address);  // 2. 分配/获取 P4D
    vmf.pud = pud_alloc(mm, p4d, address);  // 3. 分配/获取 PUD

    // 尝试 PUD 级大页 (1GB)
    if (pud_none(*vmf.pud) && thp_vma_allowable_order(..., PUD_ORDER))
        ret = create_huge_pud(&vmf);

    vmf.pmd = pmd_alloc(mm, vmf.pud, address);  // 4. 分配/获取 PMD

    // 尝试 PMD 级大页 (2MB)
    if (pmd_none(*vmf.pmd) && thp_vma_allowable_order(..., PMD_ORDER))
        ret = create_huge_pmd(&vmf);

    // 降级到 PTE 处理
    return handle_pte_fault(&vmf);
}

PTE 缺页分发

// mm/memory.c:6302-6376
static vm_fault_t handle_pte_fault(struct vm_fault *vmf)
{
    if (unlikely(pmd_none(*vmf->pmd))) {
        vmf->pte = NULL;  // PMD 空，PTE 页表未分配
    } else {
        vmf->pte = pte_offset_map_rw_nolock(...);  // 定位 PTE
        vmf->orig_pte = ptep_get_lockless(vmf->pte);
        if (pte_none(vmf->orig_pte))
            vmf->pte = NULL;  // PTE 为空
    }

    if (!vmf->pte)
        return do_pte_missing(vmf);      // 页面缺失
    if (!pte_present(vmf->orig_pte))
        return do_swap_page(vmf);         // 在 swap 中
    if (pte_protnone(vmf->orig_pte) && vma_is_accessible(vmf->vma))
        return do_numa_page(vmf);         // NUMA 迁移

    // 页面存在但权限不足
    if (vmf->flags & (FAULT_FLAG_WRITE|FAULT_FLAG_UNSHARE)) {
        if (!pte_write(entry))
            return do_wp_page(vmf);       // 写时复制
    }

    // 设置 accessed 位
    entry = pte_mkyoung(entry);
    ptep_set_access_flags(...);
}

写时复制（do_wp_page）

mm/memory.c:4178-4271

do_wp_page(vmf) {
    // 1. 共享映射 (VM_SHARED)
    if (vma->vm_flags & (VM_SHARED | VM_MAYSHARE)) {
        if (!vmf->page || is_fsdax_page(vmf->page))
            return wp_pfn_shared(vmf);   // PFN 映射
        return wp_page_shared(vmf, folio);  // 文件页
    }

    // 2. 匿名页 + 可复用（唯一引用）
    if (folio && folio_test_anon(folio) &&
        (PageAnonExclusive(vmf->page) || wp_can_reuse_anon_folio(folio, vma))) {
        wp_page_reuse(vmf, folio);  // 直接标记可写
        return 0;
    }

    // 3. 需要复制
    return wp_page_copy(vmf);  // 分配新页 + memcpy
}

06 TLB 管理

switch_mm — 进程切换时的地址空间切换

arch/x86/mm/tlb.c:783

void switch_mm_irqs_off(struct mm_struct *prev, struct mm_struct *next,
                         struct task_struct *tsk)
{
    if (prev == next) {
        // 同一进程的线程切换，可能无需 TLB flush
        if (!was_lazy) return;
    }

    // 选择/分配 ASID
    ns = choose_new_asid(next, next_tlb_gen);

    // 构建 CR3 值 = PGD 物理地址 | PCID | LAM
    new_cr3 = build_cr3(next->pgd, ns.asid, lam);

    // 写入 CR3 — 这是一个串行化指令，提供 full memory barrier
    write_cr3(new_cr3);

    // 更新 per-CPU TLB 状态
    this_cpu_write(cpu_tlbstate.loaded_mm, next);
    this_cpu_write(cpu_tlbstate.loaded_mm_asid, ns.asid);
}

CR3 构建

// arch/x86/mm/tlb.c:161-172
static inline unsigned long build_cr3(pgd_t *pgd, u16 asid, unsigned long lam)
{
    unsigned long cr3 = __sme_pa(pgd) | lam;  // SMEP 加密 + LAM
    if (static_cpu_has(X86_FEATURE_PCID))
        cr3 |= kern_pcid(asid);  // ASID+1
    return cr3;
}

// 快速路径：不刷新 TLB
static inline unsigned long build_cr3_noflush(...)
{
    return build_cr3(pgd, asid, lam) | CR3_NOFLUSH;  // 设置 NOFLUSH 位
}

延迟 TLB 刷新

tlb_gen 计数器：每个 mm 有一个 atomic64_t tlb_gen 计数器。代码先修改页表，再递增 tlb_gen，最后触发 TLB flush。低级 flush 代码通过比较本地缓存的 gen 与 mm 的 gen 来判断是否需要刷新，避免不必要的 IPI。

mmu_gather 批量处理

mm/mmu_gather.c

用于 munmap() 等批量释放场景：

  1. tlb_gather_mmu() — 初始化 gather 上下文
  2. tlb_remove_page() — 收集待释放页面（避免逐个 flush）
  3. tlb_flush_mmu() — 批量 flush + 释放
  4. tlb_finish_mmu() — 最终清理

关键优化：使用 per-CPU 本地 batch（栈上分配），避免频繁内存分配。SMP 下延迟 rmap 移除直到 TLB flush 完成。

07 内存保护

硬件 NX (No-Execute)

PTE bit 63，标记数据页不可执行。防止缓冲区溢出攻击的代码注入。通过 _PAGE_NX 控制。

SMAP：防止内核意外访问用户页面（CR4.SMAP）。
SMEP：防止内核执行用户页面的代码。

4 位硬件保护键（bit 59-62），每个进程独立的权限位图。无需修改页表即可实现细粒度内存保护。
