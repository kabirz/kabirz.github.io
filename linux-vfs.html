01 概述

VFS（Virtual File System）是 Linux 的虚拟文件系统层，为所有文件系统（ext4、XFS、procfs、sysfs 等）提供统一的接口。用户空间的 open()/read()/write()/close() 通过 VFS 转发到具体的文件系统实现。

核心设计
  • 统一抽象：所有文件系统使用相同的接口
  • 面向对象：通过函数指针实现多态
  • 缓存机制：DCache/ICache 加速查找
  • 命名空间：每个进程独立的挂载树
  • 异步 I/O：io_uring/AIO 支持

04 DCache

DCache（目录项缓存）是 VFS 的性能核心，缓存路径名到 inode 的映射，避免重复的磁盘查找。

缓存类型

缓存说明
哈希表d_hash — 快速查找（O(1)）
LRU 链表d_lru — 按使用时间排序，淘汰冷条目
目录链表d_children — 父→子关系
别名链表d_alias — 硬链接共享 inode

查找流程

path_lookup("/home/user/file")
↓
hash("home") → dcache 查找
↓ 命中
hash("user") → dcache 查找
↓ 命中
hash("file") → dcache 查找
↓ 未命中 → 调用 inode->i_op->lookup()
磁盘读取目录块 → 填充 dcache
负 dentry：当查找的文件不存在时，内核创建一个 d_inode = NULL 的负 dentry 缓存。下次查找同一路径时直接返回"不存在"，无需访问磁盘。

lookup 优化

优化说明
RCU walk无锁快速路径，通过 d_seq 保证一致性
ref walk有锁慢速路径，需要持有 dentry 引用
RCU-walk 失败自动降级到 ref-walk
DCache 命中直接返回，不调用底层 lookup

08 sysfs / procfs

sysfs 和 procfs 是 VFS 的伪文件系统实现，不存储在磁盘上，而是由内核动态生成。

注册方式

sysfs (/sys)
// 设备属性
DEVICE_ATTR_RO(my_attr);
device_create_file(&dev, &dev_attr_my_attr);

// 总线属性
BUS_ATTR_RO(my_bus);

// 驱动属性
DRIVER_ATTR_RO(my_drv);
procfs (/proc)
// seq_file 模式
proc_create_single("info", 0644,
    NULL, my_show);

// 自定义操作
proc_create("my_file", 0644,
    NULL, &my_ops);

伪文件系统 vs 真实文件系统

特性真实文件系统伪文件系统
存储磁盘/NVMe内存
持久化重启后保留重启后消失
数据生成读取磁盘块内核动态生成
写入修改磁盘数据修改内核状态
inode磁盘存储内存分配
