01 概述

Linux 内核提供了完整的追踪工具链，从静态 tracepoint 到动态 kprobe，从函数跟踪 ftrace 到可编程 BPF。本篇全景梳理所有追踪工具的关系和使用场景。

02 工具分类

Linux 追踪工具分类：

  ┌─────────────────────────────────────────────────────────┐
  │                    静态追踪                              │
  │  Tracepoint  ──  编译时定义，零开销（未启用时）           │
  │  perf trace   ──  基于 tracepoint 的用户态工具           │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │                    动态追踪                              │
  │  Kprobe      ──  内核函数入口/返回断点                    │
  │  Uprobe      ──  用户态函数入口/返回断点                  │
  │  ftrace      ──  函数级跟踪（mcount/fentry）             │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │                    可编程追踪                             │
  │  BPF         ──  可编程内核沙箱，动态加载                  │
  │  BPF + tracepoint ──  在 tracepoint 上运行 BPF 程序      │
  │  BPF + kprobe  ──  在 kprobe 上运行 BPF 程序             │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │                    用户态工具                             │
  │  perf        ──  采样、计数、tracepoint                    │
  │  trace-cmd   ──  ftrace 前端                              │
  │  bpftrace    ──  高级 BPF 追踪语言                        │
  │  bcc         ──  BPF 编译器集合                           │
  └─────────────────────────────────────────────────────────┘

03 追踪流程

追踪数据流：

  内核/用户态代码
        │
        ├── Tracepoint 触发
        │   → trace_event_call()
        │   → 记录到 ring buffer
        │
        ├── Kprobe 命中断点
        │   → kprobe_handler()
        │   → 记录到 ring buffer
        │
        └── BPF 程序执行
            → bpf_prog_run()
            → 更新 BPF Map / 输出到 ring buffer
                    │
                    ↓
  Ring Buffer（tracefs/共享内存）
                    │
                    ↓
  用户态工具读取
  ├── perf record → perf.data
  ├── trace-cmd → trace_pipe
  ├── bpftrace → stdout
  └── bcc → stdout

需求推荐工具原因
