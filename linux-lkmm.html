01 概述

LKMM（Linux Kernel Memory Model）定义了多处理器系统中内存操作的排序规则。编译器和 CPU 可能重排内存访问顺序以提高性能，LKMM 通过内存屏障（memory barriers）确保并发代码的正确性。它是 RCU、spinlock、原子操作等同步原语的基础。

02 内存屏障

内存屏障类型：

  无屏障（可能发生重排）：
  CPU 0:              CPU 1:
  WRITE A = 1         if (B == 1)
  WRITE B = 1              A = 1  ← 可能先看到 A=1，再看到 B=0

  有写屏障（smp_wmb）：
  CPU 0:              CPU 1:
  WRITE A = 1         if (B == 1) {
  smp_wmb()                smp_rmb()
  WRITE B = 1              A == 1  ← 保证看到 A=1
                       }

屏障类型：

  smp_wmb()  — 写屏障：之前的写操作在之后的写操作之前对其他 CPU 可见
  smp_rmb()  — 读屏障：之前的读操作在之后的读操作之前完成
  smp_mb()   — 全屏障：之前的所有读写在之后的所有读写之前完成
  smp_store_release() — 释放语义：之前的读写在本次写之前完成
  smp_load_acquire()  — 获取语义：本次读在之后的读写之前完成

03 排序规则

场景需要的屏障说明
生产者-消费者smp_wmb + smp_rmb确保数据在标记之前可见
队列操作smp_wmb确保入队顺序正确
引用计数原子操作自带atomic_inc/dec 提供排序
RCU 读端rcu_dereference()编译器屏障 + 控制依赖
RCU 写端rcu_assign_pointer()smp_store_release 语义

04 API 与使用

// 内存屏障 API
smp_wmb();          // 写屏障
smp_rmb();          // 读屏障
smp_mb();           // 全屏障
smp_store_release(&ptr, val);  // 释放语义写
smp_load_acquire(&ptr);        // 获取语义读

// 编译器屏障（阻止编译器重排，但不阻止 CPU 重排）
barrier();

// 不可预测的值（防止编译器优化）
READ_ONCE(x);    // 防止编译器缓存
WRITE_ONCE(x, v); // 防止编译器合并/拆分

// 生产者-消费者示例
// 生产者
data = 42;
smp_wmb();        // 确保 data 在 ready 之前可见
ready = 1;

// 消费者
if (READ_ONCE(ready)) {
    smp_rmb();    // 确保读 ready 之后再读 data
    printk("%d\n", data);  // 保证看到 data=42
}
