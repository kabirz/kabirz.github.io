01 架构全景

Zephyr 是一个面向资源受限设备的实时操作系统（RTOS），由 Linux 基金会托管，支持从 8-bit MCU 到 64-bit SoC 的广泛硬件平台。其架构采用分层设计，从底层硬件抽象到上层应用框架，每一层都有明确的职责边界。

02 内核层

Zephyr 内核是整个系统的核心，提供线程管理、调度、同步、内存管理和时间管理等基础服务。内核代码位于 kernel/ 目录，约 50 个源文件。

内核启动流程

// 内核初始化入口（arch/xxx/head.S → start_kernel）
void z_cstart(void)
{
    // 1. 关中断，初始化架构
    z_irq_lock();
    z_arch_kernel_init();

    // 2. 初始化内核数据结构
    z_sched_init();        // 调度器初始化
    k_mem_heap_init();     // 堆初始化
    z_timeout_init();      // 超时队列初始化

    // 3. 执行 SYS_INIT 钩子（PRE_KERNEL_1/2）
    z_sys_init_run_level(SYS_INIT_EARLIEST);
    z_sys_init_run_level(SYS_INIT_PRE_KERNEL_1);
    z_sys_init_run_level(SYS_INIT_PRE_KERNEL_2);

    // 4. 创建 main 线程
    z_prepare_multithreading();

    // 5. 执行 POST_KERNEL 钩子
    z_sys_init_run_level(SYS_INIT_POST_KERNEL);

    // 6. 启动调度器，永不返回
    z_swap_unlocked();
}

调度优先级体系

Zephyr 使用数值越小优先级越高的模型，分为协作式和抢占式两大类：

// 优先级定义（include/zephyr/kernel.h）
// 协作线程：负数优先级（-N ~ -1）
#define K_PRIO_COOP(x)    (-(CONFIG_NUM_COOP_PRIORITIES - (x)))

// 抢占线程：非负优先级（0 ~ N-1）
#define K_PRIO_PREEMPT(x) (x)

// 最高/最低优先级
#define K_HIGHEST_THREAD_PRIO  (-CONFIG_NUM_COOP_PRIORITIES)
#define K_LOWEST_THREAD_PRIO   CONFIG_NUM_PREEMPT_PRIORITIES
#define K_IDLE_PRIO            K_LOWEST_THREAD_PRIO
优先级范围类型特点
-CONFIG_NUM_COOP_PRIORITIES ~ -1协作式不可被抢占，主动让出 CPU
0 ~ CONFIG_NUM_PREEMPT_PRIORITIES-1抢占式可被更高优先级线程抢占
K_LOWEST_THREAD_PRIO空闲线程最低优先级，执行 idle 回调

04 HAL 与 BSP

硬件抽象层（HAL）和板级支持包（BSP）是 Zephyr 多平台支持的基石。HAL 定义了架构相关的接口，BSP 则封装了具体硬件的配置。

架构抽象层（arch/）

Zephyr 支持 14 种处理器架构，每种架构在 arch/ 下有独立的实现：

架构目录支持的 SoC
ARM Cortex-Marm/STM32、NXP、Nordic、TI
ARM Cortex-A/Rarm64/i.MX 8、RPi 4
RISC-Vriscv/ESP32、GD32、SiFive
x86x86/IA-32、x86_64
ARCarc/Synopsys ARC EM/HS
Xtensaxtensa/ESP32 系列

架构接口定义

每个架构必须实现以下核心接口（定义在 include/zephyr/arch/）：

// 架构必须实现的接口
void z_arch_kernel_init(void);           // 架构初始化
void z_arch_switch(void *switch_to, void **switch_from);
                                         // 上下文切换
void z_arch_isr_install(int irq, void (*routine)(void *), void *arg);
                                         // 中断安装
void z_irq_lock(unsigned int *flags);    // 中断屏蔽
void z_irq_unlock(unsigned int flags);   // 中断恢复

编译时优化

Zephyr 大量使用编译时决策来避免运行时开销：

// 1. 编译时设备树访问（零开销）
#define DT_ALIAS(x) DT_NODELABEL(DT_ALIAS(x))
// 编译时解析，生成常量地址
// DT_REG_ADDR(DT_NODELABEL(spi1)) → 0x40013000

// 2. 条件编译（CONFIG_XXX）
#ifdef CONFIG_USERSPACE
    // 仅在启用用户态时编译
#endif

// 3. 迭代器段（可变数组，编译时构建）
STRUCT_SECTION_FOREACH(device, dev) {
    // 遍历所有已注册设备
}

08 与 Linux 对比

Zephyr 和 Linux 虽然都源自嵌入式领域，但设计目标和架构有本质区别：

维度ZephyrLinux
定位资源受限 MCU/RTOS通用 OS / 服务器
内存需求最低 8 KB RAM最低 4 MB RAM
进程模型无进程，仅线程进程 + 线程
调度优先级抢占 + 协作CFS/EEVDF 抢占式
虚拟内存可选（MPU/MMU）必需（虚拟地址空间）
设备模型编译时静态注册运行时动态发现
构建系统CMake + KconfigKbuild + Kconfig
驱动框架轻量级 API完整子系统
用户态可选（系统调用）必需
实时性硬实时软实时（PREEMPT_RT）

关键差异

设备注册方式：Linux 在运行时通过总线枚举动态注册设备，Zephyr 在编译时通过设备树静态定义设备。这使得 Zephyr 的设备初始化开销为零，但不支持热插拔。

// Zephyr：编译时设备定义
DEVICE_DT_INST_DEFINE(n, init_fn, pm, &data, &config, POST_KERNEL,
                      CONFIG_GPIO_INIT_PRIORITY, &api);

// Linux：运行时动态注册（platform_device_register）
platform_device_register(&pdev);
