01 概述

Linux 设备模型（Device Model）是内核中统一管理硬件设备和驱动程序的框架。它通过总线（bus）、设备（device）、驱动（driver）三要素的分离与匹配，实现了设备发现、驱动绑定、电源管理、热插拔等功能。

核心设计目标
  • 统一抽象：所有设备使用统一的 device 结构
  • 自动匹配：设备与驱动通过总线自动绑定
  • 分层管理：总线/设备/驱动/类四层分离
  • sysfs 导出：设备层次结构在 /sys 中可见
  • 电源管理：统一的 PM 框架
  • 热插拔：通过 uevent 通知用户空间
三要素关系
总线
bus_type
→
设备
device
↔
驱动
device_driver

设备注册到总线 → 驱动注册到总线 → 总线执行 match → 绑定 → probe()

02 kobject

kobject 是设备模型的基础对象，提供引用计数、sysfs 表示和层次结构。

struct kobject

include/linux/kobject.h:64-82

struct kobject {
    const char        *name;           // 对象名称
    struct list_head   entry;           // 链入 kset 链表
    struct kobject     *parent;         // 父对象
    struct kset        *kset;           // 所属 kset
    const struct kobj_type *ktype;     // 类型（sysfs 操作）
    struct kernfs_node *sd;             // sysfs 目录条目
    struct kref        kref;            // 引用计数
    unsigned int state_initialized:1;
    unsigned int state_in_sysfs:1;
    unsigned int state_add_uevent_sent:1;
    unsigned int state_remove_uevent_sent:1;
    unsigned int uevent_suppress:1;
};

kobject 层次

kset (bus)
  ├── kobject /sys/bus/pci
  ├── kobject devices/
  │     ├── kobject 0000:00:1f.2
  │     │     └── kobject → 在 /sys/bus/pci/devices/0000:00:1f.2
  │     └── kobject 0000:00:02.0
  └── kobject drivers/
        ├── kobject ehci-pci/
        └── kobject xhci-hcd/
kobject 的作用：1) 引用计数（kref）确保对象在使用时不被释放；2) 自动在 sysfs 中创建目录；3) 通过 kset 形成层次结构；4) 通过 uevent 机制通知用户空间。

类名sysfs 路径说明

类 vs 总线：总线关注设备-驱动的匹配和绑定；类关注用户空间的分组和接口。一个设备可以同时属于某个总线和某个类。

07 Probe 流程

probe 是设备模型中最关键的环节 — 当驱动匹配到设备时调用 probe() 初始化硬件。

完整流程

设备注册 / 驱动注册
↓

↓ 匹配成功

↓ 或通过总线

↓

驱动特定初始化

Deferred Probe（延迟探测）

drivers/base/dd.c:37-59

// 当依赖的资源尚未就绪时
static int my_probe(struct device *dev)
{
    struct regulator *vcc;

    vcc = devm_regulator_get(dev, "vcc");
    if (IS_ERR(vcc)) {
        if (PTR_ERR(vcc) == -EPROBE_DEFER) {
            // 稍后重试
            return -EPROBE_DEFER;
        }
        return PTR_ERR(vcc);
    }
    // 初始化硬件...
    return 0;
}

// 内核维护 deferred_probe_pending_list
// 每次有新设备/驱动注册时重试

异步 Probe

ASYNC_PROBE：设置 driver.probe_type = PROBE_PREFER_ASYNCHRONOUS 后，probe 在异步工作队列中执行，加速启动。适用于不影响启动顺序的慢速设备。

08 sysfs

sysfs（/sys）是设备模型的用户空间接口，将内核对象层次导出为文件系统。

sysfs 目录映射

sysfs 路径内核对象
/sys/bus/所有注册的总线
/sys/class/所有注册的设备类
/sys/devices/所有设备（全局设备树）
/sys/dev/按设备号索引的符号链接
/sys/firmware/固件接口
/sys/module/已加载模块
/sys/power/系统电源状态

属性导出

// 设备属性
static DEVICE_ATTR_RO(my_status);   // /sys/devices/.../my_status
static DEVICE_ATTR_RW(my_config);  // 可读写
static DEVICE_ATTR_WO(my_reset);   // 只写

// 总线属性
static BUS_ATTR_RO(my_bus_attr);

// 驱动属性
static DRIVER_ATTR_RO(my_drv_attr);

// 创建/移除
device_create_file(&dev, &dev_attr_my_status);
device_remove_file(&dev, &dev_attr_my_status);

devtmpfs

devtmpfs：内核在 /dev/ 中自动创建设备节点。设备注册时，内核调用 devtmpfs_create_node() 根据设备号和类信息创建 /dev/xxx 文件，无需 udev 即可工作。

09 电源管理

设备模型集成了统一的电源管理（PM）框架，支持系统级和运行时 PM。

PM 回调层次

层次结构体回调
总线级bus_type.pmsuspend/resume
驱动级device_driver.pmsuspend/resume
设备级dev_pm_opssuspend_late/resume_early
域级dev_pm_domainpoweroff/wakeup

Runtime PM

// 运行时 PM：设备空闲时自动挂起
pm_runtime_set_autosuspend_delay(&dev, 500); // 500ms 延迟
pm_runtime_use_autosuspend(&dev);
pm_runtime_enable(&dev);

// 获取/释放设备时自动管理电源
pm_runtime_get_sync(&dev);     // 恢复电源
// ... 使用设备 ...
pm_runtime_put_autosuspend(&dev); // 自动挂起
