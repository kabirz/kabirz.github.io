01 概述

pinctrl（Pin Control）子系统是 Linux 内核中管理硬件引脚复用（pinmux）和引脚电气配置（pinconf）的统一框架。在现代 SoC 上，一个物理引脚通常可以被复用为多种功能（GPIO、UART TX、SPI CLK 等），pinctrl 提供了标准化的接口来控制这些行为。

pinctrl 的核心职责
  • 引脚复用（pinmux）：选择引脚的功能（GPIO/UART/SPI 等）
  • 引脚配置（pinconf）：设置电气属性（上下拉、驱动强度等）
  • 引脚组（pingroup）：将相关引脚分组管理
  • 冲突检测：防止多个驱动同时使用同一引脚
  • GPIO 集成：与 GPIO 子系统无缝协作
为什么需要 pinctrl
  • SoC 引脚资源有限，需要高效复用
  • 不同外设需要不同的引脚配置
  • 电源管理需要在 sleep 时切换引脚状态
  • 设备树提供了声明式的引脚描述
  • 统一接口避免每个驱动重复实现
典型场景：一个 UART 外设需要 4 个引脚（TX、RX、RTS、CTS），这些引脚可能同时是 SPI、I2C 的候选引脚。pinctrl 确保在 UART 激活时将这些引脚配置为 UART 功能，在休眠时切换为低功耗状态。

04 引脚复用

pinmux 层负责将物理引脚映射到特定的硬件功能（如 UART、SPI、I2C）。

核心概念

函数（Function）

一个引脚可以执行的功能，如 "uart0"、"spi1"、"gpio"。每个函数可以对应多个引脚组。

struct pinfunction

组（Group）

一组必须一起配置的引脚。如 UART 需要 TX+RX 作为一个组，SPI 需要 CLK+MOSI+MISO+CS 作为一组。

struct pingroup

struct pinmux_ops

struct pinmux_ops {
    int (*request)(struct pinctrl_dev *pctldev, unsigned int offset);
    void (*free)(struct pinctrl_dev *pctldev, unsigned int offset);
    int (*get_functions_count)(struct pinctrl_dev *pctldev);
    const char *(*get_function_name)(struct pinctrl_dev *pctldev, unsigned int selector);
    int (*get_function_groups)(struct pinctrl_dev *pctldev, unsigned int selector,
                              const char const **groups, unsigned int *num_groups);
    bool (*function_is_gpio)(struct pinctrl_dev *pctldev, unsigned int selector);
    int (*set_mux)(struct pinctrl_dev *pctldev, unsigned int func_selector,
                   unsigned int group_selector);
    int (*gpio_request_enable)(...);
    void (*gpio_disable_free)(...);
    int (*gpio_set_direction)(...);
    bool strict;  // 是否严格检查 GPIO/mux 冲突
};

复用选择流程

pinctrl_select_state("default")
↓
pinctrl_commit_state()
↓ 遍历 settings
pinmux_enable_setting(setting)
↓

set_mux() // 写硬件寄存器

冲突检测

严格模式：当 pinmux_ops.strict = true 时，pinctrl 会同时检查 gpio_owner 和 mux_owner，防止同一引脚同时被 GPIO 和其他功能使用。非严格模式允许 GPIO 和 mux 共存（某些 SoC 需要）。

06 设备树集成

pinctrl 通过设备树实现声明式的引脚配置，无需修改驱动代码。

设备树语法

// 引脚控制器节点
pinctrl0: pinctrl@e0000000 {
    compatible = "vendor,soc-pinctrl";
    reg = <0xe0000000 0x1000>;

    // 引脚组定义
    uart0_pins: uart0-pins {
        pins = "gpio0_12", "gpio0_13";
        function = "uart0";
        drive-strength = <8>;        // 8mA 驱动强度
        bias-pull-up;                  // 上拉
    };

    spi0_pins: spi0-pins {
        pins = "gpio0_20", "gpio0_21", "gpio0_22", "gpio0_23";
        function = "spi0";
    };
};

// 设备节点引用
uart0: serial@e0001000 {
    compatible = "vendor,soc-uart";
    reg = <0xe0001000 0x100>;

    // 引用引脚配置
    pinctrl-names = "default", "sleep";
    pinctrl-0 = &uart0_pins;        // default 状态
    pinctrl-1 = &uart0_sleep_pins;  // sleep 状态
};

07 消费者 API

设备驱动通过消费者 API 获取和切换 pinctrl 状态。

核心 API

include/linux/pinctrl/consumer.h

函数说明
pinctrl_get(dev)获取设备的 pinctrl 句柄（首次调用会创建）
devm_pinctrl_get(dev)资源管理版本（设备移除时自动释放）
pinctrl_put(p)释放 pinctrl 句柄
pinctrl_lookup_state(p, name)查找指定名称的状态
pinctrl_select_state(p, s)切换到指定状态
pinctrl_select_default_state(dev)切换到 default 状态

典型使用模式

static int my_driver_probe(struct platform_device *pdev)
{
    struct pinctrl *pctl;
    struct pinctrl_state *state_default, *state_sleep;

    // 1. 获取 pinctrl 句柄
    pctl = devm_pinctrl_get(&pdev->dev);

    // 2. 查找默认状态（通常设备核心已自动完成）
    state_default = pinctrl_lookup_state(pctl, PINCTRL_STATE_DEFAULT);

    // 3. 应用默认状态
    pinctrl_select_state(pctl, state_default);

    // 4. 查找 sleep 状态
    state_sleep = pinctrl_lookup_state(pctl, PINCTRL_STATE_SLEEP);

    // 5. 在 suspend 时切换到 sleep 状态
    // pinctrl_select_state(pctl, state_sleep);
}

PM 集成

// 电源管理状态切换
pinctrl_pm_select_default_state(dev);  // resume 时
pinctrl_pm_select_sleep_state(dev);    // suspend 时
pinctrl_pm_select_idle_state(dev);     // idle 时
pinctrl_pm_select_init_state(dev);     // probe 前

自动状态管理

设备核心集成：在 driver_probe_device() 中，设备核心会自动调用 pinctrl_bind_pins()，它会：
1. 调用 pinctrl_get() 获取句柄
2. 依次尝试 "init" → "default" 状态
3. 将 pinctrl 和 pinctrl_default_state 存入 dev->pinctrl
因此大多数驱动不需要手动调用 pinctrl_get()。
