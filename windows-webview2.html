01 概述

WebView2 是微软提供的现代 Web 视图控件，它把Microsoft Edge（Chromium 内核）的完整渲染引擎以控件形式嵌入到任意 Windows 应用中。开发者既拥有原生外壳的全部能力（文件系统、硬件、原生 API），又能用 Web 技术栈（HTML/CSS/JS/React/Vue）构建 UI——这就是混合应用（Hybrid App）模式。

核心特性
  • Chromium 渲染：与 Edge 同源，完整 Web 兼容性
  • 多框架支持：WPF / WinForms / WinUI 3 / Win32 / MFC
  • 双向通信：WebMessage + 主机对象投影
  • 运行时复用：共享系统 Edge，无需自带浏览器
  • 自动更新：Evergreen 模式由微软推送安全更新
  • 本地资源映射：虚拟主机让 Web 加载本地文件
典型适用场景
场景说明
现代 UI 改造遗留 MFC/Win32 用 Web 重构界面
跨端复用复用已有的 Web 前端代码
富内容展示复杂排版/图表/多媒体
快速迭代UI 改动无需重新编译原生
混合应用Teams / Edge 自身的 PWAs
定位：WebView2 与原生 GUI 框架互补而非替代——它是嵌入到 WPF/WinForms/WinUI/Win32/MFC 容器中的一个控件，原生负责窗口与系统集成，Web 负责 UI 渲染。

02 架构与运行时

WebView2 的运行时（Runtime）是独立的 Edge Chromium 浏览器引擎，由微软通过 Windows Update 分发。客户端应用通过 SDK 调用，SDK 内部以 COM 桥接到运行时的多进程模型。

两种运行时分发模式

Evergreen（常青）
  • 随 Windows 更新自动升级
  • 多 App 共享一份运行时
  • 微软负责安全补丁
  • 包体最小（SDK 仅几 MB）
  • 当前主流推荐方式
FixedVersion（固定版本）
  • 解压一个固定版本的运行时随 App 分发
  • 完全可控，无外部依赖
  • 需自行处理更新与安全
  • 体积大（~150MB）
  • 适用于受控环境/合规要求

组件与进程模型

// WebView2 整体架构
┌─────────────────────────────────────────────────────────┐
│  客户端 App（WPF / WinForms / Win32 / WinUI）          │
│    └── WebView2 SDK（托管包装 + COM 互操作）           │
└──────────────────────────┬──────────────────────────────┘
                           │ COM / Project Reunion
                           ▼
┌─────────────────────────────────────────────────────────┐
│  WebView2 Runtime（Edge Chromium 内核，多进程）        │
│    ├── Browser Process（主进程，1 个）              │
│    │     └── 管理所有 WebView 实例、网络、GPU 调度          │
│    ├── Renderer Process（渲染进程，每站点 1 个）     │
│    │     └── Blink 渲染 + V8 JS 引擎（沙箱隔离）          │
│    ├── GPU Process（图形合成，全局共享）            │
│    └── Utility Process（网络/音频/存储服务）         │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
                UserDataFolder（每 App 独立）
                  缓存 / Cookie / localStorage / IndexedDB
进程隔离：每个 WebView2 实例对应独立的渲染进程，与 Chrome 同源——单页崩溃不影响其他 WebView 或宿主进程。但所有实例共享同一个 UserDataFolder，不同 App 必须指定不同 UserDataFolder，否则 Cookie/缓存冲突。

03 环境与创建

WebView2 的初始化遵循三层对象模型：先创建 Environment（环境），再创建 Controller（控制器，承载 HWND/父控件），最后通过 Controller.CoreWebView2 拿到核心 WebView 用于操作。

// 三层对象层级（C# 概念）
CoreWebView2Environment        // 运行时环境（进程/数据目录/选项）
  └── CoreWebView2Controller      // 控制器：管理窗口、焦点、缩放
        └── CoreWebView2             // 核心：导航、JS、消息、设置
              └── CoreWebView2Settings  // 行为开关（DevTools/脚本/状态栏）

C# 异步初始化

// WPF / WinForms 中最常见的初始化模式
public partial class MainWindow : Window
{
    public MainWindow() {
        InitializeComponent();
        InitializeAsync();
    }

    private async Task InitializeAsync()
    {
        // 1. 显式指定 UserDataFolder（多实例隔离的关键）
        var options = new CoreWebView2EnvironmentOptions {
            AdditionalBrowserArguments = "--disable-features=msHubApps",
            Language = "zh-CN"
        };
        var env = await CoreWebView2Environment.CreateAsync(
            browserExecutableFolder: null,             // null = Evergreen
            userDataFolder: @"%LOCALAPPDATA%\MyApp\WebView",
            options: options);

        // 2. 在指定环境下初始化控件
        await webView.EnsureCoreWebView2Async(env);

        // 3. 此后 CoreWebView2 可用
        webView.CoreWebView2.Navigate("https://app.local/index.html");
    }
}
// 或使用最简形式（默认 Evergreen + 默认 UserDataFolder）
public MainWindow() {
    InitializeComponent();
    webView.CoreWebView2InitializationCompleted += (s, e) => {
        if (e.IsSuccess) {
            webView.CoreWebView2.Navigate("https://localhost:8080");
        }
    };
    _ = webView.EnsureCoreWebView2Async();
}
三层关系：Environment 是进程级对象（一个 App 可共享一个），Controller 是窗口级对象（每控件一个，绑 HWND），CoreWebView2 是页面级对象（导航/JS/消息都在这里）。CoreWebView2InitializationCompleted 事件触发后才能安全使用 CoreWebView2，在此之前访问会抛 InvalidOperationException。

WebView2 为每个主流 GUI 框架提供专门的包装控件，统一底层 API 但适配各自的布局、事件、句柄体系。

// 直接绑定到 HWND
HRESULT CreateWebView(HWND hwndParent) {
    return CreateCoreWebView2EnvironmentWithOptions(
        L"C:\\Runtime",         // 运行时路径
        L"C:\\UserData",        // UserDataFolder
        nullptr,                // 选项
        Callback<ICoreWebView2CreateCoreWebView2EnvironmentCompletedHandler>(
            [this](HRESULT, ICoreWebView2Environment* env) {
                env->CreateCoreWebView2Controller(hwndParent, ...);
                return S_OK;
            }).Get());
}

05 核心 API

CoreWebView2 暴露的核心能力分四类：导航、脚本执行、消息通信、辅助功能（截图/打印）。

// 导航相关
webView.CoreWebView2.Navigate("https://example.com");           // 加载 URL
webView.CoreWebView2.NavigateToString(htmlContent);              // 加载 HTML 字符串
string url = webView.CoreWebView2.Source;                           // 当前 URL
webView.CoreWebView2.GoBack();   webView.CoreWebView2.GoForward();
bool canBack = webView.CoreWebView2.CanGoBack();

// 事件：导航各阶段
webView.CoreWebView2.NavigationStarting    += OnNavStarting;     // 可取消
webView.CoreWebView2.NavigationCompleted   += OnNavCompleted;    // 成功/失败
webView.CoreWebView2.SourceChanged         += OnSourceChanged;
webView.CoreWebView2.HistoryChanged        += OnHistoryChanged;

// 执行 JS 表达式，返回 JSON 字符串
string result = await webView.CoreWebView2.ExecuteScriptAsync(
    "JSON.stringify({ width: window.innerWidth })");

// 截图（PNG 字节数组）
Stream stream = await webView.CoreWebView2.CapturePreviewAsync(
    CoreWebView2CapturePreviewImageFormat.Png);

// 打印为 PDF
Stream pdf = await webView.CoreWebView2.PrintToPdfAsync(
    @"C:\out.pdf", null);

C++ COM 等价写法

wil::com_ptr<ICoreWebView2> m_webView;

HRESULT NavigateAndExec(LPCWSTR url) {
    HRESULT hr = m_webView->Navigate(url);
    if (FAILED(hr)) return hr;

    // 注册导航完成回调
    m_webView->add_NavigationCompleted(
        Callback<ICoreWebView2NavigationCompletedEventHandler>(
            [](ICoreWebView2*, ICoreWebView2NavigationCompletedEventArgs* args) {
                BOOL success;
                args->get_IsSuccess(&success);
                if (success) { /* 页面就绪 */ }
                return S_OK;
            }).Get(),
        &m_navCompletedToken);
    return S_OK;
}

// 执行 JS
m_webView->ExecuteScript(L"document.title",
    Callback<ICoreWebView2ExecuteScriptCompletedHandler>(
        [](HRESULT err, LPCWSTR resultJson) {
            if (SUCCEEDED(err)) /* resultJson 形如 "\"My Page\"" */
            return S_OK;
        }).Get());
ExecuteScriptAsync 的返回：执行结果总是JSON 编码的字符串——"42" 是 "42"，true 是 "true"，对象需 JSON.stringify。这是常见坑：你不能直接拿到强类型值，必须 JsonSerializer.Deserialize。

06 原生 ↔ Web 通信

WebView2 提供两种原生与 Web 双向通信机制，分别适用于不同复杂度场景。

WebMessage（消息传递）
  • JSON 字符串双向收发
  • 异步、解耦、简单
  • 无强类型，需手动协议
  • 适合：通知、事件、轻量调用
HostObject（对象投影）
  • 把 COM 对象暴露给 JS
  • JS 直接 hostObj.Method()
  • 同步 IPC，强类型签名
  • 适合：紧密集成、复杂调用

方式一：WebMessage

// ── C# 端 ──
// 接收 Web 发来的消息
webView.CoreWebView2.WebMessageReceived += (s, e) => {
    string json = e.TryGetWebMessageAsString();
    var msg = JsonSerializer.Deserialize<Msg>(json);
    if (msg.type == "save") SaveToFile(msg.data);
};

// 向 Web 发送消息
webView.CoreWebView2.PostWebMessageAsJson(JsonSerializer.Serialize(new {
    type = "fileSaved", path = "C:\\a.txt"
}));

// ── Web 端（JS）──
window.chrome.webview.postMessage(JSON.stringify({ type: "save", data: content }));

window.chrome.webview.addEventListener('message', event => {
    const msg = JSON.parse(event.data);
    if (msg.type === 'fileSaved') showToast('已保存到 ' + msg.path);
});

方式二：主机对象投影

// ── C#：定义并暴露一个对象 ──
[ClassInterface(ClassInterfaceType.AutoDual)]
[ComVisible(true)]
public class FileBridge {
    public string ReadAllText(string path) => File.ReadAllText(path);
    public void WriteAllText(string path, string content) => File.WriteAllText(path, content);
}

// 注册：JS 中通过 chrome.webview.hostObjects.file 访问
webView.CoreWebView2.AddHostObjectToScript("file", new FileBridge());

// ── Web 端（JS）：直接调用原生方法 ──
async function loadConfig() {
    const { file } = window.chrome.webview.hostObjects;
    const text = await file.ReadAllText("C:\\config.json");  // 同步 IPC，强类型
    return JSON.parse(text);
}
HostObject 安全：暴露的 COM 对象所有公共方法都对页面中的任意 JS 可见，包括第三方脚本与 iframe（除非设置 AddHostObjectToScript 的同名参数限定同源）。仅暴露最小必要接口，绝不要把整个 File、Process 之类的危险对象直接投影——这等于把本地代码执行能力交给了 Web。
选型：消息传值简单安全但需要手写协议；HostObject 调用像本地代码但放大了攻击面。生产实践通常是WebMessage 负责高频事件流，HostObject 仅封装受控的、参数校验过的业务接口。

07 虚拟主机与本地资源

WebView2 提供一套机制让 Web 端用标准 https 协议加载本地文件，而无需 file://（CORS 受限、路径暴露）或 NavigateToString（单文件限制）。

SetVirtualHostNameToFolderMapping

// 把本地目录映射为虚拟主机
webView.CoreWebView2.SetVirtualHostNameToFolderMapping(
    hostName: "app.local",
    folderPath: @"C:\App\dist",           // 或 AppContext.BaseDirectory
    accessKind: CoreWebView2HostResourceAccessKind.Allow);

// 之后 Web 端即可：
//   fetch("https://app.local/config.json")
//   <link rel="stylesheet" href="https://app.local/style.css">
//   import './module.js' from https://app.local/index.js
webView.CoreWebView2.Navigate("https://app.local/index.html");

注入脚本与拦截请求

// 页面加载时注入全局 JS（每次导航都会执行）
webView.CoreWebView2.AddScriptToExecuteOnDocumentCreatedAsync(
    "window.NATIVE = { version: '1.0', platform: 'win32' };");

// 拦截 Web 发起的请求（虚拟主机/真实网络都拦截）
webView.CoreWebView2.WebResourceRequested += (s, e) => {
    var uri = e.Request.Uri;
    if (uri.StartsWith("https://app.local/api/")) {
        var response = new MemoryStream(Encoding.UTF8.GetBytes("{\"ok\":true}"));
        e.GetDeferral().Complete();
        // 用环境创建 Response 并赋给 e.Response
    }
};
// 必须先添加 filter，否则事件不触发
webView.CoreWebView2.AddWebResourceRequestedFilter(
    "https://app.local/api/*", CoreWebView2WebResourceContext.All);

自定义右键菜单

// 移除/添加右键菜单项
var menu = webView.CoreWebView2.ContextMenuRequested;
webView.CoreWebView2.ContextMenuRequested += (s, e) => {
    var items = e.MenuItems;
    // 移除"检查"和"重新加载"
    items.RemoveAll(i => i.Name == "inspect" || i.Name == "reload");
    // 添加自定义项
    items.Add(new CoreWebView2ContextMenuItem {
        Label = "导出为 PDF", CommandId = 1001
    });
};
虚拟主机的价值：本地资源以 https:// 访问意味着所有 Web 标准 API（fetch、ES Modules、Service Worker 概念）都正常工作，CORS 不再是障碍；同时虚拟域名对页面透明，前端代码无需改动即可在本地/远程之间切换。

08 安全

WebView2 继承 Chromium 的全部安全基础设施（沙箱、站点隔离、同源策略），但主机对象的暴露、运行时版本、导航控制是应用层必须自己把关的部分。

安全配置项API / 设置建议
开发者工具Settings.AreDevToolsEnabled发布版关闭
浏览器快捷键Settings.AreBrowserAcceleratorKeysEnabled发布版关闭（避免 F5/Ctrl+N）
右键菜单AreDefaultContextMenusEnabled按需关闭
状态栏IsStatusBarEnabled按需关闭
导航拦截NavigationStarting白名单过滤 URL
主机对象范围AddHostObjectToScript 同源参数限制为本地资源
脚本执行IsScriptEnabled纯 HTML 展示时关闭
UserDataFolder 权限目录 ACL每用户独立目录
// 发布版本的典型安全配置
var settings = webView.CoreWebView2.Settings;
settings.AreDevToolsEnabled = false;                  // 关闭 DevTools
settings.AreBrowserAcceleratorKeysEnabled = false;  // 关闭浏览器快捷键
settings.AreDefaultContextMenusEnabled = false;
settings.IsStatusBarEnabled = false;
settings.IsBuiltInErrorPageEnabled = true;

// 导航白名单
webView.CoreWebView2.NavigationStarting += (s, e) => {
    var allowed = new[] { "https://app.local", "https://login.example.com" };
    if (!allowed.Any(p => e.Uri.StartsWith(p))) {
        e.Cancel = true;                       // 拦截外部跳转
    }
};
主机对象是最大风险面：通过 AddHostObjectToScript 暴露的任意方法都能被页面中的任意 JS 调用——包括 XSS 注入的脚本、第三方广告、跨域 iframe。绝不要暴露 File、Process、Registry、PowerShell 等危险对象，应封装一个最小接口，对参数做严格校验。一旦页面被 XSS，整个宿主机随之沦陷。
SmartScreen：Edge 运行时默认集成 Microsoft Defender SmartScreen，对导航到的 URL 做钓鱼/恶意软件检查，可在运行时策略中关闭但不建议。

09 打包与部署

WebView2 运行时的部署方式直接决定应用的体积、可控性和更新策略。微软提供三种官方分发方案。

分发方式体积更新可控性典型用途
Evergreen Bootstrapper~2MB（在线引导）自动（微软推送）低消费应用，联网安装
Evergreen Standalone Installer~150MB（离线包）自动低内网/离线部署
FixedVersion~150MB（随 App 解压）手动完全合规/受控环境

运行时检测与安装

// C#：检测运行时版本
string version = await CoreWebView2Environment.GetAvailableBrowserVersionStringAsync();
if (string.IsNullOrEmpty(version)) {
    // 未安装：触发 Bootstrapper 静默安装
    Process.Start("MicrosoftEdgeWebview2Setup.exe", "/silent /install");
}

// C++：通过导出函数 GetAvailableCoreWebView2BrowserVersionString
LPWSTR version = nullptr;
GetAvailableCoreWebView2BrowserVersionString(nullptr, &version);
if (version == nullptr) { /* 未安装，启动安装器 */ }
CoTaskMemFree(version);

企业部署

Intune / 组策略
  • 通过 MDM 下发 Evergreen 安装包
  • 组策略控制更新通道
  • 可禁用自动更新（合规）
  • 强制使用 FixedVersion
FixedVersion 目录结构
MyApp/
├── MyApp.exe
├── WebView2Loader.dll      SDK 加载器
└── runtime/                解压的 FixedVersion
    ├── msedgewebview2.exe
    ├── r.130.0.2849.46/
    └── ...
Evergreen 优势：消费级应用首选——微软持续推送安全补丁，应用无需自管更新。FixedVersion 警告：你会失去自动安全更新，必须自行跟踪 CVE 并重新打包发布。

10 完整示例

一个完整的 WPF 应用：嵌入 WebView2、加载本地 HTML（通过虚拟主机）、实现 JS 与 C# 双向消息通信、调用前端函数获取返回值。

// ── MainWindow.xaml ──
<Window x:Class="MyApp.MainWindow"
        xmlns:wv2="clr-namespace:Microsoft.Web.WebView2.Wpf;assembly=Microsoft.Web.WebView2.Wpf">
  <Grid>
    <wv2:WebView2 Name="webView" />
  </Grid>
</Window>

// ── MainWindow.xaml.cs ──
using System.IO;
using System.Text.Json;
using Microsoft.Web.WebView2.Core;

public partial class MainWindow : Window
{
    public MainWindow() {
        InitializeComponent();
        Loaded += async (_, _) => await InitAsync();
    }

    private async Task InitAsync()
    {
        // 1. 初始化（指定 UserDataFolder 隔离）
        await webView.EnsureCoreWebView2Async(
            await CoreWebView2Environment.CreateAsync(
                browserExecutableFolder: null,
                userDataFolder: Path.Combine(
                    Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData),
                    "MyApp", "WebView")));

        var core = webView.CoreWebView2;

        // 2. 发布版安全配置
        core.Settings.AreDevToolsEnabled = Debugger.IsAttached;

        // 3. 虚拟主机：本地 ./dist 映射为 https://app.local
        var dist = Path.Combine(AppContext.BaseDirectory, "dist");
        core.SetVirtualHostNameToFolderMapping(
            "app.local", dist,
            CoreWebView2HostResourceAccessKind.Allow);

        // 4. 接收 Web 消息
        core.WebMessageReceived += (_, e) => {
            var msg = JsonSerializer.Deserialize<JsonElement>(e.TryGetWebMessageAsString());
            if (msg.GetProperty("type").GetString() == "save") {
                File.WriteAllText("data.json",
                    msg.GetProperty("data").GetRawText());
                core.PostWebMessageAsJson("{\"type\":\"saved\"}");
            }
        };

        // 5. 注入启动脚本
        await core.AddScriptToExecuteOnDocumentCreatedAsync(
            "window.NATIVE = { save: d => chrome.webview.postMessage(JSON.stringify({type:'save',data:d})) };");

        // 6. 加载首页
        core.Navigate("https://app.local/index.html");

        // 7. 页面就绪后调用前端函数
        core.NavigationCompleted += async (_, e) => {
            if (!e.IsSuccess) return;
            string json = await core.ExecuteScriptAsync(
                "JSON.stringify(getStats())");   // 返回 JSON 字符串
            var stats = JsonSerializer.Deserialize<Stats>(json);
            UpdateTitleBar(stats);
        };
    }
}
// ── dist/index.html（本地 Web 资源）──
<!DOCTYPE html>
<html><body>
  <button onclick="NATIVE.save({count:42})">保存到本地</button>
  <script>
    function getStats() { return { users: 100, active: 30 }; }
    window.chrome.webview.addEventListener('message', e => {
      const { type } = JSON.parse(e.data);
      if (type === 'saved') alert('已保存');
    });
  </script>
</body></html>
本例体现的完整闭环：本地 HTML 通过虚拟主机加载 → JS 调 NATIVE.save() 触发 WebMessage → C# 处理后写文件并回消息 → JS 弹窗。同时 C# 通过 ExecuteScriptAsync 主动调用前端函数取返回值，覆盖了双向通信的全部典型用法。

11 对比 Electron / CEF

三大嵌入式 Web 方案各有取舍：Electron 自带 Node.js + Chromium，CEF 是 C++ 嵌入，WebView2 复用系统 Edge。

维度WebView2ElectronCEF
内核来源系统 Edge（共享）自带 Chromium自带 Chromium
运行时体积~2MB（Evergreen）/ ~150MB（Fixed）~150MB+~100MB+
内存占用低（共享系统进程）高（独立 Chromium）高（独立 Chromium）
主语言C++ / C# / 任意JavaScript / Node.jsC++（chromiumembed）
原生能力Win32 / .NET（强原生）Node.js API（跨平台）C++ 原生（跨平台）
安全更新微软 Evergreen 自动需升 Electron 大版本需升 CEF 版本
跨平台仅 WindowsWin/macOS/LinuxWin/macOS/Linux
启动速度快（运行时常驻）中（启 Node + Chromium）中（启 Chromium）
生态Win 原生 + Webnpm 巨大生态C++ 工程化生态
何时选 WebView2
  • 仅 Windows 平台
  • 已有 WPF/WinForms/Win32/MFC 工程
  • 追求小体积、低内存
  • 需要原生系统集成（驱动/硬件）
  • 安全更新由微软兜底
何时选 Electron / CEF
  • 需要跨平台（CEF/Electron）
  • 全栈 JS（Electron）
  • 完全自控内核版本（CEF）
  • 已有大量 npm 依赖（Electron）
  • 不依赖系统浏览器（CEF）
WebView2 的核心优势：复用系统 Edge 运行时——包体最小、内存最低、安全更新由微软维护。VS Code、Teams 客户端、Edge 自身的部分组件已转向 WebView2。代价是锁定 Windows 平台和运行时可能未预装（Win10 需 Bootstrapper，Win11 默认有）。
