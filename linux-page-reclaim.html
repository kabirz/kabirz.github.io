01 概述

页面回收（Page Reclaim）是 Linux 内核的内存回收机制。当物理内存不足时，内核将不活跃的页面从内存中回收：匿名页面换出到 swap，文件页面丢弃（因为可以从磁盘重新读取）。它是 overcommit 和 OOM 之间的缓冲机制。

02 LRU 链表

Linux 页面 LRU 管理：

  每个 Zone 维护 5 条 LRU 链表：

  ┌──────────────────────────────────────────┐
  │ Zone Normal                               │
  │                                           │
  │  LRU_INACTIVE_ANON  ← 匿名页面（不可见）  │
  │  LRU_ACTIVE_ANON    ← 匿名页面（活跃）    │
  │  LRU_INACTIVE_FILE  ← 文件页面（不可见）  │
  │  LRU_ACTIVE_FILE    ← 文件页面（活跃）    │
  │  LRU_UNEVICTABLE    ← 不可驱逐（锁定）   │
  └──────────────────────────────────────────┘

  页面状态转换：
  新页面 → ACTIVE_ANON/FILE（活跃）
           │ 定期降级
           ↓
         INACTIVE_ANON/FILE（不可见）
           │ 回收候选
           ↓
         被回收（换出/丢弃）

03 回收流程

kswapd 后台回收（异步）：

  内存空闲量低于 low 水位线
        │
        ↓
  唤醒 kswapd 守护进程
        │
        ↓
  kswapd() → shrink_node()
        │
        ├── 1. 扫描 LRU 链表（inactive 优先）
        │       遍历 inactive list，找到可回收页面
        │
        ├── 2. 检查页面状态
        │       ├── clean 文件页面 → 直接丢弃
        │       ├── dirty 文件页面 → 回写到磁盘
        │       └── 匿名页面 → 换出到 swap
        │
        ├── 3. 回收页面
        │       shrink_folio_list() → 逐个回收
        │
        ├── 4. 检查回收量
        │       空闲量 ≥ high 水位线 → 停止
        │       空闲量 < high → 继续回收
        │
        └── 5. 释放回收的页面
            交给伙伴系统

直接回收（同步）：

  分配页面失败 + kswapd 未在运行
        │
        ↓
  当前进程直接调用 shrink_node()
  （阻塞进程，影响延迟）

04 kswapd 守护进程

水位线说明
min最低保留（紧急分配用）
low低水位线：触发 kswapd 回收
high高水位线：kswapd 停止回收
zone_balanced平衡水位线
  水位线关系：

  高水位（high） ──── kswapd 停止
        │
        │  kswapd 回收区域
        │
  低水位（low）  ──── kswapd 开始回收
        │
        │  紧急分配区域
        │
  最低水位（min） ──── 紧急保留（预留不可动）

05 配置与调优

# 查看内存水位线
$ cat /proc/zoneinfo | grep min\|low\|high
        min    1024
        low    3584
        high   6144

# 查看回收统计
$ cat /proc/vmstat | grep -E "pgpgin|pgpgout|pswpin|pswpout|pgsteal|pgscan"
pswpin     12345    # 从 swap 读入的页数
pswpout    67890    # 写入 swap 的页数
pgsteal_kswapd 123456 # kswapd 回收的页数

# 调整 min_free_kbytes
$ sysctl -w vm.min_free_kbytes=65536  # 64MB 保留

# 调整 watermark_boost_factor
$ sysctl -w vm.watermark_boost_factor=15000
