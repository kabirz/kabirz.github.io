01 概述

路由子系统是 Linux 网络栈的决策中心。每个进出本机的数据包都要经过路由决策：确定从哪个接口发出、下一跳是谁、是否需要转发。Linux 的路由实现基于 FIB（Forwarding Information Base），使用 LC-trie 数据结构实现高效最长前缀匹配。

路由核心职责
  • 最长前缀匹配：从路由表中找到最精确的路由
  • 下一跳解析：确定数据包发往哪个网关或接口
  • 多路径负载均衡：ECMP 等价多路径
  • 策略路由：基于源 IP、DSCP、fwmark 选择路由表
  • PMTU：路径 MTU 发现

02 路由决策流程

收包路由决策（ip_rcv）：

  数据包到达 IP 层
        │
        ↓
  ip_rcv() → NF_INET_PRE_ROUTING
        │
        ↓
  ip_route_input() — route.c
        │
        ├── 1. 查路由缓存（dst_cache）
        │       命中 → 直接返回 rtable（快速路径）
        │
        ├── 2. 缓存未命中 → ip_route_input_slow()
        │       │
        │       ├── 3. fib_table_lookup() → FIB 查找
        │       │       最长前缀匹配，得到 fib_result
        │       │
        │       ├── 4. 判断路由类型
        │       │       RTN_LOCAL  → 本机接收
        │       │       RTN_UNICAST → 转发
        │       │       RTN_MULTICAST → 组播
        │       │       RTN_BLACKHOLE → 丢弃
        │       │
        │       └── 5. 创建 rtable，加入路由缓存
        │
        ↓
  ┌─────────────────────────────────────┐
  │ 路由类型判断                          │
  ├─────────┬─────────┬─────────────────┤
  │ 本地    │ 转发    │ 丢弃/其他       │
  │ ip_local_deliver → ip_forward →    │
  │ → UDP/TCP  → netfilter → 发送     │
  └─────────┴─────────┴─────────────────┘

发包路由决策（ip_output）：

  socket 发送数据
        │
        ↓
  ip_route_output_flow() — route.c
        │
        ├── 查路由缓存 → 命中 → 返回
        │
        └── 缓存未命中 → ip_route_output_key_hash()
                │
                ├── fib_table_lookup() → FIB 查找
                ├── 创建 rtable，加入缓存
                └── 返回 skb->dst = rtable
        │
        ↓
  ip_output → ip_finish_output → dev_queue_xmit

03 核心数据结构

// include/net/flow.h:68 — 路由查找键
struct flowi4 {
    __be32  saddr;      // 源 IP
    __be32  daddr;      // 目的 IP（路由查找键）
    // 来自 flowi_common：
    int     flowi4_oif; // 出接口（可选）
    int     flowi4_iif; // 入接口
    __u8    flowi4_tos; // TOS/DSCP
    __u8    flowi4_proto;// 协议（TCP/UDP/ICMP）
    union flowi_uli uli; // 源/目的端口（用于 ECMP 哈希）
};
// include/net/route.h:57 — 路由缓存条目
struct rtable {
    struct dst_entry  dst;          // 目的缓存（含邻居引用）
    int               rt_genid;      // 路由表代号
    unsigned int      rt_flags;      // RTCF_BROADCAST 等
    __u16             rt_type;       // RTN_LOCAL / RTN_UNICAST
    __u8              rt_is_input;   // 是否是收包路由
    __u8              rt_uses_gateway;// 是否使用网关
    int               rt_iif;        // 入接口索引
    __be32            rt_gw4;        // IPv4 下一跳网关
    u32               rt_pmtu:31;    // 路径 MTU
};
// include/net/ip_fib.h:173 — FIB 查找结果
struct fib_result {
    __be32              prefix;      // 匹配的路由前缀
    unsigned char       prefixlen;   // 前缀长度（最长匹配）
    unsigned char       nh_sel;      // 下一跳选择（ECMP）
    unsigned char       type;        // 路由类型（UNICAST/LOCAL...）
    unsigned char       scope;       // 路由作用域
    struct fib_nh_common *nhc;       // 下一跳信息
    struct fib_info     *fi;         // 路由详细信息
    struct fib_table    *table;      // 匹配的路由表
};

04 FIB 转发表

FIB 表结构

// include/net/ip_fib.h:257
struct fib_table {
    u32             tb_id;        // 表 ID（255=main, 254=local, 0=unspec）
    int             tb_num_hoists; // 被引用次数
    struct rhlist_head tb_rh;    // RHASH 节点
    struct fib_table_ops __rcu *tb_data; // 操作函数表
};

// 常用路由表：
// RT_TABLE_LOCAL (254) — 本地路由（本机 IP）
// RT_TABLE_MAIN  (255) — 主路由表
// RT_TABLE_UNSPEC (0)  — 未指定

LC-trie 前缀匹配

FIB 使用 LC-trie（Level Compressed Trie）数据结构：

  路由表：
  10.0.0.0/8      → dev eth0        (scope link)
  192.168.1.0/24  → via 10.0.0.1    (gw)
  192.168.1.10/32 → dev eth0        (scope host, 本地)

  LC-trie 结构（简化）：
                        ┌───┐
                        │ 1 │ ← 第 0 位
                        └─┬─┘
                    0 /     \ 1
                ┌───┐         ┌───┐
                │ 0 │         │ 1 │
                └─┬─┘         └─┬─┘
             ...    ...
                      ┌─────┐
                      │10.0 │ ← 匹配 10.0.0.0/8
                      └─────┘
                      ┌───────────┐
                      │192.168.1.0│ ← 匹配 192.168.1.0/24
                      └───────────┘
                      ┌───────────────┐
                      │192.168.1.10   │ ← 匹配 192.168.1.10/32（最长匹配）
                      └───────────────┘

  查找结果：最长前缀匹配
  192.168.1.10 → 匹配 /32（精确），不匹配 /24 或 /8

路由表类型

类型说明示例
RTN_UNICAST单播路由默认路由、直连路由
RTN_LOCAL本地路由本机 IP 地址
RTN_BROADCAST广播路由子网广播地址
RTN_MULTICAST组播路由224.0.0.0/4
RTN_BLACKHOLE黑洞路由丢弃所有匹配包
RTN_UNREACHABLE不可达返回 ICMP unreachable

05 路由缓存

现代 Linux（3.6+）使用 dst_cache 替代传统路由缓存：

  传统路由缓存（已废弃）：
  ┌──────────────────────────────┐
  │ 全局路由缓存哈希表            │
  │ (saddr, daddr, tos, ...) →   │
  │         rtable               │
  └──────────────────────────────┘
  问题：可被伪造 IP 攻击（CVE-2013-2224）→ 3.6 移除

  现代 dst_cache（当前实现）：
  ┌──────────────────────────────┐
  │ 每个 skb 内嵌 dst_cache      │
  │ rtable 缓存在 skb->dst 中    │
  │ 相同 flow 的后续包可复用      │
  │ 无需全局哈希表               │
  └──────────────────────────────┘
  优势：无锁、无全局竞争、安全

06 策略路由

普通路由：仅根据目的 IP 选择路由

  数据包 → 查找 main 表 → 选择路由

策略路由：根据多种条件选择不同的路由表

  数据包 → ip_rule 查找
      │
      ├── 匹配 fwmark=100 → 查表 100
      ├── 匹配 src=10.0.0.0/8 → 查表 200
      ├── 匹配 dscp=46 → 查表 300
      └── 默认 → 查 main 表

多路由表示例：

  表 100 (isp1):
  default via 10.0.1.1 dev eth0

  表 200 (isp2):
  default via 10.0.2.1 dev eth1

  ip rule:
  from 10.0.1.0/24  table 100  (ISP1 用户 → ISP1 路由)
  from 10.0.2.0/24  table 200  (ISP2 用户 → ISP2 路由)

# 配置策略路由
$ ip rule show
0:	from all lookup local
32766:	from all lookup main
32767:	from all lookup default

# 添加策略规则
$ ip rule add from 10.0.1.0/24 table 100
$ ip rule add fwmark 100 table 200

# 添加路由到指定表
$ ip route add default via 10.0.1.1 dev eth0 table 100
$ ip route add default via 10.0.2.1 dev eth1 table 200

# 用 iptables 标记数据包
$ iptables -t mangle -A PREROUTING -s 10.0.1.0/24 -j MARK --set-mark 100
