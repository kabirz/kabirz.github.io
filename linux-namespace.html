01 概述

Namespace 是 Linux 内核提供的资源隔离机制。每个命名空间将全局资源（进程 ID、网络、文件系统等）封装为独立的视图，使不同命名空间中的进程看到不同的"世界"。它是 Docker、LXC、Kubernetes 等容器技术的基石。Linux 支持 7 种命名空间。

02 七种命名空间

类型CLONE 标志隔离内容典型用途
MNTCLONE_NEWNS挂载点容器文件系统
PIDCLONE_NEWPID进程 ID容器 PID 隔离
NETCLONE_NEWNET网络栈容器网络（已有专题）
UTSCLONE_NEWUTS主机名/域名容器主机名
IPCCLONE_NEWIPCIPC 资源隔离 System V IPC
USERCLONE_NEWUSER用户/组 ID非特权容器
CGROUPCLONE_NEWCGROUPcgroup 根目录cgroup 视图隔离
Docker 容器使用全部 7 种命名空间：

  host namespace                    container namespace
  ┌────────────────────┐           ┌────────────────────┐
  │ PID 1 (init)       │           │ PID 1 (container)  │  ← PID
  │ PID 2 (kthreadd)   │           │ PID 2              │
  │ ...                 │           │                    │
  │                     │           │                    │
  │ hostname: prod-srv  │           │ hostname: web-01   │  ← UTS
  │                     │           │                    │
  │ 192.168.1.100       │           │ 10.0.0.2           │  ← NET
  │ eth0, docker0       │           │ eth0               │
  │                     │           │                    │
  │ / (rootfs)          │           │ / (overlayfs)      │  ← MNT
  │ /var/lib/docker     │           │                    │
  └────────────────────┘           └────────────────────┘

04 创建与切换

创建新命名空间：

  方式 1：clone() 创建新进程
  clone(CLONE_NEWPID | CLONE_NEWNET | CLONE_NEWNS, ...)
  → 新进程获得独立的 PID/NET/MNT 命名空间

  方式 2：unshare() 当前进程
  unshare(CLONE_NEWUTS)
  → 当前进程获得独立的 UTS 命名空间
  → 不影响其他进程

切换到已有命名空间：

  setns(fd, CLONE_NEWPID)
  → 当前进程进入 fd 指定的命名空间
  → 需要 CAP_SYS_ADMIN 权限

05 Docker 中的应用

Docker 容器创建时的命名空间设置：

  docker run -d --name web nginx
        │
        ↓
  runc 容器运行时
        │
        ├── clone(CLONE_NEWPID | CLONE_NEWNET | CLONE_NEWNS
        │         | CLONE_NEWUTS | CLONE_NEWIPC
        │         | CLONE_NEWUSER | CLONE_NEWCGROUP)
        │
        ├── 创建所有 7 种命名空间
        │
        ├── 配置每个命名空间：
        │   PID:  容器内 PID 1 = nginx 进程
        │   NET:  创建 veth → br0 → 宿主机网络
        │   MNT:  挂载 overlayfs 作为根文件系统
        │   UTS:  设置 hostname = 容器 ID
        │   IPC:  隔离 System V IPC
        │   USER: 映射 UID（非特权容器）
        │   CGROUP: 设置 cgroup 限制
        │
        └── 启动容器进程
