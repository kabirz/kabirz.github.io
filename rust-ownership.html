01 概述

所有权 (Ownership) 是 Rust 的核心机制，通过编译期检查保证内存安全，无需 GC。这是 Rust 区别于其他语言的本质。

C/C++ 方式
  • 手动 new/delete
  • 双重释放、悬挂指针
  • 内存泄漏
  • use-after-free
  • 靠开发者纪律
GC 语言（Java/Go）
  • 运行时跟踪引用
  • stop-the-world 暂停
  • 不可预测延迟
  • 资源占用高
  • 难以做实时系统

Rust 的第三条路：

  • 编译期静态分析所有权转移
  • 无运行时开销
  • 无 GC
  • 确定性析构（RAII）
  • 并发安全（Send/Sync）

三大核心概念：

概念含义编译期检查
Ownership每个值有唯一所有者Move 时旧引用失效
Borrowing临时借用引用多个不可变 OR 一个可变
Lifetime引用的有效范围不能悬垂

02 所有权规则

Rust 所有权三大规则：

// 规则 1: 每个值有所有者（变量）
let s = String::from("hello");
// s 拥有 String 的内存

// 规则 2: 同一时刻只有一个所有者
let s1 = s;          // s1 接管所有权，s 失效
println!("{}", s);   // ❌ 编译错误：value borrowed after move

// 规则 3: 所有者离开作用域时自动 drop
{
    let s = String::from("hello");
}   // s 在这里 drop，内存释放

// RAII 模式（资源获取即初始化）
fn example() {
    let f = File::open("a.txt")?;   // 析构时关闭
    let mutex = Mutex::new(0);
    let _g = mutex.lock().unwrap();  // 离开作用域自动解锁
}

Copy 与 Drop 类型：

// Copy 类型（栈上复制，不需要所有权转移）：
// - 所有整数/浮点（i32, f64）
// - bool, char
// - 不含堆数据的 tuple
let a = 5;
let b = a;     // a 被复制（i32 是 Copy）
println!("{}", a);  // ✅ 仍可用

// 自定义 Copy：
#[derive(Clone, Copy)]
struct Point { x: i32, y: i32 }
// 必须：所有字段都是 Copy
// 不能：实现了 Drop

// Move 类型（堆数据）：
// - String, Vec, HashMap, Box
// - 自定义 struct 默认 Move
let s1 = String::from("a");
let s2 = s1;   // s1 move 到 s2

// 想复制：必须显式 clone
let s3 = s2.clone();

03 Move 语义

Move 是所有权转移，不是内存复制：

// 函数参数：move 进函数
fn take(s: String) {
    println!("{}", s);
}   // s 在这里 drop

let s = String::from("hello");
take(s);
println!("{}", s);   // ❌ 已被 move

// 想保留：传引用或 clone
fn borrow(s: &String) { println!("{}", s); }
fn with_clone(s: String) { ... }

let s = String::from("hello");
borrow(&s);          // ✅ 借用，s 仍可用
with_clone(s.clone()); // ✅ 克隆

// 返回值的所有权：
fn make() -> String {
    String::from("made")
}
let s = make();   // 所有权从函数 move 到调用方

// 函数返回引用（必须标注 lifetime）：
fn first_word(s: &str) -> &str {
    &s[0..5]
}

Move 在集合中的影响：

// Vec 内部是堆指针 + 长度 + 容量
let v1 = vec![1, 2, 3];
let v2 = v1;   // v1 整体 move
// v1 完全失效（不只是数据，连指针都失效）

// 引用 (&) 是 Copy（不拥有数据）
let v = vec![1, 2, 3];
let r1 = &v;
let r2 = &v;   // r1 仍然有效

// 部分移动：
let p = (String::from("a"), 5);
let s = p.0;   // p.0 被 move
// println!("{:?}", p);  // ❌ p 部分被 move
println!("{}", p.1);  // ✅ 未移动的部分仍可用

Move 在闭包中：

let s = String::from("hello");
let f = move || {           // move 关键字强制捕获所有权
    println!("{}", s);
};
// s 被 move 进闭包
// println!("{}", s);  // ❌

// 不带 move 的闭包：按需借用
let s = String::from("hello");
let f = || println!("{}", s);   // 借用 s
println!("{}", s);              // ✅ 仍可用
f();

04 借用与引用

借用 (Borrow) 通过引用 (&) 临时访问数据，不获取所有权：

// 不可变引用 &T
fn length(s: &String) -> usize { s.len() }
let s = String::from("hello");
let n = length(&s);   // 借用
println!("{}", s);     // ✅ 仍可用

// 可变引用 &mut T
fn push(s: &mut String) { s.push_str(" world"); }
let mut s = String::from("hello");
push(&mut s);
println!("{}", s);  // hello world

// 引用规则：
// 1. 任意数量的不可变引用
// 2. 或一个可变引用
// 3. 不能同时有

let mut s = String::from("hello");
let r1 = &s;
let r2 = &s;        // ✅ 多个不可变
let r3 = &mut s;    // ❌ 不能同时

let mut s = String::from("hello");
{
    let r1 = &mut s;
}   // r1 离开作用域
let r2 = &mut s;    // ✅ r1 已释放

// NLL (Non-Lexical Lifetimes)：
let mut s = String::from("hello");
let r1 = &s;
let r2 = &s;
println!("{} {}", r1, r2);   // r1, r2 最后一次使用
let r3 = &mut s;             // ✅ NLL 知道 r1/r2 不再使用

解引用与 Deref：

// * 操作符：解引用
let x = 5;
let r = &x;
assert_eq!(*r, 5);

// Deref trait：自定义解引用（智能指针）
use std::ops::Deref;

struct MyBox<T>(T);
impl<T> Deref for MyBox<T> {
    type Target = T;
    fn deref(&self) -> &Self::Target { &self.0 }
}

let x = MyBox(String::from("hi"));
// 自动 deref
fn print(s: &str) { println!("{}", s); }
print(&x);   // MyBox → String → str（多层 deref coercions）

// DerefMut：可变解引用
impl<T> DerefMut for MyBox<T> {
    fn deref_mut(&mut self) -> &mut Self::Target { &mut self.0 }
}

05 生命周期

生命周期 (Lifetime) 是编译期对引用有效范围的标注，防止悬垂引用：

// 悬垂引用示例：
{
    let r;
    {
        let x = 5;
        r = &x;       // ❌ 编译错误：x 生命周期短于 r
    }
    println!("{}", r);
}

// 函数返回引用（必须标注）：
// 这个引用的存活时间必须 ≥ 输入引用
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
// 'a 是生命周期参数
// 含义：返回引用的存活时间 = x 和 y 中较短的那个

// 结构体中的引用：
struct Excerpt<'a> {
    part: &'a str,
}
let novel = String::from("call me Ishmael...");
let first_sentence;
{
    let words = novel.as_str();
    let ex = Excerpt { part: &words[0..4] };
    first_sentence = ex.part;
}   // ❌ ex 的 part 需要比 words 活得久

// 'static 生命周期：
// 存活整个程序期间
let s: &'static str = "hello";   // 字符串字面量是 'static
fn static_fn() -> &'static str { "hello" }

生命周期可视化：

// 函数签名：
// fn foo<'a, 'b>(x: &'a i32, y: &'b i32) -> &'a i32
//     │          │              │              │
//     │          │              │              └─ 返回值寿命 >= a
//     │          │              └─ 第二个参数寿命 b（独立）
//     │          └─ 第一个参数寿命 a
//     └─ 生命周期参数声明

// 函数体中编译器追踪：
//   - 每个引用的"出生点"和"最后使用点"
//   - 确保返回的引用在所有路径上都 valid
//   - 失败 → E0597 / E0106 等编译错误

// 借用检查器（Borrow Checker）：
//   compiler/rustc_borrowck
//   基于 MIR (Mid-level IR) 做流敏感分析

06 生命周期省略

为减少样板代码，编译器在三种规则下自动推导生命周期：

// 三条省略规则：

// 规则 1: 每个引用参数自动获得独立 lifetime
fn foo(x: &i32)  ⟶  fn foo<'a>(x: &'a i32)
fn foo(x: &i32, y: &i32)  ⟶  fn foo<'a, 'b>(x: &'a i32, y: &'b i32)

// 规则 2: 如果只有一个输入 lifetime，赋给所有输出
fn foo(x: &i32) -> &i32  ⟶  fn foo<'a>(x: &'a i32) -> &'a i32

// 规则 3: 如果有 &self 或 &mut self，self 的 lifetime 赋给所有输出
fn foo(&self, x: &i32) -> &i32
   ⟶  fn foo<'a, 'b>(&'a self, x: &'b i32) -> &'a i32

// 不满足任何规则 → 必须显式标注
fn longest(x: &str, y: &str) -> &str  // ❌ 编译错误
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str  // ✅

// 例子：
fn first_word(s: &str) -> &str { ... }   // 应用规则 1+2
struct Parser { ... }
impl Parser {
    fn parse(&self, input: &str) -> &str { ... }  // 规则 1+3
}

Lifetime in HRTB（Higher-Ranked Trait Bounds）：

// for<'a> 表示"对所有 lifetime 'a"
// 常见于闭包类型：

// 等价：
fn f(x: &i32) -> &i32 { x }
let _: fn(_) -> _ = f;  // 类型推断

// 显式 HRTB：
fn apply<F>(f: F) where F: for<'a> Fn(&'a str) -> &'a str {}

// 闭包类型自动有 HRTB
let f = |x: &str| x;
apply(f);

07 协变与逆变

Variance 决定生命周期之间的子类型关系：

// 'static: 'a (static 是任何 'a 的子类型)
// 即：'static 可以替代任何 'a

// 三种 Variance：
// - 协变 (covariant):     &'a T  -- 'a 长 → 可替代 'a 短
// - 逆变 (contravariant): fn(T)  -- 参数相反
// - 不变 (invariant):     &'a mut T -- 必须严格匹配

// 协变（最常见）：
fn covariant() {
    let s: &'static str = "hi";
    let r: &'a str = s;   // ✅ 'static 可替代 'a
}

// 逆变（函数参数）：
fn contravariant(f1: fn(&'static str)) {
    let f2: fn(&'a str) = ...;
    f1 = f2;   // ✅ 接收更短 lifetime 的函数可替代
}

// 不变（&mut T）：
fn invariant() {
    let mut x: &'static str = "hi";
    let y: &'a str = ...;
    x = y;   // ❌ &mut 严格匹配，不能放宽
}

// Variance 表：
//   &'a T          covariant in 'a and T
//   &'a mut T      covariant in 'a, invariant in T
//   Box<T>         covariant in T
//   Vec<T>          covariant in T
//   UnsafeCell<T>   invariant in T
//   fn(T) -> R     contravariant in T, covariant in R
//   *mut T         invariant

08 内部可变性

正常规则：不可变引用 → 不能改数据。内部可变性模式打破此规则，通过运行时检查实现安全修改：

// RefCell<T> — 单线程内部可变性
use std::cell::RefCell;

let cell = RefCell::new(5);
{
    let mut b = cell.borrow_mut();   // 运行时借用
    *b += 1;
}   // borrow 释放
{
    let b1 = cell.borrow();
    let b2 = cell.borrow_mut();  // ❌ panic! 违反借用规则
}

// Cell<T> — Copy 类型的内部可变性
use std::cell::Cell;
let c = Cell::new(5);
let v = c.get();
c.set(10);

// Mutex<T> — 多线程内部可变性
use std::sync::Mutex;
let m = Mutex::new(5);
{
    let mut g = m.lock().unwrap();
    *g += 1;
}

// RwLock<T> — 读多写少
use std::sync::RwLock;
let lock = RwLock::new(5);
{
    let r1 = lock.read().unwrap();
    let r2 = lock.read().unwrap();   // ✅ 多读
}
{
    let w = lock.write().unwrap();
    // 独占写
}

选择指南：

类型何时用检查
&T / &mut T默认编译期
Cell<T>Copy 类型无（编译期保证）
RefCell<T>单线程复杂借用运行时
Mutex<T>多线程运行时（阻塞）
RwLock<T>多线程读多运行时（阻塞）
AtomicXxx多线程简单值硬件原子

09 智能指针

Rust 智能指针通过 Deref + Drop 实现 RAII 资源管理：

// Box<T> — 堆分配
let b = Box::new(5);   // 在堆上分配
// 适合：
//   - 递归类型（链表、树）
//   - 大数据避免栈拷贝
//   - trait 对象（Box<dyn Trait>）

// Rc<T> — 单线程引用计数
use std::rc::Rc;
let r1 = Rc::new(String::from("hi"));
let r2 = Rc::clone(&r1);   // 引用计数 +1
let r3 = Rc::clone(&r1);   // 引用计数 = 3
// 最后一个 drop 时释放

// Arc<T> — 原子引用计数（多线程）
use std::sync::Arc;
let a = Arc::new(vec![1, 2, 3]);
let a2 = Arc::clone(&a);
std::thread::spawn(move || {
    println!("{:?}", a2);
});

// RefCell<T> — 单线程内部可变性
// 配合 Rc 形成可变共享：
use std::rc::{Rc, Weak};
use std::cell::RefCell;
let shared = Rc::new(RefCell::new(0));
let s1 = Rc::clone(&shared);
let s2 = Rc::clone(&shared);
std::thread::spawn(move || {
    *s1.borrow_mut() = 5;   // ❌ Rc 跨线程！
});

// Arc<Mutex<T>> — 多线程共享可变
let shared = Arc::new(Mutex::new(0));
let s = Arc::clone(&shared);
std::thread::spawn(move || {
    *s.lock().unwrap() = 5;
});

// Weak<T> — 弱引用（避免循环）
use std::rc::{Rc, Weak};
let strong = Rc::new(5);
let weak: Weak<_> = Rc::downgrade(&strong);
if let Some(v) = weak.upgrade() {
    println!("{}", v);
}

// Cow<T> — 写时复制
use std::borrow::Cow;
fn process(input: &str) -> Cow<str> {
    if input.contains("bad") {
        Cow::Owned(input.replace("bad", "good"))
    } else {
        Cow::Borrowed(input)
    }
}

10 底层代码生成

所有权在底层生成什么代码？

// 编译期 Rust：
//   源代码 → HIR → MIR → LLVM IR → 机器码

// MIR (Mid-level IR) 是 borrow checker 工作的 IR
// 例子：
fn example() {
    let s = String::from("a");
    let r = &s;
    println!("{}", r);
}

// MIR（简化）：
//   let mut _0: ();
//   let _1: String;
//   let _2: &String;
//   
//   bb0: {
//       _1 = String::const("a");
//       _2 = &_1;
//       ... (println 调用)
//       drop(_1);          // 编译器插入 drop
//       return;
//   }

// 借用检查在 MIR 上做：
//   构建每个变量的"lifetime region"
//   流敏感地分析每个借用
//   确保引用不会比数据活得长

// LLVM IR：
//   - 引用变成普通指针
//   - Drop 变成函数调用
//   - 无 GC、无 runtime 检查
//   - 接近手写 C++ 性能

// 例：MIR → LLVM IR（简化）
//   %s = alloca %String
//   call void @String_new(%String* %s)
//   %r = alloca %String*
//   store %String* %s, %String** %r
//   ; println!
//   call void @drop_in_place(%String* %s)
//   ret void

// 零成本抽象验证：
//   1. fn length(s: &String) → 与 C 函数指针等价
//   2. Vec<T> 操作 → C 数组操作
//   3. Iterator chain → 编译期展开为循环
//   4. Box → 等价 malloc/free

借用检查器演进：

// 2015 (1.0): 词法 lifetime（按作用域）
//   fn example() {
//       let mut s = String::from("a");
//       let r = &s;
//       // r 在作用域结束才失效
//       s.push_str("b");  // ❌
//   }

// 2018 (NLL): 非词法 lifetime
//   按使用点算 lifetime
//   let mut s = String::from("a");
//   let r = &s;
//   println!("{}", r);  // 最后使用 r
//   s.push_str("b");    // ✅ NLL 允许

// 2024+ (Polonius): 新一代 borrow checker
//   更智能，支持更多模式
//   处理之前需要 unsafe 的场景
所有权系统总结:
1. 无运行时开销：所有检查在编译期完成
2. 无数据竞争：所有权规则 + Send/Sync 保证
3. 确定性析构：RAII 自动管理资源
4. 无内存泄漏：循环引用需显式 Weak 处理
5. 并发安全：编译期阻止 data race
代价：学习曲线陡，但回报是编程级安全保证。
