01 概述

内核模块（kernel module）是可动态加载和卸载的内核代码。驱动程序、文件系统、网络协议等通常以内核模块形式实现，无需重新编译整个内核。模块通过 insmod/modprobe 加载，通过 rmmod 卸载。

02 模块生命周期

内核模块完整生命周期：

  insmod mymodule.ko
        │
        ├── 1. 分配内存，读取 ELF 文件
        │
        ├── 2. 解析符号引用（resolve symbols）
        │       找到内核导出的函数地址
        │
        ├── 3. 重定位代码和数据
        │
        ├── 4. 调用 init 函数
        │       module_init(my_init)
        │       → my_init() 执行初始化
        │
        └── 5. 模块注册成功，内核功能扩展

  模块运行中...
  │
  │  模块注册的函数被内核调用
  │  如：驱动 probe、网络协议、文件系统
  │
  ↓

  rmmod mymodule
        │
        ├── 1. 检查模块引用计数
        │       module_refcount == 0 → 可以卸载
        │       module_refcount > 0 → 拒绝（有用户在使用）
        │
        ├── 2. 调用 exit 函数
        │       module_exit(my_exit)
        │       → my_exit() 执行清理
        │
        ├── 3. 注销模块注册的所有功能
        │       驱动注销、网络协议移除等
        │
        └── 4. 释放模块内存

03 模块 API

// 最简单的内核模块
#include <linux/module.h>
#include <linux/kernel.h>

static int __init my_init(void)
{
    printk(KERN_INFO "mymodule loaded\n");
    return 0;
}

static void __exit my_exit(void)
{
    printk(KERN_INFO "mymodule unloaded\n");
}

module_init(my_init);
module_exit(my_exit);
MODULE_LICENSE("GPL");
MODULE_AUTHOR("Author");
MODULE_DESCRIPTION("A simple module");

关键 API

API功能
module_init(func)注册模块初始化函数
module_exit(func)注册模块清理函数
try_module_get(mod)增加模块引用计数
module_put(mod)减少模块引用计数
EXPORT_SYMBOL_GPL(sym)导出符号供其他模块使用

04 编译与加载

# Makefile
obj-m := mymodule.o
mymodule-objs := main.o helper.o

# 编译
$ make -C /lib/modules/$(uname -r)/build M=$(pwd) modules

# 加载/卸载
$ sudo insmod mymodule.ko      # 加载
$ sudo modprobe mymodule       # 加载（自动处理依赖）
$ sudo rmmod mymodule          # 卸载
$ sudo modprobe -r mymodule    # 卸载（自动处理依赖）

# 查看已加载模块
$ lsmod
$ lsmod | grep mymodule

# 查看模块信息
$ modinfo mymodule.ko

# 查看模块日志
$ dmesg | tail -5
