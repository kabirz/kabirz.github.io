01 概述

消息队列是 Linux 的异步消息传递 IPC 机制。与管道（字节流）不同，消息队列传递的是有边界的消息——发送方发送固定大小的消息，接收方按消息边界读取。Linux 支持 System V 消息队列（传统）和 POSIX 消息队列（推荐）两种实现。

02 两种消息队列

System V 消息队列（传统）：

  key = ftok("/tmp", 'a');
  msqid = msgget(key, IPC_CREAT | 0666);

  // 发送
  struct msgbuf { long mtype; char mtext[256]; };
  msgsnd(msqid, &msg, len, 0);

  // 接收
  msgrcv(msqid, &msg, 256, type, 0);

POSIX 消息队列（推荐）：

  mqd_t mq = mq_open("/myqueue", O_CREAT | O_RDWR, 0644, &attr);

  // 发送
  mq_send(mq, "hello", 5, 0);

  // 接收
  char buf[256];
  mq_receive(mq, buf, sizeof(buf), NULL);

对比：

  System V: 基于 key + msqid, 传统接口
  POSIX:    基于文件路径, 标准接口, 支持优先级

03 发送接收流程

消息队列发送流程：

  发送方调用 msgsnd/mq_send
        │
        ├── 1. 检查消息大小是否超过限制
        │
        ├── 2. 检查队列是否已满
        │       已满 → 阻塞或返回 EAGAIN
        │
        ├── 3. 分配消息缓冲区
        │       从内核 slab 分配
        │
        ├── 4. 复制消息数据
        │       copy_from_user()
        │
        ├── 5. 添加到队列尾部
        │       链表操作
        │
        └── 6. 唤醒等待接收的进程
            wake_up_interruptible()

消息队列接收流程：

  接收方调用 msgrcv/mq_receive
        │
        ├── 1. 检查队列中是否有匹配的消息
        │       type 匹配（System V）
        │
        ├── 2. 有匹配消息 → 取出并返回
        │
        └── 3. 无匹配消息 → 阻塞等待
            wait_event_interruptible()

共享内存原始内存双向需手动删除最高
