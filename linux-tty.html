01 概述

TTY（Teletypewriter）是 Linux 的终端子系统，提供字符设备接口用于终端 I/O。它支持物理串口（/dev/ttyS0）、虚拟终端（/dev/tty1）、伪终端（/dev/pts/0）等多种终端类型。SSH、Docker exec、screen/tmux 都依赖 TTY。

02 TTY 类型

类型设备说明
控制台/dev/console内核消息输出
虚拟终端/dev/tty1 ~ /dev/tty63Ctrl+Alt+F1 切换
串口终端/dev/ttyS0, /dev/ttyAMA0物理串口连接
伪终端（PTY）/dev/pts/0, /dev/pts/1SSH、screen、tmux
当前终端/dev/tty当前进程的控制终端
当前控制终端/dev/ctty会话的控制终端

PTY（伪终端）工作原理

  伪终端是 SSH、screen 的核心机制：

  SSH 客户端              SSH 服务器
  │                        │
  ├── ssh user@host        │
  │   → 打开 /dev/ptmx    │
  │   → 分配 /dev/pts/0    │
  │                        │
  ├── read(pty_master)     ├── fork()
  │   ← SSH 服务器输出     │   → 子进程 exec bash
  │                        │   → bash 使用 /dev/pts/0
  ├── write(pty_master)    │
  │   → SSH 客户端输入     │   ← bash 读取 /dev/pts/0
  │                        │
  pty_master ←──→ pty_slave
  （SSH 客户端）  （bash 进程）

04 应用场景

# 查看当前 TTY
$ tty
/dev/pts/0

# 查看所有终端
$ ls /dev/tty*
$ ls /dev/pts/

# 查看串口设备
$ ls /dev/ttyS*
$ ls /dev/ttyUSB*    # USB 串口转换器
$ ls /dev/ttyACM*    # USB CDC ACM 设备

# 修改终端属性
$ stty -a            # 查看终端设置
$ stty speed 115200  # 设置波特率

# screen/tmux 使用 PTY
$ screen /dev/ttyS0   # 连接串口
$ tmux new -s work    # 创建伪终端会话
