01 概述

fanotify（File Access Notify）是 Linux 内核提供的高级文件系统事件通知机制。与 inotify 不同，fanotify 可以监控整个挂载点、过滤特定进程、并且支持事件响应（如杀毒软件在文件打开前扫描）。它是 ClamAV、CrowdStrike 等安全产品的内核接口。

02 fanotify vs inotify

特性inotifyfanotify
监控粒度单个文件/目录整个挂载点
权限需要读权限CAP_SYS_ADMIN
事件响应仅通知FAN_ALLOW/FAN_DENY
PID 过滤不支持支持（仅监控特定进程）
文件信息文件名fid + file handle
性能快略慢
典型用途IDE 文件监视杀毒/DLP/完整性检查

03 API 与使用

// fanotify API
#include <sys/fanotify.h>

// 1. 创建 fanotify 实例
int fd = fanotify_init(
    FAN_CLASS_CONTENT | FAN_NONBLOCK, // 标志
    O_RDONLY);                         // 操作权限

// 2. 添加监控（监控整个 /home 目录）
struct fanotify_mark_data mark = {
    .fd = AT_FDCWD,
    .mask = FAN_OPEN | FAN_ACCESS | FAN_MODIFY,
    .ignore_mask = 0,
    .path = "/home",
};
fanotify_mark(fd, FAN_MARK_ADD | FAN_MARK_MOUNT, ...);

// 3. 读取事件
struct fanotify_event_metadata event;
read(fd, &event, sizeof(event));
// event.mask = 事件类型（FAN_OPEN 等）
// event.pid = 触发进程的 PID
// event.fd = 文件描述符（可读取文件内容）

// 4. 响应事件（权限模式）
struct fanotify_response resp = {
    .fd = event.fd,
    .response = FAN_ALLOW, // 或 FAN_DENY
};
write(fd, &resp, sizeof(resp));

04 事件类型

事件说明
FAN_OPEN文件被打开
FAN_CLOSE_WRITE可写文件被关闭
FAN_CLOSE_NOWRITE只读文件被关闭
FAN_ACCESS文件被访问
FAN_MODIFY文件被修改
FAN_CREATE文件被创建
FAN_DELETE文件被删除
FAN_MOVED_FROM文件被移出
FAN_MOVED_TO文件被移入
