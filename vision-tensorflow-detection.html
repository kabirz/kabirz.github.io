01 TF Object Detection API 概述

TensorFlow Object Detection API 是 Google 开源的目标检测框架，基于 TensorFlow 2.x 构建。它提供了完整的训练、评估和部署流水线，支持 SSD、Faster R-CNN、EfficientDet 等多种检测架构。

整体架构

├── research/object_detection/
│   ├── model_main_tf2.py         主训练/评估入口
│   ├── exporter_main_v2.py       模型导出
│   ├── configs/tf2/              pipeline 配置文件
│   ├── models/                   检测模型实现
│   │   ├── ssd/                  SSD 系列
│   │   ├── faster_rcnn/          Faster R-CNN 系列
│   │   ├── efficientdet/        EfficientDet 系列
│   │   └── center_net/           CenterNet 系列
│   ├── datalab/                 数据标注工具
│   ├── inference/               推理工具
│   └── utils/                   工具函数

关键概念

TF Object Detection API 将目标检测任务分解为以下组件：

  • 特征提取器 — 通常为预训练图像分类网络（ResNet、MobileNet、EfficientNet）
  • 检测头 — 在特征图上预测边界框和类别概率
  • 锚框生成器 — 生成预定义的参考框
  • 后处理 — NMS、得分过滤、类别映射
  • 损失函数 — 定位损失（Smooth L1）+ 分类损失（Softmax/Focal Loss）
工作流程:
配置 pipeline.config 准备 TFRecord 数据 model_main_tf2.py 训练 exporter_main_v2.py 导出 TensorFlow Serving / TF Lite 部署。

03 模型选择

TF Object Detection API 提供多种预训练模型，在精度（mAP）和速度（FPS）之间提供不同权衡方案。

模型系列对比

# SSD MobileNet -- 轻量级，适合移动端
#   ssd_mobilenet_v2_fpnlite_320x320    22 mAP
#   ssd_mobilenet_v2_fpnlite_640x640    28 mAP

# SSD ResNet -- 精度与速度平衡
#   ssd_resnet50_v1_fpn_640x640         34 mAP
#   ssd_resnet101_v1_fpn_640x640        36 mAP

# EfficientDet -- 高效多尺度特征融合
#   efficientdet_d0_coco17_tpu-32       34 mAP
#   efficientdet_d1_coco17_tpu-32       37 mAP
#   efficientdet_d4_coco17_tpu-32       44 mAP
#   efficientdet_d7_coco17_tpu-32       52 mAP

# Faster R-CNN -- 高精度，速度较慢
#   faster_rcnn_resnet50_v1_640x640     37 mAP
#   faster_rcnn_resnet101_v1_640x640    39 mAP
#   faster_rcnn_inception_resnet_v2     43 mAP

# CenterNet -- 无锚框检测
#   centernet_hourglass_512x512         41 mAP
#   centernet_resnet50_v1_fpn_512x512   31 mAP

模型选择指南

选择标准取决于部署环境和精度要求。部署环境决定速度需求，目标特征决定模型复杂度。

# 选择标准：
# 1. 部署环境：移动端 SSD MobileNet
#              Web/边缘 EfficientDet-D0~D2
#              服务器 Faster R-CNN / EfficientDet-D4+

# 2. 精度要求：高精度 Faster R-CNN / EfficientDet-D7
#              中等 SSD ResNet / EfficientDet-D2

# 3. 输入分辨率：
#   320x320 快速但对小目标不友好
#   640x640 较好的综合平衡
#   1024x1024 高精度但计算量大

# 4. 下载预训练模型
wget http://download.tensorflow.org/models/object_detection/\
tf2/ssd_mobilenet_v2_fpnlite_320x320_coco17_tpu-8.tar.gz
tar -xzf ssd_mobilenet_v2_fpnlite_320x320_coco17_tpu-8.tar.gz
推荐搭配:
快速原型：SSD MobileNet V2 FPNLite 320x320。
生产部署：EfficientDet-D2 640x640。
学术竞赛：Faster R-CNN ResNet-152 + 多尺度测试。
移动端：SSD MobileNet V2 FPNLite 320x320 + TF Lite 量化。

04 pipeline.config 详解

pipeline.config 是 TF Object Detection API 的核心配置文件，采用 protobuf 文本格式定义整个训练和评估流程。所有训练参数、数据路径、模型架构和后处理设置都在此文件集中管理。

模型配置

model {
  ssd {
    feature_extractor {
      type: "ssd_mobilenet_v2_fpn_keras"
      depth_multiplier: 1.0
      min_depth: 16
      conv_hyperparams {
        regularizer { l2_regularizer { weight: 4e-5 }}
        initializer { random_normal_initializer { stddev: 0.01 }}
        activation: RELU_6
        batch_norm { decay: 0.997, center: true, scale: true }
      }
      use_depthwise: true
    }

    anchor_generator {
      ssd_anchor_generator {
        num_layers: 6
        min_scale: 0.2
        max_scale: 0.95
        aspect_ratios: [1.0, 2.0, 0.5, 3.0, 0.33]
      }
    }

    box_predictor {
      convolutional_box_predictor {
        conv_hyperparams { ... }
        use_dropout: false
        kernel_size: 3
      }
    }

    loss {
      localization_loss { weighted_smooth_l1 { delta: 1.0 }}
      classification_loss { weighted_softmax {} }
      hard_example_miner {
        num_hard_examples: 64
        iou_threshold: 0.99
        loss_type: BOTH
        max_negatives_per_positive: 3
        min_negatives_per_image: 0
      }
      classification_weight: 1.0
      localization_weight: 1.0
    }

    post_processing {
      batch_non_max_suppression {
        score_threshold: 1e-8
        iou_threshold: 0.6
        max_detections_per_class: 100
        max_total_detections: 100
      }
      score_converter: SOFTMAX
    }
  }
}

关键参数说明:
fine_tune_checkpoint_type 需与预训练模型匹配（detection / classification）。
batch_size 受 GPU 显存限制，A100 80G 可支持 64，V100 32G 建议 16-32。
num_steps 建议 SSD 系列 100K-200K，Faster R-CNN 系列 200K-500K。

06 迁移学习

从 COCO 预训练模型微调到自定义数据集是目标检测中最常用的实践。微调可以显著减少训练时间并提高精度，尤其在数据量有限时。

微调配置

# pipeline.config 关键修改：

# 1. 修改类别数
model {
  ssd {
    num_classes: 5   # 改为自定义数据集类别数
  }
}

# 2. 设置预训练检查点
train_config {
  fine_tune_checkpoint: "ssd_mobilenet_v2_fpnlite_\
    320x320_coco17_tpu-8/checkpoint/ckpt-0"
  fine_tune_checkpoint_version: V2
  fine_tune_checkpoint_type: "detection"
}

# 3. 调整学习率
train_config {
  optimizer {
    momentum_optimizer {
      learning_rate {
        cosine_decay_learning_rate {
          learning_rate_base: 0.01
          warmup_learning_rate: 0.003
          warmup_steps: 500
          total_steps: 50000
        }
      }
    }
  }
  num_steps: 50000
}

微调策略

# 策略一：冻结特征提取器
train_config {
  freeze_variables: "FeatureExtractor"
}

# 策略二：全部微调（推荐）
# 不设置 freeze_variables，用较低学习率训练所有权重

# 策略三：分阶段微调
# 阶段 1：只训练分类头
# 阶段 2：解冻部分 backbone 层
# 阶段 3：全部微调

07 模型评估

目标检测模型评估使用 COCO 标准指标，包括不同 IoU 阈值和物体尺寸下的平均精度（mAP）。

COCO 评估指标详解

# 主要指标：
#   Average Precision (AP) -- IoU=0.50:0.95 的均值
#   AP_50 -- IoU=0.50 时的 AP
#   AP_75 -- IoU=0.75 时的 AP（更严格）
#
# 按物体大小：
#   AP_small -- 面积 < 32^2 像素
#   AP_medium -- 面积 32^2 ~ 96^2
#   AP_large -- 面积 > 96^2
#
# 召回率：
#   AR_max=1 -- 每张图像最多 1 个检测时的召回率
#   AR_max=10 -- 每张图像最多 10 个检测
#   AR_max=100 -- 每张图像最多 100 个检测
#
# 典型 COCO mAP 参考：
# SSD MobileNet V2 FPNLite 320: 22.3 mAP
# SSD ResNet-50 FPN:           34.8 mAP
# EfficientDet-D2:             37.0 mAP
# EfficientDet-D7:             53.7 mAP
# Faster R-CNN ResNet-101:     39.5 mAP

可视化评估结果

import matplotlib.pyplot as plt
import matplotlib.patches as patches

def visualize_detections(image, boxes, scores, classes,
                         category_index, threshold=0.5):
    fig, ax = plt.subplots(1, figsize=(10, 10))
    ax.imshow(image)

    for i in range(len(boxes)):
        if scores[i] < threshold:
            continue
        ymin, xmin, ymax, xmax = boxes[i]
        h, w = image.shape[:2]
        xmin, xmax = xmin * w, xmax * w
        ymin, ymax = ymin * h, ymax * h

        rect = patches.Rectangle(
            (xmin, ymin), xmax - xmin, ymax - ymin,
            linewidth=2, edgecolor="r", facecolor="none"
        )
        ax.add_patch(rect)
        ax.text(xmin, ymin - 5,
                f"{category_index[classes[i]]['name']}: "
                f"{scores[i]:.2f}",
                color="white", fontsize=12,
                bbox=dict(facecolor="red", alpha=0.7))

    plt.axis("off")
    plt.show()

    "exported_model/saved_model"

推理代码

import numpy as np
import tensorflow as tf

class ObjectDetector:
    def __init__(self, model_path):
        self.model = tf.saved_model.load(model_path)
        self.infer = self.model.signatures["serving_default"]

    def detect(self, image_np):
        input_tensor = tf.convert_to_tensor(
            np.expand_dims(image_np, 0), dtype=tf.uint8
        )
        detections = self.infer(input_tensor)

        return {
            "boxes": detections["detection_boxes"][0].numpy(),
            "scores": detections["detection_scores"][0].numpy(),
            "classes": detections["detection_classes"][0].numpy()
                       .astype(np.int32),
            "num": int(
                detections["num_detections"][0].numpy()
            ),
        }

# 使用
detector = ObjectDetector("exported_model/saved_model")
result = detector.detect(cv2.imread("image.jpg"))
print(f"Detected {result['num']} objects")
