01 概述

Docker 不是单一进程，而是一组协作的组件。从 Docker 1.11（2016）开始架构演进为三层模型：

┌─────────────────────────────────────────┐
│   docker CLI                            │
│   /usr/bin/docker                       │
└────────────────┬────────────────────────┘
                 │ REST API (/var/run/docker.sock)
┌────────────────▼────────────────────────┐
│   dockerd (Docker Daemon)               │
│   - HTTP API server                     │
│   - 镜像管理（layer store）              │
│   - Volume/Network 逻辑                 │
│   - BuildKit 调度                       │
└────────────────┬────────────────────────┘
                 │ gRPC (Unix socket)
┌────────────────▼────────────────────────┐
│   containerd                           │
│   - 容器生命周期管理                     │
│   - 镜像分发（distribution）             │
│   - snapshot 管理                       │
│   - containerd-shim 启动                │
└────────────────┬────────────────────────┘
                 │ execve
┌────────────────▼────────────────────────┐
│   containerd-shim (每容器一个)           │
│   - runc 调用                           │
│   - 收集容器退出码                       │
│   - 守护 runc（runc 是一次性的）          │
└────────────────┬────────────────────────┘
                 │ execve
┌────────────────▼────────────────────────┐
│   runc                                  │
│   - 创建 namespace/cgroup               │
│   - 调用 OCI runtime spec               │
│   - 启动容器进程                         │
└─────────────────────────────────────────┘

历史演进：

版本架构关键变化

Docker 1.11+引入 containerd+runcOCI runtime 标准

Docker 18.06+BuildKit 默认并行构建

02 组件分工

// 1. dockerd（moby/daemon）
//    二进制：/usr/bin/dockerd
//    Socket：/var/run/docker.sock
//    工作目录：/var/lib/docker/
//    
//    职责：
//    - HTTP/HTTPS API server
//    - REST API（v1.xx）兼容
//    - 镜像层管理（image store）
//    - 网络驱动（bridge/overlay/...）
//    - Volume 驱动
//    - 镜像构建（BuildKit daemon）
//    - 安全策略（seccomp、AppArmor、cap）
//    - 事件聚合

// 2. containerd
//    二进制：/usr/bin/containerd
//    Socket：/run/containerd/containerd.sock
//    工作目录：/var/lib/containerd/
//    
//    职责：
//    - 容器生命周期（创建、启动、停止、删除）
//    - 镜像 pull/push（content store）
//    - 快照管理（snapshotter）
//    - 任务（task）管理
//    - containerd-shim 守护
//    - 事件、命名空间、租约（lease）

// 3. containerd-shim
//    二进制：/usr/bin/containerd-shim-runc-v2
//    每个容器一个 shim 进程
//    
//    职责：
//    - 启动 runc create
//    - 持有容器 stdin/stdout/stderr
//    - 监控容器进程，记录退出码
//    - 让 dockerd 重启不影响容器
//    - resize TTY
//    - exec 子进程

// 4. runc
//    二进制：/usr/bin/runc
//    
//    职责：
//    - 读取 OCI runtime spec (config.json)
//    - 创建 namespaces
//    - 设置 cgroups
//    - 准备 rootfs (mount, pivot_root)
//    - 启动容器进程
//    - 退出（让 shim 接管）

05 创建容器全链路

// docker run nginx

// 1. CLI 调用 dockerd HTTP API
//    POST /v1.45/containers/create
//    body: { Image: "nginx", HostConfig: {...} }

// 2. dockerd 处理（daemon/server/router_routes.go）
//    - 解析 Image name → 解析为 reference
//    - 查找本地镜像
//    - 如果不存在 → 自动 pull
//    - 生成容器配置
//    - 计算容器 ID (随机 SHA256)
//    - 在 image store 创建记录
//    
//    关键步骤：
//    a. m.Create(client.ContainerCreate, options)  // daemon/create.go
//    b. daemon.createContainer(config)             // 创建 container 对象
//    c. daemon.registerName(name, container)       // 注册名字
//    d. daemon.Register(container)                 // 注册到 state
//    
//    内部：container.ToDisk() 持久化配置到
//    /var/lib/docker/containers/<id>/config.v2.json

// 3. dockerd 调用 containerd
//    dockerd → containerd.ContainerService.Create
//    gRPC request:
//    {
//      id: "<container_id>",
//      image: "docker.io/library/nginx:latest",
//      runtime: "io.containerd.runc.v2",
//      spec: <OCI spec>,  // 生成 OCI runtime spec
//      rootfs: {
//        type: "layers",
//        diff_ids: [...]
//      },
//      snapshotter: "overlayfs"
//    }
//    
//    containerd 处理：
//    a. 解析 spec
//    b. 调用 snapshotter.Prepare()
//       → 创建 overlayfs mountpoint
//       → /var/lib/containerd/io.containerd.snapshotter.v1.overlayfs/...
//    c. 持久化容器元数据
//    d. 返回 container id

// 4. 用户调用 start
//    POST /containers/{id}/start

// 5. dockerd 调用 containerd.TaskService.Create
//    gRPC request:
//    {
//      container_id: "<container_id>",
//      rootfs: { mounts: [...] },         // 来自 snapshotter
//      terminal: false,
//      stdin: "/dev/null",
//      stdout: "/var/lib/.../stdout",
//      stderr: "/var/lib/.../stderr"
//    }
//    
//    containerd 处理：
//    a. 启动 containerd-shim-runc-v2（独立进程）
//    b. 通过 shim Create RPC
//       → shim 调用 runc binary
//       → runc create <id>
//       → 容器进程已创建（处于 created 状态）

// 6. dockerd 调用 containerd.TaskService.Start
//    gRPC request: { container_id: ... }
//    
//    containerd 转发到 shim:
//    a. shim.Start RPC
//    b. shim 调用 runc start <id>
//    c. 容器进程真正执行 ENTRYPOINT

// 7. 容器退出
//    runc 已退出（创建后立即退出）
//    shim 守护容器，wait() 容器进程
//    容器退出 → shim 通过 gRPC 通知 containerd
//    containerd → publish event → dockerd
//    dockerd → 更新容器状态 → 触发事件给用户

06 启动容器全链路

// 深入看 containerd → shim → runc

// 1. dockerd 调用 containerd.Create (Task)
//    containerd/internal/oci/task.go
func (l *local) Create(ctx context.Context, r *CreateTaskRequest, ...) {
    container, _ := l.getContainer(r.ContainerID)
    return container.NewTask(ctx, ...)
}

// container.NewTask（containerd/internal/oci/spec.go）：
func (c *container) NewTask(ctx, ...) {
    // 1. 调用 snapshotter.Prepare
    mounts, err := c.runtime.snapshotter.Prepare(ctx, key, c.Image.Target.Digest)
    
    // 2. 设置 rootfs mount
    s.Mounts = mounts
    
    // 3. 调用 runtime（io.containerd.runc.v2）
    task, err := c.runtime.Create(ctx, c.ID, s)
    // c.runtime 是 io.containerd.runc.v2.Runtime
    // 它会启动 shim
}

// 2. shim 启动（containerd/runtime/v2/runc/runtime.go）
func (r *Runtime) Create(ctx, id, opts) (Task, error) {
    // 1. 启动 shim 进程
    shim, err := r.startShim(ctx, bundle, id, opts)
    
    // 2. 通过 ttrpc 调用 shim.Create
    return shim.Create(ctx, opts)
}

// 3. shim 进程
//    containerd-shim-runc-v2 -namespace ... -id ... -address ...
//    
//    shim 二进制 main()：
//    - 创建 ttrpc server
//    - 注册 task service
//    - 监听 Unix socket

// 4. shim.Create（runtime/v2/runc/task/service.go）
func (s *service) Create(ctx, req) error {
    // 准备 OCI spec 文件
    // 调用 runc 命令
    cmd := exec.Command("runc", "create", id)
    cmd.Stdin = ...
    err := cmd.Run()
    // runc create 后容器进程已存在（suspended）
}

// 5. shim.Start
func (s *service) Start(ctx, req) error {
    cmd := exec.Command("runc", "start", id)
    return cmd.Run()
}

// 6. runc create 内部（libcontainer/container_linux.go）
func (c *linuxContainer) Start(process *Process) error {
    // 1. 创建 cgroup 路径
    // 2. 设置 namespaces (clone with CLONE_NEW*)
    // 3. mount rootfs
    // 4. pivot_root
    // 5. setup networking
    // 6. 执行 ENTRYPOINT
    return c.start(process)
}

// 整个调用链：
// CLI → HTTP API → dockerd → gRPC → containerd 
//      → shim spawn → runc exec → kernel namespace/cgroup

// docker pull nginx

// 1. dockerd 接收
POST /images/create?fromImage=nginx&tag=latest

// 2. dockerd 解析：
//    nginx → docker.io/library/nginx:latest
//    （默认添加 docker.io/library/ 前缀）
//    
//    镜像引用格式：
//    docker.io/library/nginx:latest
//    registry / namespace / repo : tag

// 3. dockerd 委托 containerd pull（如果 containerd image store 模式）
//    containerd.PullImage():
//    a. resolver 解析 registry
//    b. HTTP 请求 GET /v2/library/nginx/manifests/latest
//    c. 获取 image manifest
//    d. 检查本地是否已有所有 layer
//    e. 顺序/并行下载缺失的 layer
//    f. 校验 SHA256
//    g. 解压 layer 到 snapshotter

// 4. containerd pull 实现（containerd/pkg/...）
//    containerd.remotes.docker.Fetcher
//    
//    单个 layer 下载：
//    GET /v2/library/nginx/blobs/sha256:abc...
//    Range: bytes=0-1048575  (分块)
//    → 流式写入 content store
//    
//    content store：/var/lib/containerd/io.containerd.content.v1.content/blobs/

// 5. 解压 layer
//    containerd/snapshots/overlayfs/overlayfs.go
//    
//    snapshotter.Prepare():
//    a. 读 layer tar
//    b. 解压到 /var/lib/containerd/.../snapshots/<key>/fs/
//    c. 在 upper 层准备 mount
//    
//    snapshotter.Commit():
//    a. 标记 snapshot 为不可变
//    b. 后续容器以此为基础

// 6. 注册 image
//    containerd/images/image.go
//    a. 在 image store 注册引用
//    b. digest = manifest digest
//    c. 后续 docker images 可见

08 containerd-shim

containerd-shim 是每容器独立守护进程，承担关键角色：

// 为什么需要 shim？

// runc 设计：runc 是"一次性"工具
//   1. runc create → 创建容器进程，runc 立即退出
//   2. runc start → 唤醒容器进程，runc 立即退出
//   3. 容器进程的 stdin/stdout 需要"父亲"
//      如果父进程退出，fd 关闭

// shim 的作用：
//   1. 作为容器进程的"父亲"
//   2. 持有 stdin/stdout/stderr
//   3. 收集容器退出码
//   4. 即使 dockerd/containerd 重启，容器仍运行
//   5. 提供 exec 接口（在容器内启动新进程）
//   6. 监控容器状态

// shim 二进制变体：
//   containerd-shim-runc-v1   ← 老 v1 (per-container shim instance)
//   containerd-shim-runc-v2   ← 现代 v2 (一个 shim 管多个容器)
//   containerd-shim-kata-v2   ← Kata Containers
//   containerd-shim-runsc-v1  ← gVisor

// v2 的优势：
//   - 一个 shim 管多个容器（容器组 sandbox）
//   - 减少进程数
//   - 共享上下文

// shim 启动参数：
containerd-shim-runc-v2 \
  -namespace moby \                      # namespace
  -id <container_id> \                   # 容器 ID
  -address /run/containerd/.../socket.ttrpc \  # ttrpc 地址
  -publish-binary /usr/bin/containerd \  # 事件发布
  -bundle /run/containerd/io.containerd.runtime.v2.task/moby/<id>

// shim 接口（ttrpc）：
// service Task {
//   rpc Create(CreateTaskRequest)       returns (CreateTaskResponse);
//   rpc Start(StartRequest)              returns (StartResponse);
//   rpc Delete(DeleteRequest)            returns (DeleteResponse);
//   rpc Exec(ExecRequest)                returns (ExecResponse);
//   rpc Pids(PidsRequest)                returns (PidsResponse);
//   rpc ResizePty(ResizePtyRequest)      returns (Empty);
//   rpc Kill(KillRequest)                returns (Empty);
//   rpc Pause / Resume / Checkpoint / Update ...
// }
Daemonless 容器:
shim 设计让容器与 daemon 解耦：
1. dockerd 崩溃 → 容器继续运行（shim 接管）
2. containerd 重启 → 容器继续运行，containerd 重连 shim
3. 升级 dockerd 不影响在跑的容器
这是相比 Docker 早期版本的重大架构改进。
