01 概述

XDP（eXpress Data Path）是 Linux 内核中的高性能可编程网络数据路径。它允许在网卡驱动层直接运行 eBPF 程序处理数据包，绕过整个协议栈，实现百万级 PPS（Packets Per Second）的数据包处理能力。

核心优势
  • 零拷贝：数据包在 DMA 缓冲区原地处理
  • 绕过协议栈：不经过 sk_buff 分配
  • 可编程：eBPF 程序动态加载
  • 安全：JIT 编译 + 验证器检查
  • 毫秒级生效：无需重启内核或应用
传统收包 vs XDP
特性传统路径XDP
处理位置NAPI poll → 协议栈驱动层 DMA 后立即
内存分配sk_buff 分配零分配
PPS~1M~24M+（取决于硬件）
可编程性内核模块eBPF 程序动态加载

02 eBPF 基础

eBPF（extended Berkeley Packet Filter）是 XDP 的编程基础。用户编写 C 程序，经过验证器检查后 JIT 编译为本地机器码，在内核沙箱中安全执行。

eBPF 执行流程

用户态
┌──────────────────────────────────────────┐
│  用户编写 C 程序（如 XDP 过滤器）         │
│  ↓                                       │
│  clang/llvm 编译为 BPF 字节码（.o 文件）  │
│  ↓                                       │
│  bpftool 加载到内核                       │
└──────────────────────────────────────────┘
                   │
                   │ bpf() 系统调用
                   ↓
内核态
┌──────────────────────────────────────────┐
│  1. BPF Verifier（验证器）                │
│     - 检查程序安全性                      │
│     - 确保无无限循环                      │
│     - 确保内存访问合法                    │
│                                          │
│  2. JIT Compiler                         │
│     - 将 BPF 字节码编译为 x86_64 机器码   │
│     - 运行时直接执行，无解释开销          │
│                                          │
│  3. Attach 到钩子点                      │

03 XDP 架构

XDP 在收包路径中的位置

传统收包路径（慢）：

  网线 → NIC → DMA → IRQ → softIRQ → NAPI poll → sk_buff 分配 → 协议栈 → Socket
                            ↑ 大量内存分配和协议处理

XDP 快速路径（快）：

  网线 → NIC → DMA → XDP 程序 → 决定动作
                      ↑ 在驱动层，DMA 后立即执行
                      ↑ 无 sk_buff，零内存分配
                      │
                      ├── XDP_PASS → 走传统协议栈（需要时才升级为 sk_buff）
                      ├── XDP_DROP → 直接丢弃（DDoS 防护）
                      ├── XDP_TX  → 原路发回（负载均衡器）
                      ├── XDP_REDIRECT → 转发到其他网卡/BPF 程序
                      └── XDP_ABORTED → 丢弃 + trace（调试）

XDP 运行模式

模式说明性能限制
generic通用模式，所有网卡都支持低（仍需 sk_buff）仅用于测试
native/driver网卡驱动原生支持高（真正的零拷贝）需要驱动支持
offload网卡硬件卸载（SmartNIC）最高（硬件处理）需要特定硬件
generic 模式：协议栈内部执行（需分配 sk_buff）
  NIC → DMA → 分配 sk_buff → XDP 程序 → 协议栈

driver 模式：驱动层执行（无 sk_buff，原地处理）
  NIC → DMA → XDP 程序 → 按动作处理

offload 模式：SmartNIC 硬件执行
  NIC → 硬件 XDP → 按动作处理（CPU 完全不参与）

04 XDP 五种动作

// include/uapi/linux/bpf.h:6546
enum xdp_action {
    XDP_ABORTED  = 0,  // 丢弃 + 触发 trace（调试用）
    XDP_DROP,           // 丢弃数据包
    XDP_PASS,           // 传递给协议栈
    XDP_TX,             // 原路发回同一网卡
    XDP_REDIRECT,       // 转发到其他网卡 / CPU / BPF 程序
};
动作说明典型场景
XDP_DROP直接丢弃，不分配任何内存DDoS 防护、黑名单过滤
XDP_TX数据包从同一网卡原路发回负载均衡器（DR 模式）
XDP_REDIRECT转发到另一个网卡或 CPU跨网卡转发、AF_XDP
XDP_PASS升级为 sk_buff，进入协议栈混合模式（部分包走 XDP，部分走协议栈）
XDP_ABORTED丢弃 + 记录 trace 事件调试和异常检测

05 核心数据结构

// include/net/xdp.h:86 — XDP 数据包缓冲区
struct xdp_buff {
    void *data;            // 数据起始指针
    void *data_end;        // 数据结束指针
    void *data_meta;       // 元数据起始指针
    void *data_hard_start; // 硬件起始位置（以太网头之前）
    struct xdp_rxq_info *rxq;  // 接收队列信息
    struct xdp_txq_info *txq;  // 发送队列信息
    u32 frame_sz;          // 帧大小
    u32 flags;             // 标志（是否有 fragments）
};

// include/uapi/linux/bpf.h:6557 — 用户态可见的元数据
struct xdp_md {
    __u32 data;             // 数据偏移
    __u32 data_end;         // 数据结束偏移
    __u32 data_meta;        // 元数据偏移
    __u32 ingress_ifindex;  // 接收网卡 ifindex
    __u32 rx_queue_index;   // 接收队列索引
    __u32 egress_ifindex;   // 发送网卡 ifindex
};

06 性能对比

数据包处理路径长度对比：

iptables（Netfilter）：
  NIC → IRQ → softIRQ → sk_buff 分配 → NF_PRE_ROUTING → NF_LOCAL_IN
  → conntrack → filter → socket → 用户态
  路径长度: ~15 步                PPS: ~1M

DPDK（用户态）：
  NIC → UIO/VFIO 绕过内核 → 用户态轮询处理
  路径长度: ~3 步                PPS: ~30M+
  代价: 需要独占网卡，内核无法使用

XDP（内核态快速路径）：
  NIC → DMA → XDP 程序（eBPF JIT） → 按动作处理
  路径长度: ~4 步                PPS: ~24M+
  优势: 内核保留，与协议栈共存
方案PPS（百万包/秒）延迟内核兼容适用场景
iptables~0.5-1M高完全兼容常规防火墙
nftables~1-2M中高完全兼容新一代防火墙
XDP (driver)~15-24M低兼容DDoS 防护、负载均衡
XDP (offload)~100M+最低兼容SmartNIC 硬件处理
DPDK~30M+低独占网卡用户态网络栈

07 应用场景

DDoS 防护
// XDP_DROP: 在驱动层直接丢弃
// 14.8M PPS 丢弃（单核）
SEC("xdp")
int xdp_drop(struct xdp_md *ctx) {
    return XDP_DROP;
}

// 实际场景：丢弃黑名单 IP
SEC("xdp")
int xdp_filter(struct xdp_md *ctx) {
    struct iphdr *ip = ...;
    if (bpf_map_lookup_elem(&blacklist, &ip->saddr))
        return XDP_DROP;
    return XDP_PASS;
}
负载均衡器
// XDP_TX: 原路发回（DR 模式）
SEC("xdp")
int xdp_lbal(struct xdp_md *ctx) {
    struct iphdr *ip = ...;
    u32 key = ip->daddr & 0xFF;
    u32 *srv = bpf_map_lookup_elem(
        &servers, &key);
    if (srv)
        bpf_redirect(*srv, 0);
    return XDP_PASS;
}
跨网卡转发
// XDP_REDIRECT: 转发到其他网卡
SEC("xdp_fwd")
int xdp_forward(struct xdp_md *ctx) {
    u32 ifindex = bpf_map_lookup_elem(
        &redirect_map, &ctx->rx_queue_index);
    if (ifindex)
        return bpf_redirect(ifindex, 0);
    return XDP_PASS;
}
