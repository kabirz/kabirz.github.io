01 概述

net_device 是 Linux 内核中所有网络设备的统一抽象。无论是物理网卡（eth0）、虚拟设备（veth0）、桥接（br0）、隧道（tun/tap），还是 VLAN（eth0.10），在内核中都以 net_device 表示。它是 5678 行的 include/linux/netdevice.h 中定义的最大结构体之一。

net_device 统一模型
  • 物理网卡：eth0, ens33, enp0s3
  • 虚拟设备：veth0, dummy0
  • 桥接：br0（bridge）
  • 隧道：tun0, tap0, vxlan0
  • VLAN：eth0.10
  • 回环：lo
核心职责
职责说明
收包NAPI poll → sk_buff
发包sk_buff → ndo_start_xmit
MTU 管理最大传输单元

邻居管理ARP/NDP 表

02 net_device_ops 操作表

// include/linux/netdevice.h:1426 — 驱动核心操作表
struct net_device_ops {
    // 生命周期
    int  (*ndo_init)(struct net_device *dev);        // 设备初始化
    int  (*ndo_open)(struct net_device *dev);        // 启动（ifconfig up）
    int  (*ndo_stop)(struct net_device *dev);        // 停止（ifconfig down）

    // 发送路径
    netdev_tx_t (*ndo_start_xmit)(struct sk_buff *skb,
                                    struct net_device *dev); // ★ 发送入口
    u16 (*ndo_select_queue)(struct net_device *dev,
                             struct sk_buff *skb, ...); // TX 队列选择

    // 配置
    int (*ndo_set_mac_address)(struct net_device *dev, void *addr);
    int (*ndo_change_mtu)(struct net_device *dev, int new_mtu);
    void (*ndo_set_rx_mode)(struct net_device *dev); // 组播/混杂

    // 统计
    void (*ndo_get_stats64)(struct net_device *dev,
                           struct rtnl_link_stats64 *stats);

    // VLAN / TC / XDP
    int (*ndo_vlan_rx_add_vid)(struct net_device *dev, __be16 proto, u16 vid);
    int (*ndo_setup_tc)(struct net_device *dev, ...); // tc 调用
};

不同设备的 ops 实现

  net_device_ops 是多态的关键：

  eth0（物理网卡）     → e1000_netdev_ops（Intel 网卡驱动）
  veth0（虚拟网卡）    → veth_netdev_ops
  br0（网桥）          → br_dev_ops
  tun0（隧道）         → tun_netdev_ops
  vxlan0（overlay）    → vxlan_netdev_ops
  lo（回环）           → loopback_ops

  每种设备实现自己的 ndo_start_xmit / ndo_open / ndo_stop
  内核通过统一的 net_device 调用不同驱动的 ops → 多态

03 设备生命周期

网络设备从创建到销毁的完整生命周期：

  1. 分配
  alloc_netdev() / alloc_netdev_mqs()
  → 分配 struct net_device + 驱动私有数据
  → 设置默认值（MTU=1500, name="eth%d"）

  2. 初始化
  register_netdevice()
  → dev->netdev_ops->ndo_init()    // 驱动初始化
  → 分配 RX/TX 队列
  → 加入 dev_base_head 链表
  → 注册到网络命名空间
  → 发送 RTM_NEWLINK 通知（Netlink）

  3. 启动
  dev_open()（ifconfig eth0 up）
  → dev->flags |= IFF_UP
  → dev->netdev_ops->ndo_open()   // 驱动启动
  → 启用 NAPI / 队列
  → 注册中断处理
  → 设备状态 → __LINK_STATE_START

  4. 运行
  收包：NAPI poll → netif_receive_skb()
  发包：dev_queue_xmit() → ndo_start_xmit()
  配置：dev_set_mtu() / dev_set_mac_address() 等

  5. 停止
  dev_close()（ifconfig eth0 down）
  → dev->flags &= ~IFF_UP
  → dev->netdev_ops->ndo_stop()   // 驱动停止
  → 停用 NAPI / 释放队列

  6. 注销
  unregister_netdevice()
  → 从 dev_base_head 移除
  → dev->netdev_ops->ndo_uninit()
  → 释放所有资源
  → 发送 RTM_DELLINK 通知

04 收发数据路径

发送路径（协议栈 → 网卡）：

  TCP/UDP → ip_output → dev_queue_xmit()
        │
        ↓
  net_device._tx[queue] → qdisc
        │
        ↓
  dev_hard_start_xmit()
        │
        ↓
  netdev_ops->ndo_start_xmit(skb, dev)
        │
        └── 驱动实现：DMA 映射 → 网卡 TX ring → 通知网卡发送

接收路径（网卡 → 协议栈）：

  网卡 → DMA → 驱动分配 skb → NAPI poll
        │
        ↓
  napi_gro_receive() → __netif_receive_skb_core()
        │
        ├── AF_PACKET 抓包点（tcpdump）
        │
        ├── rx_handler（bridge 会注册）
        │
        └── 协议栈分发（IP/ARP/...）
                │
                ↓
        eth0: net_device.protocol = ETH_P_IP
        → ip_rcv()
