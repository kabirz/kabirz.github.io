01 概述

Tracepoint 是内核中编译时定义的静态跟踪点。开发者在内核代码中插入 TRACE_EVENT 宏，定义跟踪事件。未启用时零开销（通过 static key 实现），启用时可以记录详细的内核事件信息。perf、ftrace、BPF 都可以使用 tracepoint。

02 API 与使用

// 定义 tracepoint
TRACE_EVENT(sched_switch,
    TP_PROTO(struct task_struct *prev, struct task_struct *next),
    TP_STRUCT__entry(
        __array(char, prev_comm, 16)
        __field(pid_t, prev_pid)
        __array(char, next_comm, 16)
        __field(pid_t, next_pid)
    ),
    TP_fast_assign(
        memcpy(__entry->prev_comm, prev->comm, 16);
        __entry->prev_pid = prev->pid;
    ),
    TP_printk("prev=%s pid=%d next=%s pid=%d",
        __entry->prev_comm, __entry->prev_pid,
        __entry->next_comm, __entry->next_pid)
);

// 在内核代码中插入 tracepoint
trace_sched_switch(prev, next);

# 列出所有可用 tracepoint

# 启用特定 tracepoint
$ echo 1 > /sys/kernel/debug/tracing/events/sched/sched_switch/enable

# 查看 tracepoint 输出
$ cat /sys/kernel/debug/tracing/trace_pipe

# 使用 perf 读取 tracepoint
$ perf record -e sched:sched_switch -a sleep 5
$ perf script
