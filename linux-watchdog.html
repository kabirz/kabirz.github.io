01 概述

Watchdog（看门狗）是硬件或软件定时器，用于监控系统是否正常运行。如果在超时时间内没有"喂狗"（kick），看门狗会触发系统重启。Linux 支持硬件看门狗（CPU/芯片内置）和软件看门狗（softdog），是嵌入式系统和服务器高可用的关键机制。

02 硬件看门狗

硬件看门狗工作原理：

  ┌──────────────┐        ┌──────────────┐
  │    CPU       │        │   Watchdog   │
  │   (正常)     │        │   (定时器)   │
  │              │  kick  │              │
  │  定期喂狗 ──→│──────→ │  重置计数器  │
  │              │        │              │
  └──────────────┘        └──────┬───────┘
                                 │
                          超时未 kick?
                                 │
                                 ↓
                          ┌──────────────┐
                          │   系统重启    │
                          └──────────────┘

常见硬件看门狗：

  Intel iTCO      → Intel 主板
  NVIDIA Tegra    → ARM SoC
  Broadcom WDT    → 树莓派
  Qualcomm WDT    → 手机 SoC
  Synopsys DW-WDT → 通用 SoC

03 软件看门狗（softdog）

softdog（纯软件实现）：

  加载 softdog 模块
  $ sudo modprobe softdog

  用户态守护进程定期喂狗
  $ echo 1 > /dev/watchdog

  如果守护进程挂了 → 内核 softdog 超时 → 重启

  适用场景：没有硬件看门狗的服务器

04 API 与配置

# 查看可用的看门狗设备
$ ls /dev/watchdog*
/dev/watchdog  /dev/watchdog0

# 查看看门狗信息
$ cat /sys/class/watchdog/watchdog0/identity
i6300ESB watchdog

# 查看超时时间
$ cat /sys/class/watchdog/watchdog0/timeout
30  # 30 秒

# 使用 watchdog（喂狗）
$ echo 1 > /dev/watchdog   # 喂狗一次
$ echo V > /dev/watchdog   # 停用看门狗（Magic 字符 'V'）

# 使用 watchdog-test 工具
$ sudo modprobe softdog
$ sudo watchdog-test /dev/watchdog

# systemd watchdog 集成
$ systemctl status watchdog.service
