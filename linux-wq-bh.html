01 概述

WQ_BH（Bottom Half Workqueue）是 Linux 7.x 引入的在 softirq 上下文中执行工作的机制。它解决了 tasklet 的已知问题（不可睡眠、并发限制），同时保留了 softirq 的低延迟特性。WQ_BH 是 tasklet 的现代替代方案。

核心特性
  • softirq 上下文：在软中断中执行
  • cmwq 框架：复用工作队列基础设施
  • per-CPU：每个 CPU 独立的 worker pool
  • 替代 tasklet：解决 tasklet 的已知缺陷
  • 系统预定义：system_bh_wq / system_bh_highpri_wq
适用场景
场景说明
网络栈高性能网络处理
驱动底半部中断后的快速处理
替代 tasklet所有 tasklet 用例
延迟处理需要 softirq 但不能用 tasklet

02 数据结构

// include/linux/workqueue.h:370-413
enum wq_flags {
    WQ_BH          = 1 << 0,  // 在 softirq 上下文执行
    WQ_UNBOUND     = 1 << 1,  // 不绑定 CPU
    WQ_FREEZABLE   = 1 << 2,  // 挂起时冻结
    WQ_MEM_RECLAIM = 1 << 3,  // 内存回收时保证推进
    WQ_HIGHPRI     = 1 << 4,  // 高优先级
    // BH wq 只允许以下标志组合
    __WQ_BH_ALLOWS = WQ_BH | WQ_HIGHPRI | WQ_PERCPU,
};

03 核心 API

// 提交 BH 工作
queue_work(system_bh_wq, &my_work);

// 提交高优先级 BH 工作
queue_work(system_bh_highpri_wq, &my_work);

// 自定义 BH 工作队列
struct workqueue_struct *my_bh_wq;
my_bh_wq = alloc_workqueue("my_bh", WQ_BH | WQ_HIGHPRI, 0);
queue_work(my_bh_wq, &my_work);
限制：BH 工作队列只允许 WQ_BH | WQ_HIGHPRI | WQ_PERCPU 标志组合。不能使用 WQ_UNBOUND、WQ_FREEZABLE 等。BH 工作不能睡眠（在 softirq 上下文中执行）。

04 BH Worker

BH worker 是在 softirq 上下文中执行工作的特殊 worker，通过 workqueue_softirq_action() 触发。

// kernel/workqueue.c:3649-3685
static void bh_worker(struct worker *worker)
{
    struct worker_pool *pool = worker->pool;
    int nr_restarts = BH_WORKER_RESTARTS;  // 最多处理 10 个工作
    unsigned long end = jiffies + BH_WORKER_JIFFIES; // 2ms 超时

    // 从 worker pool 中取出工作执行
    do {
        struct work_struct *work =
            list_first_entry(&pool->worklist, struct work_struct, entry);
        process_scheduled_works(worker);
    } while (keep_working(pool) &&
             --nr_restarts && time_before(jiffies, end));
}

// 由 softirq 触发
void workqueue_softirq_action(bool highpri)
{
    struct worker_pool *pool =
        &per_cpu(bh_worker_pools, smp_processor_id())[highpri];
    if (need_more_worker(pool))
        bh_worker(list_first_entry(&pool->workers, struct worker, node));
}

05 使用模式

// 模式一：替代 tasklet
// 旧代码（tasklet，已废弃）：
tasklet_schedule(&my_tasklet);

// 新代码（WQ_BH）：
queue_work(system_bh_wq, &my_work);

// 模式二：网络栈底半部
static void net_rx_action(struct softirq_action *h)
{
    // 由 softirq 触发，执行 BH worker
    workqueue_softirq_action(false);
}

// 模式三：高优先级 BH
queue_work(system_bh_highpri_wq, &urgent_work);
tasklet 替代：所有使用 tasklet_schedule() 的代码都应该迁移到 queue_work(system_bh_wq, ...)。BH 工作队列提供了更好的并发管理和更清晰的语义。
