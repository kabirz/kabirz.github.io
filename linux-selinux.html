01 概述

SELinux（Security-Enhanced Linux）是 Linux 内核中的强制访问控制（MAC）安全模块。与传统 DAC（自主访问控制）不同，SELinux 为每个进程和资源分配安全标签，通过策略规则决定访问权限。即使 root 进程也被 SELinux 策略约束，是 Android、RHEL/Fedora 的核心安全机制。

02 安全标签

SELinux 标签系统：

  每个对象（进程/文件/端口）都有一个安全上下文：

  user:role:type:level

  示例：
  system_u:system_r:httpd_t:s0-s0:c0.c1023

  ├── user    : 系统用户身份
  ├── role    : 角色（进程可扮演的角色）
  ├── type    : ★ 类型（核心，策略基于类型）
  └── level   : MLS/MCS 级别（多级安全）

  类型是最关键的部分：
  httpd_t  → Apache 进程的类型
  httpd_sys_content_t → Apache 可读的文件类型
  httpd_port_t → Apache 可绑定的端口类型

  策略规则只检查类型，不检查具体用户/角色

03 访问检查流程

进程访问文件时的 SELinux 检查：

  进程 httpd (httpd_t) 打开 /var/www/html/index.html
        │
        ↓
  VFS 层 → security_inode_permission()
        │
        ├── 1. 获取进程安全上下文
        │       current->cred->security → httpd_t
        │
        ├── 2. 获取文件安全上下文
        │       inode->i_security → httpd_sys_content_t
        │
        ├── 3. 查询策略数据库
        │       type_httpd_t : type_httpd_sys_content_t : file : read → ALLOW
        │
        └── 4. 允许访问 → 继续执行

如果策略不允许：

  策略查询 → DENY
        │
        ├── 1. 记录审计日志
        │       audit.log: avc: denied { read } for ...
        │
        └── 2. 返回 -EACCES（权限拒绝）
                进程无法访问文件

04 策略语言

# SELinux 策略规则示例

# 允许 httpd 读取 web 内容文件
allow httpd_t httpd_sys_content_t:file { read open getattr };

# 允许 httpd 绑定 80/443 端口
allow httpd_t httpd_port_t:tcp_socket { name_bind };

# 允许 httpd 发起网络连接
allow httpd_t self:tcp_socket { create connect };

# 禁止 httpd 访问 ssh 密钥
# （默认 deny，无需显式写规则）

# 类型转移（进程创建子进程时的类型）
allow httpd_t httpd_t:process { fork };
# 子进程继承 httpd_t 类型

05 管理与调优

# 查看 SELinux 状态
$ getenforce
Enforcing

$ sestatus
SELinux status:                 enabled
SELinuxfs mount:                /sys/fs/selinux
SELinux root directory:         /etc/selinux
Loaded policy name:             targeted
Current mode:                   enforcing
Mode from config file:          enforcing

# 临时切换模式
$ setenforce 0   # Permissive（仅记录，不阻止）
$ setenforce 1   # Enforcing（默认，阻止违规）

# 查看文件的 SELinux 标签
$ ls -Z /var/www/html/index.html
-rw-r--r--. root root system_u:object_r:httpd_sys_content_t:s0 /var/www/html/index.html

# 修改文件标签
$ chcon -t httpd_sys_content_t /var/www/html/index.html

# 永久修改文件标签（通过 restorecon）
$ semanage fcontext -a -t httpd_sys_content_t "/var/www/html(/.*)?"
$ restorecon -Rv /var/www/html

# 查看审计日志
$ ausearch -m avc -ts recent

问题原因解决
