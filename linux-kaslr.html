01 概述

KASLR（Kernel Address Space Layout Randomization）是 Linux 内核的地址空间随机化安全机制。每次启动时，内核代码、模块、页面等基地址随机偏移，使攻击者无法预测内核函数地址，大幅增加 ROP（Return-Oriented Programming）攻击难度。

02 实现原理

无 KASLR（固定地址）：

  内核映射地址始终固定：
  ┌──────────────────────────────┐ 0xffffffff80000000
  │ 内核代码（.text）             │  ← 攻击者知道地址
  │ 内核数据（.data/.bss）        │
  │ 内核模块加载区域              │
  └──────────────────────────────┘

有 KASLR（随机化地址）：

  每次启动内核基地址随机偏移：
  ┌──────────────────────────────┐ 0xffffffff82400000 (随机)
  │  内核代码（.text）            │  ← 攻击者不知道地址
  │  内核数据（.data/.bss）       │
  │  内核模块加载区域             │
  └──────────────────────────────┘

  偏移范围：16MB（x86_64）
  每次重启地址不同 → ROP gadget 无法预测

03 启动流程

  Bootloader 加载压缩内核
        │
        ↓
  kaslr_early_init() — arch/x86/boot/compressed/kaslr.c
        │
        ├── 1. 从硬件随机源获取随机数
        │       RDRAND / RDSEED 指令
        │       或 BIOS/UEFI 提供的随机数
        │
        ├── 2. 计算随机偏移
        │       偏移 = random() & 0x00FFFFFF  // 对齐到 16MB
        │
        ├── 3. 调整内核加载地址
        │       输出地址 = 内核原始地址 + 偏移
        │
        └── 4. 后续内核初始化使用随机化后的地址

  /proc/cmdline 包含 kaslr 信息：
  $ cat /proc/cmdline
  BOOT_IMAGE=/boot/vmlinuz-... kaslr ...

04 安全价值

攻击类型KASLR 防护
ROP/JOP无法预测 gadget 地址
内核漏洞利用增加地址猜测难度
内核信息泄露地址不固定，泄露一次无效
模块加载攻击模块加载地址随机
KASLR 的局限
KASLR 只是增加了攻击难度，不是完全防御：
① 如果有信息泄露漏洞，攻击者可以读取随机化后的地址
② 一些侧信道攻击可以绕过 KASLR
③ KASLR 通常与 SMEP/SMAP/PXN 等其他防护机制配合使用
