01 概述

Linux 内核提供了完整的性能分析工具链。本文全景梳理所有性能分析工具的关系和使用场景，帮助开发者快速定位性能问题。

02 工具分类

Linux 性能分析工具分类：

  ┌─────────────────────────────────────────────────────────┐
  │                    CPU 性能分析                          │
  │  perf top     ──  实时热点函数                           │
  │  perf record  ──  采样分析 → perf report                │
  │  perf stat    ──  CPU 计数器统计                         │
  │  火焰图       ──  可视化调用栈                           │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │                    内核函数追踪                           │
  │  ftrace       ──  函数级跟踪（已有专题）                 │
  │  tracepoint   ──  静态跟踪点（已有专题）                 │
  │  kprobe       ──  动态探针（已有专题）                   │
  │  perf trace   ──  系统调用跟踪                          │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │                    可编程分析                             │
  │  BPF          ──  可编程内核沙箱                         │
  │  bpftrace     ──  高级 BPF 追踪语言                     │
  │  bcc          ──  BPF 编译器集合                        │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │                    系统级分析                             │
  │  vmstat       ──  系统活动统计                           │
  │  iostat       ──  I/O 统计                              │
  │  mpstat       ──  多处理器统计                           │
  │  sar          ──  历史数据收集                           │
  └─────────────────────────────────────────────────────────┘

03 分析流程

性能分析决策树：

  性能问题类型？
  │
  ├── CPU 热点 → perf top / perf record
  ├── CPU 利用率高 → perf stat
  ├── I/O 瓶颈 → iostat / blktrace
  ├── 内存问题 → slabtop / vmstat
  ├── 锁竞争 → lockstat / perf lock
  ├── 调度延迟 → perf sched
  ├── 网络延迟 → perf network / bpftrace
  └── 延迟分析 → ftrace + function_graph
