01 概述

读写锁（Read-Write Lock）允许多个读者同时访问共享资源，但写者必须独占。这是读多写少场景的最优同步机制。

核心思想
  • 读并行：多个读者可同时持有
  • 写互斥：写者独占，阻塞所有其他访问
  • 写者优先：写者等待时新读者也被阻塞
  • 零开销读：理想情况下读者无额外开销
适用场景
场景原因
路由表查找读极多，写极少
模块列表加载少，查找多
配置数据偶尔修改，频繁读取

02 锁类型

类型上下文等待方式说明
rwlock_t不可睡眠自旋忙等（中断/软中断可用）
rw_semaphore可睡眠睡眠进程上下文（推荐）
percpu_rw_semaphore可睡眠睡眠per-CPU 读写信号量

03 核心 API

// ===== rwlock_t（自旋，不可睡眠）=====
static DEFINE_RWLOCK(my_rwlock);

read_lock(&my_rwlock);          // 读者获取
read_unlock(&my_rwlock);        // 读者释放
write_lock(&my_rwlock);         // 写者获取
write_unlock(&my_rwlock);       // 写者释放

read_lock_irqsave(&my_rwlock, flags);
write_lock_irqsave(&my_rwlock, flags);
read_trylock(&my_rwlock);
write_trylock(&my_rwlock);

// ===== rw_semaphore（可睡眠，推荐）=====
static DEFINE_RWSEM(my_rwsem);

down_read(&my_rwsem);          // 读者获取
up_read(&my_rwsem);            // 读者释放
down_write(&my_rwsem);         // 写者获取
up_write(&my_rwsem);           // 写者释放
down_read_trylock(&my_rwsem);
down_write_trylock(&my_rwsem);

04 rwlock 详解

rwlock_t 是自旋读写锁，适用于不可睡眠的上下文。

实现原理

// 基于原子计数器
// 正数 = 读者数量
// 负数 = 写者已获取
// 0 = 空闲

// read_lock: 原子增加计数
// write_lock: CAS 设置为 -1
// write_unlock: 设置为 0
// read_unlock: 原子减少计数
注意：rwlock_t 在 PREEMPT_RT 内核中变为可睡眠（rt_mutex）。新代码应优先使用 rw_semaphore。

05 rwsem 详解

rw_semaphore 是可睡眠读写信号量，适用于进程上下文。

与 mmap_lock 的关系

// mm->mmap_lock 就是 rw_semaphore
// 用于保护进程的虚拟地址空间

// 遍历 VMA（读锁）
down_read(&mm->mmap_lock);
for_each_vma(vma) { /* 只读遍历 */ }
up_read(&mm->mmap_lock);

// 修改 VMA（写锁）
down_write(&mm->mmap_lock);
do_mmap(...);  /* 创建新 VMA */
up_write(&mm->mmap_lock);

写者优先实现

写者优先：当有写者等待时，新的读取请求会被阻塞，防止写者饥饿。这是 Linux rwsem 的默认行为。通过 current_is_kswapd() 等检查，内核会对 kswapd 等关键线程做特殊处理。

06 使用模式

// 模式一：保护配置数据
static DEFINE_RWSEM(config_lock);
static struct config_data *config;

// 读取配置
down_read(&config_lock);
struct config_data *c = config;
val = c->some_field;
up_read(&config_lock);

// 更新配置
down_write(&config_lock);
config = new_config;
up_write(&config_lock);

// 模式二：嵌套锁（必须固定顺序）
down_write(&lock_a);
down_write_nested(&lock_b, SINGLE_DEPTH_NESTING);
// ...
up_write(&lock_b);
up_write(&lock_a);
