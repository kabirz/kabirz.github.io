01 概述

内核线程（kthread）是仅在内核态运行的线程，没有用户态地址空间。它们用于执行需要持续运行或延迟执行的内核任务：页面回收（kswapd）、软中断处理（ksoftirqd）、块设备回写（kblockd）、工作队列 worker 线程等。

02 内核线程类型

内核线程功能调度
kthreadd（PID 2）所有内核线程的父线程常驻
ksoftirqd/N每个 CPU 一个，处理软中断常驻
kswapd/N页面回收（后台内存回收）按需
kcompactd/N内存压缩按需
kblockd/N块设备 I/O 调度按需
watchdog/NCPU 看门狗常驻
migration/N进程在 CPU 间迁移常驻
rcu_gp/NRCU 宽限期管理常驻
rcu_par_gp/NRCU 并行宽限期常驻
khungtaskd检测 D 状态（不可中断睡眠）进程常驻

03 创建 API

// 创建内核线程（推荐方式）
struct task_struct *kthread_create(
    int (*threadfn)(void *data),  // 线程函数
    void *data,                    // 参数
    const char namefmt[], ...);    // 名称（显示在 ps 中）
// 返回 task_struct，需要 wake_up_process() 启动

// 创建并立即运行
struct task_struct *kthread_run(
    int (*threadfn)(void *data),
    void *data,
    const char namefmt[], ...);

// 停止线程
kthread_stop(task);

// 示例：创建一个后台监控线程
static int my_monitor_thread(void *data) {
    while (!kthread_should_stop()) {
        // 执行监控任务
        msleep(1000);  // 每秒检查一次
    }
    return 0;
}

struct task_struct *task = kthread_run(
    my_monitor_thread, NULL, "my_monitor");

04 生命周期

内核线程生命周期：

  kthread_create() / kthread_run()
        │
        ↓
  创建：分配 task_struct，设置内核栈
        │
        ↓
  就绪：wake_up_process() → 加入运行队列
        │
        ↓
  运行：在内核态执行 threadfn()
        │
        ├── kthread_should_stop() → true? → 退出
        ├── msleep() / schedule() → 让出 CPU
        └── 信号处理 → 可被信号中断
        │
        ↓
  退出：kthread_stop() 或 threadfn 返回
        │
        └── 释放资源，从进程列表移除
