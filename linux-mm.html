01 概述

Linux 内存管理子系统（mm/）是内核中最庞大、最复杂的子系统之一，负责虚拟内存管理、物理页面分配、页面回收、交换、内存压缩、OOM等核心功能。

核心职责
  • 虚拟内存管理：VMA 管理、mmap/munmap
  • 物理页面分配：伙伴系统、zone 管理
  • 小对象分配：SLUB slab 分配器
  • 页面回收：kswapd、直接回收、LRU
  • 交换：swap 设备管理、页面换入换出
  • OOM：内存不足时杀死进程
  • 内存压缩：碎片整理、大页支持
  • NUMA：非统一内存访问优化

03 页面与 Folio

struct page（页面描述符）

include/linux/mm_types.h:79-178

struct page {
    memdesc_flags_t flags;    // 状态标志（PG_locked, PG_dirty 等）
    union {
        struct {  // 页面缓存/匿名页
            union {
                struct list_head lru;   // LRU 链表
                struct list_head buddy_list; // 伙伴系统链表
            };
            struct address_space *mapping; // 地址空间
            pgoff_t __folio_index;         // 页偏移
            unsigned long private;          // 私有数据
        };
        struct {  // 复合页尾页
            unsigned long compound_head;
        };
    };
    union {
        unsigned int page_type;  // 类型化 folio
        atomic_t _mapcount;     // 映射计数
    };
    atomic_t _refcount;       // 引用计数
};

struct folio（大页描述符）

include/linux/mm_types.h:401-480

struct folio {
    memdesc_flags_t flags;
    struct address_space *mapping;
    pgoff_t index;
    union { void *private; swp_entry_t swap; };
    atomic_t _mapcount;
    atomic_t _refcount;
    // ...
};
Folio 的意义：folio 是对 page 的升级，支持多页（compound page）作为单一单元。它解决了 page 的 union 复杂性，为大页（THP）、页缓存、NUMA 提供了统一的抽象。现代内核代码优先使用 folio API。

页面标志

标志说明
PG_locked页面被锁定（IO 中）
PG_dirty页面已修改
PG_lru在 LRU 链表中
PG_referenced页面被访问过
PG_uptodate页面数据有效
PG_slab属于 slab 分配器
PG_buddy在伙伴系统空闲链表中
PG_swapbacked可交换的匿名页

07 页面回收

当系统内存不足时，页面回收器（vmscan.c，7910 行）从 LRU 链表中选择页面回收。

回收触发条件

异步回收（kswapd）
  • 水位线降到 watermark_low 以下
  • 后台守护进程 kswapd 被唤醒
  • 回收到 watermark_high 时停止
  • 不阻塞分配请求
直接回收（direct reclaim）
  • 分配时水位线低于 watermark_min
  • 当前进程直接执行回收
  • 可能触发 OOM
  • 阻塞分配直到回收完成

回收策略

类型处理方式
文件页（clean）直接丢弃（可从磁盘重读）
文件页（dirty）回写到磁盘后丢弃
匿名页交换到 swap 设备
mlocked 页永不回收（锁定在内存）
共享页通过反向映射（rmap）跟踪

08 Swap

Swap 将匿名页面换出到磁盘/分区，释放物理内存。由 swapfile.c 和 swap_state.c 管理。

swap 类型

类型说明
传统 swap 分区独立磁盘分区
swap 文件文件系统上的文件
zswap内存中的压缩 swap 缓存
zram压缩 RAM 设备
swap 缓存：页面在换入换出时可能同时存在于物理内存和 swap 中（swap cache），避免重复 IO。通过 add_to_swap() / delete_from_swap_cache() 管理。

09 OOM Killer

当系统内存严重不足且无法回收时，OOM Killer（oom_kill.c，1273 行）选择一个进程杀死以释放内存。

OOM 评分

// 基于内存使用量的评分
long oom_badness(struct task_struct *p, unsigned long totalpages)
{
    long points;

    // 基础分 = 进程 RSS + 页表 + swap 使用量
    points = get_mm_rss(p->mm) +
             get_mm_counter(p->mm, MM_SWAPENTS) +
             mm_pgtables_bytes(p->mm) / PAGE_SIZE;

    // oom_score_adj 调整（-1000 ~ +1000）
    points += points * p->signal->oom_score_adj / 1000;

    // oom_score_adj = -1000 的进程永不被杀
    // oom_score_adj = +1000 的进程优先被杀
    return points > 0 ? points : 1;
}

10 内存 cgroup

内存 cgroup（memcontrol.c）为进程组提供内存使用限制和统计。

核心控制文件

文件说明
memory.max内存上限
memory.low内存保护（best-effort）
memory.high触发回收的高水位
memory.min内存硬保护
memory.current当前使用量
memory.stat详细统计信息
memory.pressurePSI 压力信息
memory.swap.maxswap 使用上限
memory.oom.groupcgroup 级 OOM

内存分层

memory.low vs memory.min：
memory.min = 硬保护，即使系统内存紧张也不会回收该 cgroup 的内存
memory.low = 软保护，仅在其他 cgroup 无法满足时才回收
适用于多租户环境中的资源隔离。
