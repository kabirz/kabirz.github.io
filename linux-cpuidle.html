01 概述

cpuidle 是 Linux 的CPU 空闲管理框架。当 CPU 没有任务可运行时，cpuidle 决定让 CPU 进入哪种深度的休眠状态。休眠越深，省电越多，但唤醒延迟也越高。cpuidle 与 cpufreq 配合，实现完整的 CPU 功耗管理。

02 CPU 空闲状态

CPU C-states（深度递增）：

  ┌──────────┬──────────────┬──────────┬──────────┐
  │ 状态     │ 功耗节省      │ 唤醒延迟  │ 说明     │
  ├──────────┼──────────────┼──────────┼──────────┤
  │ C0       │ 无           │ 0        │ 运行中   │
  │ C1       │ ~10%         │ 1 μs     │ HLT     │
  │ C1E      │ ~30%         │ 1 μs     │ 增强暂停 │
  │ C2       │ ~50%         │ 10 μs    │ 停止时钟 │
  │ C3       │ ~70%         │ 50 μs    │ 睡眠     │
  │ C6       │ ~90%         │ 100 μs   │ 深度睡眠 │
  │ C7/C8    │ ~95%+        │ 300 μs+  │ 最深休眠 │
  └──────────┴──────────────┴──────────┴──────────┘

  服务器：通常只用 C1/C2（延迟敏感）
  笔记本：使用 C3-C6（省电优先）
  嵌入式：使用最深 C-state（最低功耗）

03 进入空闲流程

  调度器发现 CPU 无任务
        │
        ↓
  cpuidle_idle_call() — 调用 cpuidle 框架
        │
        ├── 1. 调用调节器选择目标状态
        │       menu Governor / TEO Governor
        │       基于历史空闲时间预测最佳状态
        │
        ├── 2. 调用 driver->enter()
        │       ├── C1: 执行 HLT 指令
        │       ├── C3: 关闭 L1/L2 缓存
        │       └── C6: 进入深度睡眠（关闭更多电路）
        │
        ├── 3. CPU 进入休眠状态
        │       功耗降低
        │
        └── 4. 唤醒（中断到达）
                恢复执行
                记录实际空闲时间
                更新统计

04 空闲调节器

调节器策略适用场景
menu基于预测的空闲时间选择默认，通用
teo基于定时器误差的优化推荐（5.4+）
haltpoll先轮询再休眠虚拟化环境
ladder阶梯式状态选择已废弃
cpuidle 与 cpufreq 的关系
cpufreq：CPU 运行时调节频率（动态电压频率调节 DVFS）
cpuidle：CPU 空闲时进入休眠状态
两者配合实现完整的 CPU 功耗管理：
• 有任务时：cpufreq 按负载调节频率
• 无任务时：cpuidle 选择深度休眠
