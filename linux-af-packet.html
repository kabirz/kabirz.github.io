01 概述

AF_PACKET 是 Linux 提供的原始套接字地址族，允许用户态程序直接收发二层以太网帧（跳过协议栈）。它是 tcpdump、libpcap、scapy、dpdk 等网络工具的底层机制。在收发包流程和 Netfilter 文章中，AF_PACKET 多次被提及——它就是"tcpdump 抓包点"。

AF_PACKET 核心能力
  • 零协议栈：直接操作二层帧
  • 混杂模式：捕获所有经过网卡的数据
  • mmap 零拷贝：高性能收包
  • 环形缓冲区：批量处理数据包
  • FANOUT：多 socket 负载均衡
  • QDISC_BYPASS：绕过 tc 流量控制

02 三种原始套接字

类型socket 参数层级能抓到什么典型用途
AF_PACKETSOCK_RAWL2 以太网以太网帧（含 MAC 头）tcpdump
AF_PACKETSOCK_DGRAML2 以太网以太网帧（不含 MAC 头）简化抓包
AF_INETSOCK_RAWL3 IPIP 数据包（无 L2 头）ping (ICMP)、traceroute
AF_PACKET + SOCK_RAW（tcpdump 使用）：

  收到的完整以太网帧：
  ┌──────────┬──────────┬─────────┬──────────┐
  │ DST MAC  │ SRC MAC  │  Type   │ IP Data  │
  │ (6B)     │ (6B)     │ (2B)    │          │
  └──────────┴──────────┴─────────┴──────────┘
  ↑ 用户态可以看到完整的以太网帧（14 字节头 + 负载）

AF_PACKET + SOCK_DGRAM：

  收到的帧去掉以太网头后：
  ┌─────────┬──────────┐
  │  Type   │ IP Data  │
  │ (2B)    │          │
  └─────────┴──────────┘
  ↑ 用户态看不到 MAC 头，但更简单

AF_INET + SOCK_RAW（ping 使用）：

  收到的 IP 数据包：
  ┌──────────┬─────────┬──────────┐
  │ IP Header│ICMP/Hdr │ Payload  │
  └──────────┴─────────┴──────────┘
  ↑ 用户态只能看到 IP 层以上
tcpdump 为什么需要 AF_PACKET + SOCK_RAW？
tcpdump 需要看到完整的以太网帧，包括 MAC 地址和所有协议头。只有 AF_PACKET + SOCK_RAW 能提供完整的 L2 帧。AF_INET + SOCK_RAW 只能看到 L3 及以上。

03 核心数据结构

// net/packet/internal.h:104 — AF_PACKET socket
struct packet_sock {
    struct sock          sk;             // 继承自 sock
    struct packet_fanout *fanout;        // FANOUT 负载均衡
    struct packet_ring_buffer rx_ring;   // ★ mmap 接收环形缓冲区
    struct packet_ring_buffer tx_ring;   // ★ mmap 发送环形缓冲区
    int                   ifindex;       // 绑定的网络设备
    struct packet_mclist *mclist;       // 多播/组播过滤
    enum tpacket_versions tp_version;  // TPACKET 版本（v1/v2/v3）
    struct packet_type   prot_hook ____cacheline_aligned_in_smp;
                                          // ★ 注册到协议栈的钩子
};

AF_PACKET 在协议栈中的位置

  收包路径：
  NIC → 驱动 → NAPI poll
    │
    ├── → 协议栈 → Socket（TCP/UDP）
    │
    └── → packet_type.prot_hook → AF_PACKET socket
                ↑
                这就是 tcpdump 的抓包点！
                在 __netif_receive_skb_core() 中检查

  发包路径：
  AF_PACKET socket → packet_snd()
    │
    ├── QDISC_BYPASS=0 → dev_queue_xmit（经过 tc）
    └── QDISC_BYPASS=1 → dev_hard_start_xmit（绕过 tc）
                │
                ↓
            网卡发送

04 收包路径

  数据包到达 __netif_receive_skb_core()
        │
        ├── 1. 检查是否有 AF_PACKET socket 注册
        │       遍历 ptype_all / ptype_specific
        │       packet_type->func() = packet_rcv()
        │
        ├── 2. packet_rcv() — af_packet.c
        │       │
        │       ├── 检查过滤条件（protocol、接口）
        │       │
        │       ├── 检查 socket 接收缓冲区是否满
        │       │       满 → 丢弃（tp_drops++）
        │       │
        │       ├── 方式 A：直接拷贝
        │       │       skb_clone() + skb_queue_tail()
        │       │       → 传递给 socket 等待的 recvmsg()
        │       │
        │       ├── 方式 B：mmap 环形缓冲区（TPACKET）
        │       │       写入 rx_ring
        │       │       用户态通过 poll() 检测新包
        │       │
        │       └── 方式 C：QDISC_BYPASS
        │               绕过 tc 直接到网卡
        │
        └── 3. 数据包同时继续传递给协议栈
                AF_PACKET 是"旁路"抓包，不影响正常流程

05 发包路径

  应用调用 sendto(fd, frame, len, ...)
        │
        ↓
  packet_snd() — af_packet.c
        │
        ├── 1. 构造 sk_buff
        │       分配 skb + 复制用户数据到 skb
        │
        ├── 2. 设置 skb->protocol = ETH_P_ALL
        │       设置 skb->dev = 绑定的网络设备
        │
        ├── 3. 发送
        │       ├── QDISC_BYPASS=0 → dev_queue_xmit(skb)
        │       │       经过 tc 流量控制（默认）
        │       │
        │       └── QDISC_BYPASS=1 → dev_hard_start_xmit(skb)
        │               绕过 tc（高性能模式）
        │
        ↓
  网卡驱动发送原始以太网帧

06 mmap 零拷贝

传统 recvmsg()（有拷贝）：

  内核 → skb_clone() → skb_queue → recvmsg() → memcpy → 用户态
                            ↑              ↑
                        内核态拷贝      用户态拷贝

mmap TPACKET（零拷贝）：

  内核 rx_ring ←──→ 用户态 rx_ring（共享内存）
       ↑                   ↑
  驱动直接写入         用户态直接读取
  无 memcpy！         无 memcpy！

TPACKET 版本

版本缓冲区特点
TPACKET_V1固定帧大小简单，但浪费内存（小包也占整帧）
TPACKET_V2可变帧大小支持时间戳精度提升
TPACKET_V3块模式批量处理，性能最优，tcpdump 默认
// 用户态代码示例
int fd = socket(AF_PACKET, SOCK_RAW, htons(ETH_P_ALL));

// 设置 TPACKET_V3
struct tpacket_req3 req = {
    .tp_block_size = 4096,
    .tp_block_nr   = 64,
    .tp_frame_size = 2048,
    .tp_frame_nr   = 128,
};
setsockopt(fd, SOL_PACKET, PACKET_VERSION, &v3, sizeof(v3));
setsockopt(fd, SOL_PACKET, PACKET_RX_RING, &req, sizeof(req));

// mmap 共享内存
void *ring = mmap(NULL, total_size, PROT_READ, MAP_SHARED, fd, 0);

// poll 等待新包
poll(&pfd, 1, -1);
// 直接读 ring，无 memcpy
