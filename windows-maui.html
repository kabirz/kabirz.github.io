01 概述

.NET MAUI（Multi-platform App UI）是微软官方推出的跨平台原生 UI 框架，由 Xamarin.Forms 演进而来。它允许使用单一 C# 代码库 + XAML 构建可在 Windows、macOS、iOS、Android 上运行的应用，每个平台都映射到原生控件而非自绘——Windows 端实际由 WinUI 3 承载，Android 用原生 View，iOS 用 UIKit，macOS 用 AppKit。

核心特性
  • 单项目多目标：一个 .csproj 用 net9.0-android/ios/maccatalyst/windows 多目标编译
  • 原生后端：每个控件映射到平台原生，外观行为与系统一致
  • Handler 取代 Renderer：轻量属性映射，按需定制
  • 统一资源：字体、图片、样式、主题跨平台共用
  • 跨平台 API：Microsoft.Maui 提供设备、文件、网络等抽象
  • 现代 .NET：AOT、Trimming、Source Generator、C# 12+
适用场景
  • 企业 LOB 应用（Windows + 移动端共用代码）
  • 跨平台工具/客户端，需原生外观
  • 从 Xamarin.Forms 升级迁移的项目
  • 微软生态 .NET 技能栈团队的跨平台方案
  • 对自绘（Flutter/Skia）不感冒、强调原生一致
定位：MAUI 是抽象层 + 平台后端而非渲染引擎。它不画 UI，而是把跨平台控件（如 Entry）映射到平台原生控件（WinUI 的 TextBox、iOS 的 UITextField）。这意味着你能免费获得原生无障碍、键盘、IME、动画与系统主题。

02 架构层次

MAUI 架构可分为三层：底层是 Microsoft.Maui（跨平台抽象，定义 IView/IApplication 等接口），中层是 Microsoft.Maui.Controls（XAML 控件层，如 Button/Entry），上层是平台实现（Windows=WinUI 3，iOS=UIKit，macOS=AppKit，Android=Android View）。控件与平台之间通过 Handler 解耦。

MauiProgram 启动

所有 MAUI 应用由 MauiProgram 配置构建，使用 MauiAppBuilder 注册字体、Handler、平台服务、依赖注入、第三方工具包。

启动入口 CreateMauiApp() 调用 UseMauiApp<TApp>()，TApp 是 Application 派生类，其 MainPage 是应用首页。

Handler vs Renderer：Xamarin.Forms 的 Custom Renderer 把整个控件包装成一个类，重；MAUI 改用 Handler + PropertyMapper，一个 Handler 处理一个跨平台控件，按属性逐项映射到原生调用，可按需扩展而无需继承。

03 XAML 与代码

MAUI 用标记语言 XAML 描述 UI 结构，用 C# code-behind 处理逻辑。一个页面派生自 ContentPage，其 Content 是单个根视图；控件派生自 View。XAML 顶部的 x:Class 指向同名 partial C# 类。

// ── MauiProgram.cs：应用入口 ──
public static class MauiProgram {
    public static MauiApp CreateMauiApp() {
        var builder = MauiApp.CreateBuilder();
        builder
            .UseMauiApp<App>()                    // 设置根应用类
            .ConfigureFonts(fonts => {
                fonts.AddFont("OpenSans-Regular.ttf", "OpenSansRegular");
                fonts.AddFont("OpenSans-SemiBold.ttf", "OpenSansSemiBold");
            })
            .ConfigureMauiHandlers(handlers => {
                // 注册/替换自定义 Handler
            });
        return builder.Build();
    }
}

// ── App.xaml.cs：根应用类 ──
public partial class App : Application {
    public App() {
        InitializeComponent();          // 加载 App.xaml 资源
        MainPage = new AppShell();      // 设主页面（Shell 路由）
    }
}

}
启动链：平台入口（如 MauiProgram.cs）→ CreateMauiApp → UseMauiApp<App> → App 构造函数 → 设置 MainPage → 框架创建 Handler → Handler 创建原生控件 → 显示。

05 Handler 模式

Handler 是 MAUI 架构的核心抽象。它把一个跨平台控件（IView）与一个平台原生控件桥接起来：每个跨平台控件对应一个 IViewHandler，Handler 内部用 PropertyMapper 把控件属性变更映射到原生控件的设置调用。这是 MAUI 相对 Xamarin.Forms Renderer 的最大改进。

Xamarin Renderer
  • 每个控件一个 Renderer 类，整体替换
  • 需派生 + 重写 OnElementChanged
  • 属性变更触发频繁的虚函数调用
  • 定制复杂：要继承整个 Renderer
MAUI Handler
  • PropertyMapper：属性 → 原生调用映射表
  • 按需调用：属性没变就不触发
  • ModifyMapper / AppendToMapping 局部扩展
  • 无需继承整个 Handler，侵入性低
  • 类型安全，性能更好

PropertyMapper 工作原理

// 每个 Handler 内部维护一个 PropertyMapper（简化）
public static PropertyMapper<IEntry, EntryHandler> EntryMapper = new() {
    [nameof(IEntry.Text)]            = MapText,
    [nameof(IEntry.TextColor)]    = MapTextColor,
    [nameof(IEntry.FontSize)]      = MapFontSize,
    [nameof(IEntry.IsPassword)]    = MapIsPassword,
    [nameof(IEntry.Keyboard)]      = MapKeyboard,
    // ... 还有 30+ 属性
};

// 框架仅在属性变更时调用对应 Map 方法
public static void MapText(EntryHandler handler, IEntry entry) {
    handler.PlatformView.Text = entry.Text;   // PlatformView 是 WinUI TextBox
}

定制原生控件：自定义 Entry 圆角边框

// ── 方式一：局部 AppendToMapping（推荐）──
public static class EntryExtensions {
    public static void UseRoundedEntry(this MauiAppBuilder builder) {
        EntryHandler.EntryMapper.AppendToMapping(
            "CustomBorder", (handler, entry) => {
#if WINDOWS
                var textBox = (Microsoft.UI.Xaml.Controls.TextBox)handler.PlatformView;
                textBox.BorderThickness = new Thickness(1);
                textBox.CornerRadius = 12;
                textBox.BorderBrush = Microsoft.UI.Xaml.Media.SolidColorBrush.CadetBlue;
#elif IOS
                handler.PlatformView.Layer.BorderWidth = 1;
                handler.PlatformView.Layer.CornerRadius = 12;
#endif
            });
    }
}

// 在 MauiProgram 中启用
builder.UseRoundedEntry();

// ── 方式二：在 XAML/code-behind 中临时取原生控件 ──
#if WINDOWS
var native = handler.PlatformView as Microsoft.UI.Xaml.Controls.TextBox;
native.CornerRadius = 12;
#endif
核心认知：Handler + PropertyMapper 是 MAUI 架构的灵魂。理解它你就理解了 MAUI 如何把一个 Button 同时变成 WinUI Button、UIKit UIButton、Android Button——而且各处定制互不污染。handler.PlatformView 拿到的是真正的原生控件，所有原生 API 都能用。

06 数据绑定与 MVVM

MAUI 内置数据绑定：用 XAML 的 {Binding} 表达式把视图属性连到 BindingContext 的属性。BindingContext 沿可视化树继承——父页设一次，子控件全用。注意：MAUI 不支持 WPF/UWP 的 {x:Bind}，统一用经典 {Binding}。

// ── 经典手写 ViewModel ──
public class CounterViewModel : INotifyPropertyChanged {
    private int _count;
    public int Count {
        get => _count;
        set { if (_count != value) { _count = value; OnPropertyChanged(); } }
    }

    public ICommand Increment { get; }

    public CounterViewModel() {
        Increment = new Command(() => Count++);
    }

    public event PropertyChangedEventHandler? PropertyChanged;
    protected void OnPropertyChanged([CallerMemberName] string? n = null)
        => PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(n));
}

CommunityToolkit.Mvvm（官方推荐）

实际项目几乎都用 CommunityToolkit.Mvvm（前名 Microsoft.Toolkit.Mvvm）。它用源生成器从 [ObservableProperty] 特性自动生成带通知的属性，从 [RelayCommand] 生成 ICommand，样板代码减少 80%。

public partial class TodoViewModel : ObservableObject {
    ObservableCollection<TodoItem> _items = new();

    public ObservableCollection<TodoItem> Items => _items;

    [ObservableProperty]
    private string? _newTodo;                       // 自动生成 NewTodo 属性 + INPC

    [ObservableProperty]
    [NotifyCanExecuteChangedFor(nameof(AddCommand))]
    private bool _canAdd;                            // 自动通知 AddCommand 重算 CanExecute

    [RelayCommand(CanExecute = nameof(CanExecuteAdd))]
    private void Add() {
        _items.Add(new TodoItem { Title = NewTodo });
        NewTodo = string.Empty;
    }
    private bool CanExecuteAdd() => !string.IsNullOrWhiteSpace(NewTodo);
}

<!-- XAML 绑定 -->
<ContentPage xmlns="..." x:Class="App.TodoPage">
  <ContentPage.BindingContext>
    <local:TodoViewModel />
  </ContentPage.BindingContext>

  <VerticalStackLayout Padding="20">
    <Entry Text="{Binding NewTodo, Mode=TwoWay}" Placeholder="新待办" />
    <Button Text="添加" Command="{Binding AddCommand}" />
    <CollectionView ItemsSource="{Binding Items}">
      <CollectionView.ItemTemplate>
        <DataTemplate>
          <Label Text="{Binding Title}" />
        </DataTemplate>
      </CollectionView.ItemTemplate>
    </CollectionView>
  </VerticalStackLayout>
</ContentPage>
Binding 关键概念
  • BindingContext：绑定上下文，沿树继承
  • Mode：OneWay / TwoWay / OneWayToSource / OneTime
  • Source：显式指定绑定源
  • Converter：值转换（IValueConverter）
  • StringFormat：格式化（如 {Binding Price, StringFormat='{}{0:C}'}）
  • FallbackValue / TargetNullValue：降级显示
集合通知
  • ObservableCollection<T>：增删时通知 UI
  • 仅项的增删通知；项内部属性变更需项实现 INPC
  • 绑到 CollectionView.ItemsSource 自动虚拟化
  • 批量操作用 ObservableRangeCollection（CommunityToolkit）
  • 跨线程改 UI：用 MainThread.BeginInvokeOnMainThread
无 x:Bind：MAUI 不支持 WPF/UWP 编译期 {x:Bind}——所有绑定都是运行时反射 {Binding}。性能敏感场景可启用 XamlCompilation（编译 XAML 到 IL）并避免在循环中频繁修改绑定属性。

07 导航

MAUI 提供两种导航模型：Shell（路由式，官方主推，支持 URI、查询参数、深链、Flyout+Tab）和 NavigationPage（传统栈式，PushAsync/PopAsync）。Shell 内部仍维护一个导航栈，但抽象层次更高。

Shell 路由导航

<<!-- AppShell.xaml：声明 Shell 结构 -->
<Shell x:Class="App.AppShell"
       xmlns="..." xmlns:x="...">
  <FlyoutItem Title="主页" Icon="home.svg">
    <ShellContent Title="首页" Route="home"
                  ContentTemplate="{DataTemplate local:HomePage}" />
  </FlyoutItem>

  <ShellContent Title="设置" Route="settings"
                ContentTemplate="{DataTemplate local:SettingsPage}" />
</Shell>

// ── 注册子路由（无 XAML 模板的动态页）──
public partial class AppShell : Shell {
    public AppShell() {
        InitializeComponent();
        Routing.RegisterRoute("details", typeof(DetailsPage));
        Routing.RegisterRoute("item/edit", typeof(EditItemPage));
    }
}

// ── 导航调用：URI + 查询参数 ──
await Shell.Current.GoToAsync("details");
await Shell.Current.GoToAsync($"details?id={item.Id}&title={item.Title}");

// ── 目标页用 QueryProperty 接收参数 ──
[QueryProperty(nameof(ItemId), "id")]
[QueryProperty(nameof(Title), "title")]
public partial class DetailsPage : ContentPage {
    public string? ItemId { set => Load(value); }
    public new string? Title { set => SetTitle(value); }
}

// ── 后退 ──
await Shell.Current.GoToAsync("..");            // 弹回上一页
await Shell.Current.GoToAsync("../../home");  // 弹多层 + 跳转

NavigationPage 传统栈导航

// 适用于不用 Shell 的简单场景
var navPage = new NavigationPage(new HomePage());
App.Current.MainPage = navPage;

// 推入新页
await Navigation.PushAsync(new DetailsPage(id));

// 弹出
await Navigation.PopAsync();
await Navigation.PopToRootAsync();         // 回根

// 模态（覆盖整屏，自带动画）
await Navigation.PushModalAsync(new LoginPage());
await Navigation.PopModalAsync();
Shell 适用场景
  • 移动端 Flyout + Tab 布局
  • 深链 / URI 路由
  • 统一桌面端（WinUI 侧栏）与移动
  • 新项目首选
FlyoutPage 主从
  • 桌面端典型主从视图
  • FlyoutPage 有 Flyout + Detail
  • 移动端自动转抽屉
  • WinUI 映射到 SplitView

08 平台特定代码

MAUI 用多目标（multi-target）把同一个项目编译成 4 套二进制：net9.0-android、net9.0-ios、net9.0-maccatalyst、net9.0-windows。在共享代码里用 #if 条件编译区分平台；也可以把平台专属代码放在 Platforms/ 子文件夹，由编译系统按目标自动包含。

// 共享文件中的条件编译
public string GetPlatformName() {
#if WINDOWS
    return Environment.OSVersion.ToString();     // Win32 路径
#elif ANDROID
    return Android.OS.Build.Model;             // Android 路径
#elif IOS
    return UIKit.UIDevice.CurrentDevice.Name;
#elif MACCATALYST
    return AppKit.NSHost.Current.Name;
#else
    return "Unknown";
#endif
}

// ── 在 Windows 平台调 Win32 API（置顶窗口）──
#if WINDOWS
using WinRT.Interop;
using System.Runtime.InteropServices;

public static class WindowExtensions {
    [DllImport("user32.dll")]
    static extern bool SetWindowPos(IntPtr hWnd, IntPtr hAfter, int x, int y, int cx, int cy, uint flags);

    public static void BringToTop(this Microsoft.UI.Xaml.Window window) {
        var hwnd = WindowNative.GetWindowHandle(window);
        const uint HWND_TOPMOST = 0xFFFFFFFF;
        const uint SWP_NOMOVE = 0x0002;
        const uint SWP_NOSIZE = 0x0001;
        SetWindowPos(hwnd, (IntPtr)HWND_TOPMOST, 0, 0, 0, 0, SWP_NOMOVE | SWP_NOSIZE);
    }
}
#endif

DependencyService 跨平台服务

// 共享接口
public interface IPlatformInfo {
    string GetDeviceId();
}

// 平台实现（自动注册）
[assembly: Dependency(typeof(Platforms.Windows.WindowsInfo))]
namespace Platforms.Windows;

public class WindowsInfo : IPlatformInfo {
    public string GetDeviceId() => Environment.MachineName;
}

// 共享侧解析

// .NET 8+ 也可用 MauiAppBuilder 的依赖注入：

09 资源与主题

MAUI 把样式、字体、图片、颜色、应用主题统一放在跨平台 XAML 资源字典中。每个 MAUI 应用根 App.xaml 有 Application.Resources，里面定义全局样式；也可在 Resources/Styles.xaml 拆分按需 Merge。

样式与资源

<!-- App.xaml：全局资源 -->
<Application x:Class="App.App">
  <Application.Resources>
    <ResourceDictionary>
      <Color x:Key="Primary">#512BD4</Color>
      <Color x:Key="Secondary">#DFD8F7</Color>
      <x:Double x:Key="FontSizeMedium">16</x:Double>

      <Style x:Key="PrimaryButton" TargetType="Button">
        <Setter Property="BackgroundColor" Value="{StaticResource Primary}" />
        <Setter Property="TextColor" Value=White" />
        <Setter Property="FontSize" Value="{StaticResource FontSizeMedium}" />
        <Setter Property="CornerRadius" Value=8" />
      </Style>
    </ResourceDictionary>
  </Application.Resources>
</Application>

<!-- 使用 -->
<Button Text="OK" Style="{StaticResource PrimaryButton}" />
<Label TextColor="{DynamicResource Primary}" />
StaticResource vs DynamicResource
  • StaticResource：编译期解析，查找一次
  • DynamicResource：运行期监听，主题切换时自动更新
  • 主题色用 DynamicResource
  • 静态常量用 StaticResource（性能更好）
主题
  • Application.Current.UserAppTheme：强制指定
  • 值：Light / Dark / Unspecified（跟系统）
  • AppThemeColor / AppThemeResource 双色资源
  • 系统主题变更事件 Application.RequestedThemeChanged

暗黑主题示例

<Application.Resources>
  <ResourceDictionary>
    <Color x:Key="PageBackground">White</Color>
    <Color x:Key="TextPrimary">#1F1F1F</Color>
  </ResourceDictionary>
</Application.Resources>

<Application.Resources>

10 完整示例

下面是一个最小但完整的 MAUI 应用骨架，体现 MauiProgram + Shell + 页面 + ViewModel + 数据绑定 + 命令的协作。Windows 端编译为 WinUI 3 应用直接双击运行。

// ── MauiProgram.cs ──
public static class MauiProgram {
    public static MauiApp CreateMauiApp() {
        var builder = MauiApp.CreateBuilder();
        builder.UseMauiApp<App>()
               .ConfigureFonts(f => f.AddFont("OpenSans-Regular.ttf", "OpenSansRegular"))
               .UseMauiCommunityToolkit();
        builder.Services.AddSingleton<CounterViewModel>();
        builder.Services.AddTransient<CounterPage>();
        return builder.Build();
    }
}

// ── App.xaml.cs ──
public partial class App : Application {
    public App() {
        InitializeComponent();
        MainPage = new AppShell();
    }
}

// ── AppShell.xaml.cs ──
public partial class AppShell : Shell {
    public AppShell() {
        InitializeComponent();
        Routing.RegisterRoute("counter", typeof(CounterPage));
        Shell.Current.GoToAsync("counter");
    }
}

// ── CounterViewModel.cs ──
public partial class CounterViewModel : ObservableObject {
    [ObservableProperty]
    private int _count;

    [ObservableProperty]
    private string _message = "点击下方按钮开始计数";

    [RelayCommand]
    private void Increment() {
        Count++;
        Message = $"当前计数：{Count}";
    }

    [RelayCommand]
    private async Task Reset() {
        Count = 0;
        Message = "已重置";
        await Application.Current!.MainPage!.DisplayAlert("提示", "计数已重置", "OK");
    }
}

11 对比 WPF/Avalonia/Uno/Flutter

同为 UI 框架，几者定位截然不同。WPF 是单平台（仅 Windows）.NET 富客户端，Avalonia 是第三方跨平台 .NET 框架（自绘），Uno 把 WinUI 的 XAML 带到 iOS/Android/Web/macOS，Flutter 是非 .NET 的自绘方案。

框架语言渲染跨平台微软背书定位
.NET MAUIC#/XAML原生控件映射Win/macOS/iOS/Android官方跨平台原生 UI
WPFC#/XAMLDirectX 自绘仅 Windows官方Windows 富客户端
AvaloniaC#/XAMLSkia 自绘Win/macOS/Linux/iOS/Android/Web第三方跨平台自绘
Uno PlatformC#/XAML原生 / SkiaWin/macOS/iOS/Android/Web/WebAssembly第三方WinUI XAML 跨平台
FlutterDartSkia/Impeller 自绘全平台 + WebGoogle非 .NET 自绘
MAUI 优势
  • 微软官方支持，长期路线图明确
  • 原生外观行为，免费获得系统主题、IME、无障碍
  • 同一代码覆盖 Win/macOS/iOS/Android
  • 与 .NET 生态（EF Core、ASP.NET）无缝
  • Handler 模式易于深度定制原生能力
MAUI 短板
  • 不支持 Linux 与 Web
  • Windows 上仍是 WinUI 3 包，部署有一定门槛
  • 启动与包体相比原生有额外开销
  • 复杂动画/特效需靠各平台原生实现
  • 无 {x:Bind} 编译期绑定
选型：微软生态 .NET 团队的跨平台原生 UI 首选 MAUI。需要 Linux/Web 或极致自绘一致性选 Avalonia。WPF 适合纯 Windows 富客户端。已有 WinUI/UWP 经验且要 Web/Linux 选 Uno。
