01 概述

futex（Fast Userspace muTEX）是 Linux 实现高性能互斥锁的核心机制。它在无竞争时仅使用原子操作（CAS），不进入内核态；只有在竞争时才通过系统调用挂起线程。pthread_mutex、pthread_cond、pthread_rwlock 的底层都依赖 futex。

02 快速路径与慢速路径

快速路径（无竞争，~10ns）：

  用户态 pthread_mutex_lock()
        │
        ├── CAS 尝试将 futex word 从 0 → 1
        │   成功 → 获取锁，直接返回！不进内核
        │
        └── CAS 失败 → 进入慢速路径

慢速路径（有竞争，~1000ns）：

  futex(FUTEX_WAIT, ...) → 系统调用
        │
        ↓
  内核检查 futex word 值
        │
        ├── 值仍为 1 → 将当前线程挂到 futex 等待队列
        │               schedule() → 让出 CPU
        │
        └── 值已变 → 快速返回（锁已被释放）

  另一线程释放锁：
        │
        ├── CAS 将 futex word 从 1 → 0
        │
        ├── futex(FUTEX_WAKE, 1, ...) → 唤醒等待线程
        │   内核从等待队列取出一个线程 → wake_up_process()
        │
        └── 被唤醒的线程重新 CAS 获取锁

03 核心数据结构

// futex word — 用户态 32 位整数
// 状态：0=未锁, 1=已锁, 2=已锁且有等待者
u32 futex_word = 0;

// futex_q — 内核等待队列条目
struct futex_q {
    struct plist_node   node;        // 优先级队列节点
    struct rt_mutex_waiter *rt_waiter; // PI futex
    struct futex_hash_bucket *hb; // 所属哈希桶
    u32              key;         // futex key（地址 + mm）
    struct futex_pi_state *pi_state; // PI 状态
};

// futex_key — 唯一标识一个 futex
struct futex_key {
    union {
        struct task_struct *rgctxid; // robust list
        struct mm_struct *mm;          // 进程地址空间
    };
    unsigned long  address;   // futex 在用户态的地址
    unsigned int   offset;    // 偏移量
};

// FUTEX_WAIT：检查 *uaddr == val，如果是则挂起
// FUTEX_WAKE：唤醒 val 个等待者
// FUTEX_REQUEUE：从一个 futex 移动到另一个
// FUTEX_CMP_REQUEUE：带比较的 requeue

// 用户态使用（简化版 pthread_mutex_lock）
static inline void mutex_lock(int *lock) {
    if (cmpxchg(lock, 0, 1) != 0) {  // 快速路径
        if (*lock == 2)
            futex_wait(lock, 2);     // 已有等待者，直接等待
        while (cmpxchg(lock, 0, 2) != 0)
            futex_wait(lock, 2);     // 慢速路径
    }
}

// 用户态使用（简化版 pthread_mutex_unlock）
static inline void mutex_unlock(int *lock) {
    if (atomic_dec(lock) != 1) {  // 不是最后一个
        *lock = 0;
        futex_wake(lock, 1);    // 唤醒一个等待者
    }
}

为什么 futex 快？
99% 的锁操作是无竞争的（只有当前线程在使用锁）。futex 的快速路径仅使用 CAS 原子操作（~10ns），不进入内核态。只有在有竞争时才通过系统调用（~1000ns）挂起线程。这比纯内核态的 mutex 快 100 倍。
