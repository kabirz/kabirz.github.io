01 概述

Linux 提供了丰富的进程间通信（IPC）机制。本文全景梳理所有 IPC 机制的关系、适用场景和性能特征。

02 机制分类

Linux IPC 机制全景图：

  ┌─────────────────────────────────────────────────────────┐
  │  管道（Pipe）— 字节流，单向                              │
  │  ├── pipe()           — 匿名管道（父子进程）             │
  │  └── FIFO (mkfifo)   — 命名管道（任意进程）             │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │  UNIX Domain Socket — 字节流/数据报，双向               │
  │  ├── SOCK_STREAM     — 类似 TCP（可靠有序）             │
  │  ├── SOCK_DGRAM      — 类似 UDP（无连接）               │
  │  └── SCM_RIGHTS      — 文件描述符传递                   │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │  共享内存 — 最快的 IPC，需同步                           │
  │  ├── mmap + MAP_SHARED — 文件/匿名共享内存              │
  │  ├── POSIX shm         — /dev/shm 共享内存              │
  │  └── SysV shm          — 传统共享内存                    │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │  消息传递 — 有边界的消息                                 │
  │  ├── POSIX 消息队列   — 标准接口                         │
  │  └── SysV 消息队列    — 传统接口                         │
  └─────────────────────────────────────────────────────────┘

  ┌─────────────────────────────────────────────────────────┐
  │  同步机制 — 配合 IPC 使用                                │
  │  ├── Signal           — 异步通知                         │
  │  ├── Semaphore        — 计数信号量                       │
  │  └── Futex            — 快速用户空间互斥锁               │
  └─────────────────────────────────────────────────────────┘

03 选择指南

场景推荐 IPC原因
Shell 管道pipe / FIFO简单，标准
父子进程通信pipe最简单
任意进程通信UNIX Domain Socket灵活，支持 fd 传递
高性能数据共享共享内存 + 同步零拷贝
异步事件通知Signal简单通知
生产者-消费者消息队列有消息边界
数据库共享内存 + futex最高性能
