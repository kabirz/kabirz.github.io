01 概述

Docker Compose 是多容器应用编排工具。用 YAML 文件定义应用的所有服务、网络、卷，一键启动/停止整个应用栈。

历史：

版本实现特点
Fig (2013)PythonCompose 的前身
docker-compose 1.xPython独立 CLI
docker-compose v2 (2021+)Go集成到 docker CLI 的 plugin

典型场景：

  • 开发环境：一键拉起后端 + DB + 缓存 + 代理
  • CI/CD：测试环境
  • 小型生产（配合 Swarm）
  • 演示和文档

最小示例：

# docker-compose.yml
services:
  web:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: secret
    volumes:
      - dbdata:/var/lib/postgresql/data

volumes:
  dbdata:

# 启动
$ docker compose up -d

# 状态
$ docker compose ps
NAME                SERVICE   STATUS     PORTS
myapp-web-1         web       Up         0.0.0.0:80->80/tcp
myapp-db-1          db        Up         5432/tcp

# 日志
$ docker compose logs -f web

# 停止
$ docker compose down

# 停止并删卷
$ docker compose down -v

02 Compose 文件格式

// Compose 文件顶层结构（v3+）：
{
  // 版本（v2 后可选）
  "version": "3.9",

  // 服务定义（核心）
  "services": {
    "<service_name>": { ServiceConfig },
    ...
  },

  // 自定义网络
  "networks": {
    "<network_name>": { NetworkConfig },
    ...
  },

  // 自定义卷
  "volumes": {
    "<volume_name>": { VolumeConfig },
    ...
  },

  // Swarm secrets
  "secrets": {
    "<secret_name>": { SecretConfig },
    ...
  },

  // Swarm configs
  "configs": {
    "<config_name>": { ConfigConfig },
    ...
  }
}

多文件合并规则：

# Compose 文件优先级（从低到高）：
# 1. 默认：docker-compose.yml + docker-compose.override.yml
# 2. -f 指定的多个文件，按顺序合并
# 3. COMPOSE_FILE 环境变量
# 4. -f 覆盖

# 多环境示例：
docker compose -f docker-compose.yml -f docker-compose.prod.yml up

# docker-compose.yml - 基础
# docker-compose.prod.yml - 生产 override
#   覆盖/扩展基础配置

# 合并规则：
# - services: 同名服务合并配置
# - networks: 同名网络合并
# - 大部分字段：override 覆盖
# - 列表字段（如 ports）：append
# - 环境变量：合并（override 优先）

# COMPOSE_FILE：
export COMPOSE_FILE=docker-compose.yml:docker-compose.dev.yml
docker compose up

变量插值：

# .env 文件（自动加载）
POSTGRES_PASSWORD=secret
APP_PORT=8080

# docker-compose.yml
services:
  db:
    image: postgres
    environment:
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
  web:
    ports:
      - "${APP_PORT}:80"

# 优先级（从高到低）：
# 1. shell environment
# 2. .env file
# 3. Dockerfile ARG
# 4. 默认值

# 默认值语法：
image: ${IMAGE:-nginx:latest}
port: ${PORT:?required}

03 服务定义

services:
  web:
    # 镜像（image 或 build 二选一）
    image: nginx:alpine
    # OR 构建
    build:
      context: ./web
      dockerfile: Dockerfile
      args:
        VERSION: 1.0
      target: production
      cache_from:
        - myapp:cache
      labels:
        - "com.myapp.description=Web"

    # 容器名（默认 <project>-<service>-<index>）
    container_name: web-server

    # 副本数（仅 Swarm 启用）
    deploy:
      replicas: 3
    
    # 端口
    ports:
      - "80:80"
      - "443:443"
      - target: 8080          # 长语法
        published: "8080"
        protocol: tcp
        mode: host
    
    # 暴露端口（仅容器间）
    expose:
      - "8080"
    
    # 卷
    volumes:
      - ./html:/usr/share/nginx/html:ro
      - dbdata:/var/lib/mysql
      - type: volume
        source: mydata
        target: /data
        read_only: true
    
    # 环境
    environment:
      - DEBUG=1
      - DB_HOST=db
      RAILS_ENV: production   # 字典语法
    
    # 从文件加载环境
    env_file:
      - .env
      - .env.production
    
    # 依赖（启动顺序）
    depends_on:
      - db
      - cache
      # 健康检查：
      - db:
          condition: service_healthy
      - cache:
          condition: service_started
    
    # 网络配置
    networks:
      - frontend
      - backend:
          aliases:
            - my-service
    
    # 重启策略
    restart: always  # no | always | on-failure | unless-stopped
    
    # 健康检查
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
    
    # 资源限制（仅 Swarm）
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 512M
        reservations:
          cpus: '0.25'
          memory: 256M
    
    # 本地资源限制（compose v2 支持）
    mem_limit: 512m
    cpus: 0.5
    
    # 用户/权限
    user: "1000:1000"
    cap_add:
      - NET_ADMIN
    cap_drop:
      - ALL
    privileged: false
    read_only: true
    
    # 命令
    command: ["nginx", "-g", "daemon off;"]
    entrypoint: /custom-entrypoint.sh
    
    # 工作目录
    working_dir: /app
    
    # 标签
    labels:
      - "com.myapp.role=web"
    
    # 日志
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"
    
    # sysctls
    sysctls:
      net.core.somaxconn: 1024
    
    # ulimits
    ulimits:
      nproc: 65535
      nofile:
        soft: 20000
        hard: 40000
    
    # tmpfs
    tmpfs:
      - /run
      - /tmp
    
    # TTY/STDIN
    tty: true
    stdin_open: true
    
    # extra_hosts（/etc/hosts 添加）
    extra_hosts:
      - "host.docker.internal:host-gateway"
    
    # DNS
    dns:
      - 8.8.8.8
      - 8.8.4.4
    dns_search: example.com
    
    # 外部链接（连接非本 compose 的容器）
    external_links:
      - some_container
    
    # 网络模式
    network_mode: bridge  # bridge | host | none | service:[service name] | container:[name/id]
    
    # PID 共享
    pid: "service:db"  # 或 "host"
    
    # IPC 共享
    ipc: "service:cache"

04 卷与网络

volumes:
  # 默认（自动创建）
  dbdata:
  
  # 命名卷（外部存在）
  existing_volume:
    external: true
  
  # driver 配置
  nfs_data:
    driver: local
    driver_opts:
      type: nfs
      o: addr=nfs.local,rw
      device: ":/share"
  
  # 标签
  labeled_vol:
    labels:
      - "com.myapp.type=data"

networks:
  # 默认 bridge
  frontend:
  
  # 自定义子网
  backend:
    driver: bridge
    ipam:
      driver: default
      config:
        - subnet: 172.20.0.0/16
          ip_range: 172.20.0.0/24
          gateway: 172.20.0.1
  
  # 外部网络
  existing:
    external: true
    name: shared_net
  
  # overlay（swarm）
  overlay_net:
    driver: overlay
    attachable: true   # 允许独立容器加入
  
  # macvlan
  macvlan_net:
    driver: macvlan
    driver_opts:
      parent: eth0
    ipam:
      config:
        - subnet: 192.168.1.0/24

05 部署与资源

services:
  web:
    image: nginx
    deploy:
      # 副本数
      mode: replicated  # replicated | global
      replicas: 3
      
      # 资源
      resources:
        limits:
          cpus: '1.0'
          memory: 1G
        reservations:
          cpus: '0.5'
          memory: 512M
      
      # 重启策略
      restart_policy:
        condition: on-failure
        delay: 5s
        max_attempts: 3
        window: 120s
      
      # 更新策略
      update_config:
        parallelism: 2
        delay: 10s
        failure_action: rollback
        monitor: 30s
        max_failure_ratio: 0.3
        order: start-first  # start-first | stop-first
      
      # 回滚策略
      rollback_config:
        parallelism: 0
        order: stop-first
      
      # 放置约束
      placement:
        constraints:
          - node.role == manager
          - engine.labels.ssd == true
        preferences:
          - spread: node.labels.zone
      
      # 端口模式
      endpoint_mode: vip  # vip | dnsrr
      
      # 标签
      labels:
        - "com.myapp.service=web"

非 Swarm 警告：

# deploy 字段仅在 swarm 模式生效（docker stack deploy）
# 普通 compose up 时会被忽略（除 resources 限制）

# 单机资源限制（compose v2 真正生效）：
services:
  web:
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 512M
    # 在 compose v2 + Linux 上能生效

06 Profile 与多环境

services:
  web:
    image: nginx
    profiles:
      - prod
      - staging
  
  debug:
    image: nginx-debug
    profiles:
      - debug

# 启动 prod 服务
docker compose --profile prod up

# 启动 debug 服务
docker compose --profile debug up

# 启动所有服务（所有 profile）
docker compose --profile "*" up

# 多 profile
docker compose --profile prod --profile monitor up

# 没指定 profile 的服务默认启动

多环境模式（推荐）：

# docker-compose.yml（基础）
# docker-compose.override.yml（开发默认）
# docker-compose.prod.yml（生产）

# 调用：
docker compose up                      # 用 override
docker compose -f docker-compose.yml \
               -f docker-compose.prod.yml up

# .env 区分
# .env      - 共享
# .env.dev  - 开发
# .env.prod - 生产
docker compose --env-file .env.prod up

07 Compose v2 架构

Compose v2 完全用 Go 重写，集成到 docker CLI 作为 plugin：

// 调用方式：
docker compose up    # 新（v2）
docker-compose up    # 老（v1，Python）

// Plugin 位置：
// /usr/lib/docker/cli-plugins/docker-compose
// 或 ~/.docker/cli-plugins/docker-compose

// Compose v2 优势：
//   1. 速度（Go 编译，比 Python 快 10 倍）
//   2. 与 docker CLI 集成
//   3. 更好的错误信息
//   4. 与 BuildKit 集成
//   5. 持续维护（v1 已停止）

Compose v2 内部架构：

08 主要命令

命令作用
up启动整个应用（构建、创建、启动）
down停止并移除容器、网络（-v 同时删卷）
start启动已存在的容器
stop停止容器（不删除）
restart重启容器
pause / unpause暂停 / 恢复
build构建或重建服务镜像
pull拉取服务镜像
push推送服务镜像
logs查看日志（-f 跟随）
ps列出容器
top查看进程
exec在运行容器执行命令
run启动一次性容器（如迁移）
scale扩展服务（swarm 或多副本）
events实时事件流
config校验并显示合并后的配置
images列出使用的镜像
port查看端口映射
cp在容器和宿主间复制文件

常用选项：

docker compose up -d                 # 后台运行
docker compose up --build            # 强制重新构建
docker compose up --force-recreate   # 强制重建容器
docker compose up --scale web=3      # 启动 3 个 web 实例

docker compose exec web sh           # 进入容器
docker compose run --rm web rake db:migrate  # 一次性任务

docker compose logs -f --tail=100 web

docker compose ps --services         # 仅列服务名
docker compose ps -a                 # 包含已停止

docker compose config                # 显示合并后配置（含变量替换）
docker compose convert               # 同上（v2 名字）

09 Swarm 模式

Compose 是 Swarm 的声明式输入。可以部署到集群：

# 启用 swarm
docker swarm init
docker swarm join --token ... <manager_ip>:2377

# 部署 stack
docker stack deploy -c docker-compose.yml myapp

# 查看
docker stack services myapp
docker service ls

# 删除
docker stack rm myapp

Swarm 特有概念：

// 1. Stack
//    一组相关服务的集合
//    来自 compose 文件

// 2. Service
//    服务模板（如 web）
//    可有多个副本（task）

// 3. Task
//    单个容器实例
//    调度到 node 上

// 4. Node
//    swarm 集群的机器
//    manager / worker

// Compose → Swarm：
//   Compose 用于开发/小生产
//   Swarm 用于集群部署
//   K8s 是更高级的集群方案（与 Swarm 同类）

// Swarm vs K8s：
//   Swarm: Docker 内置，简单
//   K8s:   功能丰富，生态广
//   现代生产几乎都选 K8s
