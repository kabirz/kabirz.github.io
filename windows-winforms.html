01 概述

Windows Forms（WinForms）是微软在 .NET 平台上提供的托管桌面 GUI 框架。它以 System.Windows.Forms 命名空间为核心，把 Win32 公共控件库（ComCtl32.dll）与 GDI+ 包装成一组 .NET 类，让 C#/VB.NET 开发者用设计器拖拽 + 事件处理器的方式快速构建表单型应用。自 2002 年随 .NET Framework 1.0 发布以来，它一直是企业内部工具、数据库前端、工业上位机领域最高产的桌面方案。

核心特性
  • Win32 薄封装：每个控件封装一个 HWND，消息驱动忠实映射
  • 设计器优先：VS 拖拽即生成 InitializeComponent
  • 事件驱动：标准 EventHandler 委托模式
  • 属性即样式：Text/Font/BackColor 是一等公民
  • 稳定向后兼容：20 年前的程序在 .NET 9 上常可直接运行
  • 跨平台尝试：.NET 7+ 通过 Mono 移植支持 Linux/macOS
适用场景
场景契合度
企业内部数据表单极佳
工业上位机/HMI极佳
遗留系统改造极佳
工具类小应用好
富媒体/复杂动画弱（用 WPF）
跨平台桌面弱（用 Avalonia/MAUI）
定位：WinForms 是“用最少的代码做出一个能用的 Windows 表单”的方案。它的设计目标是生产力而非视觉表现力——这与 WPF（声明式 + 矢量渲染）形成鲜明对比。

02 核心架构

WinForms 的所有可视元素都派生自 System.Windows.Forms.Control，它封装一个 HWND（窗口句柄），并把 Win32 消息循环映射到 .NET 事件。最外层的入口是 System.Windows.Forms.Application，它托管了消息泵（GetMessage/TranslateMessage/DispatchMessage）。

控件树模型

WinForms 是严格的父子树：每个 Control 有零或一个 Parent，Controls 集合持有子控件。父控件销毁时子控件自动销毁；父控件移动时子控件坐标在父客户区内计算。

属性 / 事件 双驱动

WinForms 没有命令绑定，所有交互靠属性赋值与事件订阅。设置 Text 立即触发 SetWindowText；订阅 button.Click += handler 即把委托挂到底层消息处理路径上。

HWND 即生命：Control.Handle 是懒创建的——首次访问或调用 CreateControl 时才真正调用 CreateWindowEx。在此之前，许多依赖句柄的属性/方法会强制触发创建。批量初始化时用 SuspendLayout() 包住可显著提升启动速度。

03 应用启动

WinForms 应用的入口是 static void Main()，必须标注 [STAThread]——这是 OLE/COM 公共对话框、剪贴板、拖放子系统的硬性要求。Application.Run 启动消息循环直到主窗体关闭；ApplicationConfiguration.Initialize()（.NET 6+ 的自动生成入口）负责高 DPI、默认字体、应用程序级设置。

using System;
using System.Windows.Forms;

internal static class Program
{
    [STAThread]
    static void Main()
    {
        // .NET 6+ 由项目模板生成：高 DPI、默认字体 Segoe UI 9pt
        ApplicationConfiguration.Initialize();

        // 启动消息循环，主窗体关闭时退出
        Application.Run(new MainForm());
    }
}

ApplicationContext：自定义生命周期

Application.Run 有一个重载接受 ApplicationContext。当应用没有主窗体（如纯托盘程序、单实例守护进程）时，可继承 ApplicationContext 并在合适时机调用 ExitThread。

class TrayContext : ApplicationContext
{
    private readonly NotifyIcon _tray;

    public TrayContext()
    {
        _tray = new NotifyIcon
        {
            Icon = new Icon("app.ico"),
            Visible = true,
            Text = "后台服务",
            ContextMenuStrip = BuildMenu()
        };
    }

    private ContextMenuStrip BuildMenu()
    {
        var menu = new ContextMenuStrip();
        menu.Items.Add("退出", null, (_, _) => {
            _tray.Visible = false;
            Application.Exit();          // 触发 ExitThread → 退出循环
        });
        return menu;
    }

    protected override void Dispose(bool disposing) { /* 释放 _tray */ }
}

// Program.Main
Application.Run(new TrayContext());

Application 关键 API

API作用
Application.Run(Form)启动消息循环，主窗体关闭即退出
Application.Run(ApplicationContext)自定义生命周期，无固定主窗体
Application.Exit()优雅退出：发 WM_QUIT，处理完当前消息后退出
Application.ExitThread()仅退出当前线程的消息循环
Application.DoEvents()抽出当前队列中的消息处理一遍（慎用）
Application.AddMessageFilter在分发前拦截消息（实现快捷键/全局过滤）
Application.SetUnhandledExceptionMode控制未捕获异常的派发方式
Application.ThreadExceptionUI 线程未捕获异常的全局事件
Application.Idle消息队列空闲时触发（替代 MFC 的 OnIdle）
DoEvents 的陷阱：Application.DoEvents() 会重入消息循环，可能在不期望的时机触发事件处理器（如按钮再次被点击），造成重入 bug。现代代码应使用 async/await 或 Task.Run 替代。

示例：一个简单的录入窗体

public partial class EditForm : Form
{
    private readonly Label      _lblName;
    private readonly TextBox    _txtName;
    private readonly CheckBox   _chkActive;
    private readonly Button     _btnOK;
    private readonly Button     _btnCancel;

    public string NameValue => _txtName.Text;
    public bool   IsActive => _chkActive.Checked;

    public EditForm()
    {
        Text = "编辑用户";
        FormBorderStyle = FormBorderStyle.FixedDialog;
        StartPosition   = FormStartPosition.CenterParent;
        MaximizeBox     = false;
        ClientSize      = new Size(320, 160);

        _lblName = new Label { Text = "姓名(&N):", Location = new Point(12, 15), AutoSize = true };
        _txtName = new TextBox { Location = new Point(80, 12), Width = 220 };
        _chkActive = new CheckBox { Text = "启用", Location = new Point(80, 45), AutoSize = true, Checked = true };

        _btnOK = new Button { Text = "确定", DialogResult = DialogResult.OK, Location = new Point(144, 108), Size = new Size(75, 28) };
        _btnCancel = new Button { Text = "取消", DialogResult = DialogResult.Cancel, Location = new Point(225, 108), Size = new Size(75, 28) };

        AcceptButton = _btnOK;       // 回车触发 OK
        CancelButton = _btnCancel;   // Esc 触发 Cancel

        Controls.AddRange(new Control[] { _lblName, _txtName, _chkActive, _btnOK, _btnCancel });

        _btnOK.Click += (_, _) =>
        {
            if (string.IsNullOrWhiteSpace(_txtName.Text))
            {
                MessageBox.Show(this, "姓名不能为空", "提示", MessageBoxButtons.OK, MessageBoxIcon.Warning);
                this.DialogResult = DialogResult.None;   // 阻止关闭
            }
        };
    }
}

// 调用方
using var dlg = new EditForm();
if (dlg.ShowDialog(this) == DialogResult.OK)
{
    Console.WriteLine($"name={dlg.NameValue} active={dlg.IsActive}");
}
模态 vs 非模态：form.ShowDialog() 阻塞调用直到关闭（自动调用 Dispose），用于对话框；form.Show() 立即返回，生命周期需自行管理（关闭不等于释放，需显式 Dispose）。

05 布局系统

WinForms 的默认坐标模型是绝对像素定位——控件 Location/Size 直接对应客户区像素。这在小屏与单一 DPI 下没问题，但分辨率/DPI 变化时会错乱。布局系统提供两层补偿：Anchor/Dock（锚定/停靠，基于父容器）与布局容器（TableLayoutPanel / FlowLayoutPanel）。

Anchor 与 Dock

// Anchor：到父容器四条边的距离在父调整时保持不变
// 默认是 Top|Left，即“贴左上角”
_txtName.Anchor = AnchorStyles.Top | AnchorStyles.Left | AnchorStyles.Right;
// 父宽度变大 → 文本框右侧跟随扩展，左侧固定

// Dock：整个控件贴向父容器某一边，并占满该方向的剩余空间
_menu.Dock = DockStyle.Top;
_status.Dock = DockStyle.Bottom;
_grid.Dock = DockStyle.Fill;       // ★ 填满剩余区域

布局容器对比

容器布局规则典型用途
Panel容器，子控件自由定位分组容器、滚动区
GroupBox带标题边框的 Panel单选按钮分组
SplitContainer两个面板中间可拖拽分隔条主从视图、资源管理器布局
TableLayoutPanel行列网格，按百分比/绝对/自动分配表单对齐、响应式行高列宽
FlowLayoutPanel子控件按方向流式排列，超出自动换行动态项集合、工具条
TabControl标签页切换，每页独立容器设置面板、分类表单

TableLayoutPanel：WinForms 的网格布局利器

var table = new TableLayoutPanel
{
    Dock = DockStyle.Fill,
    ColumnCount = 2,
    RowCount = 4,
};
table.ColumnStyles.Add(new ColumnStyle(SizeType.Absolute, 80));   // 第 1 列固定 80px
table.ColumnStyles.Add(new ColumnStyle(SizeType.Percent, 100)); // 第 2 列占剩余
table.RowStyles.Add(new RowStyle(SizeType.AutoSize));
table.RowStyles.Add(new RowStyle(SizeType.AutoSize));
table.RowStyles.Add(new RowStyle(SizeType.Percent, 100)); // 第 3 行吃掉纵向空间
table.RowStyles.Add(new RowStyle(SizeType.AutoSize));

// 通过 SetChildIndex + SetColumn/SetRow 定位子控件
table.Controls.Add(new Label { Text = "姓名" }, 0, 0);
table.Controls.Add(_txtName, 1, 0);
table.Controls.Add(new Label { Text = "备注" }, 0, 1);
table.Controls.Add(_txtMemo, 1, 1);
table.Controls.Add(_grid, 0, 2);
table.SetColumnSpan(_grid, 2);    // 跨两列
table.Controls.Add(_btnPanel, 0, 3);
table.SetColumnSpan(_btnPanel, 2);

Padding/Margin/AutoSize

  • Margin：本控件外缘到相邻控件/容器的最小间距（布局容器生效）
  • Padding：本控件内边距（容器生效；某些控件如 GroupBox 在设计器中体现）
  • AutoSize：为 true 时控件按内容自适应；常配合 AutoSizeMode（GrowOnly / GrowAndShrink）
设计建议：所有用户可调整大小的窗体都应使用 TableLayoutPanel + Dock/Anchor，避免绝对坐标堆砌。绝对坐标只用在固定尺寸对话框（FormBorderStyle = FixedDialog）。

06 事件驱动模型

WinForms 的事件基于 .NET 委托（delegate）与多播事件机制。EventHandler 与泛型版 EventHandler<TEventArgs> 是几乎所有 UI 事件的标准签名：(object sender, EventArgs e)。sender 是事件源控件本身，e 携带事件数据。

标准事件签名

// 最基础的事件委托
public delegate void EventHandler(object? sender, EventArgs e);

// 泛型版本：e 是 EventArgs 的派生
public delegate void EventHandler<in TEventArgs>(object? sender, TEventArgs e) where TEventArgs : EventArgs;

// 鼠标、键盘、取消 事件携带的数据
public class MouseEventArgs : EventArgs { public MouseButtons Button; public Point Location; public int Clicks; public int Delta; }
public class KeyEventArgs    : EventArgs { public Keys KeyCode; public bool Alt/Control/Shift; public bool Handled; public bool SuppressKeyPress; }
public class CancelEventArgs : EventArgs { public bool Cancel; }
public class FormClosingEventArgs : CancelEventArgs { public CloseReason CloseReason; }

订阅与取消订阅

// 订阅：+= 把委托加入事件的多播调用列表
_btnOK.Click += BtnOK_Click;

// 简写：lambda 表达式
_btnCancel.Click += (_, _) => Close();

// 取消订阅：必须 -= 同一个委托实例（lambda 不能取消）
_btnOK.Click -= BtnOK_Click;

private void BtnOK_Click(object? sender, EventArgs e)
{
    var btn = (Button)sender!;
    btn.Enabled = false;
    // ... 业务
}

// 泛型事件：携带具体数据
_form.FormClosing += (s, e) =>
{
    if (_dirty && MessageBox.Show("放弃修改?", "", MessageBoxButtons.YesNo) == DialogResult.No)
        e.Cancel = true;     // 阻止关闭
};

_grid.CellDoubleClick += (s, e) =>
{
    var row = ((DataGridView)s!).Rows[e.RowIndex];
    OpenEditor((Customer)row.DataBoundItem);
};

自定义事件的标准模式

public class Scanner : Component
{
    // 1. 声明事件（event 关键字保证外部只能 += / -=，不能直接赋值/调用）
    public event EventHandler<BarcodeReadEventArgs>? BarcodeRead;

    // 2. 受保护的虚 OnXxx 方法，子类可重写
    protected virtual void OnBarcodeRead(BarcodeReadEventArgs e)
    {
        // .NET 4.0+ 推荐写法：先取局部变量，避免并发空引用
        BarcodeRead?.Invoke(this, e);
    }

    // 3. 内部触发
    private void OnDeviceData(string code)
    {
        OnBarcodeRead(new BarcodeReadEventArgs(code));
    }
}

public class BarcodeReadEventArgs : EventArgs
{
    public string Barcode { get; }
    public BarcodeReadEventArgs(string barcode) => Barcode = barcode;
}

// 使用
_scanner.BarcodeRead += (s, e) => _txtCode.Text = e.Barcode;
内存泄漏：长生命周期对象（静态单例、Application.Idle、事件总线）订阅短生命周期对象的事件时，必须显式 -=。否则长对象会通过事件委托一直引用短对象，导致 GC 无法回收。

07 设计器与代码分离

WinForms 设计器是其生产力的核心来源。它的实现基于 C# 的 partial class：用户业务代码放在 MainForm.cs，由设计器自动生成、随时可被重写的代码放在 MainForm.Designer.cs。两者在编译期合并成同一个类。

一对 partial 文件

// ── MainForm.Designer.cs（设计器自动维护，禁止手改）──
namespace App;

partial class MainForm
{
    private System.ComponentModel.IContainer? components = null;

    protected override void Dispose(bool disposing)
    {
        if (disposing && (components != null))
            components.Dispose();
        base.Dispose(disposing);
    }

    // ★ 设计器入口：构造函数调用它
    private void InitializeComponent()
    {
        this.components = new System.ComponentModel.Container();
        this._btnOK = new System.Windows.Forms.Button();
        this._txtName = new System.Windows.Forms.TextBox();
        this.SuspendLayout();                       // ★ 暂停布局，批量创建

        // btnOK
        this._btnOK.Location = new System.Drawing.Point(220, 12);
        this._btnOK.Name = "_btnOK";
        this._btnOK.Size = new System.Drawing.Size(75, 28);
        this._btnOK.TabIndex = 0;
        this._btnOK.Text = "确定";
        this._btnOK.Click += new System.EventHandler(this.btnOK_Click);

        // txtName
        this._txtName.Location = new System.Drawing.Point(12, 15);
        this._txtName.Name = "_txtName";
        this._txtName.Size = new System.Drawing.Size(200, 23);
        this._txtName.TabIndex = 1;

        // MainForm
        this.AutoScaleDimensions = new System.Drawing.SizeF(7F, 15F);
        this.AutoScaleMode = System.Windows.Forms.AutoScaleMode.Font;
        this.ClientSize = new System.Drawing.Size(312, 53);
        this.Controls.Add(this._txtName);
        this.Controls.Add(this._btnOK);
        this.Name = "MainForm";
        this.Text = "主窗体";
        this.ResumeLayout(false);                 // ★ 恢复布局
        this.PerformLayout();
    }

    private System.Windows.Forms.Button _btnOK = null!;
    private System.Windows.Forms.TextBox _txtName = null!;
}

// ── MainForm.cs（业务逻辑，手写）──
namespace App;

{
    public MainForm()
    {
        InitializeComponent();      // ★ 必须是构造函数第一条
        // 之后写业务初始化
    }

    private void btnOK_Click(object sender, EventArgs e)
    {
        MessageBox.Show($"Hello {_txtName.Text}");

SuspendLayout / ResumeLayoutManager

批量添加/修改控件时，布局引擎会反复触发——轻则闪烁，重则计算爆 O(n²)。包一层 SuspendLayout()/ResumeLayout() 让布局延迟到最后统一计算，既快又稳。设计器自动生成这段代码，手写代码也应养成习惯。

命名约定
  • 设计器默认 button1、textBox1——立刻改
  • 常见前缀：字段 _btnOK、_txtName、_gridCustomers
  • 事件处理 btnOK_Click（控件名_事件名）
  • VS 选项可设默认前缀风格
.resx 资源
  • MainForm.resx 存本地化字符串、图标、图像
  • 多语言：每文化一份 MainForm.en-US.resx
  • 编译为 satellite assembly，按 Thread.CurrentThread.CurrentUICulture 加载

08 数据绑定

数据绑定让控件属性与数据源对象自动同步。WinForms 数据绑定分两类：简单绑定（控件单一属性到对象属性）与复杂绑定（列表控件绑定整个集合）。BindingSource 是核心中介——它包装数据源、管理当前项位置、提供排序/过滤，几乎总是绑定的最佳起点。

简单绑定 vs 复杂绑定

// ── 简单绑定：控件.属性 ⇄ 对象.属性 ──
public class Customer : INotifyPropertyChanged
{
    private string _name = "";
    public string Name
    {
        get => _name;
        set
        {
            if (_name != value)
            {
                _name = value;
                PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(nameof(Name)));
            }
        }
    }

    public event PropertyChangedEventHandler? PropertyChanged;
}

// 把 TextBox.Text 绑定到 customer.Name；双向更新

// ── 复杂绑定：列表控件 DataSource = 集合 ──

_combo.ValueMember   = nameof(Customer.Id);     // 实际值（SelectedValue）

09 GDI+ 绘图

WinForms 自绘基于 GDI+（System.Drawing）。核心是 Graphics 对象——它代表一块绘图表面（窗口客户区、位图、打印页），所有绘图原语（线、矩形、椭圆、文字、图像）都通过它进行。Graphics 与所有绘图对象（Pen/Brush/Font/Bitmap）都实现 IDisposable，必须正确释放。

获取 Graphics 的两种方式

// 1. OnPaint 的 e.Graphics（推荐，自动成对，自动失效区裁剪）
protected override void OnPaint(PaintEventArgs e)
{
    base.OnPaint(e);
    Graphics g = e.Graphics;
    g.DrawString("Hi", Control.DefaultFont, Brushes.Black, 10, 10);
}

// 2. CreateGraphics() ——临时绘制（不推荐，不持久化）
// 仅适合实时显示（如选区橡皮筋），重绘后会消失
using Graphics g = _panel.CreateGraphics();
g.DrawRectangle(Pens.Red, 0, 0, 100, 100);

public class Gauge : Control
{
    private float _value = 50f;
    public float Value
    {
        get => _value;
        set { _value = Math.Clamp(value, 0f, 100f); Invalidate(); }
    }

    public Gauge()
    {
        DoubleBuffered = true;       // ★ 双缓冲，杜绝闪烁
        ResizeRedraw   = true;       // 尺寸变化即重绘
        SetStyle(ControlStyles.AllPaintingInWmPaint | ControlStyles.UserPaint | ControlStyles.OptimizedDoubleBuffer, true);
    }

    protected override void OnPaint(PaintEventArgs e)
    {
        Graphics g = e.Graphics;
        g.SmoothingMode = System.Drawing.Drawing2D.SmoothingMode.AntiAlias;   // ★ 抗锯齿

        var rect = new Rectangle(8, 8, Width - 16, Height - 16);

        // 所有 GDI 资源必须释放 —— 推荐 using 模式
        using var bgBrush   = new SolidBrush(Color.FromArgb(245, 246, 250));
        using var valuePen = new Pen(Color.FromArgb(80, 130, 240), 10f);
        using var tickPen  = new Pen(Color.FromArgb(180, 184, 200), 2f);
        using var font     = new Font("Segoe UI", 16f, FontStyle.Bold);
        using var textBrush = new SolidBrush(ForeColor);

        g.FillEllipse(bgBrush, rect);

        // 进度弧（从 -90° 顺时针）
        float sweep = 360f * (_value / 100f);
        g.DrawArc(valuePen, rect, -90f, sweep);

        // 刻度
        var center = new PointF(rect.X + rect.Width / 2f, rect.Y + rect.Height / 2f);
        float rOuter = rect.Width / 2f;
        float rInner = rOuter - 12f;
        for (int i = 0; i <= 10; i++)
        {
            float angle = (float)((-90 + i * 36) * Math.PI / 180);
            var p1 = new PointF(center.X + Math.Cos(angle) * rOuter, center.Y + Math.Sin(angle) * rOuter);
            var p2 = new PointF(center.X + Math.Cos(angle) * rInner, center.Y + Math.Sin(angle) * rInner);
            g.DrawLine(tickPen, p1, p2);
        }

        // 文字
        string label = $"{_value:F0}%";
        var sz = g.MeasureString(label, font);
        g.DrawString(label, font, textBrush, center.X - sz.Width / 2f, center.Y - sz.Height / 2f);
    }
}

双缓冲与防闪烁

  • 窗体/控件 DoubleBuffered = true：先画到内存位图再 BitBlt 到屏幕
  • 更细粒度：SetStyle(ControlStyles.OptimizedDoubleBuffer | AllPaintingInWmPaint | UserPaint, true)
  • 复杂动画：BufferedGraphicsManager.Allocate 显式管理缓冲区
  • OnPaintBackground 不擦背景（重写为空）可避免背景与前景分别绘制造成的闪烁
资源生命周期：GDI 句柄是系统级稀缺资源。每个进程约 10000 个上限，Pen/Brush/Font/Bitmap 不释放会快速耗尽。using 或 try/finally Dispose 是底线， Pens.Black/ Brushes.White/SystemFonts.DefaultFont 等系统库存对象不可释放。

10 线程模型与异步

WinForms 是严格的单线程 UI 模型：所有控件只能在创建它的 UI 线程上访问。[STAThread] 标注主入口；任何后台线程直接读写控件属性，运行时会抛出 InvalidOperationException: Cross-thread operation not valid（.NET Framework 默认检测，.NET Core+ 通过 Control.CheckForIllegalCrossThreadCalls）。

铁律：后台线程绝不能直接接触控件。违反会立刻抛异常（运气好）或在随机时刻产生状态损坏、死锁、半绘制（运气差，最难排查）。所有跨线程 UI 更新必须 Invoke 切回 UI 线程。

Invoke / BeginInvoke

// ❌ 错误：后台线程直接更新 UI
Task.Run(() =>
{
    Thread.Sleep(1000);
    _lblStatus.Text = "完成";        // 抛 InvalidOperationException
});

// ✅ 正确：用 Invoke 同步切回 UI 线程
Task.Run(() =>
{
    Thread.Sleep(1000);
    if (_lblStatus.InvokeRequired)           // 检测当前是否在 UI 线程
        _lblStatus.Invoke(() => _lblStatus.Text = "完成");
    else
        _lblStatus.Text = "完成";
});

// ✅ 异步投递（不等返回）
_lblStatus.BeginInvoke(new Action(() => _lblStatus.Text = "完成"));

SynchronizationContext 捕获上下文

// 在 UI 线程捕获上下文（含 UI 线程的同步语义）
var ui = SynchronizationContext.Current!;   // 必须在 UI 线程调用

Task.Run(() =>
{
    // 后台计算
    double result = HeavyCompute();

    // 用 Post 异步切回 UI 线程
    ui.Post(_ => _lblResult.Text = $"{result:F2}", null);
    // 用 Send 同步切回（会阻塞直到 UI 线程处理）
    ui.Send(_ => _lblResult.Text = $"{result:F2}", null);
});

async/await：现代推荐写法

private async void btnLoad_Click(object sender, EventArgs e)
{
    _btnLoad.Enabled = false;
    try
    {
        // await 自动用 captured SynchronizationContext 切回 UI 线程
        var data = await LoadDataAsync();
        _grid.DataSource = data;
        _lblStatus.Text = $"加载 {data.Count} 条";
    }
    catch (Exception ex)
    {
        MessageBox.Show(this, ex.Message, "错误", MessageBoxButtons.OK, MessageBoxIcon.Error);
    }
    finally
    {
        _btnLoad.Enabled = true;
    }
}

private async Task<List<Customer>> LoadDataAsync()
{
    await Task.Delay(800);                  // 模拟 IO
    return await Task.Run(() => QueryFromDb());
}

// 进度反馈：IProgress<T> + Progress<T> 自动切回 UI 线程
private async void btnImport_Click(object s, EventArgs e)
{
    IProgress<int> progress = new Progress<int>(v => _progress.Value = v);
    await Task.Run(() =>
    {
        for (int i = 0; i <= 100; i++)
        {
            Thread.Sleep(20);
            progress.Report(i);          // Progress<T> 内部用 captured SynchronizationContext 回到 UI 线程
        }
    });
    _progress.Value = 0;
    MessageBox.Show("完成");
}

using System.ComponentModel;
using System.Runtime.CompilerServices;

public class Customer : INotifyPropertyChanged, IEditableObject
{
    public int Id { get; set; }

    private string _name = "";
    public string Name
    {
        get => _name;
        set => SetField(ref _name, value);
    }

    private bool _active = true;
    public bool Active
    {
        get => _active;
        set => SetField(ref _active, value);
    }

    public event PropertyChangedEventHandler? PropertyChanged;

    protected bool SetField<T>(ref T field, T value, [CallerMemberName] string? name = null)
    {
        if (EqualityComparer<T>.Default.Equals(field, value)) return false;
        field = value;
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(name));
        return true;
    }

    private Customer? _backup;
    public void BeginEdit() => _backup = (Customer)MemberwiseClone();
    public void CancelEdit() { if (_backup != null) { _name = _backup._name; _active = _backup._active; } }
    public void EndEdit() => _backup = null;
}

12 对比 WPF

WinForms 与 WPF 是 .NET 桌面上的两大并行框架。微软官方说法是“两者都长期支持，互不替代”，但技术路线、设计哲学、渲染原理截然不同。

维度WinFormsWPF
渲染引擎GDI+（位图、句柄、即时模式）DirectX（矢量、保留模式）
控件本质每个控件一个 HWND无窗口，一个 HWND 装整个窗口的合成
UI 定义代码 + 设计器（partial class）XAML（声明式标记）
布局Anchor/Dock/TableLayoutPanelStackPanel/Grid/Canvas/DockPanel（保留模式）
数据绑定较弱，单向/双向需手写极强，命令系统 + 模板 + 多向
样式与模板几乎没有Style/ControlTemplate/DataTemplate
矢量/动画/3D弱强（矢量原生、动画系统、3D 支持）
透明/混合受限（Need layered window）原生支持透明、混合模式
设计器成熟稳定、所见即所得存在但相对复杂
学习曲线平缓（拖控件即用）陡峭（XAML + MVVM + 数据上下文）
性能（控件少）优良好
性能（控件极多）差（HWND 限制）优（虚拟化）
跨平台实验性（Mono/.NET 7+）否（深度绑定 DirectX/Windows）
现代化路径WinForms → MAUI（不直接）WPF → WinUI 3 → WinAppSDK
选 WinForms 当
  • 表单为主、控件少、数据库前端
  • 团队 C# 经验多但 UI 经验少
  • 维护大量遗留 .NET Framework 代码
  • 对接 Win32/COM/旧组件（OCX）
  • 工业上位机、设备控制面板
选 WPF 当
  • 需要富媒体（动画、视频、3D）
  • 大量自定义控件、复杂主题
  • 严格 MVVM、单元测试 UI 逻辑
  • 大数据量列表（虚拟化必须）
  • 新项目且团队接受 XAML 学习成本
选型建议：内部业务工具、配置界面、上位机选 WinForms，5 分钟出原型；面向终端用户、强调视觉、复杂交互选 WPF。两者可以同进程共存（ElementHost 把 WPF 控件嵌进 WinForms，WindowsFormsHost 反之）。

阅读建议:
System.Windows.Forms.Application 消息循环与生命周期管理的入口，理解 WinForms 程序如何启动与退出。
System.Windows.Forms.Control 所有控件的基类，封装 HWND、属性、事件、布局，是 WinForms 的根本。
System.Windows.Forms.Form 顶层窗口，模态/非模态对话框与生命周期事件。
System.ComponentModel.BindingSource 数据绑定的核心中介，配合 BindingList<T> 实现增删改查的全自动同步。
System.Drawing.Graphics GDI+ 绘图表面，所有自绘的入口，配合 Pen/Brush/Font。
System.Windows.Forms.TableLayoutPanel 响应式布局的主力容器，行/列百分比分配。
微软官方《Windows Forms documentation》与 System.Windows.Forms 源码（github.com/dotnet/winforms）为最权威参考。
